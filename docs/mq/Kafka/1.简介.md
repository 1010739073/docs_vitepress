# Kafka 简介

## 什么是 Kafka

Apache Kafka 是一个分布式流处理平台，具有以下三个核心功能：

1. **发布和订阅记录流** - 类似于消息队列或企业消息系统
2. **以容错、持久的方式存储记录流**
3. **处理记录流** - 在记录生成时即进行处理

## Kafka 的特点

### 1. 高吞吐量
- 单机每秒可处理百万级消息
- 面向批处理设计，支持高效的消息传输

### 2. 低延迟
- 消息延迟可以控制在毫秒级
- 适合实时数据处理场景

### 3. 高可用性
- 分布式架构，支持副本机制
- 依赖 ZooKeeper 实现集群协调

### 4. 可扩展性
- 支持在线扩容
- 数据分片存储

### 5. 持久化
- 消息持久化到磁盘
- 支持消息回溯

### 6. 支持多种客户端
- 提供 Java、Python、Go、C++ 等多种语言客户端
- 支持 REST API

## Kafka vs 传统消息队列

| 特性 | Kafka | RabbitMQ | RocketMQ |
|------|-------|----------|----------|
| 吞吐量 | 极高 | 中等 | 高 |
| 延迟 | 低 | 低 | 低 |
| 可靠性 | 高 | 高 | 高 |
| 消息回溯 | 支持 | 不支持 | 支持 |
| 消息堆积 | 支持 | 受内存限制 | 支持 |
| 协议 | 自定义 TCP 协议 | AMQP | 自定义协议 |

## Kafka 的应用场景

### 1. 日志收集
- 收集分布式系统产生的日志
- 集中存储和分析

### 2. 流式处理
- 实时数据分析
- 复杂事件处理

### 3. 消息队列
- 异步通信
- 系统解耦

### 4. 事件溯源
- 存储系统状态变更历史
- 支持状态重建

### 5. CQRS
- 命令查询责任分离
- 读写分离的数据同步

## Kafka 版本演进

- **0.8.x** - 引入副本机制
- **0.9.x** - 引入安全特性
- **0.10.x** - 引入 Streams API
- **0.11.x** - 引入事务支持
- **2.0.x** - 重构架构
- **2.8.x** - 移除 ZooKeeper 依赖（KRaft 模式）
- **3.x+** - 完全移除 ZooKeeper
