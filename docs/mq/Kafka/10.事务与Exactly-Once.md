# Kafka 事务与 Exactly-Once 语义

## 消息传递语义

### 三种语义

| 语义 | 描述 | 实现难度 |
|------|------|----------|
| At Most Once | 消息可能丢失，但不会重复 | 低 |
| At Least Once | 消息不会丢失，但可能重复 | 中 |
| Exactly Once | 消息不丢失也不重复 | 高 |

### Kafka 中的实现

| 场景 | 默认语义 | Exactly Once 实现 |
|------|----------|-------------------|
| 生产者 → Broker | At Least Once | 幂等性 + 事务 |
| Broker → 消费者 | At Least Once | 事务 + 手动提交 |
| 生产者 → Broker → 消费者 | At Least Once | 端到端事务 |

## 幂等性（Idempotence）

### 概念

幂等性保证生产者发送的消息在 Broker 端不会重复写入。

### 实现原理

```
1. 每个 Producer 分配唯一的 PID（Producer ID）
2. 每个 PID 维护一个序列号（Sequence Number）
3. Broker 检查 PID + Sequence Number，拒绝重复消息
```

### PID 和 Sequence Number

```
Producer 1:
├── PID = 1001
├── TopicA-Partition0: SN = [0, 1, 2, 3, ...]
└── TopicA-Partition1: SN = [0, 1, 2, 3, ...]

Broker 检查：
- 收到 PID=1001, Partition=0, SN=5 的消息
- 检查该 PID 在该 Partition 的最新 SN
- 如果 SN <= 5，说明是重复消息，拒绝写入
```

### 配置

```java
// 启用幂等性
props.put("enable.idempotence", "true");

// 幂等性会自动设置以下参数：
// acks = all
// retries = Integer.MAX_VALUE
// max.in.flight.requests.per.connection = 1（或 5，如果启用幂等性）
```

### 幂等性限制

1. 只能保证单个 Producer 实例的幂等性
2. 只能保证单个 Partition 的幂等性
3. Producer 重启后，PID 会变化

## 事务（Transaction）

### 概念

事务允许生产者将多个消息写入多个 Partition，保证原子性：要么全部成功，要么全部失败。

### 事务特性

1. **原子性** - 多条消息要么全部成功，要么全部失败
2. **跨 Partition** - 支持跨多个 Partition
3. **跨 Topic** - 支持跨多个 Topic
4. **持久性** - 事务提交后，数据持久化

### 事务协调器（Transaction Coordinator）

每个 Producer 绑定一个 Transaction Coordinator，负责：

1. 分配 PID
2. 管理事务状态
3. 写入事务日志（__transaction_state）

### 事务日志

Kafka 内部 Topic `__transaction_state` 存储事务状态：

```
事务状态转换：
Empty → Ongoing → PrepareCommit → CompleteCommit
                → PrepareAbort → CompleteAbort
```

### 事务流程

```
1. Producer 发送 InitProducerId 请求
2. Transaction Coordinator 分配 PID
3. Producer 发送 BeginTransaction 请求
4. Producer 发送多条消息
5. Producer 发送 EndTxn 请求（Commit 或 Abort）
6. Transaction Coordinator 写入事务日志
7. Transaction Coordinator 通知所有 Partition 标记事务完成
```

### 事务示例

```java
// 配置
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("transactional.id", "my-transactional-id");
props.put("key.serializer", StringSerializer.class.getName());
props.put("value.serializer", StringSerializer.class.getName());

// 创建生产者
KafkaProducer<String, String> producer = new KafkaProducer<>(props);

// 初始化事务
producer.initTransactions();

try {
    // 开始事务
    producer.beginTransaction();

    // 发送多条消息（跨 Topic、跨 Partition）
    producer.send(new ProducerRecord<>("topic1", "key1", "value1"));
    producer.send(new ProducerRecord<>("topic1", "key2", "value2"));
    producer.send(new ProducerRecord<>("topic2", "key3", "value3"));

    // 提交事务
    producer.commitTransaction();
} catch (Exception e) {
    // 回滚事务
    producer.abortTransaction();
} finally {
    producer.close();
}
```

## 消费者事务隔离

### 隔离级别

```java
// 读未提交（默认）
props.put("isolation.level", "read_uncommitted");

// 读已提交
props.put("isolation.level", "read_committed");
```

### 控制消息（Control Message）

Kafka 使用控制消息标记事务的边界：

```
Partition 数据：
├── 普通消息 1
├── 普通消息 2
├── Control Message (COMMIT) ← 事务 1 提交
├── 普通消息 3
├── 普通消息 4
├── Control Message (ABORT) ← 事务 2 回滚
├── 普通消息 5
└── Control Message (COMMIT) ← 事务 3 提交

read_committed 消费者只能看到：
├── 普通消息 1
├── 普通消息 2
├── 普通消息 5
```

### LSO（Last Stable Offset）

LSO 表示所有已提交事务的最大 Offset：

```
消费者消费范围：
- read_uncommitted：LEO 之前的所有消息
- read_committed：LSO 之前的所有消息（排除 ABORT 事务的消息）
```

## 端到端 Exactly-Once

### 实现

Kafka Streams 提供端到端的 Exactly-Once 语义：

```java
Properties props = new Properties();
props.put(StreamsConfig.APPLICATION_ID_CONFIG, "my-app");
props.put(StreamsConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
props.put(StreamsConfig.PROCESSING_GUARANTEE_CONFIG, StreamsConfig.EXACTLY_ONCE_V2);

StreamsBuilder builder = new StreamsBuilder();
builder.stream("input-topic")
    .mapValues(value -> value.toString().toUpperCase())
    .to("output-topic");

KafkaStreams streams = new KafkaStreams(builder.build(), props);
streams.start();
```

### consume-transform-produce 模式

```java
KafkaConsumer<String, String> consumer = new KafkaConsumer<>(consumerProps);
KafkaProducer<String, String> producer = new KafkaProducer<>(producerProps);

producer.initTransactions();

while (true) {
    ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));

    if (!records.isEmpty()) {
        producer.beginTransaction();

        try {
            for (ConsumerRecord<String, String> record : records) {
                // 处理消息
                String newValue = process(record.value());

                // 发送到输出 Topic
                producer.send(new ProducerRecord<>("output-topic", record.key(), newValue));
            }

            // 提交事务（包括消费 Offset 和输出消息）
            Map<TopicPartition, OffsetAndMetadata> offsets = new HashMap<>();
            for (ConsumerRecord<String, String> record : records) {
                offsets.put(new TopicPartition(record.topic(), record.partition()),
                    new OffsetAndMetadata(record.offset() + 1));
            }
            producer.sendOffsetsToTransaction(offsets, "my-consumer-group");

            producer.commitTransaction();
        } catch (Exception e) {
            producer.abortTransaction();
        }
    }
}
```

## 事务配置详解

### 生产者配置

```properties
# 事务 ID（必须）
transactional.id=my-transactional-id

# 事务超时时间
transaction.timeout.ms=60000

# 幂等性（自动开启）
enable.idempotence=true
```

### 消费者配置

```properties
# 隔离级别
isolation.level=read_committed

# 消费者组
group.id=my-group

# 禁用自动提交
enable.auto.commit=false
```

### Broker 配置

```properties
# 事务日志分区数
transactional.id.expiration.ms=604800000

# 事务日志副本数
transaction.state.log.replication.factor=3

# 事务日志最小 ISR
transaction.state.log.min.isr=2
```

## 事务监控

### 相控指标

| 指标 | 描述 | 建议阈值 |
|------|------|----------|
| transaction-rate | 事务速率 | - |
| transaction-latency-avg | 事务延迟 | < 100ms |
| transaction-abort-rate | 事务中止率 | < 1% |
| transaction-timeout-rate | 事务超时率 | < 0.1% |

## 最佳实践

### 1. 合理设置 transactional.id

```java
// transactional.id 应该唯一且持久
// 建议格式：application-name + instance-id
props.put("transactional.id", "my-app-instance-1");
```

### 2. 合理设置事务超时

```java
// 默认 60 秒，根据业务处理时间调整
props.put("transaction.timeout.ms", "120000");
```

### 3. 事务大小控制

```java
// 控制单个事务包含的消息数量
// 避免大事务导致性能问题
if (messageCount >= 1000) {
    producer.commitTransaction();
    producer.beginTransaction();
}
```

### 4. 异常处理

```java
try {
    producer.beginTransaction();
    // ...
    producer.commitTransaction();
} catch (ProducerFencedException e) {
    // 另一个 Producer 使用了相同的 transactional.id
    producer.close();
} catch (OutOfOrderSequenceException e) {
    // 序列号错误，需要重新创建 Producer
    producer.close();
} catch (AuthorizationException e) {
    // 权限错误
    producer.close();
} catch (KafkaException e) {
    // 其他 Kafka 异常
    producer.abortTransaction();
}
```

### 5. 消费者配置

```java
// 使用 read_committed 隔离级别
props.put("isolation.level", "read_committed");

// 禁用自动提交，使用事务提交 Offset
props.put("enable.auto.commit", "false");
```

## 总结

| 特性 | 幂等性 | 事务 |
|------|--------|------|
| 保证范围 | 单个 Partition | 跨 Partition、跨 Topic |
| Producer 重启 | PID 变化，不保证幂等 | transactional.id 不变，保证幂等 |
| 消费者感知 | 无 | read_committed 隔离级别 |
| 性能开销 | 低 | 中 |
| 适用场景 | 简单生产者 | 复杂业务场景 |

**选择建议**：
- 简单场景：仅启用幂等性
- 复杂场景：使用事务
- 端到端 Exactly-Once：使用 Kafka Streams
