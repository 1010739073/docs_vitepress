# Kafka 核心概念

## 基本概念图

```
Producer → Topic (Partition 0, Partition 1, Partition 2) → Consumer Group
```

## 1. Producer（生产者）

生产者是向 Kafka 集群发布消息的客户端应用。

### 生产者的特点
- 负责将消息发送到指定的 Topic
- 可以选择发送到哪个 Partition
- 支持批量发送以提高吞吐量
- 支持消息压缩

### 生产者配置示例

```java
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");

Producer<String, String> producer = new KafkaProducer<>(props);
producer.send(new ProducerRecord<>("my-topic", "key", "value"));
```

## 2. Consumer（消费者）

消费者是从 Kafka 集群订阅消息并处理的客户端应用。

### 消费者的特点
- 订阅一个或多个 Topic
- 可以加入 Consumer Group 实现负载均衡
- 支持自动提交或手动提交 offset
- 支持消息回溯

### 消费者配置示例

```java
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("group.id", "my-group");
props.put("key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
props.put("value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");

Consumer<String, String> consumer = new KafkaConsumer<>(props);
consumer.subscribe(Arrays.asList("my-topic"));
```

## 3. Topic（主题）

Topic 是 Kafka 中消息的逻辑分类，类似于数据库中的表。

### Topic 的特点
- 每个消息都有一个 Topic
- Topic 可以有多个 Partition
- 支持消息保留策略（时间、大小）
- 支持消息压缩

### 创建 Topic 示例

```bash
# 创建一个包含 3 个分区、2 个副本的 Topic
bin/kafka-topics.sh --create \
  --topic my-topic \
  --partitions 3 \
  --replication-factor 2 \
  --bootstrap-server localhost:9092
```

## 4. Partition（分区）

Partition 是 Topic 的物理分片，用于实现消息的并行处理和负载均衡。

### Partition 的特点
- 每个 Partition 是一个有序的、不可变的消息序列
- 每个 Partition 都有唯一的 offset
- 消息按顺序追加到 Partition 末尾
- 同一个 Partition 内消息有序，不同 Partition 之间无序

### Partition 的作用
1. **水平扩展** - 通过增加 Partition 实现扩展
2. **负载均衡** - 将数据分散到多个 Broker
3. **并行消费** - 允许多个消费者并行消费

### Partition 选择策略

| 策略 | 描述 |
|------|------|
| 轮询（Round-Robin） | 轮询选择 Partition |
| 随机（Random） | 随机选择 Partition |
| Key 哈希（Hash） | 根据 Key 哈希选择，保证相同 Key 的消息到同一 Partition |
| 自定义（Custom） | 自定义 Partition 选择器 |

## 5. Offset（偏移量）

Offset 是 Partition 中每条消息的唯一标识。

### Offset 的特点
- 从 0 开始递增
- 消费者记录自己消费到的 offset
- 支持重置 offset 进行重新消费

### Offset 提交方式

```java
// 自动提交
props.put("enable.auto.commit", "true");
props.put("auto.commit.interval.ms", "1000");

// 手动提交
props.put("enable.auto.commit", "false");
consumer.commitSync(); // 同步提交
consumer.commitAsync(); // 异步提交
```

## 6. Consumer Group（消费者组）

消费者组是一组消费者的集合，用于实现负载均衡。

### 消费者组的特点
- 同一个 Group 内的消费者不会重复消费同一条消息
- 不同 Group 可以独立消费相同的消息
- Group 内的消费者数量不能超过 Partition 数量

### 消费者组示例

```
Topic: 3个Partition
Consumer Group A: 3个Consumer → 每个 Consumer 处理1个 Partition
Consumer Group B: 2个Consumer → 1个处理2个Partition，1个处理1个Partition
Consumer Group C: 4个Consumer → 1个Consumer 处于空闲状态
```

## 7. Broker（代理）

Broker 是 Kafka 集群中的服务器节点。

### Broker 的特点
- 每个 Broker 都有唯一的 ID
- 一个 Broker 可以存储多个 Partition
- Broker 之间通过 ZooKeeper 协调

## 8. Replica（副本）

副本是 Partition 的冗余备份，用于提高可用性。

### 副本类型
- **Leader（主副本）** - 处理所有的读写请求
- **Follower（从副本）** - 从 Leader 同步数据，不处理客户端请求

### 副本的作用
- 数据冗余，防止数据丢失
- 故障转移，当 Leader 故障时，选举新的 Leader

## 9. ISR（In-Sync Replica）

ISR 是与 Leader 保持同步的副本集合。

### ISR 的特点
- ISR 包含 Leader 和所有同步的 Follower
- 只有 ISR 中的 Follower 才有资格成为新的 Leader
- ISR 动态变化，落后太多的 Follower 会被移出 ISR

## 消息存储结构

```
Topic: my-topic
├── Partition 0
│   ├── Leader: Broker 1
│   ├── Replica: Broker 2 (in ISR)
│   └── Replica: Broker 3 (in ISR)
├── Partition 1
│   ├── Leader: Broker 2
│   ├── Replica: Broker 3 (in ISR)
│   └── Replica: Broker 1 (in ISR)
└── Partition 2
    ├── Leader: Broker 3
    ├── Replica: Broker 1 (in ISR)
    └── Replica: Broker 2 (in ISR)
```
