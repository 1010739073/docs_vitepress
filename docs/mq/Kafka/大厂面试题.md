### 2. 如何设计一个秒杀系统的消息队列方案？

**答案**：

```
用户请求 → API Gateway → Redis 预扣减 → Kafka 异步处理 → MySQL 落库
                    ↓
               直接返回成功
```

```properties

# 分区数根据 TPS 设置，建议单分区 TPS 不超过 1 万
num.partitions=10
replication.factor=3

# 保留时间
log.retention.hours=1

# 批量配置
```

**3. 生产者代码**

```java
// 配置
Properties props = new Properties();
props.put("bootstrap.servers", "kafka-cluster:9092");
props.put("acks", "1");  // 秒杀场景优先性能，设为 1
props.put("retries", 0);  // 快速失败
props.put("max.in.flight.requests.per.connection", 5);
props.put("compression.type", "lz4");

// 异步发送
KafkaProducer<String, String> producer = new KafkaProducer<>(props);

public void placeOrder(Order order) {
    ProducerRecord<String, String> record =
        new ProducerRecord<>("seckill-topic", order.getUserId(), JsonUtils.toJson(order));

    producer.send(record, (metadata, exception) -> {
        if (exception != null) {
            // 记录日志，后续补偿
            log.error("Send failed: {}", exception.getMessage());
            // 放入 Redis 延迟队列，后续重试
            redisTemplate.opsForList().leftPush("seckill:retry", order);
        }
    });
}
```


**5. 降级策略**

```java
// 流量控制
public class SeckillController {
    private final RateLimiter rateLimiter = RateLimiter.create(10000);

    public Result placeOrder(OrderRequest request) {
        // 限流
        if (!rateLimiter.tryAcquire()) {
            return Result.fail("系统繁忙，请稍后重试");
        }

        // Redis 预扣减
        Long remaining = redisTemplate.opsForValue()
            .decrement("seckill:stock:" + request.getProductId());
        if (remaining == null || remaining < 0) {
            // 回滚
            redisTemplate.opsForValue()
                .increment("seckill:stock:" + request.getProductId());
            return Result.fail("库存不足");
        }

        // 异步处理
        kafkaProducer.sendOrder(request);

        return Result.success();
    }
}
```

```java
public class LogPartitioner implements Partitioner {
    @Override
    public int partition(String topic, Object key, byte[] keyBytes,
                         Object value, byte[] valueBytes, Cluster cluster) {
        LogEntry log = JsonUtils.fromJson(new String(valueBytes), LogEntry.class);
        List<PartitionInfo> partitions = cluster.partitionsForTopic(topic);

        // 按 Service 分区
        int hash = Math.abs(log.getService().hashCode());
        return hash % partitions.size();
    }
}
```

## 三、容错与恢复

### 7. Kafka Broker 故障后如何自动恢复？

**答案**：

**1. 故障检测**

```
1. ZooKeeper 检测到 Broker 会话超时
2. Controller 收到 Broker 离线通知
3. Controller 开始处理故障恢复
```

**2. 恢复流程**

```
1. Controller 识别故障 Broker 的 Partition
2. Controller 从 ISR 中选举新的 Leader
3. Controller 更新元数据
4. Controller 通知所有 Broker 更新元数据
5. 新 Leader 开始处理请求
6. Follower 从新 Leader 同步数据
```

**3. 配置优化**

```properties
# 会话超时
zookeeper.session.timeout.ms=18000

# Controller 超时
controller.socket.timeout.ms=30000

# 副本恢复
num.recovery.threads.per.data.dir=4
replica.fetch.backoff.ms=1000

# 不干净选举
unclean.leader.election.enable=false
```

### 9. 如何处理 Kafka 数据不一致问题？

**答案**：

**3. 数据校验**

```java
// 使用 checksum 校验
public class MessageChecksum {
    public static String calculateChecksum(String message) {
        try {
            MessageDigest md = MessageDigest.getInstance("MD5");
            byte[] digest = md.digest(message.getBytes());
            return DatatypeConverter.printHexBinary(digest);
        } catch (NoSuchAlgorithmException e) {
            throw new RuntimeException(e);
        }
    }

    public static void validate(String message, String expectedChecksum) {
        String actualChecksum = calculateChecksum(message);
        if (!actualChecksum.equals(expectedChecksum)) {
            throw new RuntimeException("Checksum mismatch");
        }
    }
}
```

```java
public void repairInconsistentMessages(String topic, long startOffset, long endOffset) {
    Properties props = new Properties();
    props.put("bootstrap.servers", "kafka-cluster:9092");
    props.put("group.id", "repair-group");
    props.put("enable.auto.commit", "false");

    KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);

    consumer.seek(new TopicPartition(topic, 0), startOffset);

    while (true) {
        ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));

        for (ConsumerRecord<String, String> record : records) {
            if (record.offset() > endOffset) {
                break;
            }

            // 校验数据
            if (isInconsistent(record)) {
                // 重新发送
                repairMessage(record);
            }
        }
    }
}
```

### 15. Kafka 如何实现 Exactly-Once 语义？

**答案**：

**2. 消费端事务**

```java

props.put("enable.auto.commit", "false");

// 消费并生产
KafkaConsumer<String, String> consumer = new KafkaConsumer<>(consumerProps);
KafkaProducer<String, String> producer = new KafkaProducer<>(producerProps);

producer.initTransactions();

while (true) {
    ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));

    if (!records.isEmpty()) {
        producer.beginTransaction();

        try {
            for (ConsumerRecord<String, String> record : records) {
                // 处理消息
                String newValue = process(record.value());

                // 发送到输出 Topic
                producer.send(new ProducerRecord<>("output-topic", record.key(), newValue));
            }

            // 提交事务（包括消费 Offset）
            Map<TopicPartition, OffsetAndMetadata> offsets = new HashMap<>();
            for (ConsumerRecord<String, String> record : records) {
                offsets.put(new TopicPartition(record.topic(), record.partition()),
                    new OffsetAndMetadata(record.offset() + 1));
            }
            producer.sendOffsetsToTransaction(offsets, "my-group");

            producer.commitTransaction();
        } catch (Exception e) {
            producer.abortTransaction();
        }
    }
}
```
