# Kafka 生产者详解

## 生产者架构

```
应用程序 → ProducerRecord → 序列化器 → 分区器 → 批量记录累加器 → Sender 线程 → Broker
```

## 发送流程

### 1. 消息封装

```java
ProducerRecord<String, String> record = new ProducerRecord<>(
    "topic-name",           // Topic 名称
    "partition-key",        // 分区 Key（可选）
    "message-value"         // 消息内容
);
```

### 2. 拦截器（Interceptor）

在消息发送前/后进行拦截处理。

```java
public class MyProducerInterceptor implements ProducerInterceptor<String, String> {
    @Override
    public ProducerRecord<String, String> onSend(ProducerRecord<String, String> record) {
        // 发送前处理，可以修改消息
        System.out.println("Sending message: " + record.value());
        return record;
    }

    @Override
    public void onAcknowledgement(RecordMetadata metadata, Exception exception) {
        // 收到确认后处理
        if (exception == null) {
            System.out.println("Message sent successfully: " + metadata.offset());
        } else {
            System.out.println("Message failed: " + exception.getMessage());
        }
    }

    @Override
    public void close() {
        // 关闭资源
    }

    @Override
    public void configure(Map<String, ?> configs) {
        // 配置
    }
}
```

### 3. 序列化器

将对象转换为字节数组。

```java
// 自定义序列化器
public class UserSerializer implements Serializer<User> {
    @Override
    public byte[] serialize(String topic, User data) {
        try (ByteArrayOutputStream baos = new ByteArrayOutputStream();
             ObjectOutputStream oos = new ObjectOutputStream(baos)) {
            oos.writeObject(data);
            return baos.toByteArray();
        } catch (IOException e) {
            throw new SerializationException("Error serializing User", e);
        }
    }
}

// JSON 序列化器（推荐）
public class JsonSerializer<T> implements Serializer<T> {
    private final ObjectMapper objectMapper = new ObjectMapper();

    @Override
    public byte[] serialize(String topic, T data) {
        try {
            return objectMapper.writeValueAsBytes(data);
        } catch (JsonProcessingException e) {
            throw new SerializationException("Error serializing JSON", e);
        }
    }
}
```

### 4. 分区器

决定消息发送到哪个分区。

```java
// 默认分区策略
if (partition != null) {
    // 直接指定分区
} else if (key != null) {
    // 根据 Key 哈希选择分区
    partition = Math.abs(key.hashCode()) % numPartitions;
} else {
    // 轮询选择分区
    partition = nextPartition.getAndIncrement() % numPartitions;
}
```

#### 自定义分区器

```java
public class CustomPartitioner implements Partitioner {
    private Random random;

    @Override
    public void configure(Map<String, ?> configs) {
        random = new Random();
    }

    @Override
    public int partition(String topic, Object key, byte[] keyBytes,
                         Object value, byte[] valueBytes, Cluster cluster) {
        List<PartitionInfo> partitions = cluster.partitionsForTopic(topic);
        int numPartitions = partitions.size();
        if (keyBytes == null) {
            return random.nextInt(numPartitions);
        } else {
            return Math.abs(Arrays.hashCode(keyBytes)) % numPartitions;
        }
    }

    @Override
    public void close() {
        // 清理资源
    }
}
```

### 5. 批量发送

生产者将消息放入缓冲区，批量发送以提高吞吐量。

```java
// 批量发送配置
props.put("batch.size", 16384);          // 批量大小（字节）
props.put("linger.ms", 10);              // 等待时间（毫秒）
props.put("buffer.memory", 33554432);    // 缓冲区总大小（字节）
```

## 确认机制（acks）

`acks` 参数控制生产者等待多少个副本确认消息写入成功。

| acks 值 | 描述 | 可靠性 | 性能 |
|---------|------|--------|------|
| 0 | 不等待确认，发送后立即认为成功 | 最低 | 最高 |
| 1 | 只等待 Leader 确认 | 中等 | 中等 |
| all/-1 | 等待 ISR 中所有副本确认 | 最高 | 最低 |

### 代码示例

```java
// acks=0：可能丢失数据，但性能最高
props.put("acks", "0");

// acks=1：Leader 确认即可，可能丢失数据（但概率低于 acks=0）
props.put("acks", "1");

// acks=all：等待所有 ISR 副本确认，不会丢失数据
props.put("acks", "all");
```

## 消息压缩

压缩可以减少网络传输和磁盘存储开销。

### 压缩算法对比

| 算法 | 压缩比 | 压缩速度 | 解压速度 |
|------|--------|----------|----------|
| none | - | - | - |
| gzip | 高 | 慢 | 快 |
| snappy | 中 | 快 | 快 |
| lz4 | 中 | 很快 | 很快 |
| zstd | 很高 | 中等 | 很快 |

### 代码示例

```java
props.put("compression.type", "lz4");
```

## 异步发送

### 简单异步发送

```java
// 发送消息，不关心结果
producer.send(new ProducerRecord<>("topic", "value"));
```

### 带回调的异步发送

```java
producer.send(new ProducerRecord<>("topic", "value"),
    new Callback() {
        @Override
        public void onCompletion(RecordMetadata metadata, Exception exception) {
            if (exception != null) {
                // 发送失败处理
                System.err.println("Send failed: " + exception.getMessage());
            } else {
                // 发送成功处理
                System.out.printf("Sent to partition %d, offset %d%n",
                    metadata.partition(), metadata.offset());
            }
        }
    });
```

## 同步发送

```java
try {
    RecordMetadata metadata = producer.send(
        new ProducerRecord<>("topic", "value")
    ).get(); // 阻塞等待结果

    System.out.printf("Sent to partition %d, offset %d%n",
        metadata.partition(), metadata.offset());
} catch (InterruptedException | ExecutionException e) {
    e.printStackTrace();
}
```

## 事务支持

Kafka 2.0+ 支持跨多个 Topic 和 Partition 的原子操作。

### 配置事务

```java
// 开启事务
props.put("transactional.id", "my-transactional-id");

// 初始化事务
producer.initTransactions();

// 开始事务
producer.beginTransaction();

try {
    // 发送多条消息
    producer.send(new ProducerRecord<>("topic1", "value1"));
    producer.send(new ProducerRecord<>("topic2", "value2"));

    // 提交事务
    producer.commitTransaction();
} catch (Exception e) {
    // 回滚事务
    producer.abortTransaction();
}
```

## 完整示例

```java
import org.apache.kafka.clients.producer.*;
import java.util.Properties;
import java.util.concurrent.Future;

public class KafkaProducerExample {
    public static void main(String[] args) {
        // 配置
        Properties props = new Properties();
        props.put("bootstrap.servers", "localhost:9092");
        props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
        props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");

        // 可靠性配置
        props.put("acks", "all");
        props.put("retries", 3);
        props.put("enable.idempotence", "true");

        // 性能配置
        props.put("batch.size", 16384);
        props.put("linger.ms", 10);
        props.put("buffer.memory", 33554432);
        props.put("compression.type", "lz4");

        // 创建生产者
        Producer<String, String> producer = new KafkaProducer<>(props);

        try {
            // 发送消息
            for (int i = 0; i < 100; i++) {
                ProducerRecord<String, String> record =
                    new ProducerRecord<>("my-topic", "key-" + i, "value-" + i);

                Future<RecordMetadata> future = producer.send(record,
                    (metadata, exception) -> {
                        if (exception != null) {
                            System.err.println("Send failed: " + exception.getMessage());
                        } else {
                            System.out.printf("Sent to partition %d, offset %d%n",
                                metadata.partition(), metadata.offset());
                        }
                    });
            }
        } finally {
            // 关闭生产者
            producer.close();
        }
    }
}
```
