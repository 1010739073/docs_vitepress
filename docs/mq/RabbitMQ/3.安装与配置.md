# RabbitMQ 安装与配置

## 一、安装方式

### 1.1 Docker 安装（推荐）

#### 单机安装

```bash
# 拉取镜像
docker pull rabbitmq:3.12-management

# 运行容器
docker run -d \
  --name rabbitmq \
  -p 5672:5672 \
  -p 15672:15672 \
  -e RABBITMQ_DEFAULT_USER=admin \
  -e RABBITMQ_DEFAULT_PASS=admin123 \
  -v rabbitmq-data:/var/lib/rabbitmq \
  rabbitmq:3.12-management
```

**参数说明**：
- `-p 5672:5672`：AMQP 协议端口
- `-p 15672:15672`：管理界面端口
- `RABBITMQ_DEFAULT_USER`：默认用户名
- `RABBITMQ_DEFAULT_PASS`：默认密码
- `-v rabbitmq-data:/var/lib/rabbitmq`：数据持久化

#### 访问管理界面

打开浏览器访问：`http://localhost:15672`
- 用户名：`admin`
- 密码：`admin123`

#### Docker Compose 安装

```yaml
# docker-compose.yml
version: '3.8'

services:
  rabbitmq:
    image: rabbitmq:3.12-management
    container_name: rabbitmq
    hostname: rabbitmq
    ports:
      - "5672:5672"
      - "15672:15672"
    environment:
      RABBITMQ_DEFAULT_USER: admin
      RABBITMQ_DEFAULT_PASS: admin123
    volumes:
      - rabbitmq-data:/var/lib/rabbitmq
      - ./rabbitmq.conf:/etc/rabbitmq/rabbitmq.conf:ro
    restart: always

volumes:
  rabbitmq-data:
```

启动：
```bash
docker-compose up -d
```

### 1.2 Linux 安装

#### CentOS/RHEL

```bash
# 安装 Erlang
sudo yum install -y epel-release
sudo yum install -y erlang

# 添加 RabbitMQ 仓库
sudo yum install -y wget
wget https://github.com/rabbitmq/rabbitmq-server/releases/download/v3.12.10/rabbitmq-server-3.12.10-1.el8.noarch.rpm

# 安装 RabbitMQ
sudo yum install -y rabbitmq-server-3.12.10-1.el8.noarch.rpm

# 启动服务
sudo systemctl start rabbitmq-server
sudo systemctl enable rabbitmq-server

# 开启管理插件
sudo rabbitmq-plugins enable rabbitmq_management

# 重启服务
sudo systemctl restart rabbitmq-server
```

#### Ubuntu/Debian

```bash
# 安装 Erlang
sudo apt-get update
sudo apt-get install -y erlang

# 添加 RabbitMQ 仓库
curl -1sLf 'https://github.com/rabbitmq/signing-keys/releases/download/3.0/cloudsmith.rabbitmq-erlang.E495BB49CC4BBE5B.key' | sudo apt-key add -
curl -1sLf 'https://github.com/rabbitmq/signing-keys/releases/download/3.0/cloudsmith.rabbitmq-server.9F4587F226208342.key' | sudo apt-key add -

sudo tee /etc/apt/sources.list.d/rabbitmq.list <<EOF
deb https://dl.cloudsmith.io/public/rabbitmq/rabbitmq-erlang/deb/ubuntu focal main
deb-src https://dl.cloudsmith.io/public/rabbitmq/rabbitmq-erlang/deb/ubuntu focal main

deb https://dl.cloudsmith.io/public/rabbitmq/rabbitmq-server/deb/ubuntu focal main
deb-src https://dl.cloudsmith.io/public/rabbitmq/rabbitmq-server/deb/ubuntu focal main
EOF

sudo apt-get update -y

# 安装 RabbitMQ
sudo apt-get install -y rabbitmq-server

# 启动服务
sudo systemctl start rabbitmq-server
sudo systemctl enable rabbitmq-server

# 开启管理插件
sudo rabbitmq-plugins enable rabbitmq_management

# 重启服务
sudo systemctl restart rabbitmq-server
```

## 二、基础配置

### 2.1 配置文件位置

| 系统 | 配置文件位置 |
|------|-------------|
| Linux | `/etc/rabbitmq/rabbitmq.conf` |
| Windows | `%APPDATA%\RabbitMQ\rabbitmq.conf` |
| Docker | 挂载到 `/etc/rabbitmq/rabbitmq.conf` |

### 2.2 常用配置

```ini
# rabbitmq.conf

# 监听端口
listeners.tcp.default = 5672

# 管理界面端口
management.tcp.port = 15672

# 默认用户（仅用于初始化）
default_user = admin
default_pass = admin123

# 日志级别
log.console.level = info
log.file.level = info

# 磁盘空间限制（字节）
disk_free_limit.absolute = 1GB

# 内存限制（百分比）
vm_memory_high_watermark.relative = 0.4

# 心跳超时（秒）
heartbeat = 60

# 连接数限制
channel_max = 2048
connection_max = infinity

# 消息 TTL（毫秒）
default_vhost = /

# 集群配置
cluster_formation.peer_discovery_backend = rabbit_peer_discovery_classic_config
cluster_formation.classic_config.nodes.1 = rabbit@node1
cluster_formation.classic_config.nodes.2 = rabbit@node2

# 管理界面配置
management.load_definitions = /etc/rabbitmq/definitions.json
```

### 2.3 环境变量

```bash
# /etc/rabbitmq/rabbitmq-env.conf

# 节点名称
NODENAME=rabbit@localhost

# 配置文件路径
CONFIG_FILE=/etc/rabbitmq/rabbitmq.conf

# 日志文件路径
LOG_BASE=/var/log/rabbitmq

# 数据目录
MNESIA_BASE=/var/lib/rabbitmq/mnesia

# 插件目录
PLUGIN_DIR=/usr/lib/rabbitmq/plugins

# Erlang 节点端口
RABBITMQ_NODE_PORT=5672

# 启用安全特性
RABBITMQ_SERVICENAME=RabbitMQ
```

## 三、用户管理

### 3.1 命令行操作

```bash
# 查看所有用户
rabbitmqctl list_users

# 添加用户
rabbitmqctl add_user username password

# 删除用户
rabbitmqctl delete_user username

# 修改密码
rabbitmqctl change_password username newpassword

# 设置用户标签（administrator 可以访问管理界面）
rabbitmqctl set_user_tags username administrator

# 查看用户权限
rabbitmqctl list_permissions

# 授予用户权限
rabbitmqctl set_permissions -p / username ".*" ".*" ".*"

# 清除用户权限
rabbitmqctl clear_permissions -p / username
```

### 3.2 用户标签

| 标签 | 说明 |
|------|------|
| `management` | 可以访问管理界面 |
| `policymaker` | 可以访问管理界面，可以管理策略 |
| `monitoring` | 可以访问管理界面，可以查看监控信息 |
| `administrator` | 拥有所有权限 |

```bash
# 设置管理员
rabbitmqctl set_user_tags admin administrator

# 设置多个标签
rabbitmqctl set_user_tags user1 management,monitoring
```

### 3.3 权限配置

权限格式：`configure write read`

```bash
# 完全权限
rabbitmqctl set_permissions -p / username ".*" ".*" ".*"

# 只读权限
rabbitmqctl set_permissions -p / username "^$" "^$" ".*"

# 特定队列权限
rabbitmqctl set_permissions -p / username "^(my-queue)$" "^(my-queue)$" "^(my-queue)$"

# 查看用户在特定 vhost 的权限
rabbitmqctl list_permissions -p /
```

## 四、Virtual Host 管理

### 4.1 命令行操作

```bash
# 查看所有 vhost
rabbitmqctl list_vhosts

# 添加 vhost
rabbitmqctl add_vhost /my_app

# 删除 vhost
rabbitmqctl delete_vhost /my_app

# 设置 vhost 权限
rabbitmqctl set_permissions -p /my_app username ".*" ".*" ".*"

# 清除 vhost 权限
rabbitmqctl clear_permissions -p /my_app username
```

### 4.2 创建应用专用 vhost

```bash
# 创建生产环境 vhost
rabbitmqctl add_vhost /prod

# 创建测试环境 vhost
rabbitmqctl add_vhost /test

# 创建开发环境 vhost
rabbitmqctl add_vhost /dev

# 为每个环境创建用户
rabbitmqctl add_user prod_user prod_pass
rabbitmqctl add_user test_user test_pass
rabbitmqctl add_user dev_user dev_pass

# 分配权限
rabbitmqctl set_permissions -p /prod prod_user ".*" ".*" ".*"
rabbitmqctl set_permissions -p /test test_user ".*" ".*" ".*"
rabbitmqctl set_permissions -p /dev dev_user ".*" ".*" ".*"
```

## 五、插件管理

### 5.1 查看插件

```bash
# 查看所有插件
rabbitmq-plugins list

# 查看已启用的插件
rabbitmq-plugins list -e
```

### 5.2 启用/禁用插件

```bash
# 启用插件
rabbitmq-plugins enable rabbitmq_management
rabbitmq-plugins enable rabbitmq_delayed_message_exchange
rabbitmq-plugins enable rabbitmq_shovel
rabbitmq-plugins enable rabbitmq_federation

# 禁用插件
rabbitmq-plugins disable rabbitmq_delayed_message_exchange
```

### 5.3 常用插件

| 插件名 | 功能 |
|--------|------|
| `rabbitmq_management` | Web 管理界面 |
| `rabbitmq_delayed_message_exchange` | 延迟消息 |
| `rabbitmq_shovel` | 消息复制/迁移 |
| `rabbitmq_federation` | 联邦队列 |
| `rabbitmq_consistent_hash_exchange` | 一致性哈希交换机 |
| `rabbitmq_prometheus` | Prometheus 监控 |
| `rabbitmq_top` | 实时监控消息速率 |
| `rabbitmq_tracing` | 消息追踪 |

## 六、集群部署

### 6.1 集群架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Node 1    │─────│   Node 2    │─────│   Node 3    │
│  (Primary)  │     │  (Mirror)   │     │  (Mirror)   │
│  Port: 5672 │     │  Port: 5673 │     │  Port: 5674 │
└─────────────┘     └─────────────┘     └─────────────┘
       │                   │                   │
       └───────────────────┴───────────────────┘
                            │
                      ┌─────▼─────┐
                      │  负载均衡  │
                      │  (Nginx/LB)│
                      └─────┬─────┘
                            │
                      ┌─────▼─────┐
                      │   客户端   │
                      └───────────┘
```

### 6.2 Docker Compose 集群

```yaml
# docker-compose-cluster.yml
version: '3.8'

services:
  rabbitmq1:
    image: rabbitmq:3.12-management
    container_name: rabbitmq1
    hostname: rabbitmq1
    ports:
      - "5672:5672"
      - "15672:15672"
    environment:
      RABBITMQ_ERLANG_COOKIE: rabbitmq_cookie
      RABBITMQ_DEFAULT_USER: admin
      RABBITMQ_DEFAULT_PASS: admin123
    volumes:
      - rabbitmq1-data:/var/lib/rabbitmq
      - ./rabbitmq.conf:/etc/rabbitmq/rabbitmq.conf:ro
    command: >
      bash -c "rabbitmq-server & \
               sleep 10 && \
               rabbitmqctl stop_app && \
               rabbitmqctl reset && \
               rabbitmqctl start_app && \
               tail -f /dev/null"

  rabbitmq2:
    image: rabbitmq:3.12-management
    container_name: rabbitmq2
    hostname: rabbitmq2
    ports:
      - "5673:5672"
      - "15673:15672"
    environment:
      RABBITMQ_ERLANG_COOKIE: rabbitmq_cookie
      RABBITMQ_DEFAULT_USER: admin
      RABBITMQ_DEFAULT_PASS: admin123
    volumes:
      - rabbitmq2-data:/var/lib/rabbitmq
    command: >
      bash -c "rabbitmq-server & \
               sleep 10 && \
               rabbitmqctl stop_app && \
               rabbitmqctl reset && \
               rabbitmqctl join_cluster rabbit@rabbitmq1 && \
               rabbitmqctl start_app && \
               tail -f /dev/null"
    depends_on:
      - rabbitmq1

  rabbitmq3:
    image: rabbitmq:3.12-management
    container_name: rabbitmq3
    hostname: rabbitmq3
    ports:
      - "5674:5672"
      - "15674:15672"
    environment:
      RABBITMQ_ERLANG_COOKIE: rabbitmq_cookie
      RABBITMQ_DEFAULT_USER: admin
      RABBITMQ_DEFAULT_PASS: admin123
    volumes:
      - rabbitmq3-data:/var/lib/rabbitmq
    command: >
      bash -c "rabbitmq-server & \
               sleep 10 && \
               rabbitmqctl stop_app && \
               rabbitmqctl reset && \
               rabbitmqctl join_cluster rabbit@rabbitmq1 && \
               rabbitmqctl start_app && \
               tail -f /dev/null"
    depends_on:
      - rabbitmq1

volumes:
  rabbitmq1-data:
  rabbitmq2-data:
  rabbitmq3-data:
```

启动集群：
```bash
docker-compose -f docker-compose-cluster.yml up -d
```

### 6.3 集群管理命令

```bash
# 查看集群状态
rabbitmqctl cluster_status

# 添加节点到集群
rabbitmqctl stop_app
rabbitmqctl reset
rabbitmqctl join_cluster rabbit@node1
rabbitmqctl start_app

# 移除节点
rabbitmqctl stop_app
rabbitmqctl reset
rabbitmqctl start_app

# 集群中移除节点（在其他节点执行）
rabbitmqctl forget_cluster_node rabbit@node3

# 修改节点类型
rabbitmqctl change_cluster_node_type disc
rabbitmqctl change_cluster_node_type ram
```

## 七、镜像队列

### 7.1 配置镜像队列

```bash
# 设置策略：所有以 "ha." 开头的队列都是镜像队列
rabbitmqctl set_policy ha-all "^ha\." '{"ha-mode":"all","ha-sync-mode":"automatic"}'

# 查看策略
rabbitmqctl list_policies

# 删除策略
rabbitmqctl clear_policy ha-all
```

### 7.2 镜像模式

| 模式 | 说明 |
|------|------|
| `ha-mode: all` | 所有节点都有镜像 |
| `ha-mode: exactly` | 指定数量的节点有镜像 |
| `ha-mode: nodes` | 指定节点有镜像 |

```bash
# 指定镜像节点数
rabbitmqctl set_policy ha-two "^ha\." '{"ha-mode":"exactly","ha-count":2}'

# 指定镜像节点
rabbitmqctl set_policy ha-nodes "^ha\." '{"ha-mode":"nodes","ha-params":["rabbit@node1","rabbit@node2"]}'
```

## 八、监控与日志

### 8.1 日志配置

```ini
# rabbitmq.conf

# 日志级别
log.console.level = info
log.file.level = info

# 日志文件路径
log.file = /var/log/rabbitmq/rabbit.log

# 日志轮转
log.rotation.size = 50MB
log.rotation.count = 5

# 分类日志
log.connection.level = info
log.channel.level = info
```

### 8.2 监控指标

| 指标 | 说明 |
|------|------|
| `queue_messages` | 队列中消息数量 |
| `queue_messages_ready` | 准备投递的消息数 |
| `queue_messages_unacked` | 未确认的消息数 |
| `channel_consumers` | 消费者数量 |
| `connection_count` | 连接数 |
| `channel_count` | 通道数 |

### 8.3 Prometheus 集成

```bash

# 访问指标
# http://localhost:15692/metrics
```

## 九、安全配置

### 9.1 启用 TLS

```bash
# 生成证书
openssl req -newkey rsa:2048 -nodes -keyout private_key.pem -x509 -days 365 -out certificate.pem

# 配置 TLS
# rabbitmq.conf
listeners.ssl.default = 5671
ssl_options.cacertfile = /path/to/cacert.pem
ssl_options.certfile = /path/to/certificate.pem
ssl_options.keyfile = /path/to/private_key.pem
ssl_options.verify = verify_peer
ssl_options.fail_if_no_peer_cert = false
```

### 9.2 管理界面安全

```ini
# rabbitmq.conf

# 禁用默认用户
management.load_definitions = /etc/rabbitmq/load_definitions.json

# 限制访问 IP
management.tcp.ip = 127.0.0.1
```

### 9.3 防火墙配置

```bash
# 开放端口
sudo firewall-cmd --permanent --add-port=5672/tcp
sudo firewall-cmd --permanent --add-port=15672/tcp
sudo firewall-cmd --reload
```
