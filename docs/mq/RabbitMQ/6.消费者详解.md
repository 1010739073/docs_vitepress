# RabbitMQ 消费者详解

## 一、消息消费流程

### 1.1 消费流程图

```
Queue
   │
   ├─ 1. 投递消息到 Consumer
   │
   ├─ 2. Consumer 接收消息
   │
   ├─ 3. 处理业务逻辑
   │
   ├─ 4. 发送 ACK/NACK
   │
   └─ 5. RabbitMQ 处理确认
        ├─ ACK: 删除消息
        ├─ NACK+Requeue: 重新入队
        └─ NACK: 删除消息
```

### 1.2 基本消费

```java
@Component
public class SimpleConsumer {

    @RabbitListener(queues = "my.queue")
    public void receiveMessage(String message) {
        System.out.println("Received: " + message);
    }
}
```

## 二、消费确认模式

### 2.1 自动确认（Auto Ack）

消息投递后立即确认，不管业务是否成功。

```yaml
spring:
  rabbitmq:
    listener:
      simple:
        acknowledge-mode: auto  # 或 none（自动确认）
```

```java
@RabbitListener(queues = "my.queue")
public void receiveMessage(String message) {
    try {
        // 处理消息
        processMessage(message);
    } catch (Exception e) {
        // 即使抛异常，消息也会被确认（丢失）
        log.error("处理失败", e);
    }
}
```

**缺点**：消息可能丢失

### 2.2 手动确认（Manual Ack）

业务处理成功后手动确认。

```yaml
spring:
  rabbitmq:
    listener:
      simple:
        acknowledge-mode: manual
```

```java
@RabbitListener(queues = "my.queue")
public void receiveMessage(String message,
                         Channel channel,
                         @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
    try {
        // 处理消息
        processMessage(message);

        // 手动 ACK
        channel.basicAck(deliveryTag, false);
        log.info("消息处理成功");

    } catch (Exception e) {
        log.error("消息处理失败", e);

        try {
            // 拒绝消息，重新入队
            channel.basicNack(deliveryTag, false, true);
        } catch (IOException ex) {
            log.error("NACK 失败", ex);
        }
    }
}
```

**ACK 参数说明**：
- `deliveryTag`：消息标签
- `multiple`：是否批量确认（true：确认该 deliveryTag 及之前的所有消息）

### 2.3 手动确认最佳实践

```java
@Component
public class ReliableConsumer {

    @RabbitListener(queues = "my.queue")
    public void receiveMessage(Message message,
                             Channel channel,
                             @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
        String msgBody = new String(message.getBody());
        long startTime = System.currentTimeMillis();

        try {
            // 1. 解析消息
            BusinessMessage businessMessage = JSON.parseObject(msgBody, BusinessMessage.class);

            // 2. 验证消息
            validateMessage(businessMessage);

            // 3. 幂等性检查
            if (isProcessed(businessMessage.getId())) {
                log.info("消息已处理，直接确认: {}", businessMessage.getId());
                channel.basicAck(deliveryTag, false);
                return;
            }

            // 4. 处理业务
            processBusiness(businessMessage);

            // 5. 标记已处理
            markAsProcessed(businessMessage.getId());

            // 6. ACK
            channel.basicAck(deliveryTag, false);

            long cost = System.currentTimeMillis() - startTime;
            log.info("消息处理成功, messageId: {}, 耗时: {}ms",
                businessMessage.getId(), cost);

        } catch (BusinessException e) {
            // 业务异常，不重试
            log.warn("业务异常，直接确认: {}", e.getMessage());
            try {
                channel.basicAck(deliveryTag, false);
            } catch (IOException ex) {
                log.error("ACK 失败", ex);
            }

        } catch (Exception e) {
            // 系统异常，重试
            log.error("系统异常，消息重新入队", e);
            try {
                channel.basicNack(deliveryTag, false, true);
            } catch (IOException ex) {
                log.error("NACK 失败", ex);
            }
        }
    }

    private void validateMessage(BusinessMessage message) {
        if (message == null || message.getId() == null) {
            throw new BusinessException("消息格式错误");
        }
    }

    private boolean isProcessed(String messageId) {
        // 检查 Redis 或数据库
        return redisTemplate.hasKey("processed:" + messageId);
    }

    private void markAsProcessed(String messageId) {
        // 标记已处理
        redisTemplate.opsForValue().set("processed:" + messageId, "1", 24, TimeUnit.HOURS);
    }
}
```

## 三、Prefetch（预取数量）

### 3.1 原理

Prefetch 控制消费者未确认的消息数量。

```
prefetch = 1
┌─────────────────────────────────────┐
│ 消费者                              │
│  ┌─────┐  ┌─────┐  ┌─────┐         │
│  │消息1│  │消息2│  │消息3│ ...     │
│  │处理中│  │等待中│  │等待中│         │
│  └─────┘  └─────┘  └─────┘         │
│   ↑ 只有一条消息在处理              │
└─────────────────────────────────────┘

prefetch = 10
┌─────────────────────────────────────┐
│ 消费者                              │
│  ┌─────┐ ┌─────┐ ┌─────┐ ...      │
│  │消息1│ │消息2│ │消息3│ ...      │
│  │处理中│ │等待中│ │等待中│         │
│  └─────┘ └─────┘ └─────┘          │
│   ↑ 最多 10 条未确认消息            │
└─────────────────────────────────────┘
```

### 3.2 配置 Prefetch

```yaml
spring:
  rabbitmq:
    listener:
      simple:
        prefetch: 1  # 公平分发
```

或代码配置：

```java
@Configuration
public class RabbitMQConfig {

    @Bean
    public SimpleRabbitListenerContainerFactory rabbitListenerContainerFactory(
            ConnectionFactory connectionFactory) {
        SimpleRabbitListenerContainerFactory factory =
            new SimpleRabbitListenerContainerFactory();
        factory.setConnectionFactory(connectionFactory);
        factory.setAcknowledgeMode(AcknowledgeMode.MANUAL);
        factory.setPrefetch(1);  // 每次只接收一条消息
        return factory;
    }
}
```

### 3.3 Prefetch 选择建议

| 场景 | 推荐值 | 说明 |
|------|--------|------|
| 处理时间短 | 10-20 | 提高吞吐量 |
| 处理时间长 | 1 | 公平分发 |
| 不确定 | 5 | 平衡性能和公平性 |

## 四、消费限流

### 4.1 限流策略

```java
@Component
public class RateLimitedConsumer {

    private final RateLimiter rateLimiter = RateLimiter.create(100); // 100 QPS

    @RabbitListener(queues = "my.queue")
    public void receiveMessage(String message) {
        // 限流
        if (!rateLimiter.tryAcquire()) {
            log.warn("限流，消息处理延迟");
            try {
                Thread.sleep(10);  // 等待后再处理
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }

        // 处理消息
        processMessage(message);
    }
}
```

### 4.2 动态限流

```java
@Component
public class DynamicRateLimiter {

    private double currentRate = 100.0;
    private final AtomicLong successCount = new AtomicLong(0);
    private final AtomicLong errorCount = new AtomicLong(0);
    private final ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(1);

    @PostConstruct
    public void init() {
        // 每分钟调整限流阈值
        scheduler.scheduleAtFixedRate(this::adjustRate, 1, 1, TimeUnit.MINUTES);
    }

    private void adjustRate() {
        long success = successCount.getAndSet(0);
        long error = errorCount.getAndSet(0);
        long total = success + error;

        if (total == 0) return;

        double errorRate = (double) error / total;

        // 错误率超过 10%，降低限流
        if (errorRate > 0.1) {
            currentRate = Math.max(10, currentRate * 0.8);
            log.warn("错误率过高({:.2f}%)，降低限流至: {}", errorRate * 100, currentRate);
        }
        // 错误率低于 1%，提高限流
        else if (errorRate < 0.01) {
            currentRate = Math.min(1000, currentRate * 1.2);
            log.info("错误率较低({:.2f}%)，提高限流至: {}", errorRate * 100, currentRate);
        }
    }

    public boolean tryAcquire() {
        return RateLimiter.create(currentRate).tryAcquire();
    }

    @PreDestroy
    public void destroy() {
        scheduler.shutdown();
    }
}
```

## 五、消费重试

### 5.1 自动重试

```yaml
spring:
  rabbitmq:
    listener:
      simple:
        retry:
          enabled: true
          max-attempts: 3
          initial-interval: 1000
          multiplier: 2.0
          max-interval: 10000
          stateless: false
```

### 5.2 自定义重试

```java
@Component
public class RetryConsumer {

    private static final int MAX_RETRY = 3;

    @RabbitListener(queues = "my.queue")
    public void receiveMessage(Message message,
                             Channel channel,
                             @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
        String msgBody = new String(message.getBody());

        try {
            // 获取重试次数
            int retryCount = getRetryCount(message);

            if (retryCount >= MAX_RETRY) {
                log.warn("达到最大重试次数，发送到死信队列");
                sendToDeadLetter(message);
                channel.basicAck(deliveryTag, false);
                return;
            }

            // 处理消息
            processMessage(msgBody);

            // 成功，ACK
            channel.basicAck(deliveryTag, false);

        } catch (Exception e) {
            log.error("处理失败，准备重试", e);

            // 增加重试次数
            incrementRetryCount(message);

            try {
                // 拒绝，重新入队
                channel.basicNack(deliveryTag, false, true);
            } catch (IOException ex) {
                log.error("NACK 失败", ex);
            }
        }
    }

    private int getRetryCount(Message message) {
        Map<String, Object> headers = message.getMessageProperties().getHeaders();
        Object retryCount = headers.get("x-retry-count");
        return retryCount != null ? (int) retryCount : 0;
    }

    private void incrementRetryCount(Message message) {
        Map<String, Object> headers = message.getMessageProperties().getHeaders();
        int retryCount = getRetryCount(message);
        headers.put("x-retry-count", retryCount + 1);
    }
}
```

### 5.3 延迟重试

```java
@Component
public class DelayedRetryConsumer {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    @RabbitListener(queues = "my.queue")
    public void receiveMessage(Message message,
                             Channel channel,
                             @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
        String msgBody = new String(message.getBody());

        try {
            processMessage(msgBody);
            channel.basicAck(deliveryTag, false);

        } catch (Exception e) {
            int retryCount = getRetryCount(message);

            if (retryCount < MAX_RETRY) {
                // 延迟重试
                long delay = (long) Math.pow(2, retryCount) * 1000; // 指数退避

                MessageProperties properties = message.getMessageProperties();
                properties.getHeaders().put("x-retry-count", retryCount + 1);
                properties.setExpiration(String.valueOf(delay));

                // 发送到延迟队列
                rabbitTemplate.send("delayed.exchange", "retry.key", message);

                // ACK 原始消息
                channel.basicAck(deliveryTag, false);
            } else {
                // 发送到死信队列
                sendToDeadLetter(message);
                channel.basicAck(deliveryTag, false);
            }
        }
    }
}
```

@Component
public class ManualBatchConsumer {

    @RabbitListener(queues = "my.queue")
    public void receiveMessages(List<Message> messages,
                               Channel channel) {
        List<Long> deliveryTags = new ArrayList<>();
        List<String> messageIds = new ArrayList<>();

        try {
            // 收集 deliveryTag
            for (Message message : messages) {
                long deliveryTag = message.getMessageProperties().getDeliveryTag();
                deliveryTags.add(deliveryTag);

                String messageId = message.getMessageProperties().getMessageId();
                messageIds.add(messageId);
            }

            // 批量处理
            batchProcess(messages);

            // 批量 ACK
            if (!deliveryTags.isEmpty()) {
                long lastDeliveryTag = deliveryTags.get(deliveryTags.size() - 1);
                channel.basicAck(lastDeliveryTag, true);  // multiple=true
            }

            log.info("批量消费成功，消息数量: {}", messages.size());

        } catch (Exception e) {
            log.error("批量消费失败", e);

            // 全部 NACK
            if (!deliveryTags.isEmpty()) {
                long lastDeliveryTag = deliveryTags.get(deliveryTags.size() - 1);
                try {
                    channel.basicNack(lastDeliveryTag, true, true);  // 重新入队
                } catch (IOException ex) {
                    log.error("NACK 失败", ex);
                }
            }
        }
    }
}

### 7.2 动态并发调整

```java
@Component
public class DynamicConcurrencyAdjuster {

    @Autowired
    private SimpleRabbitListenerContainerFactory factory;

    @Autowired
    private RabbitAdmin rabbitAdmin;

    private final ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(1);

    @PostConstruct
    public void init() {
        // 每 30 秒调整一次并发数
        scheduler.scheduleAtFixedRate(this::adjustConcurrency, 30, 30, TimeUnit.SECONDS);
    }

    private void adjustConcurrency() {
        // 获取队列消息数
        GetResponse response = rabbitAdmin.getRabbitTemplate().execute(channel ->
            channel.basicGet("my.queue", false)
        );

        if (response == null) return;

        long messageCount = response.getMessageCount();

        // 根据消息数调整并发
        int currentConcurrency = factory.getConcurrentConsumers();
        int targetConcurrency = calculateConcurrency(messageCount);

        if (targetConcurrency != currentConcurrency) {
            log.info("调整并发数: {} -> {}", currentConcurrency, targetConcurrency);
            factory.setConcurrentConsumers(targetConcurrency);
        }
    }

    private int calculateConcurrency(long messageCount) {
        if (messageCount < 100) return 5;
        if (messageCount < 1000) return 10;
        if (messageCount < 10000) return 20;
        return 50;
    }

    @PreDestroy
    public void destroy() {
        scheduler.shutdown();
    }
}
```

## 八、消费监控

### 8.1 消费统计

```java
@Component
public class ConsumerMonitor {

    private final AtomicLong consumeCount = new AtomicLong(0);
    private final AtomicLong successCount = new AtomicLong(0);
    private final AtomicLong failureCount = new AtomicLong(0);
    private final AtomicLong totalLatency = new AtomicLong(0);

    @EventListener
    public void handleConsumeEvent(ConsumeEvent event) {
        long count = consumeCount.incrementAndGet();

        if (event.isSuccess()) {
            successCount.incrementAndGet();
            totalLatency.addAndGet(event.getLatency());
        } else {
            failureCount.incrementAndGet();
        }

        // 每 1000 条输出一次统计
        if (count % 1000 == 0) {
            long avgLatency = totalLatency.get() / count;
            log.info("消费统计 - 总数: {}, 成功: {}, 失败: {}, 平均耗时: {}ms",
                count, successCount.get(), failureCount.get(), avgLatency);
        }
    }
}
```

## 九、最佳实践

### 9.1 消费建议

| 建议 | 说明 |
|------|------|
| 使用手动确认 | 确保消息可靠处理 |
| 设置合理的 Prefetch | 平衡性能和公平性 |
| 实现幂等性 | 避免重复消费问题 |
| 处理异常 | 区分业务异常和系统异常 |
| 监控消费指标 | 及时发现问题 |
| 合理设置重试 | 避免无限重试 |
| 使用批量消费 | 提高吞吐量 |

### 9.2 完整示例

```java
@Component
@Slf4j
public class ReliableConsumer {

    private static final int MAX_RETRY = 3;
    private static final long SLOW_THRESHOLD = 1000;

    @Autowired
    private MessageProcessor messageProcessor;

    @RabbitListener(
        queues = "my.queue",
        concurrency = "5-10"
    )
    public void receiveMessage(Message message,
                             Channel channel,
                             @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
        long startTime = System.currentTimeMillis();
        String messageId = message.getMessageProperties().getMessageId();

        try {
            // 1. 解析消息
            String msgBody = new String(message.getBody());
            BusinessMessage businessMessage = JSON.parseObject(msgBody, BusinessMessage.class);

            // 2. 幂等性检查
            if (messageProcessor.isProcessed(messageId)) {
                log.info("消息已处理，跳过: {}", messageId);
                channel.basicAck(deliveryTag, false);
                return;
            }

            // 3. 获取重试次数
            int retryCount = getRetryCount(message);

            // 4. 检查重试次数
            if (retryCount >= MAX_RETRY) {
                log.warn("达到最大重试次数，发送到死信队列: {}", messageId);
                messageProcessor.sendToDeadLetter(message);
                channel.basicAck(deliveryTag, false);
                return;
            }

            // 5. 处理业务
            messageProcessor.process(businessMessage);

            // 6. 标记已处理
            messageProcessor.markAsProcessed(messageId);

            // 7. ACK
            channel.basicAck(deliveryTag, false);

            // 8. 记录耗时
            long latency = System.currentTimeMillis() - startTime;
            if (latency > SLOW_THRESHOLD) {
                log.warn("慢消息 - messageId: {}, 耗时: {}ms", messageId, latency);
            }

        } catch (BusinessException e) {
            // 业务异常，直接 ACK
            log.warn("业务异常: {}", e.getMessage());
            try {
                channel.basicAck(deliveryTag, false);
            } catch (IOException ex) {
                log.error("ACK 失败", ex);
            }

        } catch (Exception e) {
            // 系统异常，NACK 重试
            log.error("系统异常，准备重试", e);

            // 增加重试次数
            incrementRetryCount(message);

            try {
                channel.basicNack(deliveryTag, false, true);
            } catch (IOException ex) {
                log.error("NACK 失败", ex);
            }
        }
    }

    private int getRetryCount(Message message) {
        Object retryCount = message.getMessageProperties().getHeaders().get("x-retry-count");
        return retryCount != null ? (int) retryCount : 0;
    }

    private void incrementRetryCount(Message message) {
        int retryCount = getRetryCount(message);
        message.getMessageProperties().getHeaders().put("x-retry-count", retryCount + 1);
    }
}
```
