# RabbitMQ 核心概念

## 一、Exchange（交换机）

Exchange 是消息到达 RabbitMQ 的第一站，负责接收生产者发送的消息，并根据路由规则将消息分发到绑定到它的 Queue。

### 1.1 Exchange 类型

RabbitMQ 提供了四种内置的 Exchange 类型：

| 类型 | 说明 | 路由规则 | 适用场景 |
|------|------|----------|----------|
| **Direct** | 直连交换机 | 完全匹配 Routing Key | 点对点消息路由 |
| **Fanout** | 扇形交换机 | 忽略 Routing Key，广播到所有绑定的 Queue | 消息广播 |
| **Topic** | 主题交换机 | 模糊匹配 Routing Key（支持 * 和 #）| 灵活的消息路由 |
| **Headers** | 头交换机 | 根据消息头属性匹配 | 复杂的路由规则 |

### 1.2 Direct Exchange

**工作原理**：

消息的 Routing Key 与 Queue 的 Binding Key 完全一致时，消息才会被路由到该 Queue。

```
Exchange (Direct)
    ├── Queue A (binding key: "error")  ← 消息 (routing key: "error")
    ├── Queue B (binding key: "info")   ← 消息 (routing key: "info")
    └── Queue C (binding key: "error")  ← 消息 (routing key: "error")
```

**代码示例**：
```java
// 声明 Direct Exchange
@Bean
public DirectExchange directExchange() {
    return new DirectExchange("direct.exchange");
}

// 声明 Queue
@Bean
public Queue errorQueue() {
    return QueueBuilder.durable("error.queue").build();
}

// 绑定
@Bean
public Binding bindingError() {
    return BindingBuilder.bind(errorQueue())
        .to(directExchange())
        .with("error");
}

// 发送消息
rabbitTemplate.convertAndSend("direct.exchange", "error", "Error message");
// 只有绑定 key 为 "error" 的 Queue 才能收到消息
```

### 1.3 Fanout Exchange

**工作原理**：

忽略 Routing Key，将消息广播到所有绑定的 Queue。

```
Exchange (Fanout)
    ├── Queue A  ← 所有消息
    ├── Queue B  ← 所有消息
    └── Queue C  ← 所有消息
```

**代码示例**：
```java
// 声明 Fanout Exchange
@Bean
public FanoutExchange fanoutExchange() {
    return new FanoutExchange("fanout.exchange");
}

// 绑定（不需要 routing key）
@Bean
public Binding bindingA() {
    return BindingBuilder.bind(queueA()).to(fanoutExchange());
}

// 发送消息
rabbitTemplate.convertAndSend("fanout.exchange", "", "Broadcast message");
// 所有绑定的 Queue 都能收到消息
```

### 1.4 Topic Exchange

**工作原理**：

使用通配符匹配 Routing Key：
- `*`：匹配一个单词
- `#`：匹配零个或多个单词

```
Exchange (Topic)
    ├── Queue A (binding key: "*.error")     ← 匹配: user.error, system.error
    ├── Queue B (binding key: "user.*")      ← 匹配: user.info, user.error
    └── Queue C (binding key: "#")           ← 匹配: 所有消息
```

**代码示例**：
```java
// 声明 Topic Exchange
@Bean
public TopicExchange topicExchange() {
    return new TopicExchange("topic.exchange");
}

// 绑定
@Bean
public Binding bindingUserError() {
    return BindingBuilder.bind(userErrorQueue())
        .to(topicExchange())
        .with("*.error");
}

// 发送消息
rabbitTemplate.convertAndSend("topic.exchange", "user.error", "User error");
// 匹配: *.error 和 # 的 Queue 会收到消息
```

### 1.5 Headers Exchange

**工作原理**：

根据消息头（headers）中的键值对进行匹配。

**代码示例**：
```java
// 声明 Headers Exchange
@Bean
public HeadersExchange headersExchange() {
    return new HeadersExchange("headers.exchange");
}

// 绑定（设置匹配规则）
@Bean
public Binding bindingHeaders() {
    Map<String, Object> headers = new HashMap<>();
    headers.put("type", "email");
    headers.put("format", "html");

    return BindingBuilder.bind(emailQueue())
        .to(headersExchange())
        .whereAll(headers).match(); // 所有键值对都匹配
}

// 发送消息
Message message = MessageBuilder.withBody("Hello".getBytes())
    .setHeader("type", "email")
    .setHeader("format", "html")
    .build();

rabbitTemplate.send("headers.exchange", "", message);
```

### 1.6 Default Exchange

每个 RabbitMQ 节点都有一个默认的 Exchange（名称为空字符串 ""）。

**特点**：
- 类型为 Direct
- 每个 Queue 都会自动绑定到 Default Exchange
- Binding Key 就是 Queue 名称

**使用示例**：
```java
// 使用 Default Exchange（不需要声明）
// Queue: "test.queue"

// 方式一：直接指定 queue 名称
rabbitTemplate.convertAndSend("", "test.queue", "message");

// 方式二：等同于上面的写法
rabbitTemplate.convertAndSend("test.queue", "message");
```

## 二、Queue（队列）

Queue 是消息的最终存储位置，消费者从 Queue 获取消息。

### 2.1 Queue 属性

| 属性 | 说明 | 默认值 |
|------|------|--------|
| **name** | 队列名称 | - |
| **durable** | 是否持久化 | false |
| **exclusive** | 是否排他（仅限当前连接）| false |
| **autoDelete** | 是否自动删除（没有消费者时）| false |
| **arguments** | 其他参数 | - |

### 2.2 队列参数

#### TTL（Time To Live）

消息存活时间。

```java
// 方式一：队列级别 TTL（所有消息）
Map<String, Object> args = new HashMap<>();
args.put("x-message-ttl", 60000); // 60 秒
return QueueBuilder.durable("my.queue").withArguments(args).build();

// 方式二：消息级别 TTL
Message message = MessageBuilder
    .withBody("Hello".getBytes())
    .setExpiration("60000") // 60 秒
    .build();
rabbitTemplate.send("", "my.queue", message);
```

#### 队列 TTL

队列存活时间（没有消费者且没有消息时自动删除）。

```java
Map<String, Object> args = new HashMap<>();
args.put("x-expires", 3600000); // 1 小时
return QueueBuilder.durable("temp.queue").withArguments(args).build();
```

#### 最大长度

限制队列中消息数量。

```java
Map<String, Object> args = new HashMap<>();
args.put("x-max-length", 10000); // 最多 10000 条消息
return QueueBuilder.durable("limit.queue").withArguments(args).build();
```

#### 最大长度字节

限制队列中消息总大小。

```java
Map<String, Object> args = new HashMap<>();
args.put("x-max-length-bytes", 10485760); // 最多 10MB
return QueueBuilder.durable("size.queue").withArguments(args).build();
```

#### 溢出行为

队列满时的处理策略。

```java
Map<String, Object> args = new HashMap<>();
args.put("x-max-length", 10000);
args.put("x-overflow", "drop-head"); // drop-head（删除最旧）、reject-publish（拒绝新消息）
return QueueBuilder.durable("overflow.queue").withArguments(args).build();
```

#### 死信交换机

消息被拒绝或过期时，转发到死信交换机。

```java
Map<String, Object> args = new HashMap<>();
args.put("x-dead-letter-exchange", "dlx.exchange");
args.put("x-dead-letter-routing-key", "dlx.routing.key");
return QueueBuilder.durable("normal.queue").withArguments(args).build();
```

### 2.3 队列类型

#### Classic Queue（经典队列）

传统队列类型，默认使用。

#### Quorum Queue（仲裁队列）

RabbitMQ 3.8+ 引入的队列类型，基于 Raft 协议。

```java
Map<String, Object> args = new HashMap<>();
args.put("x-queue-type", "quorum");
return QueueBuilder.durable("quorum.queue").withArguments(args).build();
```

**优势**：
- 数据一致性更强
- 更好的故障恢复
- 支持消息去重

#### Stream Queue（流队列）

RabbitMQ 3.9+ 引入的队列类型，类似 Kafka。

```java
Map<String, Object> args = new HashMap<>();
args.put("x-queue-type", "stream");
return QueueBuilder.durable("stream.queue").withArguments(args).build();
```

**优势**：
- 基于日志追加
- 支持消息回溯
- 高吞吐量

## 三、Binding（绑定）

Binding 定义了 Exchange 和 Queue 之间的关系，包含 Binding Key（路由键）。

### 3.1 Binding 结构

```
Exchange --[Binding: Binding Key]--> Queue
```

### 3.2 创建 Binding

```java
// Direct Binding
BindingBuilder.bind(queue).to(directExchange()).with("routing.key");

// Fanout Binding
BindingBuilder.bind(queue).to(fanoutExchange());

// Topic Binding
BindingBuilder.bind(queue).to(topicExchange()).with("*.error");

// Headers Binding
BindingBuilder.bind(queue).to(headersExchange()).whereAll(headers).match();
```

### 3.3 多个 Binding

一个 Queue 可以绑定多个 Exchange：

```
Exchange A --[binding]--> Queue
Exchange B --[binding]--> Queue
Exchange C --[binding]--> Queue
```

一个 Exchange 可以绑定多个 Queue：

```
Exchange --[binding]--> Queue A
Exchange --[binding]--> Queue B
Exchange --[binding]--> Queue C
```

## 四、Routing Key（路由键）

Routing Key 是生产者发送消息时指定的路由规则。

### 4.1 命名规范

通常使用点号分隔的单词：

```
user.registered
user.deleted
order.created
order.paid
error.user
error.system
```

### 4.2 使用场景

#### Direct Exchange

Routing Key 必须与 Binding Key 完全一致。

```java
// 发送
rabbitTemplate.convertAndSend("direct.exchange", "user.error", "message");

// 只有绑定 key 为 "user.error" 的 Queue 才能收到
```

#### Topic Exchange

Routing Key 支持通配符匹配。

```java
// 发送
rabbitTemplate.convertAndSend("topic.exchange", "user.error", "message");

// 绑定 "user.error"、"*.error"、"user.*"、"#" 的 Queue 都能收到
```

## 五、Virtual Host（虚拟主机）

Virtual Host 是 RabbitMQ 的逻辑隔离单位，类似于数据库中的数据库。

### 5.1 作用

- 资源隔离：不同 vhost 中的 Exchange、Queue 互不干扰
- 权限控制：可以为不同用户分配不同 vhost 的权限
- 多租户：一个 RabbitMQ 服务器可以服务多个应用

### 5.2 默认 vhost

RabbitMQ 默认创建一个名为 `/` 的 vhost。

### 5.3 创建 vhost

```bash
# 使用 rabbitmqctl
rabbitmqctl add_vhost /my_app

# 使用 Web 管理界面
Admin → Virtual Hosts → Add a new virtual host
```

### 5.4 配置用户权限

```bash
# 授予用户权限
rabbitmqctl set_permissions -p /my_app guest ".*" ".*" ".*"

# 权限格式
# configure: 配置权限（正则表达式）
# write: 写权限
# read: 读权限
```

### 5.5 连接指定 vhost

```java
// Spring Boot 配置
spring:
  rabbitmq:
    host: localhost
    port: 5672
    virtual-host: /my_app

# 或使用 ConnectionFactory
ConnectionFactory factory = new ConnectionFactory();
factory.setVirtualHost("/my_app");
```

## 六、Connection 和 Channel

### 6.1 Connection（连接）

Connection 是客户端与 RabbitMQ 之间的 TCP 连接。

**特点**：
- 建立 Connection 需要三次握手，开销较大
- 一个 Connection 可以创建多个 Channel
- 建议复用 Connection

**代码示例**：
```java
ConnectionFactory factory = new ConnectionFactory();
factory.setHost("localhost");
factory.setUsername("guest");
factory.setPassword("guest");

// 建立连接
Connection connection = factory.newConnection();

// 使用完成后关闭
connection.close();
```

### 6.2 Channel（通道）

Channel 是在 Connection 内部创建的逻辑连接，用于多路复用。

**特点**：
- 建立 Channel 开销小
- 推荐 AMQP 操作使用 Channel
- 一个 Connection 可以创建多个 Channel

**代码示例**：
```java
Connection connection = factory.newConnection();

// 创建多个 Channel
Channel channel1 = connection.createChannel();
Channel channel2 = connection.createChannel();
Channel channel3 = connection.createChannel();

// 使用 Channel 发送消息
channel1.basicPublish("", "queue1", null, "message1".getBytes());
channel2.basicPublish("", "queue2", null, "message2".getBytes());

// 关闭 Channel
channel1.close();
channel2.close();
channel3.close();

// 关闭 Connection
connection.close();
```

### 6.3 Channel 线程安全

Channel 不是线程安全的，每个线程应该使用自己的 Channel。

```java
// 正确做法：线程池 + 每个线程一个 Channel
ExecutorService executor = Executors.newFixedThreadPool(10);

for (int i = 0; i < 10; i++) {
    executor.submit(() -> {
        Connection connection = factory.newConnection();
        Channel channel = connection.createChannel();
        // 使用 channel
        channel.close();
        connection.close();
    });
}
```

## 七、Consumer（消费者）

Consumer 是从 Queue 获取消息并处理的客户端。

### 7.1 消费方式

#### 方式一：自动 ACK

```java
// 手动创建 Consumer
channel.basicConsume("my.queue", true, new DefaultConsumer(channel) {
    @Override
    public void handleDelivery(String consumerTag,
                               Envelope envelope,
                               AMQP.BasicProperties properties,
                               byte[] body) {
        String message = new String(body);
        System.out.println("Received: " + message);
        // 自动 ACK，无需手动调用
    }
});

// 或使用 Spring AMQP
@RabbitListener(queues = "my.queue")
public void handleMessage(String message) {
    System.out.println("Received: " + message);
    // 自动 ACK
}
```

#### 方式二：手动 ACK

```java
// Spring AMQP 手动 ACK
@RabbitListener(queues = "my.queue")
public void handleMessage(String message,
                         Channel channel,
                         @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
    try {
        // 处理消息
        processMessage(message);

        // 手动 ACK
        channel.basicAck(deliveryTag, false);
    } catch (Exception e) {
        // 拒绝消息，重新入队
        channel.basicNack(deliveryTag, false, true);
    }
}
```

### 7.2 Prefetch Count（预取数量）

控制每个消费者未 ACK 的消息数量。

```java
// 设置 prefetch count
channel.basicQos(10); // 每个消费者最多接收 10 条未 ACK 的消息

// 或使用 Spring 配置
spring:
  rabbitmq:
    listener:
      simple:
        prefetch: 10
```

### 7.3 消费者确认模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **自动确认** | 消息投递后立即确认 | 允许消息丢失 |
| **手动确认** | 消息处理完成后确认 | 需要保证消息可靠处理 |

## 八、消息属性

### 8.1 基本属性

```java
Message message = MessageBuilder
    .withBody("Hello".getBytes())
    .setContentType(MessageProperties.CONTENT_TYPE_TEXT_PLAIN)
    .setContentEncoding("UTF-8")
    .setDeliveryMode(MessageDeliveryMode.PERSISTENT) // 持久化
    .setExpiration("60000") // TTL
    .setPriority(5) // 优先级 0-9
    .setHeader("custom-header", "value") // 自定义头
    .build();
```

### 8.2 常用属性

| 属性 | 说明 |
|------|------|
| **contentType** | 内容类型（如 text/plain、application/json）|
| **contentEncoding** | 内容编码（如 UTF-8、GBK）|
| **deliveryMode** | 投递模式（1=非持久化，2=持久化）|
| **priority** | 优先级（0-9，数字越大优先级越高）|
| **correlationId** | 关联 ID（用于 RPC 模式）|
| **replyTo** | 回复队列名（用于 RPC 模式）|
| **expiration** | 过期时间 |
| **timestamp** | 时间戳 |
| **type** | 消息类型 |
| **userId** | 用户 ID |

### 8.3 自定义属性

```java
Message message = MessageBuilder
    .withBody("Hello".getBytes())
    .setHeader("business-id", "123456")
    .setHeader("retry-count", 0)
    .setHeader("source", "mobile-app")
    .build();

// 消费时获取
@RabbitListener(queues = "my.queue")
public void handleMessage(String message,
                         @Header("business-id") String businessId,
                         @Header("retry-count") int retryCount) {
    System.out.println("Business ID: " + businessId);
    System.out.println("Retry Count: " + retryCount);
}
```
