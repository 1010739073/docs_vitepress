# RabbitMQ 可靠性保证

## 一、消息可靠性保障体系

RabbitMQ 的消息可靠性保障从三个层面实现：

```
┌─────────────────────────────────────────────────────────────┐
│                     消息可靠性保障体系                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐            │
│  │发送阶段  │ -> │存储阶段  │ -> │消费阶段  │            │
│  │          │    │          │    │          │            │
│  │- Confirm│    │- 持久化  │    │- ACK     │            │
│  │- Return │    │- 副本    │    │- 重试    │            │
│  │- 事务   │    │- 集群    │    │- 死信    │            │
│  └──────────┘    └──────────┘    └──────────┘            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、发送阶段可靠性

### 2.1 发送确认（Confirm）

确保消息成功到达 Exchange。

```yaml
spring:
  rabbitmq:
    publisher-confirm-type: correlated
```

```java
@Component
public class ReliableProducer {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    public void sendMessageWithConfirm(String exchange, String routingKey, Object message) {
        String messageId = UUID.randomUUID().toString();
        CorrelationData correlationData = new CorrelationData(messageId);

        // 设置回调
        correlationData.getFuture().thenConfirm(confirm -> {
            if (confirm.isAck()) {
                log.info("消息发送成功, messageId: {}", messageId);
                // 更新消息状态为已发送
                updateMessageStatus(messageId, "SENT");
            } else {
                log.error("消息发送失败, messageId: {}, 原因: {}", messageId, confirm.getReason());
                // 处理发送失败，如重试或告警
                handleSendFailure(messageId, exchange, routingKey, message);
            }
        });

        rabbitTemplate.convertAndSend(exchange, routingKey, message, correlationData);
    }
}
```

### 2.2 路由确认（Return）

确保消息成功路由到 Queue。

```yaml
spring:
  rabbitmq:
    publisher-returns: true
    template:
      mandatory: true  # 消息无法路由时返回
```

```java
@Configuration
public class RabbitMQConfig {

    @Bean
    public RabbitTemplate rabbitTemplate(ConnectionFactory connectionFactory) {
        RabbitTemplate template = new RabbitTemplate(connectionFactory);

        // 设置 Return 回调
        template.setReturnsCallback(returnedMessage -> {
            log.error("消息未被路由 - exchange: {}, routingKey: {}, message: {}",
                returnedMessage.getExchange(),
                returnedMessage.getRoutingKey(),
                new String(returnedMessage.getMessage().getBody()));

            // 处理未路由的消息
            handleUnroutedMessage(returnedMessage);
        });

        return template;
    }
}
```

### 2.3 发送重试

```java
@Service
public class RetryableProducer {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    @Retryable(
        value = {AmqpException.class},
        maxAttempts = 3,
        backoff = @Backoff(delay = 1000, multiplier = 2)
    )
    public void sendMessageWithRetry(String exchange, String routingKey, Object message) {
        rabbitTemplate.convertAndSend(exchange, routingKey, message);
    }

    @Recover
    public void recover(AmqpException e, String exchange, String routingKey, Object message) {
        log.error("重试失败，发送到备份队列", e);

        // 发送到备份队列
        rabbitTemplate.convertAndSend("backup.exchange", "backup.key",
            new BackupMessage(exchange, routingKey, message));
    }
}
```

## 三、存储阶段可靠性

### 3.1 队列持久化

确保队列在 RabbitMQ 重启后仍然存在。

```java
@Configuration
public class DurableQueueConfig {

    @Bean
    public Queue durableQueue() {
        return QueueBuilder
            .durable("my.queue")  // 持久化队列
            .build();
    }
}
```

### 3.2 消息持久化

确保消息在 RabbitMQ 重启后仍然存在。

```java
// 方式一：全局配置
spring:
  rabbitmq:
    template:
      default-receive-queue: durable
    publisher-confirm-type: correlated

// 方式二：消息级别
MessageProperties properties = new MessageProperties();
properties.setDeliveryMode(MessageDeliveryMode.PERSISTENT);  // 持久化

Message message = new Message("Hello".getBytes(), properties);
rabbitTemplate.send("", "my.queue", message);

// 方式三：使用 MessageBuilder
Message message = MessageBuilder
    .withBody("Hello".getBytes())
    .setDeliveryMode(MessageDeliveryMode.PERSISTENT)
    .build();
```

### 3.3 交换机持久化

确保交换机在 RabbitMQ 重启后仍然存在。

```java
@Configuration
public class DurableExchangeConfig {

    @Bean
    public DirectExchange durableExchange() {
        return new DirectExchange("my.exchange", true, false);
        // 参数说明：
        // name: 交换机名称
        // durable: 是否持久化
        // autoDelete: 没有队列绑定时是否自动删除
    }
}
```

### 3.4 镜像队列

确保队列数据在多个节点上都有副本。

```bash
# 设置镜像策略
rabbitmqctl set_policy ha-all "^ha\." \
  '{"ha-mode":"all","ha-sync-mode":"automatic"}'
```

```java
// 创建镜像队列（队列名以 ha. 开头）
@Bean
public Queue mirroredQueue() {
    return QueueBuilder
        .durable("ha.my.queue")  // 以 ha. 开头，匹配策略
        .build();
}
```

### 3.5 仲裁队列（Quorum Queue）

RabbitMQ 3.8+ 推荐的队列类型，数据一致性更强。

```java
@Bean
public Queue quorumQueue() {
    return QueueBuilder
        .durable("quorum.my.queue")
        .quorum()  // 设置为仲裁队列
        .build();
}
```

**仲裁队列特性**：
- 使用 Raft 共识算法
- 数据一致性更强
- 更好的故障恢复
- 支持消息去重

## 四、消费阶段可靠性

### 4.1 手动确认

确保消息被正确处理后才确认。

```java
@RabbitListener(queues = "my.queue")
public void receiveMessage(Message message,
                         Channel channel,
                         @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
    try {
        // 处理消息
        processMessage(message);

        // 手动 ACK
        channel.basicAck(deliveryTag, false);

    } catch (Exception e) {
        log.error("处理失败", e);

        try {
            // NACK，重新入队
            channel.basicNack(deliveryTag, false, true);
        } catch (IOException ex) {
            log.error("NACK 失败", ex);
        }
    }
}
```

### 4.2 消费重试

```yaml
spring:
  rabbitmq:
    listener:
      simple:
        retry:
          enabled: true
          max-attempts: 3
          initial-interval: 1000
          multiplier: 2.0
          max-interval: 10000
```

### 4.3 死信队列

处理无法正常消费的消息。

#### 配置死信队列

```java
@Configuration
public class DeadLetterConfig {

    @Bean
    public Queue normalQueue() {
        Map<String, Object> args = new HashMap<>();
        args.put("x-dead-letter-exchange", "dlx.exchange");
        args.put("x-dead-letter-routing-key", "dlx.routing.key");
        return QueueBuilder.durable("normal.queue").withArguments(args).build();
    }

    @Bean
    public DirectExchange dlxExchange() {
        return new DirectExchange("dlx.exchange");
    }

    @Bean
    public Queue dlxQueue() {
        return QueueBuilder.durable("dlx.queue").build();
    }

    @Bean
    public Binding dlxBinding() {
        return BindingBuilder.bind(dlxQueue())
            .to(dlxExchange())
            .with("dlx.routing.key");
    }
}
```

#### 消费死信消息

```java
@RabbitListener(queues = "dlx.queue")
public void handleDeadLetter(Message message) {
    // 记录死信消息
    String messageId = message.getMessageProperties().getMessageId();
    String reason = (String) message.getMessageProperties().getHeaders()
        .get("x-death-reason");

    log.warn("死信消息 - messageId: {}, reason: {}", messageId, reason);

    // 发送告警
    sendAlert(messageId, reason);

    // 或进行人工处理
    saveToManualProcess(message);
}
```

### 4.4 幂等性保证

确保重复消费不会导致业务问题。

```java
@Component
public class IdempotentConsumer {

    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    @RabbitListener(queues = "my.queue")
    public void receiveMessage(String message) {
        String messageId = getMessageId(message);

        // 检查是否已处理
        if (redisTemplate.hasKey("processed:" + messageId)) {
            log.info("消息已处理，跳过: {}", messageId);
            return;
        }

        try {
            // 处理消息
            processMessage(message);

            // 标记已处理
            redisTemplate.opsForValue().set("processed:" + messageId, "1", 24, TimeUnit.HOURS);

        } catch (Exception e) {
            log.error("处理失败", e);
            throw e;
        }
    }

    private String getMessageId(String message) {
        // 从消息中提取 ID
        return JSON.parseObject(message).getString("id");
    }
}
```

## 五、集群可靠性

### 5.1 普通集群

数据只在一个节点，其他节点只有元数据。

```
Node 1          Node 2          Node 3
  │               │               │
  ├─ Queue 1      ├─ Queue 1(元数据) ├─ Queue 1(元数据)
  │               │               │
  └─ Queue 2      └─ Queue 2(元数据) └─ Queue 2(元数据)

Node 1 故障 → Queue 1 和 Queue 2 不可用
```

### 5.2 镜像集群

数据在多个节点有副本。

```
Node 1          Node 2          Node 3
  │               │               │
  ├─ Queue 1      ├─ Queue 1(镜像) ├─ Queue 1(镜像)
  │               │               │
  └─ Queue 2      └─ Queue 2(镜像) └─ Queue 2(镜像)

Node 1 故障 → Node 2 或 Node 3 自动接管
```

配置镜像集群：
```bash
# 设置策略
rabbitmqctl set_policy ha-all "^ha\." \
  '{"ha-mode":"all","ha-sync-mode":"automatic"}'
```

### 5.3 仲裁队列集群

使用 Raft 协议保证数据一致性。

```java
@Bean
public Queue quorumQueue() {
    return QueueBuilder
        .durable("quorum.queue")
        .quorum()
        .build();
}
```

## 六、可靠性最佳实践

### 6.1 完整的可靠性配置

```yaml
spring:
  rabbitmq:
    # 连接配置
    host: localhost
    port: 5672
    username: guest
    password: guest
    virtual-host: /

    # 发送确认
    publisher-confirm-type: correlated
    publisher-returns: true
    template:
      mandatory: true

    # 消费配置
    listener:
      simple:
        acknowledge-mode: manual
        prefetch: 1
        retry:
          enabled: true
          max-attempts: 3
          initial-interval: 1000
          multiplier: 2.0
        concurrency: 5
        max-concurrency: 10
```

### 6.2 完整的可靠性代码

```java
@Service
@Slf4j
public class ReliableMessagingService {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    /**
     * 可靠发送
     */
    public void reliableSend(String exchange, String routingKey, Object message) {
        String messageId = UUID.randomUUID().toString();
        CorrelationData correlationData = new CorrelationData(messageId);

        // 保存消息到本地（用于重试）
        saveToLocal(messageId, exchange, routingKey, message);

        // 设置回调
        correlationData.getFuture().thenConfirm(confirm -> {
            if (confirm.isAck()) {
                log.info("消息发送成功: {}", messageId);
                updateLocalStatus(messageId, "SENT");
            } else {
                log.error("消息发送失败: {}", messageId);
                updateLocalStatus(messageId, "FAILED");
            }
        });

        // 发送消息
        Message msg = MessageBuilder
            .withBody(JSON.toJSONBytes(message))
            .setContentType(MessageProperties.CONTENT_TYPE_JSON)
            .setMessageId(messageId)
            .setDeliveryMode(MessageDeliveryMode.PERSISTENT)
            .build();

        rabbitTemplate.convertAndSend(exchange, routingKey, msg, correlationData);
    }

    /**
     * 可靠消费
     */
    @RabbitListener(queues = "my.queue")
    public void reliableConsume(Message message,
                              Channel channel,
                              @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
        String messageId = message.getMessageProperties().getMessageId();

        try {
            // 幂等性检查
            if (isProcessed(messageId)) {
                log.info("消息已处理: {}", messageId);
                channel.basicAck(deliveryTag, false);
                return;
            }

            // 处理消息
            processMessage(message);

            // 标记已处理
            markAsProcessed(messageId);

            // ACK
            channel.basicAck(deliveryTag, false);

        } catch (BusinessException e) {
            // 业务异常，直接 ACK
            log.warn("业务异常: {}", e.getMessage());
            channel.basicAck(deliveryTag, false);

        } catch (Exception e) {
            // 系统异常，重试
            log.error("系统异常", e);
            channel.basicNack(deliveryTag, false, true);
        }
    }

    /**
     * 定时重试失败消息
     */
    @Scheduled(fixedDelay = 60000)
    public void retryFailedMessages() {
        List<MessageRecord> failedMessages = getFailedMessages();

        for (MessageRecord record : failedMessages) {
            if (record.getRetryCount() >= 3) {
                // 达到最大重试次数，告警
                alert(record);
                continue;
            }

            try {
                // 重新发送
                reliableSend(record.getExchange(), record.getRoutingKey(), record.getMessage());

                // 更新重试次数
                incrementRetryCount(record.getId());

            } catch (Exception e) {
                log.error("重试失败: {}", record.getId(), e);
            }
        }
    }

    private void saveToLocal(String messageId, String exchange, String routingKey, Object message) {
        // 保存到数据库
    }

    private void updateLocalStatus(String messageId, String status) {
        // 更新状态
    }

    private boolean isProcessed(String messageId) {
        return redisTemplate.hasKey("processed:" + messageId);
    }

    private void markAsProcessed(String messageId) {
        redisTemplate.opsForValue().set("processed:" + messageId, "1", 24, TimeUnit.HOURS);
    }

    private List<MessageRecord> getFailedMessages() {
        // 查询失败消息
        return Collections.emptyList();
    }

    private void incrementRetryCount(Long id) {
        // 更新重试次数
    }

    private void alert(MessageRecord record) {
        // 发送告警
    }
}
```

|--------|------|

| ✅ 开启 Return | 消息路由确认 |
| ✅ 队列持久化 | 队列重启后存在 |
| ✅ 消息持久化 | 消息重启后存在 |
