# RabbitMQ 大厂面试题

## 一、架构设计

### Q1: 如何设计一个高可用的 RabbitMQ 集群？

**答案：**

**架构设计：**

```
                       ┌─────────────┐
                       │  负载均衡    │
                       │ (HAProxy/Nginx)│
                       └──────┬──────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        │                     │                     │
   ┌────▼────┐          ┌────▼────┐          ┌────▼────┐
   │ Node 1  │          │ Node 2  │          │ Node 3  │
   │ (Master)│          │ (Slave) │          │ (Slave) │
   └────┬────┘          └────┬────┘          └────┬────┘
        │                     │                     │
        └─────────────────────┴─────────────────────┘
                              │
                    ┌─────────▼─────────┐
                    │   磁盘阵列/备份    │
                    └───────────────────┘
```

**关键点：**

1. **节点数量**：奇数个节点，至少 3 个
2. **节点分布**：不同物理机/机架
3. **镜像队列**：所有队列都是镜像队列
4. **负载均衡**：使用 HAProxy 或 Nginx
5. **监控告警**：完整的监控体系
6. **数据备份**：定期备份数据
7. **故障演练**：定期进行故障切换演练

**具体配置：**

```bash
# 1. 安装 RabbitMQ（3 个节点）
# Node 1, 2, 3

# 2. 同步 Erlang Cookie
# 确保 /var/lib/rabbitmq/.erlang.cookie 一致

# 3. 组建集群
# Node 2
rabbitmqctl stop_app
rabbitmqctl reset
rabbitmqctl join_cluster rabbit@node1
rabbitmqctl start_app

# Node 3
rabbitmqctl stop_app
rabbitmqctl reset
rabbitmqctl join_cluster rabbit@node1
rabbitmqctl start_app

# 4. 配置镜像策略
rabbitmqctl set_policy ha-all "^" \
  '{"ha-mode":"all","ha-sync-mode":"automatic"}'

# 5. 配置 HAProxy
# /etc/haproxy/haproxy.cfg
frontend rabbitmq_frontend
    bind *:5672
    default_backend rabbitmq_backend

backend rabbitmq_backend
    balance roundrobin
    server node1 192.168.1.101:5672 check
    server node2 192.168.1.102:5672 check
    server node3 192.168.1.103:5672 check
```

```java
// 批量消费
@RabbitListener(queues = "high.perf.queue")
public void batchConsume(List<Message> messages, Channel channel) {
    List<Long> deliveryTags = new ArrayList<>();

    try {
        for (Message message : messages) {
            deliveryTags.add(message.getMessageProperties().getDeliveryTag());
            processMessage(message);
        }

        // 批量 ACK
        if (!deliveryTags.isEmpty()) {
            long lastTag = deliveryTags.get(deliveryTags.size() - 1);
            channel.basicAck(lastTag, true);
        }

    } catch (Exception e) {
        // 批量 NACK
        if (!deliveryTags.isEmpty()) {
            long lastTag = deliveryTags.get(deliveryTags.size() - 1);
            channel.basicNack(lastTag, true, true);
        }
    }
}
```

## 二、底层原理

### Q4: RabbitMQ 的存储机制是什么？

**答案：**

RabbitMQ 使用 Erlang 的 Mnesia 数据库和文件系统存储消息。

**存储层次：**

```
┌─────────────────────────────────────┐
│           应用层                    │
│  Queue、Exchange、Binding 等        │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│           Mnesia 数据库             │
│  元数据（队列、交换机、绑定等）     │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│           文件系统                  │
│  消息数据（.msg 文件）             │
└─────────────────────────────────────┘
```

**消息存储格式：**

```
/var/lib/rabbitmq/mnesia/rabbit@hostname/msg_store_persistent/

1.msg
2.msg
3.msg
...
```

**存储优化：**

| 策略 | 说明 |
|------|------|
| **惰性队列** | 消息存入磁盘，不占用内存 |
| **流（Stream）** | 基于日志追加，适合大量消息 |
| **批量写入** | 减少磁盘 I/O |

**配置惰性队列：**

```java
Map<String, Object> args = new HashMap<>();
args.put("x-queue-mode", "lazy");
QueueBuilder.durable("lazy.queue").withArguments(args).build();
```

### Q5: RabbitMQ 的消息流转过程是什么？

**答案：**

```
Producer → [1.创建消息] → [2.设置属性] → [3.发送到Exchange]
                                              ↓
                           Exchange → [4.路由匹配] → [5.绑定查找]
                                              ↓
                           Queue → [6.写入消息] → [7.返回ACK]
                                              ↓
                           Consumer → [8.投递消息] → [9.处理消息]
                                              ↓
                                     → [10.发送ACK/NACK]
```

**详细流程：**

**1. Producer 发送消息**

```java
rabbitTemplate.convertAndSend("exchange", "routing.key", message);
```

**2. RabbitMQ 接收**

```erlang
%% 伪代码
handle_message(Exchange, RoutingKey, Message) ->
    case lookup_bindings(Exchange, RoutingKey) of
        [] -> handle_unrouted(Message);
        Queues -> deliver_to_queues(Queues, Message)
    end.
```

**3. 路由到 Queue**

```erlang
%% Direct Exchange
route_direct(Exchange, RoutingKey, Message) ->
    Queues = get_queues_bound(Exchange, RoutingKey),
    deliver(Queues, Message).

%% Topic Exchange
route_topic(Exchange, RoutingKey, Message) ->
    Bindings = get_bindings(Exchange),
    Queues = [Q || {_, RK, Q} <- Bindings, match(RoutingKey, RK)],
    deliver(Queues, Message).

match(RoutingKey, BindingKey) ->
    Wildcard = string:split(BindingKey, "."),
    Parts = string:split(RoutingKey, "."),
    match_wildcard(Wildcard, Parts).
```

**4. 消费者接收**

```java
@RabbitListener(queues = "my.queue")
public void receiveMessage(String message) {
    processMessage(message);
}
```

### Q6: RabbitMQ 如何实现消息确认？

**答案：**

**发送确认（Confirm）流程：**

```
Producer                Exchange
   │                         │
   │── 1. 发送消息 ────────→ │
   │                         │
   │← 2. Ack/Nack ──────────│
   │                         │
   (处理确认结果)            (处理消息)
```

**实现原理：**

```erlang
%% RabbitMQ 伪代码
handle_publish(Producer, Message) ->
    %% 写入消息
    case store_message(Message) of
        ok ->
            %% 发送 Ack
            send(Producer, {ack, CorrelationId});
        error ->
            %% 发送 Nack
            send(Producer, {nack, CorrelationId, Reason})
    end.
```

**消费确认（ACK）流程：**

```
Queue          Consumer
  │                │
  │── 1. 投递 ──→  │
  │                │
  │← 2. Ack/Nack │
  │                │
 (删除/重新入队)   (处理完成)
```

**实现原理：**

```erlang
%% RabbitMQ 伪代码
handle_delivery(Consumer, Message) ->
    %% 标记消息为 Unacked
    set_message_status(Message, unacked),

    %% 投递给消费者
    send(Consumer, {deliver, Message}),

    %% 等待 Ack/Nack
    receive
        {ack, DeliveryTag} ->
            %% 删除消息
            delete_message(DeliveryTag);
        {nack, DeliveryTag, Requeue} ->
            if Requeue ->
                %% 重新入队
                requeue_message(DeliveryTag);
            true ->
                %% 删除消息
                delete_message(DeliveryTag)
            end
    end.
```

### Q7: RabbitMQ 的事务机制是如何实现的？

**答案：**

**事务流程：**

```
Producer                RabbitMQ
   │                         │
   │── 1. TxSelect ────────→ │
   │← 2. TxOk ─────────────│
   │                         │
   │── 3. BasicPublish ───→ │
   │── 4. BasicPublish ───→ │
   │── 5. BasicPublish ───→ │
   │                         │
   │── 6. TxCommit ───────→ │
   │← 7. TxOk ─────────────│
   │                         │
 (所有消息生效)        (提交所有消息)
```

**实现原理：**

```erlang
%% RabbitMQ 伪代码
handle_tx_select(Channel) ->
    %% 创建事务上下文
    TxId = create_tx_context(Channel),
    Channel#state{tx_mode = true, tx_id = TxId}.

handle_tx_commit(Channel) ->
    TxId = Channel#state.tx_id,
    %% 提交所有未确认的消息
    Messages = get_tx_messages(TxId),
    lists:foreach(fun(Message) ->
        store_message(Message)
    end, Messages),
    %% 清除事务上下文
    Channel#state{tx_mode = false, tx_id = undefined}.

handle_basic_publish(Channel, Message, TxMode) ->
    if TxMode ->
        %% 事务模式，暂存消息
        store_tx_message(Channel#state.tx_id, Message);
    true ->
        %% 非事务模式，直接存储
        store_message(Message)
    end.
```

**性能问题：**

事务是同步阻塞的，性能较差。建议使用 Confirm 机制替代。

## 三、高并发场景

### Q8: 如何处理秒杀场景下的消息堆积？

**答案：**

**问题分析：**

```
正常流量：100 QPS
秒杀流量：10000 QPS
消费能力：500 QPS

结果：消息快速堆积
```

**解决方案：**

**1. 多级缓冲**

```
用户请求 → 限流 → MQ → 消费者 → 数据库
            ↓
        Redis 缓存
```

**2. 预扣库存**

```java
@Service
public class SeckillService {

    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    @Autowired
    private RabbitTemplate rabbitTemplate;

    public void seckill(Long productId, Long userId) {
        String stockKey = "stock:" + productId;
        String userKey = "user:" + userId + ":" + productId;

        // 检查是否已购买
        if (redisTemplate.hasKey(userKey)) {
            throw new BusinessException("已经购买过");
        }

        // 预扣库存（原子操作）
        Long stock = redisTemplate.opsForValue().decrement(stockKey);

        if (stock < 0) {
            // 库存不足，回滚
            redisTemplate.opsForValue().increment(stockKey);
            throw new BusinessException("库存不足");
        }

        // 标记用户已购买
        redisTemplate.opsForValue().set(userKey, "1", 1, TimeUnit.HOURS);

        // 发送消息到 MQ
        SeckillMessage message = new SeckillMessage(productId, userId);
        rabbitTemplate.convertAndSend("seckill.exchange", "seckill.key", message);
    }
}
```

**3. 批量消费**

```java
@RabbitListener(queues = "seckill.queue", concurrency = "20-50")
public void batchProcessSeckill(List<SeckillMessage> messages) {
    // 批量更新数据库
    List<Order> orders = new ArrayList<>();
    for (SeckillMessage msg : messages) {
        Order order = createOrder(msg);
        orders.add(order);
    }

    // 批量插入
    orderService.batchInsert(orders);
}
```

**4. 降级策略**

```java

    try {
        // 正常处理
        processOrder(message);
    } catch (Exception e) {
        // 降级：记录到数据库，后续处理
        saveToPendingTable(message);
    }
}
```

```java
@Service
public class QueueManager {

    private static final int QUEUE_COUNT = 1000;

    public void sendMessage(Object message) {
        int hash = message.hashCode();
        int queueIndex = Math.abs(hash) % QUEUE_COUNT;

        String queueName = "topic.queue." + queueIndex;
        rabbitTemplate.convertAndSend("", queueName, message);
    }

    public void createQueues() {
        for (int i = 0; i < QUEUE_COUNT; i++) {
            String queueName = "topic.queue." + i;
            rabbitAdmin.declareQueue(new Queue(queueName, true));
        }
    }
}
```

## 四、故障处理

### Q10: 如何处理 RabbitMQ 节点故障？

**答案：**

**故障场景：**

```
正常状态：
Node 1 (Master) ←→ Node 2 (Slave) ←→ Node 3 (Slave)

Node 1 故障：
Node 1 (Down)    Node 2 (Master) ←→ Node 3 (Slave)
     ↓                   ↑
   不可用           自动接管
```

**处理流程：**

**1. 故障检测**

```erlang
%% RabbitMQ 伪代码
monitor_nodes() ->
    Nodes = get_cluster_nodes(),
    lists:foreach(fun(Node) ->
        case ping(Node) of
            pong -> ok;
            pang -> handle_node_down(Node)
        end
    end, Nodes).

handle_node_down(Node) ->
    %% 检查是否为主节点
    case is_master(Node) of
        true -> elect_new_master();
        false -> ok
    end.
```

**2. 主节点选举**

```erlang
%% 镜像队列选举
elect_new_master() ->
    Mirrors = get_mirror_nodes(),
    %% 选择最同步的节点
    NewMaster = select_most_synced(Mirrors),
    promote_to_master(NewMaster).
```

**3. 客户端重连**

```java
@Configuration
public class RabbitMQConfig {
```

```java
    }
}
```

### Q11: 如何处理消息积压？

**答案：**

**分析积压原因：**

| 原因 | 解决方案 |
|------|----------|
| 消费者处理慢 | 优化处理逻辑、增加消费者 |
| 消费者数量不足 | 增加消费者数 |
| 网络问题 | 检查网络、增加超时时间 |
| 数据库瓶颈 | 优化数据库、批量处理 |

**解决方案：**

**1. 临时扩容**

```java
@Service
public class BacklogHandler {

    public void handleBacklog() {
        // 1. 创建新的 Topic，分区数扩大 10 倍
        createNewTopic("backlog.topic", 10);

        // 2. 将积压消息转发到新 Topic
        transferMessages("original.topic", "backlog.topic");

        // 3. 部署 10 倍的消费者
        deployConsumers("backlog.topic", 100);

        // 4. 消费完成后恢复
        restoreOriginalConfig();
    }
}
```

**2. 丢弃不重要消息**

```java
@RabbitListener(queues = "backlog.queue")
public void handleBacklogMessage(Message message) {
    long age = getMessageAge(message);

    // 超过 1 小时的消息直接丢弃
    if (age > 3600000) {
        log.warn("丢弃过期消息: {}", message.getMessageProperties().getMessageId());
        return;
    }

    processMessage(message);
}
```

**3. 优先处理重要消息**

```java
@RabbitListener(queues = "backlog.queue")
public void handleWithPriority(Message message) {
    int priority = message.getMessageProperties().getPriority();

    // 高优先级消息优先处理
    if (priority >= 8) {
        processImmediately(message);
    } else {
        addToBatch(message);
    }
}
```

### Q12: 如何优化 RabbitMQ 的内存使用？

**答案：**

**内存优化策略：**

**1. 配置内存限制**

```ini
# rabbitmq.conf

# 内存水位线（百分比）
vm_memory_high_watermark.relative = 0.6

# 或绝对值
vm_memory_high_watermark.absolute = 2GB

# 内存告警
vm_memory_high_watermark_paging_ratio = 0.75
```

**2. 使用惰性队列**

```java
Map<String, Object> args = new HashMap<>();
args.put("x-queue-mode", "lazy");
QueueBuilder.durable("lazy.queue").withArguments(args).build();
```

**3. 及时删除过期消息**

```java
Map<String, Object> args = new HashMap<>();
args.put("x-message-ttl", 3600000); // 1 小时
QueueBuilder.durable("ttl.queue").withArguments(args).build();
```

**4. 定期清理队列**

```java
@Scheduled(cron = "0 0 * * * ?")
public void cleanExpiredQueues() {
    List<String> queues = rabbitAdmin.getQueues();

    for (String queue : queues) {
        if (isExpiredQueue(queue)) {
            rabbitAdmin.deleteQueue(queue);
        }
    }
}
```
