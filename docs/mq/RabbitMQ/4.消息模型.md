# RabbitMQ 消息模型

RabbitMQ 提供了 5 种经典的消息模型，涵盖了大多数使用场景。

## 一、简单队列模型（Hello World）

### 1.1 模型结构

```
Producer ──────► Queue ──────► Consumer
```

**特点**：
- 一个生产者
- 一个队列
- 一个消费者
- 一对一关系

### 1.2 应用场景

- 简单的任务分发
- 日志收集
- 通知推送

### 1.3 代码实现

#### 生产者

```java
@Component
public class HelloWorldProducer {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    public void sendMessage(String message) {
        // 发送消息到默认交换机，指定队列名
        rabbitTemplate.convertAndSend("", "hello.queue", message);
        System.out.println("Sent: " + message);
    }
}
```

#### 消费者

```java
@Component
public class HelloWorldConsumer {

    @RabbitListener(queues = "hello.queue")
    public void receiveMessage(String message) {
        System.out.println("Received: " + message);
    }
}
```

#### 配置

```java
@Configuration
public class HelloWorldConfig {

    @Bean
    public Queue helloQueue() {
        return QueueBuilder.durable("hello.queue").build();
    }
}
```

## 二、工作队列模型（Work Queue）

### 2.1 模型结构

```
Producer ──────► Queue ──────► Consumer 1
                    │
                    └─────► Consumer 2
```

**特点**：
- 一个生产者
- 一个队列
- 多个消费者
- 消息竞争消费

### 2.2 消息分发策略

#### 轮询分发（Round-robin）

默认策略，消息按顺序分发给消费者。

```
消息1 → Consumer 1
消息2 → Consumer 2
消息3 → Consumer 1
消息4 → Consumer 2
...
```

#### 公平分发（Fair Dispatch）

根据消费者处理能力分发，需要设置 `prefetch`。

```
Consumer 1 处理快 → 多分消息
Consumer 2 处理慢 → 少分消息
```

### 2.3 应用场景

- 任务队列
- 图片/视频处理
- 邮件发送

### 2.4 代码实现

#### 配置

```java
@Configuration
public class WorkQueueConfig {

    @Bean
    public Queue workQueue() {
        return QueueBuilder.durable("work.queue").build();
    }

    // 公平分发：设置 prefetch
    @Bean
    public SimpleRabbitListenerContainerFactory rabbitListenerContainerFactory(
            ConnectionFactory connectionFactory) {
        SimpleRabbitListenerContainerFactory factory =
            new SimpleRabbitListenerContainerFactory();
        factory.setConnectionFactory(connectionFactory);
        factory.setPrefetch(1); // 每次只接收一条消息
        return factory;
    }
}
```

#### 生产者

```java
@Component
public class WorkQueueProducer {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    public void sendTasks(int count) {
        for (int i = 1; i <= count; i++) {
            String message = "Task " + i;
            rabbitTemplate.convertAndSend("", "work.queue", message);
            System.out.println("Sent: " + message);
        }
    }
}
```

#### 消费者

```java
@Component
public class WorkQueueConsumer {

    // 消费者 1
    @RabbitListener(queues = "work.queue")
    public void receiveTask1(String message) {
        System.out.println("Worker 1 received: " + message);
        try {
            // 模拟处理时间
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }

    // 消费者 2
    @RabbitListener(queues = "work.queue")
    public void receiveTask2(String message) {
        System.out.println("Worker 2 received: " + message);
        try {
            // 模拟处理时间
            Thread.sleep(500);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

## 三、发布订阅模型（Publish/Subscribe）

### 3.1 模型结构

```
Producer ──────► Exchange (Fanout)
                    │
        ┌───────────┼───────────┐
        ↓           ↓           ↓
      Queue 1     Queue 2     Queue 3
        │           │           │
        ↓           ↓           ↓
     Consumer 1  Consumer 2  Consumer 3
```

**特点**：
- 一个生产者
- 一个交换机（Fanout 类型）
- 多个队列
- 每个队列一个消费者
- 消息广播到所有队列

### 3.2 应用场景

- 系统广播通知
- 配置更新
- 事件发布

### 3.3 代码实现

#### 配置

```java
@Configuration
public class PublishSubscribeConfig {

    @Bean
    public FanoutExchange fanoutExchange() {
        return new FanoutExchange("fanout.exchange");
    }

    @Bean
    public Queue queueA() {
        return QueueBuilder.durable("fanout.queue.a").build();
    }

    @Bean
    public Queue queueB() {
        return QueueBuilder.durable("fanout.queue.b").build();
    }

    @Bean
    public Queue queueC() {
        return QueueBuilder.durable("fanout.queue.c").build();
    }

    @Bean
    public Binding bindingA() {
        return BindingBuilder.bind(queueA()).to(fanoutExchange());
    }

    @Bean
    public Binding bindingB() {
        return BindingBuilder.bind(queueB()).to(fanoutExchange());
    }

    @Bean
    public Binding bindingC() {
        return BindingBuilder.bind(queueC()).to(fanoutExchange());
    }
}
```

#### 生产者

```java
@Component
public class PublishSubscribeProducer {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    public void broadcast(String message) {
        rabbitTemplate.convertAndSend("fanout.exchange", "", message);
        System.out.println("Broadcasted: " + message);
    }
}
```

#### 消费者

```java
@Component
public class PublishSubscribeConsumer {

    @RabbitListener(queues = "fanout.queue.a")
    public void receiveA(String message) {
        System.out.println("Consumer A received: " + message);
    }

    @RabbitListener(queues = "fanout.queue.b")
    public void receiveB(String message) {
        System.out.println("Consumer B received: " + message);
    }

    @RabbitListener(queues = "fanout.queue.c")
    public void receiveC(String message) {
        System.out.println("Consumer C received: " + message);
    }
}
```

## 四、路由模型（Routing）

### 4.1 模型结构

```
Producer ──────► Exchange (Direct)
                    │
        ┌───────────┼───────────┐
        ↓           ↓           ↓
      Queue 1     Queue 2     Queue 3
      (error)     (info)      (warning)
        │           │           │
        ↓           ↓           ↓
     Consumer 1  Consumer 2  Consumer 3
```

**特点**：
- 一个生产者
- 一个交换机（Direct 类型）
- 多个队列，每个队列绑定不同的 routing key
- 根据路由键将消息路由到对应队列

### 4.2 应用场景

- 日志分级处理
- 按业务类型分发消息
- 错误通知

### 4.3 代码实现

#### 配置

```java
@Configuration
public class RoutingConfig {

    @Bean
    public DirectExchange directExchange() {
        return new DirectExchange("direct.exchange");
    }

    @Bean
    public Queue errorQueue() {
        return QueueBuilder.durable("error.queue").build();
    }

    @Bean
    public Queue infoQueue() {
        return QueueBuilder.durable("info.queue").build();
    }

    @Bean
    public Queue warningQueue() {
        return QueueBuilder.durable("warning.queue").build();
    }

    @Bean
    public Binding bindingError() {
        return BindingBuilder.bind(errorQueue())
            .to(directExchange())
            .with("error");
    }

    @Bean
    public Binding bindingInfo() {
        return BindingBuilder.bind(infoQueue())
            .to(directExchange())
            .with("info");
    }

    @Bean
    public Binding bindingWarning() {
        return BindingBuilder.bind(warningQueue())
            .to(directExchange())
            .with("warning");
    }
}
```

#### 生产者

```java
@Component
public class RoutingProducer {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    public void sendError(String message) {
        rabbitTemplate.convertAndSend("direct.exchange", "error", message);
    }

    public void sendInfo(String message) {
        rabbitTemplate.convertAndSend("direct.exchange", "info", message);
    }

    public void sendWarning(String message) {
        rabbitTemplate.convertAndSend("direct.exchange", "warning", message);
    }
}
```

#### 消费者

```java
@Component
public class RoutingConsumer {

    @RabbitListener(queues = "error.queue")
    public void receiveError(String message) {
        System.out.println("Error Consumer received: " + message);
    }

    @RabbitListener(queues = "info.queue")
    public void receiveInfo(String message) {
        System.out.println("Info Consumer received: " + message);
    }

    @RabbitListener(queues = "warning.queue")
    public void receiveWarning(String message) {
        System.out.println("Warning Consumer received: " + message);
    }
}
```

## 五、主题模型（Topics）

### 5.1 模型结构

```
Producer ──────► Exchange (Topic)
                    │
        ┌───────────┼───────────┐
        ↓           ↓           ↓
      Queue 1     Queue 2     Queue 3
      (*.error)   (user.*)     (#)
        │           │           │
        ↓           ↓           ↓
     Consumer 1  Consumer 2  Consumer 3
```

**特点**：
- 一个生产者
- 一个交换机（Topic 类型）
- 多个队列，支持通配符绑定
- 灵活的路由规则

### 5.2 通配符规则

| 通配符 | 说明 | 示例 |
|--------|------|------|
| `*` | 匹配一个单词 | `*.error` 匹配 `user.error`、`system.error` |
| `#` | 匹配零个或多个单词 | `user.#` 匹配 `user`、`user.info`、`user.info.detail` |

### 5.3 应用场景

- 复杂的消息路由
- 多维度分类
- 事件溯源

### 5.4 代码实现

#### 配置

```java
@Configuration
public class TopicsConfig {

    @Bean
    public TopicExchange topicExchange() {
        return new TopicExchange("topic.exchange");
    }

    @Bean
    public Queue allErrorQueue() {
        return QueueBuilder.durable("all.error.queue").build();
    }

    @Bean
    public Queue userQueue() {
        return QueueBuilder.durable("user.queue").build();
    }

    @Bean
    public Queue allQueue() {
        return QueueBuilder.durable("all.queue").build();
    }

    @Bean
    public Binding bindingAllError() {
        return BindingBuilder.bind(allErrorQueue())
            .to(topicExchange())
            .with("*.error");
    }

    @Bean
    public Binding bindingUser() {
        return BindingBuilder.bind(userQueue())
            .to(topicExchange())
            .with("user.*");
    }

    @Bean
    public Binding bindingAll() {
        return BindingBuilder.bind(allQueue())
            .to(topicExchange())
            .with("#");
    }
}
```

#### 生产者

```java
@Component
public class TopicsProducer {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    public void sendUserError(String message) {
        rabbitTemplate.convertAndSend("topic.exchange", "user.error", message);
    }

    public void sendUserLogin(String message) {
        rabbitTemplate.convertAndSend("topic.exchange", "user.login", message);
    }

    public void sendSystemError(String message) {
        rabbitTemplate.convertAndSend("topic.exchange", "system.error", message);
    }

    public void sendOrderCreated(String message) {
        rabbitTemplate.convertAndSend("topic.exchange", "order.created", message);
    }
}
```

#### 消费者

```java
@Component
public class TopicsConsumer {

    @RabbitListener(queues = "all.error.queue")
    public void receiveAllError(String message) {
        System.out.println("All Error Consumer received: " + message);
    }

    @RabbitListener(queues = "user.queue")
    public void receiveUser(String message) {
        System.out.println("User Consumer received: " + message);
    }

    @RabbitListener(queues = "all.queue")
    public void receiveAll(String message) {
        System.out.println("All Consumer received: " + message);
    }
}
```

## 六、消息模型对比

| 模型 | 交换机类型 | 路由规则 | 消息分发 | 适用场景 |
|------|-----------|----------|----------|----------|
| **简单队列** | Default | 无 | 一对一 | 简单任务 |
| **工作队列** | Default | 无 | 竞争消费 | 任务分发 |
| **发布订阅** | Fanout | 广播 | 一对多 | 广播通知 |
| **路由** | Direct | 完全匹配 | 按路由键分发 | 日志分级 |
| **主题** | Topic | 通配符匹配 | 灵活路由 | 复杂路由 |

## 七、模型选择建议

### 7.1 根据场景选择

```
场景 → 消息模型

简单任务 → 简单队列
任务分发 → 工作队列
广播通知 → 发布订阅
按类型分发 → 路由
复杂路由 → 主题
```

### 7.2 混合使用

实际项目中可以组合使用多种模型：

```java
// 用户注册流程
// 1. 发送注册事件（发布订阅）
rabbitTemplate.convertAndSend("user.registered.exchange", "", user);

// 2. 发送欢迎邮件（路由）
rabbitTemplate.convertAndSend("notification.exchange", "email.welcome", userEmail);

// 3. 记录日志（主题）
rabbitTemplate.convertAndSend("log.exchange", "user.registered", logMessage);
```

## 八、Spring Boot 整合

### 8.1 依赖配置

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-amqp</artifactId>
</dependency>
```

### 8.2 配置文件

```yaml
spring:
  rabbitmq:
    host: localhost
    port: 5672
    username: guest
    password: guest
    virtual-host: /
    listener:
      simple:
        acknowledge-mode: manual  # 手动 ACK
        prefetch: 1              # 公平分发
        concurrency: 5           # 消费者数量
        max-concurrency: 10      # 最大消费者数量
    publisher-confirm-type: correlated  # 发送确认
    publisher-returns: true
    template:
      mandatory: true
```

### 8.3 统一配置

```java
@Configuration
@EnableRabbit
public class RabbitMQConfig {

    @Bean
    public RabbitTemplate rabbitTemplate(ConnectionFactory connectionFactory) {
        RabbitTemplate template = new RabbitTemplate(connectionFactory);

        // 开启发送确认
        template.setConfirmCallback((correlationData, ack, cause) -> {
            if (ack) {
                System.out.println("消息发送成功");
            } else {
                System.out.println("消息发送失败: " + cause);
            }
        });

        // 开启返回确认
        template.setReturnsCallback(returnedMessage -> {
            System.out.println("消息未被路由: " + returnedMessage.getMessage());
        });

        return template;
    }

    @Bean
    public RabbitAdmin rabbitAdmin(ConnectionFactory connectionFactory) {
        return new RabbitAdmin(connectionFactory);
    }

    @Bean
    public SimpleMessageListenerContainer messageListenerContainer(
            ConnectionFactory connectionFactory,
            MessageListenerAdapter listenerAdapter) {
        SimpleMessageListenerContainer container =
            new SimpleMessageListenerContainer();
        container.setConnectionFactory(connectionFactory);
        container.setQueueNames("my.queue");
        container.setMessageListener(listenerAdapter);
        return container;
    }
}
```
