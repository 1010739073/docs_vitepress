# Spark 全面详解

## 1. Spark 核心原理

### 1.1 Spark 架构

Spark 采用主从架构，主要组件包括：

- **Driver**：负责作业的调度和资源管理
- **Executor**：负责执行任务，存储数据
- **Cluster Manager**：资源管理器（如 YARN、Mesos、Standalone）

### 1.2 RDD (Resilient Distributed Dataset)

RDD 是 Spark 的核心抽象，具有以下特性：

- **弹性**：数据丢失时可自动恢复
- **分布式**：数据分布在多个节点上
- **不可变**：一旦创建不可修改
- **分区**：数据被划分为多个分区

```python
# 创建 RDD 的几种方式
rdd1 = sc.parallelize([1, 2, 3, 4, 5])  # 从内存创建
rdd2 = sc.textFile("hdfs://path/to/file")  # 从文件创建
rdd3 = rdd1.map(lambda x: x * 2)  # 从其他 RDD 转换
```

### 1.3 宽依赖与窄依赖

- **窄依赖**：每个父 RDD 分区只被一个子 RDD 分区使用（如 map、filter）
- **宽依赖**：多个子 RDD 分区依赖同一个父 RDD 分区（如 reduceByKey、groupBy）

### 1.4 DAG (Directed Acyclic Graph)

Spark 会将作业转换为 DAG，然后划分为多个 Stage 执行：

1. **Stage 划分**：以宽依赖为边界
2. **Task 分配**：每个 Stage 包含多个 Task
3. **执行顺序**：按照依赖关系顺序执行

## 2. Spark 优化

### 2.1 数据存储优化

- **序列化**：使用 Kryo 序列化替代 Java 序列化
- **内存管理**：调整 executor 内存分配
- **缓存策略**：合理使用 cache() 和 persist()

```python
# 使用 Kryo 序列化
from pyspark import SparkConf
conf = SparkConf().set("spark.serializer", "org.apache.spark.serializer.KryoSerializer")

# 缓存 RDD
rdd.cache()  # 默认缓存在内存
rdd.persist(StorageLevel.MEMORY_AND_DISK)  # 内存不足时使用磁盘
```

### 2.2 计算优化

- **分区调整**：根据数据量调整分区数
- **避免 Shuffle**：使用广播变量减少数据传输
- **使用高性能算子**：如 reduceByKey 替代 groupByKey

```python
# 合理设置分区数
rdd = rdd.repartition(100)  # 增加分区
rdd = rdd.coalesce(50)  # 减少分区

# 使用广播变量
broadcast_var = sc.broadcast(large_lookup_table)
result = rdd.map(lambda x: (x, broadcast_var.value.get(x, 0)))
```

### 2.3 资源配置优化

- **executor 数量**：根据集群规模调整
- **executor 内存**：避免 OOM
- **cores 配置**：充分利用集群资源

```bash
# 提交作业时的资源配置
spark-submit \
  --num-executors 10 \
  --executor-memory 8g \
  --executor-cores 4 \
  --driver-memory 4g \
  app.py
```

### 2.4 SQL 编码优化

#### 2.4.1 查询优化

- **使用谓词下推**：将过滤条件下推到数据源
- **避免全表扫描**：使用索引和分区过滤
- **使用合适的 join 策略**：根据数据量选择 broadcast join 或 sort-merge join
- **减少 shuffle 操作**：避免不必要的 repartition

```python

# 广播小表示例

result1 = df.filter(col("age") > 18).count()
result2 = df.filter(col("age") < 60).count()
```

#### 2.4.2 数据类型优化

- **使用合适的数据类型**：避免使用过大的数据类型
- **处理空值**：合理处理空值，避免全表扫描
- **字符串处理**：使用内置函数替代自定义 UDF

#### 2.4.3 分区优化

- **合理设置分区数**：根据数据量和集群资源调整

#### 2.4.5 Spark SQL处理数据倾斜方案

数据倾斜是Spark SQL查询中常见的性能问题，当数据分布不均匀时，某些任务会处理大量数据，导致整体查询速度变慢。以下是处理Spark SQL数据倾斜的常用方案：

1. **使用广播变量（Broadcast Join）**：

   当一个表很小（通常小于1GB）时，可以使用广播变量将小表广播到所有执行器，避免shuffle操作。

   ```sql
   -- 启用广播连接
   SET spark.sql.autoBroadcastJoinThreshold=104857600; -- 100MB
   
   SELECT /*+ BROADCAST(t2) */ t1.*, t2.*
   FROM large_table t1
   JOIN small_table t2 ON t1.id = t2.id;
   ```

2. **使用随机前缀（Salting）**：

   对于倾斜的key，可以添加随机前缀，将数据分散到不同的分区，然后在处理完成后再去除前缀。

   ```sql
   -- 为倾斜的key添加随机前缀
   WITH salted_data AS (
       SELECT 
           CASE 
               WHEN key = 'skewed_key' THEN CONCAT(key, '_', CAST(FLOOR(RAND() * 10) AS STRING))
               ELSE key
           END AS salted_key,
           value
       FROM large_table
   )
   SELECT 
       SUBSTRING(salted_key, 1, INSTR(salted_key, '_') - 1) AS original_key,
       SUM(value) AS total
   FROM salted_data
   GROUP BY salted_key
   GROUP BY original_key;
   ```

3. **使用分区裁剪**：

   利用分区表的特性，只扫描相关分区，减少数据处理量。

   ```sql
   -- 只查询特定分区
   SELECT * FROM partitioned_table WHERE dt = '2023-01-01' AND region = 'China';
   ```

4. **使用桶表（Bucketed Table）**：

   对于经常进行连接操作的表，可以创建桶表，减少shuffle数据量。

   ```sql
   -- 创建桶表
   CREATE TABLE bucketed_table (
       id INT,
       name STRING,
       value DOUBLE
   ) CLUSTERED BY (id) INTO 100 BUCKETS;
   
   -- 插入数据
   INSERT INTO bucketed_table SELECT * FROM source_table;
   
   -- 桶表连接
   SELECT t1.*, t2.*
   FROM bucketed_table t1
   JOIN bucketed_table t2 ON t1.id = t2.id;
   ```

5. **调整并行度**：

   增加shuffle操作的并行度，分散负载。

   ```sql
   -- 设置shuffle并行度
   SET spark.sql.shuffle.partitions=200;
   
   SELECT key, SUM(value) AS total
   FROM large_table
   GROUP BY key;
   ```

6. **使用倾斜连接优化**：

   Spark 3.0+ 提供了倾斜连接优化，可以自动检测和处理数据倾斜。

   ```sql
   -- 启用倾斜连接优化
   SET spark.sql.adaptive.enabled=true;
   SET spark.sql.adaptive.skewedJoin.enabled=true;
   
   SELECT t1.*, t2.*
   FROM large_table t1
   JOIN large_table t2 ON t1.key = t2.key;
   ```

7. **使用近似聚合函数**：

   对于不需要精确结果的场景，使用近似聚合函数减少计算量。

   ```sql
   -- 使用近似聚合函数
   SELECT 
       APPROX_COUNT_DISTINCT(user_id) AS unique_users,
       APPROX_PERCENTILE(amount, 0.5) AS median_amount
   FROM orders;
   ```

8. **数据预处理**：

   在ETL阶段对数据进行预处理，减少倾斜程度。

   ```sql
   -- 预处理倾斜数据
   CREATE TABLE processed_table AS
   SELECT 
       key,
       value,
       CASE 
           WHEN key = 'skewed_key' THEN CAST(FLOOR(RAND() * 10) AS STRING)
           ELSE '0'
       END AS salt
   FROM original_table;
   ```

## 3. 数据倾斜处理

### 3.1 数据倾斜的表现

- 个别 Task 执行时间过长
-  executor OOM
- 作业整体运行缓慢

### 3.2 数据倾斜的原因

- 数据分布不均匀
- 某些 key 出现频率过高
- 操作设计不当

### 3.3 数据倾斜的解决方案

#### 3.3.1 预处理数据

- 过滤异常数据
- 对热点 key 进行单独处理

#### 3.3.2 加盐法

```python
# 对 key 加盐
def add_salt(key):
    import random
    return (key, random.randint(0, 9))

# 加盐后处理
rdd = rdd.map(lambda x: (add_salt(x[0]), x[1]))
         .groupByKey()
         .map(lambda x: (x[0][0], process_values(x[1])))
```

#### 3.3.3 聚合操作优化

- 使用 combineByKey 替代 groupByKey
- 使用 reduceByKey 替代 groupByKey + reduce

#### 3.3.4 广播小表

```python
# 广播小表
small_table = sc.parallelize([(1, "a"), (2, "b")]).collectAsMap()
broadcast_small = sc.broadcast(small_table)

# 大表使用广播变量
large_rdd = large_rdd.map(lambda x: (x[0], (x[1], broadcast_small.value.get(x[0], None)))
```

#### 3.3.5 SQL 层面的数据倾斜处理

1. **使用随机前缀**：

```python
from pyspark.sql.functions import col, rand, concat, lit

# 对热点 key 添加随机前缀
df = df.withColumn("random_prefix", 
                   concat(lit(""), (rand() * 10).cast("int")))
df = df.withColumn("salted_key", 
                   concat(col("random_prefix"), col("key")))

# 处理后去前缀
result = result.withColumn("key", 
                           substr(col("salted_key"), 2, length(col("salted_key"))))
```

2. **使用 skewed join**：

```python
# 识别热点 key
hot_keys = ["hot_key1", "hot_key2"]

# 单独处理热点数据
hot_df = df.filter(col("key").isin(hot_keys))
normal_df = df.filter(~col("key").isin(hot_keys))

# 热点数据单独处理
hot_result = hot_df.join(broadcast(small_df), "key")

# 正常数据常规处理
normal_result = normal_df.join(small_df, "key")

# 合并结果
final_result = hot_result.union(normal_result)
```

3. **调整 join 策略**：

```python
# 强制使用 broadcast join
spark.sql("SET spark.sql.autoBroadcastJoinThreshold=10485760")  # 10MB

df1.join(df2, "id").show()

# 强制使用 sort-merge join
spark.sql("SET spark.sql.autoBroadcastJoinThreshold=-1")
df1.join(df2, "id").show()
```

4. **使用 explode 处理**：

```python
from pyspark.sql.functions import explode, array

# 对热点 key 进行扩展
df = df.withColumn("key_array", 
                   when(col("key").isin(hot_keys), 
                        array([lit(i) for i in range(10)]))
                   .otherwise(array([lit(0)])))

df = df.withColumn("key", 
                   concat(col("key"), lit("_"), explode(col("key_array"))))

# 处理后去扩展
df = df.withColumn("key", split(col("key"), "_")[0])
```

5. **SQL 语句处理数据倾斜**：

```sql
-- 1. 识别热点 key
SELECT key, COUNT(*) as count 
FROM table 
GROUP BY key 
ORDER BY count DESC 
LIMIT 10;

-- 2. 对热点 key 进行单独处理
-- 处理热点数据
WITH hot_data AS (
    SELECT * FROM table WHERE key IN ('hot_key1', 'hot_key2')
),
normal_data AS (
    SELECT * FROM table WHERE key NOT IN ('hot_key1', 'hot_key2')
)
-- 热点数据与小表广播 join
SELECT * FROM hot_data JOIN small_table ON hot_data.key = small_table.key
UNION ALL
-- 正常数据常规 join
SELECT * FROM normal_data JOIN small_table ON normal_data.key = small_table.key;

-- 3. 使用随机前缀
SELECT 
    CONCAT(CAST(FLOOR(RAND() * 10) AS STRING), key) as salted_key,
    value
FROM table;

-- 4. 使用分桶表
CREATE TABLE bucketed_table (
    id INT,
    name STRING,
    value DOUBLE
)
CLUSTERED BY (id) INTO 32 BUCKETS;

-- 5. 调整并行度

```

## 4. Spark 面试题

### 4.1 基础概念

**Q1: Spark 与 MapReduce 的区别？**

- Spark 使用内存计算，速度更快
- Spark 支持 DAG 执行，减少磁盘 IO
- Spark 提供丰富的 API，支持更多操作
- Spark 支持流处理，而 MapReduce 主要用于批处理

**Q2: RDD、DataFrame 和 Dataset 的区别？**

- **RDD**：底层抽象，无 schema，性能较差
- **DataFrame**：有 schema，性能较好，使用 Catalyst 优化
- **Dataset**：结合了 RDD 和 DataFrame 的优点，类型安全

### 4.2 技术原理

**Q3: Spark 的 Shuffle 过程是怎样的？**

1. Map 阶段：每个 Task 生成中间结果
2. Shuffle Write：将中间结果写入本地磁盘
3. Shuffle Read：拉取其他节点的中间结果
4. Reduce 阶段：处理拉取的数据

**Q4: Spark 如何处理数据倾斜？**

- 加盐法
- 广播小表
- 调整分区策略
- 使用随机前缀

### 4.3 优化问题

**Q5: 如何优化 Spark 作业？**

- 数据序列化优化
- 内存管理优化
- 分区调整
- 避免 Shuffle
- 合理使用缓存

**Q6: Spark 中的广播变量和累加器是什么？**

- **广播变量**：将小数据集广播到所有节点，减少网络传输
- **累加器**：用于分布式环境下的计数和求和

### 5.2 数据倾斜处理示例

```python
# 处理数据倾斜的示例
from pyspark import SparkContext
import random

sc = SparkContext("local", "DataSkew")

# 模拟数据倾斜
rdd = sc.parallelize([("hot_key", i) for i in range(10000)] + 
                    [("normal_key_" + str(i), i) for i in range(100)])

# 加盐处理
def add_salt(key):
    if key == "hot_key":
        return (key, random.randint(0, 9))
    return (key, 0)

# 加盐后聚合
salted_rdd = rdd.map(lambda x: (add_salt(x[0]), x[1]))
aggregated = salted_rdd.groupByKey().mapValues(len)

# 去盐
result = aggregated.map(lambda x: (x[0][0], x[1])).reduceByKey(lambda a, b: a + b)

result.collect()
sc.stop()
```

### 5.3 性能优化示例

