# Sqoop 数据传输工具详解

## 1. Sqoop 概述

Sqoop (SQL-to-Hadoop) 是一个用于在 Hadoop 和关系型数据库之间传输数据的工具。它可以将数据从关系型数据库导入到 Hadoop，也可以将数据从 Hadoop 导出到关系型数据库。

### 1.1 Sqoop 特性

- **简单易用**：提供命令行接口，操作简单
- **高效传输**：支持并行数据传输
- **多种格式**：支持多种数据格式
- **自动化**：支持自动化任务调度
- **扩展性**：支持自定义连接器和类型映射

## 2. Sqoop 架构

### 2.1 核心组件

- **Sqoop Client**：客户端，执行 Sqoop 命令
- **Connector**：连接器，连接不同的数据源
- **Metastore**：元数据存储，存储任务信息
- **Hadoop**：底层存储和计算框架

### 2.2 架构原理

## 3. Sqoop 导入

### 3.1 基本导入

```bash
# 从 MySQL 导入数据到 HDFS
sqoop import \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table users \
  --target-dir /user/hadoop/users \
  --m 1

# 导入指定字段
sqoop import \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table users \
  --columns "id,name,email" \
  --target-dir /user/hadoop/users \
  --m 1

# 使用 WHERE 条件
sqoop import \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table users \
  --where "age > 18" \
  --target-dir /user/hadoop/adult_users \
  --m 1
```

### 3.2 增量导入

```bash
# 基于时间戳的增量导入
sqoop import \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table orders \
  --target-dir /user/hadoop/orders \
  --incremental append \
  --check-column created_at \
  --last-value '2024-01-01 00:00:00' \
  --m 1

# 基于自增 ID 的增量导入
sqoop import \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table users \
  --target-dir /user/hadoop/users \
  --incremental append \
  --check-column id \
  --last-value 1000 \
  --m 1

# 增量导入并保存最后值
sqoop import \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table orders \
  --target-dir /user/hadoop/orders \
  --incremental append \
  --check-column created_at \
  --last-value '2024-01-01 00:00:00' \
  --merge-key id \
  --m 1
```

### 3.3 导入到 Hive

```bash
# 导入数据到 Hive 表
sqoop import \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table users \
  --hive-import \
  --hive-table default.users \
  --create-hive-table \
  --m 1

# 导入到已存在的 Hive 表
sqoop import \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table users \
  --hive-import \
  --hive-table default.users \
  --m 1

# 导入到 Hive 分区表
sqoop import \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table orders \
  --hive-import \
  --hive-table default.orders \
  --hive-partition-key dt \
  --hive-partition-value '2024-01-01' \
  --m 1
```

### 3.4 自定义查询导入

```bash
# 使用自定义查询导入
sqoop import \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --query 'SELECT id, name, email FROM users WHERE age > 18 AND $CONDITIONS' \
  --target-dir /user/hadoop/adult_users \
  --m 1

# 使用 JOIN 查询
sqoop import \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --query 'SELECT u.id, u.name, o.order_id, o.amount 
          FROM users u 
          JOIN orders o ON u.id = o.user_id 
          WHERE $CONDITIONS' \
  --target-dir /user/hadoop/user_orders \
  --m 1
```

## 4. Sqoop 导出

### 4.1 基本导出

```bash
# 从 HDFS 导出到 MySQL
sqoop export \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table users \
  --export-dir /user/hadoop/users \
  --m 1

# 指定字段映射
sqoop export \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table users \
  --export-dir /user/hadoop/users \
  --columns "id,name,email" \
  --m 1

# 指定字段分隔符
sqoop export \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table users \
  --export-dir /user/hadoop/users \
  --input-fields-terminated-by ',' \
  --input-lines-terminated-by '\n' \
  --m 1
```

### 4.2 更新模式

```bash
# 使用 INSERT 模式（默认）
sqoop export \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table users \
  --export-dir /user/hadoop/users \
  --m 1

  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table users \
  --export-dir /user/hadoop/users \
  --update-key id \
  --update-mode allowinsert \
  --m 1

# 使用 CALL 模式（调用存储过程）

  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table users \
  --export-dir /user/hadoop/users \
  --call stored_procedure \
  --m 1
```

### 4.3 从 Hive 导出

## 5. Sqoop 作业管理

### 5.1 创建和保存作业

```bash
# 创建并保存作业
sqoop job --create import_users \
  -- import \
  --connect jdbc:mysql://localhost:3306/mydb \
  --username root \
  --password password \
  --table users \
  --target-dir /user/hadoop/users \
  --m 1

# 查看作业列表
sqoop job --list

# 查看作业详情
sqoop job --show import_users

# 执行作业
sqoop job --exec import_users
```

### 5.2 作业调度

```bash
# 使用 crontab 定时执行 Sqoop 作业
# 编辑 crontab
crontab -e

# 添加定时任务（每天凌晨 2 点执行）
0 2 * * * sqoop job --exec import_users >> /var/log/sqoop/import_users.log 2>&1

# 每小时执行一次
0 * * * * sqoop job --exec import_orders >> /var/log/sqoop/import_orders.log 2>&1
```

## 8. Sqoop 面试题

### 8.1 基础概念

**Q1: Sqoop 的作用是什么？**

Sqoop 是一个数据传输工具，用于：
- 在 Hadoop 和关系型数据库之间传输数据
- 从关系型数据库导入数据到 Hadoop
- 从 Hadoop 导出数据到关系型数据库
- 支持增量数据传输

**Q2: Sqoop 的核心功能有哪些？**

- 数据导入：从关系型数据库导入到 Hadoop
- 数据导出：从 Hadoop 导出到关系型数据库
- 作业管理：创建、保存和执行作业
- 类型映射：自动映射数据类型
- 并行传输：支持并行数据传输

### 8.2 技术原理

**Q3: Sqoop 的工作原理是怎样的？**

**导入流程**：
1. Sqoop 获取数据库表结构
2. 生成 Java 类（可选）
3. 将导入任务转换为 MapReduce 作业
4. 多个 Mapper 并行读取数据
5. 数据写入 HDFS 或 Hive

**导出流程**：
1. Sqoop 获取目标表结构
2. 将导出任务转换为 MapReduce 作业
3. 多个 Mapper 并行读取 HDFS 数据
4. 数据写入关系型数据库

**Q4: Sqoop 如何实现增量导入？**

- 基于时间戳的增量导入
- 基于自增 ID 的增量导入
- 保存最后导入的值
- 支持合并增量数据

### 8.3 应用场景

**Q5: Sqoop 的典型应用场景有哪些？**

- 数据仓库构建
- 数据迁移
- 数据备份
- 数据同步
- 数据分析

**Q6: 如何优化 Sqoop 性能？**

- 增加并行度（--m 参数）
- 选择合适的分割列
- 使用压缩减少数据量
- 合理设置内存参数
- 使用批量操作

## 9. 总结

Sqoop 作为 Hadoop 生态系统中重要的数据传输工具，具有以下特点：

1. **简单易用**：提供命令行接口，操作简单
2. **高效传输**：支持并行数据传输
3. **多种格式**：支持多种数据格式
4. **自动化**：支持自动化任务调度
5. **扩展性**：支持自定义连接器和类型映射

掌握 Sqoop 的使用方法和最佳实践，对于构建高效的数据传输管道具有重要意义。通过合理的配置和优化，可以充分发挥 Sqoop 的优势，满足各种数据传输需求。
