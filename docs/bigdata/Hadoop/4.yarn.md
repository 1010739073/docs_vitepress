# YARN 资源管理器详解

## 1. YARN 概述

YARN (Yet Another Resource Negotiator) 是 Hadoop 的资源管理系统，负责集群资源的统一管理和调度。它将资源管理和作业调度分离，提高了集群的利用率和扩展性。

### 1.1 YARN 特性

- **资源统一管理**：统一管理集群的 CPU、内存等资源
- **多框架支持**：支持 MapReduce、Spark、Flink 等多种计算框架
- **高可用性**：支持 ResourceManager 高可用部署
- **可扩展性**：支持大规模集群部署
- **资源隔离**：提供容器级别的资源隔离

## 2. YARN 架构

### 2.1 核心组件

- **ResourceManager**：全局资源管理器，负责整个集群的资源管理
- **NodeManager**：节点管理器，负责单个节点的资源管理
- **ApplicationMaster**：应用管理器，负责单个应用的资源申请和任务调度
- **Container**：容器，资源分配的基本单位

### 2.2 架构原理

```java
// YARN 架构示意
/*
ResourceManager (全局资源管理器)
├── Scheduler (调度器)
│   ├── FIFO Scheduler
│   ├── Capacity Scheduler
│   └── Fair Scheduler
├── ApplicationsManager (应用管理器)
└── ResourceTracker (资源跟踪器)

NodeManager (节点管理器)
├── ContainerManager (容器管理器)
├── NodeHealthChecker (健康检查器)
└── AuxiliaryServices (辅助服务)

ApplicationMaster (应用管理器)
├── 资源申请
├── 任务调度
└── 任务监控

Container (容器)
├── CPU 资源
├── 内存资源
└── 运行环境
*/
```

## 3. YARN 工作流程

### 3.1 应用提交流程

1. **客户端提交应用**：客户端向 ResourceManager 提交应用
2. **启动 ApplicationMaster**：ResourceManager 在某个 NodeManager 上启动 ApplicationMaster
3. **资源申请**：ApplicationMaster 向 ResourceManager 申请资源
4. **资源分配**：Scheduler 分配 Container 给 ApplicationMaster
5. **任务启动**：ApplicationMaster 在分配的 Container 中启动任务
6. **任务执行**：任务在 Container 中执行
7. **应用完成**：ApplicationMaster 向 ResourceManager 注销，应用结束

```java
// YARN 应用提交示例
public class YarnClientExample {
    public static void main(String[] args) throws Exception {
        YarnConfiguration conf = new YarnConfiguration();
        YarnClient yarnClient = YarnClient.createYarnClient();
        yarnClient.init(conf);
        yarnClient.start();
        
        // 创建应用提交上下文
        ApplicationSubmissionContext appContext = 
            Records.newRecord(ApplicationSubmissionContext.class);
        
        // 设置应用名称
        ApplicationId appId = ApplicationId.newInstance(0, 1);
        appContext.setApplicationId(appId);
        appContext.setApplicationName("My YARN Application");
        
        // 设置应用主类
        ContainerLaunchContext amContainer = Records.newRecord(ContainerLaunchContext.class);
        appContext.setAMContainerSpec(amContainer);
        
        // 设置资源需求
        Resource resource = Records.newRecord(Resource.class);
        resource.setMemory(1024);
        resource.setVirtualCores(1);
        appContext.setResource(resource);
        
        // 提交应用
        ApplicationId submittedAppId = yarnClient.submitApplication(appContext);
        System.out.println("Application submitted: " + submittedAppId);
        
        // 监控应用状态
        ApplicationReport appReport = yarnClient.getApplicationReport(submittedAppId);
        while (appReport.getYarnApplicationState() != YarnApplicationState.FINISHED) {
            Thread.sleep(1000);
            appReport = yarnClient.getApplicationReport(submittedAppId);
            System.out.println("Application state: " + appReport.getYarnApplicationState());
        }
        
        yarnClient.close();
    }
}
```

### 3.2 资源调度流程

1. **心跳汇报**：NodeManager 定期向 ResourceManager 发送心跳
2. **资源更新**：ResourceManager 更新集群资源状态
3. **资源请求**：ApplicationMaster 提交资源请求
4. **资源分配**：Scheduler 根据策略分配资源
5. **容器启动**：NodeManager 启动分配的 Container

```java
// ApplicationMaster 资源申请示例
public class MyApplicationMaster extends ApplicationMaster {
    
    @Override
    public void run() throws Exception {
        // 获取 AMRMClient
        AMRMClient<AMRMClient.ContainerRequest> amRMClient = 
            AMRMClient.createAMRMClient();
        amRMClient.init(getConfig());
        amRMClient.start();
        
        // 注册 ApplicationMaster
        amRMClient.registerApplicationMaster("", 0, "");
        
        // 申请资源
        Resource capability = Records.newRecord(Resource.class);
        capability.setMemory(1024);
        capability.setVirtualCores(1);
        
        Priority priority = Records.newRecord(Priority.class);
        priority.setPriority(0);
        
        AMRMClient.ContainerRequest containerRequest = 
            new AMRMClient.ContainerRequest(capability, null, null, priority);
        amRMClient.addContainerRequest(containerRequest);
        
        // 等待资源分配
        while (true) {
            List<Container> allocatedContainers = amRMClient.getMatchingContainers(
                new AMRMClient.ContainerRequest(capability, null, null, priority));
            
            if (!allocatedContainers.isEmpty()) {
                Container container = allocatedContainers.get(0);
                System.out.println("Allocated container: " + container.getId());
                
                // 在容器中启动任务
                launchTask(container);
                break;
            }
            
            Thread.sleep(1000);
        }
        
        // 注销 ApplicationMaster
        amRMClient.unregisterApplicationMaster(
            FinalApplicationStatus.SUCCEEDED, "", "");
        amRMClient.stop();
    }
    
    private void launchTask(Container container) {
        // 启动任务的逻辑
    }
}
```

## 4. YARN 调度器

### 4.1 FIFO Scheduler

FIFO (First-In-First-Out) 调度器按照应用提交的先后顺序进行调度。

```xml
<!-- yarn-site.xml FIFO Scheduler 配置 -->
<property>
    <name>yarn.resourcemanager.scheduler.class</name>
    <value>org.apache.hadoop.yarn.server.resourcemanager.scheduler.fifo.FifoScheduler</value>
</property>
```

### 4.2 Capacity Scheduler

Capacity Scheduler 将集群资源划分为多个队列，每个队列可以分配一定比例的资源。

```xml
<!-- yarn-site.xml Capacity Scheduler 配置 -->
<property>
    <name>yarn.resourcemanager.scheduler.class</name>
    <value>org.apache.hadoop.yarn.server.resourcemanager.scheduler.capacity.CapacityScheduler</value>
</property>

<property>
    <name>yarn.scheduler.capacity.root.queues</name>
    <value>default,prod,dev</value>
</property>

<property>
    <name>yarn.scheduler.capacity.root.default.capacity</name>
    <value>40</value>
</property>

<property>
    <name>yarn.scheduler.capacity.root.prod.capacity</name>
    <value>40</value>
</property>

<property>
    <name>yarn.scheduler.capacity.root.dev.capacity</name>
    <value>20</value>
</property>
```

### 4.3 Fair Scheduler

Fair Scheduler 致力于让所有应用公平地共享集群资源。

```xml
<!-- yarn-site.xml Fair Scheduler 配置 -->
<property>
    <name>yarn.resourcemanager.scheduler.class</name>
    <value>org.apache.hadoop.yarn.server.resourcemanager.scheduler.fair.FairScheduler</value>
</property>

<property>
    <name>yarn.scheduler.fair.allocation.file</name>
    <value>/etc/hadoop/conf/fair-scheduler.xml</value>
</property>
```

```xml
<!-- fair-scheduler.xml 配置示例 -->
<?xml version="1.0"?>
<allocations>
    <queue name="prod">
        <minResources>1024 mb, 4 vcores</minResources>
        <maxResources>8192 mb, 16 vcores</maxResources>
        <maxRunningApps>10</maxRunningApps>
        <weight>2.0</weight>
        <schedulingPolicy>fair</schedulingPolicy>
    </queue>
    
    <queue name="dev">
        <minResources>512 mb, 2 vcores</minResources>
        <maxResources>4096 mb, 8 vcores</maxResources>
        <maxRunningApps>20</maxRunningApps>
        <weight>1.0</weight>
        <schedulingPolicy>fair</schedulingPolicy>
    </queue>
    
    <queuePlacementPolicy>
        <rule name="specified" create="false"/>
        <rule name="default" queue="default"/>
    </queuePlacementPolicy>
</allocations>
```

## 7. YARN 面试题

### 7.1 基础概念

**Q1: YARN 的作用是什么？**

YARN 是 Hadoop 的资源管理系统，负责：
- 统一管理集群资源
- 为各种计算框架提供资源支持
- 提高集群资源利用率
- 支持多租户环境

**Q2: YARN 的核心组件有哪些？**

- **ResourceManager**：全局资源管理器
- **NodeManager**：节点管理器
- **ApplicationMaster**：应用管理器
- **Container**：资源容器

### 7.2 技术原理

**Q3: YARN 的工作流程是怎样的？**

1. 客户端提交应用
2. ResourceManager 启动 ApplicationMaster
3. ApplicationMaster 申请资源
4. Scheduler 分配 Container
5. ApplicationMaster 在 Container 中启动任务
6. 任务执行完成，ApplicationMaster 注销

**Q4: YARN 的调度器有哪些？各有什么特点？**

- **FIFO Scheduler**：简单，按提交顺序调度
- **Capacity Scheduler**：支持多队列，资源隔离
- **Fair Scheduler**：公平调度，动态分配资源

### 7.3 运维优化

**Q5: 如何优化 YARN 集群性能？**

- 合理配置调度器和队列
- 优化资源分配策略
- 监控集群资源使用情况
- 及时清理僵尸应用
- 调整心跳和超时参数

**Q6: YARN 如何实现高可用？**

通过 ResourceManager HA 实现：
- 配置多个 ResourceManager 实例
- 使用 ZooKeeper 进行 leader 选举
- 实现故障自动转移
- 保证应用状态一致性

## 8. 代码示例

### 8.1 自定义 ApplicationMaster

```java
import org.apache.hadoop.yarn.api.*;
import org.apache.hadoop.yarn.api.records.*;
import org.apache.hadoop.yarn.client.api.AMRMClient;
import org.apache.hadoop.yarn.client.api.NMClient;
import org.apache.hadoop.yarn.conf.YarnConfiguration;
import org.apache.hadoop.yarn.util.Records;

public class CustomApplicationMaster extends ApplicationMaster {
    
    private AMRMClient<AMRMClient.ContainerRequest> amRMClient;
    private NMClient nmClient;
    private int numContainersToRequest;
    private int numCompletedContainers = 0;
    
    @Override
    public void run() throws Exception {
        // 初始化客户端
        amRMClient = AMRMClient.createAMRMClient();
        amRMClient.init(getConfig());
        amRMClient.start();
        
        nmClient = NMClient.createNMClient();
        nmClient.init(getConfig());
        nmClient.start();
        
        // 注册 ApplicationMaster
        amRMClient.registerApplicationMaster("", 0, "");
        
        // 获取参数
        numContainersToRequest = Integer.parseInt(System.getenv("NUM_CONTAINERS"));
        
        // 申请并启动容器
        requestAndLaunchContainers();
        
        // 等待所有容器完成
        waitForCompletion();
        
        // 注销 ApplicationMaster
        amRMClient.unregisterApplicationMaster(
            FinalApplicationStatus.SUCCEEDED, "Application completed successfully", "");
        
        amRMClient.stop();
        nmClient.close();
    }
    
    private void requestAndLaunchContainers() throws Exception {
        // 定义资源需求
        Resource capability = Records.newRecord(Resource.class);
        capability.setMemory(1024);
        capability.setVirtualCores(1);
        
        // 定义优先级
        Priority priority = Records.newRecord(Priority.class);
        priority.setPriority(0);
        
        // 申请容器
        for (int i = 0; i < numContainersToRequest; i++) {
            AMRMClient.ContainerRequest containerRequest = 
                new AMRMClient.ContainerRequest(capability, null, null, priority);
            amRMClient.addContainerRequest(containerRequest);
        }
        
        // 等待容器分配并启动任务
        while (numCompletedContainers < numContainersToRequest) {
            // 获取分配的容器
            List<Container> allocatedContainers = amRMClient.getMatchingContainers(
                new AMRMClient.ContainerRequest(capability, null, null, priority));
            
            for (Container container : allocatedContainers) {
                // 启动任务
                launchTask(container);
            }
            
            Thread.sleep(1000);
        }
    }
    
    private void launchTask(Container container) throws Exception {
        // 创建容器启动上下文
        ContainerLaunchContext ctx = Records.newRecord(ContainerLaunchContext.class);
        
        // 设置命令
        List<String> commands = new ArrayList<>();
        commands.add("/bin/echo 'Hello from container " + container.getId() + "'");
        ctx.setCommands(commands);
        
        // 启动容器
        nmClient.startContainer(container, ctx);
        
        System.out.println("Launched container: " + container.getId());
    }
    
    private void waitForCompletion() throws Exception {
        while (numCompletedContainers < numContainersToRequest) {
            Thread.sleep(1000);
        }
    }
    
    public static void main(String[] args) throws Exception {
        CustomApplicationMaster appMaster = new CustomApplicationMaster();
        appMaster.init(args);
        appMaster.run();
    }
}
```

### 8.2 YARN 客户端示例

```java
import org.apache.hadoop.yarn.api.*;
import org.apache.hadoop.yarn.api.records.*;
import org.apache.hadoop.yarn.client.api.YarnClient;
import org.apache.hadoop.yarn.conf.YarnConfiguration;
import org.apache.hadoop.yarn.util.Records;

public class YarnClientExample {
    
    public static void main(String[] args) throws Exception {
        // 创建 YARN 客户端
        YarnConfiguration conf = new YarnConfiguration();
        YarnClient yarnClient = YarnClient.createYarnClient();
        yarnClient.init(conf);
        yarnClient.start();
        
        // 创建应用提交上下文
        ApplicationSubmissionContext appContext = 
            Records.newRecord(ApplicationSubmissionContext.class);
        
        // 设置应用 ID
        ApplicationId appId = ApplicationId.newInstance(
            System.currentTimeMillis(), 1);
        appContext.setApplicationId(appId);
        appContext.setApplicationName("Custom YARN Application");
        
        // 设置应用类型
        appContext.setApplicationType("CUSTOM");
        
        // 设置 AM 容器规格
        Resource amResource = Records.newRecord(Resource.class);
        amResource.setMemory(1024);
        amResource.setVirtualCores(1);
        appContext.setResource(amResource);
        
        // 创建 AM 容器启动上下文
        ContainerLaunchContext amContainer = 
            Records.newRecord(ContainerLaunchContext.class);
        
        // 设置 AM 命令
        List<String> commands = new ArrayList<>();
        commands.add("$JAVA_HOME/bin/java" +
                    " -Xmx256M" +
                    " com.example.CustomApplicationMaster" +
                    " 1>" + ApplicationConstants.LOG_DIR_EXPANSION_VAR + "/stdout" +
                    " 2>" + ApplicationConstants.LOG_DIR_EXPANSION_VAR + "/stderr");
        amContainer.setCommands(commands);
        
        // 设置本地资源
        Map<String, LocalResource> localResources = new HashMap<>();
        LocalResource appJarResource = Records.newRecord(LocalResource.class);
        // 设置 JAR 文件路径
        Path jarPath = new Path("hdfs://namenode:8020/user/app/custom-app.jar");
        FileStatus jarStatus = jarPath.getFileSystem(conf).getFileStatus(jarPath);
        appJarResource.setType(LocalResourceType.FILE);
        appJarResource.setVisibility(LocalResourceVisibility.APPLICATION);
        appJarResource.setResource(ConverterUtils.getYarnUrlFromPath(jarPath));
        appJarResource.setTimestamp(jarStatus.getModificationTime());
        appJarResource.setSize(jarStatus.getLen());
        localResources.put("custom-app.jar", appJarResource);
        amContainer.setLocalResources(localResources);
        
        // 设置环境变量
        Map<String, String> env = new HashMap<>();
        env.put("CLASSPATH", 
            "$HADOOP_CONF_DIR:" +
            "$HADOOP_COMMON_HOME/share/hadoop/common/*:" +
            "$HADOOP_COMMON_HOME/share/hadoop/common/lib/*:" +
            "$HADOOP_HDFS_HOME/share/hadoop/hdfs/*:" +
            "$HADOOP_HDFS_HOME/share/hadoop/hdfs/lib/*:" +
            "$HADOOP_YARN_HOME/share/hadoop/yarn/*:" +
            "$HADOOP_YARN_HOME/share/hadoop/yarn/lib/*:" +
            "./custom-app.jar");
        amContainer.setEnvironment(env);
        
        // 设置 AM 容器规格
        appContext.setAMContainerSpec(amContainer);
        
        // 提交应用
        ApplicationId submittedAppId = yarnClient.submitApplication(appContext);
        System.out.println("Application submitted with ID: " + submittedAppId);
        
        // 监控应用状态
        monitorApplication(yarnClient, submittedAppId);
        
        yarnClient.close();
    }
    
    private static void monitorApplication(YarnClient yarnClient, 
                                         ApplicationId appId) throws Exception {
        while (true) {
            try {
                ApplicationReport report = yarnClient.getApplicationReport(appId);
                YarnApplicationState state = report.getYarnApplicationState();
                
                System.out.println("Application state: " + state);
                
                if (state == YarnApplicationState.FINISHED || 
                    state == YarnApplicationState.FAILED || 
                    state == YarnApplicationState.KILLED) {
                    System.out.println("Application completed with status: " + 
                                     report.getFinalApplicationStatus());
                    return;
                }
                
                Thread.sleep(1000);
            } catch (Exception e) {
                System.err.println("Failed to monitor application: " + e.getMessage());
                throw e;
            }
        }
    }
}
```

## 9. 总结

YARN 作为 Hadoop 的资源管理系统，在大数据生态系统中扮演着重要角色：

1. **统一资源管理**：为各种计算框架提供统一的资源管理接口
2. **高可用性**：支持 ResourceManager HA，保证集群稳定性
3. **灵活调度**：提供多种调度器，满足不同场景需求
4. **可扩展性**：支持大规模集群部署

掌握 YARN 的工作原理和配置方法，对于构建和管理高效的大数据集群至关重要。通过合理的资源调度和配置优化，可以显著提高集群的利用率和应用性能。
