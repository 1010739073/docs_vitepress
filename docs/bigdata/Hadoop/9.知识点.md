# 大数据基础知识点及面试题

## 一、大数据基础概念

### 1. 什么是大数据？
大数据是指无法在合理时间内用传统IT技术和工具进行处理的海量数据集合。大数据的特点通常用4V来概括：
- **Volume（量大）**：数据量巨大，从TB级别跃升至PB级别
- **Velocity（速度快）**：数据生成和处理速度快
- **Variety（多样）**：数据类型多样，包括结构化、半结构化和非结构化数据
- **Value（价值高）**：数据价值密度低，但整体价值高

### 2. 大数据的应用场景
- **电商推荐系统**：根据用户行为数据推荐商品
- **金融风控**：分析用户交易数据，识别欺诈行为
- **智能交通**：分析交通流量数据，优化交通管理
- **医疗健康**：分析医疗数据，辅助诊断和治疗
- **智慧城市**：整合城市各类数据，提升城市管理水平

### 3. 大数据技术栈
- **数据采集**：Flume、Kafka、Logstash等
- **数据存储**：HDFS、HBase、MongoDB、Cassandra等
- **数据处理**：MapReduce、Spark、Flink等
- **数据查询**：Hive、Impala、Presto等
- **数据可视化**：Tableau、Power BI、ECharts等

## 二、Hadoop生态系统

### 1. Hadoop简介
Hadoop是一个开源的分布式计算框架，专为处理大规模数据而设计。它的核心组件包括：
- **HDFS（Hadoop Distributed File System）**：分布式文件系统，用于存储海量数据
- **MapReduce**：分布式计算框架，用于处理大规模数据
- **YARN（Yet Another Resource Negotiator）**：资源管理和调度系统
- **Common**：通用工具和库

### 2. Hadoop的优势
- **高可靠性**：数据多副本存储，容错能力强
- **高扩展性**：可以横向扩展，支持数千个节点
- **高效性**：采用分布式计算，处理速度快
- **高容错性**：节点故障自动检测和恢复
- **低成本**：使用普通硬件，成本低

### 3. Hadoop生态系统组件
- **HDFS**：分布式文件系统
- **MapReduce**：分布式计算框架
- **YARN**：资源管理和调度系统
- **Hive**：数据仓库工具，提供SQL查询能力
- **HBase**：分布式NoSQL数据库
- **Pig**：数据流处理工具
- **Spark**：内存计算框架
- **Flink**：流处理框架
- **ZooKeeper**：分布式协调服务
- **Oozie**：工作流调度系统
- **Sqoop**：数据导入/导出工具
- **Flume**：日志收集工具

## 三、HDFS详解

### 1. HDFS架构
- **NameNode**：管理文件系统的命名空间，存储元数据
- **DataNode**：存储实际数据，执行数据块的读写操作
- **SecondaryNameNode**：辅助NameNode，定期合并编辑日志，减轻NameNode负担

### 2. HDFS数据块
- 默认块大小：128MB（Hadoop 2.x）
- 块的副本数：默认3个
- 块的存储：数据块分散存储在不同的DataNode上

### 3. HDFS读写流程
- **读流程**：
  1. 客户端向NameNode请求文件元数据
  2. NameNode返回文件的数据块位置
  3. 客户端直接从DataNode读取数据

- **写流程**：
  1. 客户端向NameNode请求写入文件
  2. NameNode分配数据块和DataNode
  3. 客户端将数据写入DataNode
  4. DataNode复制数据到其他副本节点

### 4. HDFS命令行操作
```bash
# 查看HDFS目录
hadoop fs -ls /

# 创建目录
hadoop fs -mkdir /user/hadoop

# 上传文件
hadoop fs -put localfile /user/hadoop/

# 下载文件
hadoop fs -get /user/hadoop/file localfile

# 删除文件
hadoop fs -rm /user/hadoop/file
```

## 四、MapReduce详解

### 1. MapReduce原理
MapReduce是一种分布式计算模型，将计算分为两个阶段：
- **Map阶段**：将输入数据分割成小块，并行处理
- **Reduce阶段**：合并Map阶段的结果，产生最终输出

### 2. MapReduce执行流程
1. **输入分片**：将输入数据分割成多个输入分片
2. **Map任务**：并行处理每个输入分片
3. **Shuffle**：将Map输出按照key分组，发送到对应的Reduce任务
4. **Reduce任务**：处理分组后的数据，产生最终结果
5. **输出**：将结果写入HDFS

### 3. MapReduce示例：WordCount
```java
public class WordCount {
    public static class TokenizerMapper extends Mapper<Object, Text, Text, IntWritable> {
        private final static IntWritable one = new IntWritable(1);
        private Text word = new Text();

        public void map(Object key, Text value, Context context) throws IOException, InterruptedException {
            StringTokenizer itr = new StringTokenizer(value.toString());
            while (itr.hasMoreTokens()) {
                word.set(itr.nextToken());
                context.write(word, one);
            }
        }
    }

    public static class IntSumReducer extends Reducer<Text, IntWritable, Text, IntWritable> {
        private IntWritable result = new IntWritable();

        public void reduce(Text key, Iterable<IntWritable> values, Context context) throws IOException, InterruptedException {
            int sum = 0;
            for (IntWritable val : values) {
                sum += val.get();
            }
            result.set(sum);
            context.write(key, result);
        }
    }

    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        Job job = Job.getInstance(conf, "word count");
        job.setJarByClass(WordCount.class);
        job.setMapperClass(TokenizerMapper.class);
        job.setCombinerClass(IntSumReducer.class);
        job.setReducerClass(IntSumReducer.class);
        job.setOutputKeyClass(Text.class);
        job.setOutputValueClass(IntWritable.class);
        FileInputFormat.addInputPath(job, new Path(args[0]));
        FileOutputFormat.setOutputPath(job, new Path(args[1]));
        System.exit(job.waitForCompletion(true) ? 0 : 1);
    }
}
```

### 4. MapReduce优化策略
- **数据本地化**：将计算任务分配到数据所在的节点
- **Combiner**：在Map端进行局部聚合，减少网络传输
- **压缩**：对Map输出和Reduce输出进行压缩
- **合理设置Map和Reduce任务数量**
- **使用自定义分区器**：优化数据分布

## 五、YARN详解

### 1. YARN架构
- **ResourceManager**：全局资源管理器，负责资源分配和调度
- **NodeManager**：每个节点的资源管理器，负责节点上的资源管理
- **ApplicationMaster**：每个应用的管理器，负责应用的执行
- **Container**：资源容器，包含CPU、内存等资源

### 2. YARN工作流程
1. 客户端提交应用到YARN
2. ResourceManager分配第一个Container给ApplicationMaster
3. ApplicationMaster启动并向ResourceManager申请资源
4. ResourceManager分配资源给ApplicationMaster
5. ApplicationMaster启动任务并监控执行
6. 任务执行完成后，ApplicationMaster释放资源

## 六、Hive详解

### 1. Hive简介
Hive是一个基于Hadoop的数据仓库工具，提供SQL查询能力，将SQL转换为MapReduce任务执行。

### 2. Hive架构
- **用户接口**：CLI、JDBC/ODBC、Web UI
- **元数据存储**：存储表结构、分区信息等元数据
- **执行引擎**：将SQL转换为MapReduce任务
- **Hadoop**：执行MapReduce任务和存储数据

### 3. Hive数据类型
- **基本数据类型**：INT、BIGINT、FLOAT、DOUBLE、STRING、BOOLEAN等
- **复合数据类型**：ARRAY、MAP、STRUCT

### 4. HiveQL示例
```sql
-- 创建表
CREATE TABLE employees (
    id INT,
    name STRING,
    salary FLOAT,
    department STRING
) ROW FORMAT DELIMITED FIELDS TERMINATED BY '\t';

-- 加载数据
LOAD DATA LOCAL INPATH '/path/to/data' INTO TABLE employees;

-- 查询数据
SELECT department, AVG(salary) FROM employees GROUP BY department;

-- 创建分区表
CREATE TABLE sales (
    id INT,
    product STRING,
    amount FLOAT
) PARTITIONED BY (year INT, month INT) ROW FORMAT DELIMITED FIELDS TERMINATED BY '\t';

-- 插入分区数据
INSERT INTO TABLE sales PARTITION (year=2023, month=1) VALUES (1, 'A', 100.0);
```

## 七、HBase详解

### 1. HBase简介
HBase是一个分布式的、面向列的NoSQL数据库，基于HDFS存储数据。

### 2. HBase架构
- **HMaster**：管理集群，分配Region
- **RegionServer**：处理数据读写请求
- **Region**：数据分片，每个Region包含一定范围的数据
- **HDFS**：存储数据文件

### 3. HBase数据模型
- **表**：由行和列族组成
- **行**：由行键（RowKey）唯一标识
- **列族**：包含多个列
- **列**：由列族和列限定符组成
- **单元格**：行键、列族、列限定符和时间戳唯一标识的单元

### 4. HBase操作示例
```java
// 创建HBase配置
Configuration conf = HBaseConfiguration.create();

// 创建连接
Connection connection = ConnectionFactory.createConnection(conf);

// 获取表
Table table = connection.getTable(TableName.valueOf("users"));

// 插入数据
Put put = new Put(Bytes.toBytes("user1"));
put.addColumn(Bytes.toBytes("info"), Bytes.toBytes("name"), Bytes.toBytes("John"));
put.addColumn(Bytes.toBytes("info"), Bytes.toBytes("age"), Bytes.toBytes("30"));
table.put(put);

// 查询数据
Get get = new Get(Bytes.toBytes("user1"));
Result result = table.get(get);
byte[] name = result.getValue(Bytes.toBytes("info"), Bytes.toBytes("name"));
System.out.println(Bytes.toString(name));

// 扫描数据
Scan scan = new Scan();
scan.addColumn(Bytes.toBytes("info"), Bytes.toBytes("name"));
ResultScanner scanner = table.getScanner(scan);
for (Result res : scanner) {
    System.out.println(Bytes.toString(res.getRow()) + ": " + Bytes.toString(res.getValue(Bytes.toBytes("info"), Bytes.toBytes("name"))));
}

// 关闭资源
table.close();
connection.close();
```

## 八、Spark详解

### 1. Spark简介
Spark是一个快速、通用的分布式计算框架，支持批处理、流处理、机器学习等多种计算模式。

### 2. Spark核心概念
- **RDD（Resilient Distributed Dataset）**：弹性分布式数据集，Spark的基本数据抽象
- **Driver**：执行main函数，创建SparkContext
- **Executor**：在工作节点上执行任务
- **Job**：由多个Stage组成的计算任务
- **Stage**：由多个Task组成的阶段
- **Task**：在Executor上执行的最小计算单元

### 3. Spark RDD操作
- **转换操作（Transformations）**：延迟执行，如map、filter、reduceByKey等
- **行动操作（Actions）**：立即执行，如count、collect、save等

### 4. Spark示例：WordCount
```scala
import org.apache.spark.sql.SparkSession

object WordCount {
  def main(args: Array[String]): Unit = {
    val spark = SparkSession.builder()
      .appName("WordCount")
      .master("local[*]")
      .getOrCreate()
    
    val lines = spark.sparkContext.textFile("hdfs://path/to/input")
    val words = lines.flatMap(_.split(" "))
    val wordCounts = words.map((_, 1)).reduceByKey(_ + _)
    wordCounts.saveAsTextFile("hdfs://path/to/output")
    
    spark.stop()
  }
}
```

### 3. HDFS的架构是什么？NameNode和DataNode的作用分别是什么？

**详细答案**：

- **NameNode**：管理文件系统的命名空间，存储元数据（文件路径、文件大小、数据块位置等）

- 负责客户端的文件操作请求
- 管理DataNode的状态

DataNode的作用：
- 存储实际的数据块
- 执行数据块的读写操作
- 定期向NameNode汇报心跳和块信息
- 参与数据块的复制

### 4. MapReduce的工作原理是什么？

**详细答案**：
MapReduce是一种分布式计算模型，将计算分为两个阶段：

- **Map阶段**：
  1. 输入数据被分割成多个输入分片
  2. 每个输入分片由一个Map任务处理
  3. Map任务读取输入数据，生成键值对
  4. Map输出被写入本地磁盘

- **Reduce阶段**：
  1. 所有Map输出按照key分组
  2. 每个分组由一个Reduce任务处理
  3. Reduce任务读取分组数据，执行聚合操作
  4. Reduce输出被写入HDFS

### 9. 什么是数据倾斜？如何解决数据倾斜？

**详细答案**：
数据倾斜是指在分布式计算中，数据分布不均匀，导致某些任务处理的数据量远大于其他任务，从而影响整体计算性能。

解决数据倾斜的方法：
- **数据预处理**：对数据进行清洗和转换，减少倾斜
- **增加Reduce任务数量**：分散负载
- **使用自定义分区器**：根据数据特点设计分区策略
- **局部聚合**：在Map端进行局部聚合，减少数据传输
- **加盐（Salting）**：对倾斜的key添加随机前缀，分散到不同的分区
- **使用Broadcast Join**：对于小表，使用广播变量减少数据传输

**代码示例**：
```scala
// 使用加盐解决数据倾斜
val rdd = spark.sparkContext.parallelize(Seq(
  ("key1", 1), ("key1", 2), ("key1", 3), // 倾斜的key
  ("key2", 4), ("key3", 5)
))

// 加盐
val saltedRdd = rdd.map{ case (key, value) =>
  val salt = scala.util.Random.nextInt(10)
  (s"$key_$salt", value)
}

// 局部聚合
val aggregatedRdd = saltedRdd.reduceByKey(_ + _)

// 去除盐
val resultRdd = aggregatedRdd.map{ case (key, value) =>
  val originalKey = key.split("_")(0)
  (originalKey, value)
}.reduceByKey(_ + _)
```

### 11. 什么是Spark Streaming？它是如何工作的？

**详细答案**：
Spark Streaming是Spark的流处理组件，用于处理实时数据流。它的工作原理是：
- 将实时数据流分成小的批次
- 对每个批次使用Spark Core进行处理
- 输出处理结果

Spark Streaming支持多种数据源，如Kafka、Flume、TCP sockets等。

**代码示例**：
```scala
import org.apache.spark.streaming._
import org.apache.spark.streaming.kafka._

val ssc = new StreamingContext(sparkConf, Seconds(5))

val kafkaParams = Map[String, String](
  "bootstrap.servers" -> "localhost:9092",
  "key.deserializer" -> classOf[StringDeserializer].getName,
  "value.deserializer" -> classOf[StringDeserializer].getName,
  "group.id" -> "spark-streaming-group"
)

val topics = Set("input-topic")
val stream = KafkaUtils.createDirectStream[String, String, StringDeserializer, StringDeserializer](
  ssc, kafkaParams, topics
)

val wordCounts = stream
  .flatMap(_._2.split(" "))
  .map((_, 1))
  .reduceByKey(_ + _)

wordCounts.print()

ssc.start()
ssc.awaitTermination()
```

### 12. 什么是Hive分区表？为什么要使用分区表？

**详细答案**：
Hive分区表是指将表数据按照一定的规则（如时间、地区等）分割成多个分区，每个分区对应一个目录。

使用分区表的原因：
- **提高查询性能**：只扫描相关分区，减少数据扫描量
- **便于数据管理**：可以按分区进行数据加载、删除等操作
- **优化存储**：不同分区可以存储在不同的存储介质上

**代码示例**：
```sql
-- 创建分区表
CREATE TABLE sales (
    id INT,
    product STRING,
    amount FLOAT
) PARTITIONED BY (year INT, month INT) ROW FORMAT DELIMITED FIELDS TERMINATED BY '\t';

-- 插入分区数据
INSERT INTO TABLE sales PARTITION (year=2023, month=1) VALUES (1, 'A', 100.0);

-- 查询特定分区
SELECT * FROM sales WHERE year=2023 AND month=1;
```

### 13. 什么是HBase的RowKey设计？如何设计一个好的RowKey？

**详细答案**：
HBase的RowKey是行的唯一标识，设计RowKey时需要考虑以下因素：
- **唯一性**：确保RowKey唯一
- **排序性**：HBase按RowKey字典序排序，合理的排序可以提高查询性能
- **散列性**：均匀分布数据，避免热点问题
- **长度**：RowKey长度应适中，过长会增加存储开销

设计好的RowKey的方法：
- **反转时间戳**：如使用 `(Long.MAX_VALUE - timestamp)` 作为RowKey的一部分
- **加盐**：添加随机前缀，分散数据
- **哈希**：使用哈希函数对RowKey进行处理
- **组合键**：将多个字段组合成RowKey

**示例**：
设计用户行为数据的RowKey：
```
<user_id>_<timestamp>
```

这样设计可以按用户ID查询，也可以按时间范围查询。

### 15. 什么是Flink？它与Spark Streaming的区别是什么？

**详细答案**：
Flink是一个流处理框架，专注于低延迟、高吞吐的流处理。它的特点包括：
- **真正的流处理**：基于事件的流处理，而不是微批处理
- **低延迟**：毫秒级延迟
- **高吞吐**：支持每秒处理百万级事件
- **状态管理**：内置状态管理，支持容错
- **Exactly-once语义**：保证消息只被处理一次

Flink与Spark Streaming的区别：
| 特性 | Flink | Spark Streaming |
|------|-------|----------------|
| 处理模型 | 真正的流处理 | 微批处理 |
| 延迟 | 毫秒级 | 秒级 |
| 状态管理 | 内置状态管理 | 依赖外部存储 |
| 容错机制 | 检查点机制 | RDD lineage |
| 窗口操作 | 支持事件时间和处理时间 | 主要支持处理时间 |

**代码示例**：
Flink WordCount：
```java
StreamExecutionEnvironment env = StreamExecutionEnvironment.getExecutionEnvironment();

DataStream<String> text = env.socketTextStream("localhost", 9999);

DataStream<Tuple2<String, Integer>> counts = text
    .flatMap(new FlatMapFunction<String, Tuple2<String, Integer>>() {
        @Override
        public void flatMap(String value, Collector<Tuple2<String, Integer>> out) {
            for (String word : value.split(" ")) {
                out.collect(new Tuple2<>(word, 1));
            }
        }
    })
    .keyBy(0)
    .timeWindow(Time.seconds(5))
    .sum(1);

counts.print();

env.execute("WordCount");
```

## 十一、代码优化建议

1. **合理使用数据结构**：根据数据特点选择合适的数据结构，如使用RDD、DataFrame或Dataset
2. **避免 shuffle 操作**：shuffle操作会导致网络传输，影响性能

5. **使用缓存**：对于重复使用的RDD，使用cache()或persist()方法缓存

7. **避免使用collect()**：collect()会将数据拉取到Driver节点，可能导致内存溢出
8. **使用分区器**：根据数据特点选择合适的分区器
9. **合理设置批处理大小**：对于流处理，设置合适的批处理大小
