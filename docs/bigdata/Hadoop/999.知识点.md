# 大数据基础知识点及面试题

## 一、大数据基础概念

### 1. 什么是大数据？
大数据是指无法在合理时间内用传统IT技术和工具进行处理的海量数据集合。大数据的特点通常用4V来概括：
- **Volume（量大）**：数据量巨大，从TB级别跃升至PB级别
- **Velocity（速度快）**：数据生成和处理速度快
- **Variety（多样）**：数据类型多样，包括结构化、半结构化和非结构化数据
- **Value（价值高）**：数据价值密度低，但整体价值高

### 2. 大数据的应用场景
- **电商推荐系统**：根据用户行为数据推荐商品
- **金融风控**：分析用户交易数据，识别欺诈行为
- **智能交通**：分析交通流量数据，优化交通管理
- **医疗健康**：分析医疗数据，辅助诊断和治疗
- **智慧城市**：整合城市各类数据，提升城市管理水平

### 3. 大数据技术栈
- **数据采集**：Flume、Kafka、Logstash等
- **数据存储**：HDFS、HBase、MongoDB、Cassandra等
- **数据处理**：MapReduce、Spark、Flink等
- **数据查询**：Hive、Impala、Presto等
- **数据可视化**：Tableau、Power BI、ECharts等

## 二、Hadoop生态系统

### 1. Hadoop简介
Hadoop是一个开源的分布式计算框架，专为处理大规模数据而设计。它的核心组件包括：
- **HDFS（Hadoop Distributed File System）**：分布式文件系统，用于存储海量数据
- **MapReduce**：分布式计算框架，用于处理大规模数据
- **YARN（Yet Another Resource Negotiator）**：资源管理和调度系统
- **Common**：通用工具和库

### 2. Hadoop的优势
- **高可靠性**：数据多副本存储，容错能力强
- **高扩展性**：可以横向扩展，支持数千个节点
- **高效性**：采用分布式计算，处理速度快
- **高容错性**：节点故障自动检测和恢复
- **低成本**：使用普通硬件，成本低

### 3. Hadoop生态系统组件
- **HDFS**：分布式文件系统
- **MapReduce**：分布式计算框架
- **YARN**：资源管理和调度系统
- **Hive**：数据仓库工具，提供SQL查询能力
- **HBase**：分布式NoSQL数据库
- **Pig**：数据流处理工具
- **Spark**：内存计算框架
- **Flink**：流处理框架
- **ZooKeeper**：分布式协调服务
- **Oozie**：工作流调度系统
- **Sqoop**：数据导入/导出工具
- **Flume**：日志收集工具

## 三、HDFS详解

### 1. HDFS架构
- **NameNode**：管理文件系统的命名空间，存储元数据
- **DataNode**：存储实际数据，执行数据块的读写操作
- **SecondaryNameNode**：辅助NameNode，定期合并编辑日志，减轻NameNode负担

### 2. HDFS数据块
- 默认块大小：128MB（Hadoop 2.x）
- 块的副本数：默认3个
- 块的存储：数据块分散存储在不同的DataNode上

### 3. HDFS读写流程
- **读流程**：
  1. 客户端向NameNode请求文件元数据
  2. NameNode返回文件的数据块位置
  3. 客户端直接从DataNode读取数据

- **写流程**：
  1. 客户端向NameNode请求写入文件
  2. NameNode分配数据块和DataNode
  3. 客户端将数据写入DataNode
  4. DataNode复制数据到其他副本节点

### 4. HDFS命令行操作
```bash
# 查看HDFS目录
hadoop fs -ls /

# 创建目录
hadoop fs -mkdir /user/hadoop

# 上传文件
hadoop fs -put localfile /user/hadoop/

# 下载文件
hadoop fs -get /user/hadoop/file localfile

# 删除文件
hadoop fs -rm /user/hadoop/file
```

## 四、MapReduce详解

### 1. MapReduce原理
MapReduce是一种分布式计算模型，将计算分为两个阶段：
- **Map阶段**：将输入数据分割成小块，并行处理
- **Reduce阶段**：合并Map阶段的结果，产生最终输出

### 2. MapReduce执行流程
1. **输入分片**：将输入数据分割成多个输入分片
2. **Map任务**：并行处理每个输入分片
3. **Shuffle**：将Map输出按照key分组，发送到对应的Reduce任务
4. **Reduce任务**：处理分组后的数据，产生最终结果
5. **输出**：将结果写入HDFS

### 3. MapReduce示例：WordCount
```java
public class WordCount {
    public static class TokenizerMapper extends Mapper<Object, Text, Text, IntWritable> {
        private final static IntWritable one = new IntWritable(1);
        private Text word = new Text();

        public void map(Object key, Text value, Context context) throws IOException, InterruptedException {
            StringTokenizer itr = new StringTokenizer(value.toString());
            while (itr.hasMoreTokens()) {
                word.set(itr.nextToken());
                context.write(word, one);
            }
        }
    }

    public static class IntSumReducer extends Reducer<Text, IntWritable, Text, IntWritable> {
        private IntWritable result = new IntWritable();

        public void reduce(Text key, Iterable<IntWritable> values, Context context) throws IOException, InterruptedException {
            int sum = 0;
            for (IntWritable val : values) {
                sum += val.get();
            }
            result.set(sum);
            context.write(key, result);
        }
    }

    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        Job job = Job.getInstance(conf, "word count");
        job.setJarByClass(WordCount.class);
        job.setMapperClass(TokenizerMapper.class);
        job.setCombinerClass(IntSumReducer.class);
        job.setReducerClass(IntSumReducer.class);
        job.setOutputKeyClass(Text.class);
        job.setOutputValueClass(IntWritable.class);
        FileInputFormat.addInputPath(job, new Path(args[0]));
        FileOutputFormat.setOutputPath(job, new Path(args[1]));
        System.exit(job.waitForCompletion(true) ? 0 : 1);
    }
}
```

### 4. MapReduce优化策略
- **数据本地化**：将计算任务分配到数据所在的节点
- **Combiner**：在Map端进行局部聚合，减少网络传输
- **压缩**：对Map输出和Reduce输出进行压缩
- **合理设置Map和Reduce任务数量**
- **使用自定义分区器**：优化数据分布

## 五、YARN详解

### 1. YARN架构
- **ResourceManager**：全局资源管理器，负责资源分配和调度
- **NodeManager**：每个节点的资源管理器，负责节点上的资源管理
- **ApplicationMaster**：每个应用的管理器，负责应用的执行
- **Container**：资源容器，包含CPU、内存等资源

### 2. YARN工作流程
1. 客户端提交应用到YARN
2. ResourceManager分配第一个Container给ApplicationMaster
3. ApplicationMaster启动并向ResourceManager申请资源
4. ResourceManager分配资源给ApplicationMaster
5. ApplicationMaster启动任务并监控执行
6. 任务执行完成后，ApplicationMaster释放资源

## 六、Hive详解

### 1. Hive简介
Hive是一个基于Hadoop的数据仓库工具，提供SQL查询能力，将SQL转换为MapReduce任务执行。

### 2. Hive架构
- **用户接口**：CLI、JDBC/ODBC、Web UI
- **元数据存储**：存储表结构、分区信息等元数据
- **执行引擎**：将SQL转换为MapReduce任务
- **Hadoop**：执行MapReduce任务和存储数据

### 3. Hive数据类型
- **基本数据类型**：INT、BIGINT、FLOAT、DOUBLE、STRING、BOOLEAN等
- **复合数据类型**：ARRAY、MAP、STRUCT

### 4. HiveQL示例
```sql
-- 创建表
CREATE TABLE employees (
    id INT,
    name STRING,
    salary FLOAT,
    department STRING
) ROW FORMAT DELIMITED FIELDS TERMINATED BY '\t';

-- 加载数据
LOAD DATA LOCAL INPATH '/path/to/data' INTO TABLE employees;

-- 查询数据
SELECT department, AVG(salary) FROM employees GROUP BY department;

-- 创建分区表
CREATE TABLE sales (
    id INT,
    product STRING,
    amount FLOAT
) PARTITIONED BY (year INT, month INT) ROW FORMAT DELIMITED FIELDS TERMINATED BY '\t';

-- 插入分区数据
INSERT INTO TABLE sales PARTITION (year=2023, month=1) VALUES (1, 'A', 100.0);
```

## 七、HBase详解

### 1. HBase简介
HBase是一个分布式的、面向列的NoSQL数据库，基于HDFS存储数据。

### 2. HBase架构
- **HMaster**：管理集群，分配Region
- **RegionServer**：处理数据读写请求
- **Region**：数据分片，每个Region包含一定范围的数据
- **HDFS**：存储数据文件

### 3. HBase数据模型
- **表**：由行和列族组成
- **行**：由行键（RowKey）唯一标识
- **列族**：包含多个列
- **列**：由列族和列限定符组成
- **单元格**：行键、列族、列限定符和时间戳唯一标识的单元

### 4. HBase操作示例
```java
// 创建HBase配置
Configuration conf = HBaseConfiguration.create();

// 创建连接
Connection connection = ConnectionFactory.createConnection(conf);

// 获取表
Table table = connection.getTable(TableName.valueOf("users"));

// 插入数据
Put put = new Put(Bytes.toBytes("user1"));
put.addColumn(Bytes.toBytes("info"), Bytes.toBytes("name"), Bytes.toBytes("John"));
put.addColumn(Bytes.toBytes("info"), Bytes.toBytes("age"), Bytes.toBytes("30"));
table.put(put);

// 查询数据
Get get = new Get(Bytes.toBytes("user1"));
Result result = table.get(get);
byte[] name = result.getValue(Bytes.toBytes("info"), Bytes.toBytes("name"));
System.out.println(Bytes.toString(name));

// 扫描数据
Scan scan = new Scan();
scan.addColumn(Bytes.toBytes("info"), Bytes.toBytes("name"));
ResultScanner scanner = table.getScanner(scan);
for (Result res : scanner) {
    System.out.println(Bytes.toString(res.getRow()) + ": " + Bytes.toString(res.getValue(Bytes.toBytes("info"), Bytes.toBytes("name"))));
}

// 关闭资源
table.close();
connection.close();
```

## 八、Spark详解

### 1. Spark简介
Spark是一个快速、通用的分布式计算框架，支持批处理、流处理、机器学习等多种计算模式。

### 2. Spark核心概念
- **RDD（Resilient Distributed Dataset）**：弹性分布式数据集，Spark的基本数据抽象
- **Driver**：执行main函数，创建SparkContext
- **Executor**：在工作节点上执行任务
- **Job**：由多个Stage组成的计算任务
- **Stage**：由多个Task组成的阶段
- **Task**：在Executor上执行的最小计算单元

### 3. Spark RDD操作
- **转换操作（Transformations）**：延迟执行，如map、filter、reduceByKey等
- **行动操作（Actions）**：立即执行，如count、collect、save等

### 4. Spark示例：WordCount
```scala
import org.apache.spark.sql.SparkSession

object WordCount {
  def main(args: Array[String]): Unit = {
    val spark = SparkSession.builder()
      .appName("WordCount")
      .master("local[*]")
      .getOrCreate()
    
    val lines = spark.sparkContext.textFile("hdfs://path/to/input")
    val words = lines.flatMap(_.split(" "))
    val wordCounts = words.map((_, 1)).reduceByKey(_ + _)
    wordCounts.saveAsTextFile("hdfs://path/to/output")
    
    spark.stop()
  }
}
```

## 十、高频面试题及详细答案

### 1. 什么是大数据？大数据的4V特征是什么？

**详细答案**：
大数据是指无法在合理时间内用传统IT技术和工具进行处理的海量数据集合。大数据的4V特征包括：
- **Volume（量大）**：数据量巨大，从TB级别跃升至PB级别
- **Velocity（速度快）**：数据生成和处理速度快
- **Variety（多样）**：数据类型多样，包括结构化、半结构化和非结构化数据
- **Value（价值高）**：数据价值密度低，但整体价值高

**示例**：
电商平台每天产生的用户行为数据（浏览、点击、购买等）就是典型的大数据，具有数据量大、实时性要求高、数据类型多样等特点。

### 2. 介绍一下MapReduce的基本流程，shuffle的作用是什么？

**详细答案**：
MapReduce的基本流程包括以下几个步骤：

1. **输入分片**：将输入数据分割成多个输入分片（InputSplit）
2. **Map任务**：每个输入分片由一个Map任务处理，Map任务读取输入数据，生成键值对
3. **Shuffle过程**：
   - Map端排序：将Map输出按照key排序
   - 分区：将排序后的数据按照key的哈希值分配到不同的分区
   - 合并（Combine）：在Map端进行局部聚合，减少数据传输
   - Reduce端拉取：Reduce任务从各个Map任务拉取属于自己分区的数据
   - Reduce端排序：对拉取的数据进行排序
4. **Reduce任务**：处理排序后的数据，执行聚合操作，产生最终结果
5. **输出**：将结果写入HDFS

**Shuffle的作用**：
- **数据传输**：将Map任务的输出数据传输到Reduce任务
- **数据排序**：确保相同key的数据被分到同一个Reduce任务
- **数据聚合**：在Map端进行局部聚合，减少网络传输量
- **负载均衡**：通过分区策略，确保Reduce任务的负载均衡

### 3. 简单介绍一下HDFS和YARN

**详细答案**：

**HDFS（Hadoop Distributed File System）**：
- **定义**：HDFS是Hadoop的分布式文件系统，专为存储海量数据而设计
- **架构**：
  - NameNode：管理文件系统的命名空间，存储元数据
  - DataNode：存储实际数据，执行数据块的读写操作
  - SecondaryNameNode：辅助NameNode，定期合并编辑日志
- **特点**：
  - 高可靠性：数据多副本存储
  - 高扩展性：可以横向扩展
  - 高容错性：节点故障自动检测和恢复
  - 适合存储大文件：默认块大小为128MB

**YARN（Yet Another Resource Negotiator）**：
- **定义**：YARN是Hadoop的资源管理和调度系统
- **架构**：
  - ResourceManager：全局资源管理器，负责资源分配和调度
  - NodeManager：每个节点的资源管理器，负责节点上的资源管理
  - ApplicationMaster：每个应用的管理器，负责应用的执行
  - Container：资源容器，包含CPU、内存等资源
- **特点**：
  - 资源统一管理：集中管理集群资源
  - 多框架支持：支持MapReduce、Spark等多种计算框架
  - 灵活的调度策略：支持多种调度算法
  - 高扩展性：可以支持大规模集群

### 4. RDD是什么？

**详细答案**：
RDD（Resilient Distributed Dataset）是Spark的基本数据抽象，具有以下特点：

- **弹性**：数据丢失时可以自动恢复
- **分布式**：数据分布在多个节点上
- **不可变**：一旦创建就不能修改
- **分区**：数据被分为多个分区
- **Lineage**：记录了RDD的创建过程，用于容错
- **支持两种操作**：
  - 转换操作（Transformations）：延迟执行，如map、filter、reduceByKey等
  - 行动操作（Actions）：立即执行，如count、collect、save等

**示例**：
```scala
// 创建RDD
val rdd = spark.sparkContext.parallelize(Seq(1, 2, 3, 4, 5))

// 转换操作
val mappedRdd = rdd.map(_ * 2)

// 行动操作
val result = mappedRdd.collect()
```

### 5. Spark如何划分stage？

**详细答案**：
Spark根据RDD之间的依赖关系来划分stage，具体规则如下：

1. **窄依赖**：父RDD的每个分区只被一个子RDD的分区使用，如map、filter等操作，这些操作可以在同一个stage中执行
2. **宽依赖**：父RDD的每个分区被多个子RDD的分区使用，如shuffle、reduceByKey等操作，这些操作会触发stage的划分

**划分过程**：
- 从最终的RDD（通常是执行行动操作的RDD）开始，向上遍历依赖关系
- 遇到宽依赖时，就划分一个新的stage
- 每个stage包含一系列窄依赖操作
- 一个stage中的所有任务可以并行执行

**示例**：
```scala
val rdd1 = sc.textFile("input")
val rdd2 = rdd1.flatMap(_.split(" "))  // 窄依赖
val rdd3 = rdd2.map((_, 1))  // 窄依赖
val rdd4 = rdd3.reduceByKey(_ + _)  // 宽依赖，划分新stage
val rdd5 = rdd4.mapValues(_ * 2)  // 窄依赖
rdd5.collect()  // 行动操作
```

在这个例子中，stage1包含rdd1、rdd2、rdd3的操作，stage2包含rdd4、rdd5的操作。

### 6. 介绍一下Spark的宽窄依赖

**详细答案**：

**窄依赖**：
- **定义**：父RDD的每个分区只被一个子RDD的分区使用
- **特点**：
  - 数据不需要跨节点传输
  - 可以流水线执行
  - 容错成本低，只需要重新计算丢失的分区
- **示例**：map、filter、union、mapPartitions等操作

**宽依赖**：
- **定义**：父RDD的每个分区被多个子RDD的分区使用
- **特点**：
  - 数据需要跨节点传输（shuffle）
  - 不能流水线执行
  - 容错成本高，需要重新计算所有相关的父分区
- **示例**：reduceByKey、groupByKey、join等操作

**宽窄依赖的区别**：
| 特性 | 窄依赖 | 宽依赖 |
|------|--------|--------|
| 数据传输 | 不需要跨节点传输 | 需要跨节点传输（shuffle） |
| 执行方式 | 可以流水线执行 | 不能流水线执行 |
| 容错成本 | 低，只需要重新计算丢失的分区 | 高，需要重新计算所有相关的父分区 |
| 划分stage | 不会触发stage划分 | 会触发stage划分 |

### 7. Spark任务调度的流程？

**详细答案**：
Spark任务调度的流程包括以下几个步骤：

1. **作业提交**：客户端提交作业（Job）到SparkContext
2. **DAG创建**：SparkContext将作业转换为DAG（有向无环图）
3. **Stage划分**：根据RDD之间的依赖关系，将DAG划分为多个Stage
4. **Task创建**：每个Stage被拆分为多个Task（根据分区数量）
5. **Task调度**：
   - Driver将Task发送到Executor
   - Executor执行Task并返回结果
   - Driver监控Task的执行状态
6. **结果汇总**：所有Task执行完成后，Driver汇总结果并返回给客户端

**调度级别**：
- **作业（Job）**：由多个Stage组成的计算任务
- **Stage**：由多个Task组成的阶段，每个Stage对应一个宽依赖
- **Task**：在Executor上执行的最小计算单元

### 8. Spark调优，如何处理数据倾斜？

**详细答案**：
数据倾斜是指在分布式计算中，数据分布不均匀，导致某些任务处理的数据量远大于其他任务，从而影响整体计算性能。

**处理数据倾斜的方法**：

1. **数据预处理**：
   - 对数据进行清洗和转换，减少倾斜
   - 过滤掉异常数据

2. **增加并行度**：
   - 增加分区数量，分散负载
   - 设置合适的spark.default.parallelism参数

3. **使用自定义分区器**：
   - 根据数据特点设计分区策略
   - 确保数据均匀分布

4. **局部聚合**：
   - 在Map端进行局部聚合，减少数据传输
   - 使用combiner或map-side aggregation

5. **加盐（Salting）**：
   - 对倾斜的key添加随机前缀，分散到不同的分区
   - 处理完成后再去除前缀

6. **使用Broadcast Join**：
   - 对于小表，使用广播变量减少数据传输
   - 避免shuffle操作

7. **使用随机采样**：
   - 对倾斜的key进行随机采样，分析倾斜原因
   - 针对具体情况采取相应措施

**代码示例**：
```scala
// 使用加盐解决数据倾斜
val rdd = spark.sparkContext.parallelize(Seq(
  ("key1", 1), ("key1", 2), ("key1", 3), // 倾斜的key
  ("key2", 4), ("key3", 5)
))

// 加盐
val saltedRdd = rdd.map{ case (key, value) =>
  val salt = scala.util.Random.nextInt(10)
  (s"$key_$salt", value)
}

// 局部聚合
val aggregatedRdd = saltedRdd.reduceByKey(_ + _)

// 去除盐
val resultRdd = aggregatedRdd.map{ case (key, value) =>
  val originalKey = key.split("_")(0)
  (originalKey, value)
}.reduceByKey(_ + _)
```

### 11. 数仓怎么分层的，分层的目的是什么？

**详细答案**：

**数仓分层**：
1. **原始数据层（ODS）**：
   - 存储原始数据，保持数据的原始状态
   - 数据格式与来源系统一致
   - 通常按时间分区存储

2. **数据仓库层（DW）**：
   - **维度层（DIM）**：存储维度表，如用户、商品、地区等
   - **事实层（FACT）**：存储事实表，如交易、日志等
   - 按照业务主题组织数据

3. **数据集市层（DM）**：
   - 面向特定业务部门或应用的数据集合
   - 提供汇总和分析数据
   - 数据已经过预处理，便于快速查询

4. **应用层（APP）**：
   - 直接面向用户的数据，如报表、Dashboard等
   - 数据已经过高度汇总和处理

**分层的目的**：
- **数据隔离**：不同层次的数据相互隔离，便于管理和维护
- **数据质量**：通过分层处理，提高数据质量
- **性能优化**：针对不同层次的需求，优化存储和查询性能
- **灵活性**：支持不同类型的分析需求
- **可维护性**：便于数据的更新和维护
- **可扩展性**：便于系统的扩展和升级

### 12. 星型模型和雪花模型的区别，以及各自的优缺点？

**详细答案**：

**星型模型**：
- **结构**：以事实表为中心，周围环绕维度表，维度表直接连接到事实表
- **优点**：
  - 查询性能好，连接操作少
  - 模型简单，易于理解和维护
  - 适合OLAP分析
- **缺点**：
  - 数据冗余度高
  - 维度表可能变得很大

**雪花模型**：
- **结构**：维度表被进一步规范化，分解为多个关联的表
- **优点**：
  - 数据冗余度低
  - 维度表更加规范化
  - 节省存储空间
- **缺点**：
  - 查询性能较差，连接操作多
  - 模型复杂，不易理解和维护
  - 不适合复杂的OLAP分析

**区别**：
| 特性 | 星型模型 | 雪花模型 |
|------|----------|----------|
| 维度表结构 | 非规范化 | 规范化 |
| 连接操作 | 少 | 多 |
| 查询性能 | 好 | 较差 |
| 数据冗余 | 高 | 低 |
| 模型复杂度 | 简单 | 复杂 |
| 维护成本 | 低 | 高 |

### 18. 班级成绩表，取出每门课程的前三名？

**详细答案**：

**表结构假设**：
```
班级成绩表（class_score）
- student_id：学生ID
- student_name：学生姓名
- course_name：课程名称
- score：分数
```

**SQL查询（使用窗口函数）**：
```sql
SELECT
    student_id,
    student_name,
    course_name,
    score
FROM (
    SELECT
        student_id,
        student_name,
        course_name,
        score,
        ROW_NUMBER() OVER (PARTITION BY course_name ORDER BY score DESC) AS rank
    FROM
        class_score
) AS ranked
WHERE
    rank <= 3;
```

**解释**：
- `PARTITION BY course_name`：按课程分组
- `ORDER BY score DESC`：按分数降序排列
- `ROW_NUMBER()`：为每个分组内的记录分配行号
- `WHERE rank <= 3`：只取前三名

**注意**：如果有分数相同的情况，可以使用`RANK()`或`DENSE_RANK()`函数来处理：
- `RANK()`：相同分数会有相同的排名，后续排名会跳过
- `DENSE_RANK()`：相同分数会有相同的排名，后续排名不会跳过

### 20. 连续7天登录的用户？

**详细答案**：

**表结构假设**：
```
用户登录表（user_login）
- user_id：用户ID
- login_date：登录日期
```

**SQL查询（使用窗口函数和日期计算）**：
```sql
WITH user_login_ranked AS (
    SELECT
        user_id,
        login_date,
        ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY login_date) AS rn
    FROM
        user_login
),
user_login_diff AS (
    SELECT
        user_id,
        login_date,
        DATE_SUB(login_date, INTERVAL rn DAY) AS grp
    FROM
        user_login_ranked
)
SELECT
    user_id
FROM
    user_login_diff
GROUP BY
    user_id,
    grp
HAVING
    COUNT(*) >= 7;
```

**解释**：
1. **第一步**：为每个用户的登录记录按日期排序，并分配行号
2. **第二步**：计算登录日期减去行号的差值，如果连续登录，这个差值会相同
3. **第三步**：按用户ID和差值分组，统计每组的记录数，如果大于等于7，则说明连续登录了7天

**另一种方法（使用LAG函数）**：
```sql
WITH user_login_diff AS (
    SELECT
        user_id,
        login_date,
        DATEDIFF(login_date, LAG(login_date) OVER (PARTITION BY user_id ORDER BY login_date)) AS day_diff
    FROM
        user_login
),
user_login_streak AS (
    SELECT
        user_id,
        login_date,
        CASE
            WHEN day_diff = 1 THEN 1
            ELSE 0
        END AS is_consecutive
    FROM
        user_login_diff
),
user_login_streak_group AS (
    SELECT
        user_id,
        login_date,
        SUM(is_consecutive) OVER (PARTITION BY user_id ORDER BY login_date) AS streak_group
    FROM
        user_login_streak
)
SELECT
    user_id
FROM
    user_login_streak_group
GROUP BY
    user_id,
    streak_group
HAVING
    COUNT(*) >= 7;
```

**解释**：
1. **第一步**：计算每个用户相邻登录日期之间的差值
2. **第二步**：判断是否连续登录（差值为1）
3. **第三步**：计算连续登录的分组
4. **第四步**：按用户ID和分组统计连续登录的天数，如果大于等于7，则说明连续登录了7天



## 十一、代码优化建议

1. **合理使用数据结构**：根据数据特点选择合适的数据结构，如使用RDD、DataFrame或Dataset
2. **避免 shuffle 操作**：shuffle操作会导致网络传输，影响性能
3. **使用广播变量**：对于小数据集，使用广播变量减少数据传输

5. **使用缓存**：对于重复使用的RDD，使用cache()或persist()方法缓存

7. **避免使用collect()**：collect()会将数据拉取到Driver节点，可能导致内存溢出
8. **使用分区器**：根据数据特点选择合适的分区器
9. **合理设置批处理大小**：对于流处理，设置合适的批处理大小
