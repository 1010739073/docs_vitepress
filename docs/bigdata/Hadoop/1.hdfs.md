# HDFS 分布式文件系统详解

## 1. HDFS 概述

HDFS (Hadoop Distributed File System) 是 Hadoop 的核心组件，用于存储海量数据。它设计用于运行在通用硬件上的分布式文件系统，具有高容错性和高吞吐量的特点。

### 1.1 HDFS 特性

- **高容错性**：数据自动复制，节点故障时自动恢复
- **高吞吐量**：适合批处理，不适合低延迟访问
- **大文件存储**：适合存储 TB、PB 级大文件
- **硬件成本低**：运行在普通硬件上
- **流式数据访问**：一次写入，多次读取

## 2. HDFS 架构

### 2.1 核心组件

- **NameNode**：主节点，管理文件系统元数据和命名空间
- **DataNode**：从节点，存储实际数据块
- **SecondaryNameNode**：辅助 NameNode，合并编辑日志

### 2.2 架构原理

```java
// HDFS 架构示意
/*
NameNode (主节点)
├── 维护文件系统树
├── 管理元数据
├── 处理客户端请求
└── 管理数据块映射

DataNode (从节点)
├── 存储数据块
├── 向 NameNode 汇报状态
├── 处理数据读写请求
└── 执行数据块复制
*/
```

## 3. HDFS 工作原理

### 3.1 文件读取流程

1. 客户端调用 `open()` 方法打开文件
2. DistributedFileSystem 向 NameNode 请求文件块位置
3. NameNode 返回数据块及其所在 DataNode 列表
4. 客户端直接与 DataNode 交互读取数据

```java
// HDFS 文件读取示例
public class HDFSReadExample {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        FileSystem fs = FileSystem.get(conf);
        
        Path path = new Path("hdfs://namenode:8020/user/data/input.txt");
        FSDataInputStream in = fs.open(path);
        
        BufferedReader reader = new BufferedReader(new InputStreamReader(in));
        String line;
        while ((line = reader.readLine()) != null) {
            System.out.println(line);
        }
        
        reader.close();
        fs.close();
    }
}
```

### 3.2 文件写入流程

1. 客户端调用 `create()` 方法创建文件
2. DistributedFileSystem 向 NameNode 请求创建新文件
3. NameNode 执行各种检查，若文件不存在则创建文件记录
4. 客户端写入数据，DataNode 以管道形式复制数据

```java
// HDFS 文件写入示例
public class HDFSWriteExample {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        FileSystem fs = FileSystem.get(conf);
        
        Path path = new Path("hdfs://namenode:8020/user/data/output.txt");
        FSDataOutputStream out = fs.create(path);
        
        String content = "Hello, HDFS!";
        out.write(content.getBytes());
        
        out.close();
        fs.close();
    }
}
```

## 4. HDFS 关键概念

### 4.1 数据块 (Block)

HDFS 将文件分割成固定大小的块（默认 128MB），每个块在多个 DataNode 上复制（默认 3 份）。

```bash
# 查看数据块信息
hdfs fsck /user/data -files -blocks -locations
```

### 4.2 副本机制

- **默认副本数**：3
- **副本放置策略**：
  - 第一个副本：本地机架
  - 第二个副本：不同机架
  - 第三个副本：同机架不同节点

```xml
<!-- hdfs-site.xml 配置副本数 -->
<property>
    <name>dfs.replication</name>
    <value>3</value>
</property>
```

### 4.3 心跳机制

DataNode 定期向 NameNode 发送心跳信号，报告存活状态和数据块信息。

```java
// DataNode 心跳配置
dfs.heartbeat.interval = 3  // 心跳间隔（秒）
dfs.namenode.heartbeat.recheck.interval = 300000  // 心跳重检间隔（毫秒）
```

## 5. HDFS 操作命令

### 5.1 基础操作

```bash
# 创建目录
hdfs dfs -mkdir /user/data

# 上传文件
hdfs dfs -put local_file.txt /user/data/

# 下载文件
hdfs dfs -get /user/data/file.txt local_file.txt

# 查看文件内容
hdfs dfs -cat /user/data/file.txt

# 删除文件
hdfs dfs -rm /user/data/file.txt

# 查看目录结构
hdfs dfs -ls -R /user/data
```

### 5.2 管理操作

```bash
# 查看文件系统状态
hdfs dfsadmin -report

# 安全模式操作
hdfs dfsadmin -safemode enter
hdfs dfsadmin -safemode leave

# 查看数据块信息
hdfs fsck /user/data -files -blocks

# 设置副本数
hdfs dfs -setrep -w 3 /user/data/file.txt
```

## 6. HDFS 高级特性

### 6.1 HDFS Federation

解决单 NameNode 内存限制问题，支持多个 NameNode 共享 DataNode。

```xml
<!-- hdfs-site.xml Federation 配置 -->
<property>
    <name>dfs.nameservices</name>
    <value>ns1,ns2</value>
</property>

<property>
    <name>dfs.namenode.rpc-address.ns1</name>
    <value>namenode1:8020</value>
</property>
```

### 6.2 HDFS HA (High Availability)

实现 NameNode 高可用，通过 Active/Standby 模式避免单点故障。

```xml
<!-- hdfs-site.xml HA 配置 -->
<property>
    <name>dfs.ha.namenodes.ns1</name>
    <value>nn1,nn2</value>
</property>

<property>
    <name>dfs.namenode.shared.edits.dir</name>
    <value>qjournal://node1:8485;node2:8485;node3:8485/ns1</value>
</property>
```

### 6.3 快照功能

创建文件系统的时间点副本，用于数据备份和恢复。

```bash
# 创建快照
hdfs dfs -createSnapshot /user/data snapshot1

# 查看快照
hdfs lsSnapshottableDir

# 恢复快照
hdfs dfs -cp /user/data/.snapshot/snapshot1/file.txt /user/data/restored_file.txt
```

## 8. HDFS 面试题

### 8.1 基础概念

**Q1: HDFS 的设计目标是什么？**

- 存储超大数据集
- 运行在廉价硬件上
- 高容错性和高可用性
- 流式数据访问模式
- 简化的文件系统模型

**Q2: HDFS 适合存储什么类型的文件？**

- 大文件（GB、TB 级别）
- 一次写入、多次读取的文件
- 批处理场景
- 不适合存储大量小文件

### 8.2 架构原理

**Q3: NameNode 和 DataNode 的作用是什么？**

- **NameNode**：管理文件系统元数据、命名空间、数据块映射
- **DataNode**：存储实际数据块、处理读写请求、定期向 NameNode 汇报

**Q4: HDFS 如何保证数据可靠性？**

- 数据块多副本机制
- 心跳检测和故障恢复
- 校验和数据完整性检查
- 自动副本重建

### 8.3 运维优化

**Q5: 如何解决 HDFS 小文件问题？**

- 使用 HDFS Archive (HAR)
- 合并小文件为大文件
- 使用 CombineFileInputFormat
- 定期清理无用小文件

**Q6: HDFS 的读写流程是怎样的？**

**读流程**：
1. 客户端向 NameNode 请求文件元数据
2. NameNode 返回数据块位置信息
3. 客户端直接从 DataNode 读取数据

**写流程**：
1. 客户端向 NameNode 请求创建文件
2. 数据按块分割并流水线写入 DataNode
3. DataNode 确认写入完成
4. NameNode 更新元数据

## 9. 代码示例

### 9.1 Java API 操作

```java
import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.*;
import java.io.*;

public class HDFSOperations {
    private FileSystem fs;
    
    public HDFSOperations() throws Exception {
        Configuration conf = new Configuration();
        conf.set("fs.defaultFS", "hdfs://namenode:8020");
        fs = FileSystem.get(conf);
    }
    
    // 创建目录
    public void mkdir(String path) throws Exception {
        Path dirPath = new Path(path);
        if (!fs.exists(dirPath)) {
            fs.mkdirs(dirPath);
            System.out.println("目录创建成功: " + path);
        }
    }
    
    // 上传文件
    public void uploadFile(String localPath, String hdfsPath) throws Exception {
        Path local = new Path(localPath);
        Path hdfs = new Path(hdfsPath);
        fs.copyFromLocalFile(local, hdfs);
        System.out.println("文件上传成功: " + hdfsPath);
    }
    
    // 下载文件
    public void downloadFile(String hdfsPath, String localPath) throws Exception {
        Path hdfs = new Path(hdfsPath);
        Path local = new Path(localPath);
        fs.copyToLocalFile(hdfs, local);
        System.out.println("文件下载成功: " + localPath);
    }
    
    // 删除文件
    public void deleteFile(String path) throws Exception {
        Path filePath = new Path(path);
        fs.delete(filePath, false);
        System.out.println("文件删除成功: " + path);
    }
    
    // 列出文件
    public void listFiles(String path) throws Exception {
        Path dirPath = new Path(path);
        FileStatus[] files = fs.listStatus(dirPath);
        for (FileStatus file : files) {
            System.out.println(file.getPath().getName() + 
                             " - " + file.getLen() + " bytes");
        }
    }
    
    public void close() throws Exception {
        fs.close();
    }
    
    public static void main(String[] args) throws Exception {
        HDFSOperations hdfs = new HDFSOperations();
        
        hdfs.mkdir("/user/test");
        hdfs.listFiles("/user/test");
        
        hdfs.close();
    }
}
```

### 9.2 MapReduce 读取 HDFS

```java
import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.Path;
import org.apache.hadoop.io.*;
import org.apache.hadoop.mapreduce.*;
import org.apache.hadoop.mapreduce.lib.input.FileInputFormat;
import org.apache.hadoop.mapreduce.lib.output.FileOutputFormat;

public class HDFSWordCount {
    
    public static class TokenizerMapper 
        extends Mapper<Object, Text, Text, IntWritable>{
        
        private final static IntWritable one = new IntWritable(1);
        private Text word = new Text();
        
        public void map(Object key, Text value, Context context
                        ) throws IOException, InterruptedException {
            String[] tokens = value.toString().split("\\s+");
            for (String token : tokens) {
                word.set(token);
                context.write(word, one);
            }
        }
    }
    
    public static class IntSumReducer 
        extends Reducer<Text,IntWritable,Text,IntWritable> {
        
        private IntWritable result = new IntWritable();
        
        public void reduce(Text key, Iterable<IntWritable> values, 
                           Context context) throws IOException, InterruptedException {
            int sum = 0;
            for (IntWritable val : values) {
                sum += val.get();
            }
            result.set(sum);
            context.write(key, result);
        }
    }
    
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        Job job = Job.getInstance(conf, "HDFS Word Count");
        
        job.setJarByClass(HDFSWordCount.class);
        job.setMapperClass(TokenizerMapper.class);
        job.setCombinerClass(IntSumReducer.class);
        job.setReducerClass(IntSumReducer.class);
        
        job.setOutputKeyClass(Text.class);
        job.setOutputValueClass(IntWritable.class);
        
        FileInputFormat.addInputPath(job, new Path(args[0]));
        FileOutputFormat.setOutputPath(job, new Path(args[1]));
        
        System.exit(job.waitForCompletion(true) ? 0 : 1);
    }
}
```

## 10. 总结

HDFS 作为大数据存储的基础设施，具有以下关键特点：

1. **分布式存储**：将大文件分割成块存储在多个节点
2. **高容错性**：通过多副本机制保证数据可靠性
3. **高吞吐量**：适合大规模批处理场景
4. **简单架构**：主从架构，易于理解和维护

掌握 HDFS 的工作原理和操作方法，是构建大数据系统的基础。通过合理的配置和优化，可以充分发挥 HDFS 的性能优势，为上层应用提供可靠的数据存储服务。
