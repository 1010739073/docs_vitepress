# MapReduce 计算框架详解

## 1. MapReduce 概述

MapReduce 是 Hadoop 的分布式计算框架，用于处理和生成大数据集。它将复杂的并行计算过程抽象为 Map 和 Reduce 两个阶段，简化了分布式编程的复杂度。

### 1.1 MapReduce 特性

- **简单编程模型**：只需实现 Map 和 Reduce 函数
- **自动并行化**：框架自动处理任务分发和并行执行
- **容错性强**：任务失败自动重试
- **可扩展性好**：支持大规模集群
- **数据本地化**：尽量在数据所在节点执行任务

## 2. MapReduce 架构

### 2.1 核心组件

- **JobTracker**：负责任务调度和资源管理
- **TaskTracker**：执行具体任务
- **Job**：完整的 MapReduce 作业
- **Task**：Map 或 Reduce 任务

### 2.2 架构原理

```java
// MapReduce 架构示意
/*
JobTracker (主节点)
├── 接收客户端作业提交
├── 任务调度和分配
├── 监控任务执行状态
└── 处理任务失败重试

TaskTracker (从节点)
├── 接收任务分配
├── 启动和监控任务
├── 向 JobTracker 汇报状态
└── 处理任务失败
*/
```

## 3. MapReduce 工作流程

### 3.1 作业执行流程

1. **作业提交**：客户端提交作业到 JobTracker
2. **作业初始化**：JobTracker 创建作业对象
3. **任务分配**：JobTracker 将任务分配给 TaskTracker
4. **任务执行**：TaskTracker 执行 Map 和 Reduce 任务
5. **进度更新**：TaskTracker 定期汇报进度
6. **作业完成**：所有任务完成后，作业结束

```java
// MapReduce 作业提交流程
public class MapReduceJob {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        Job job = Job.getInstance(conf, "My MapReduce Job");
        
        // 设置作业配置
        job.setJarByClass(MapReduceJob.class);
        job.setMapperClass(MyMapper.class);
        job.setReducerClass(MyReducer.class);
        
        // 设置输入输出类型
        job.setOutputKeyClass(Text.class);
        job.setOutputValueClass(IntWritable.class);
        
        // 设置输入输出路径
        FileInputFormat.addInputPath(job, new Path(args[0]));
        FileOutputFormat.setOutputPath(job, new Path(args[1]));
        
        // 提交作业
        System.exit(job.waitForCompletion(true) ? 0 : 1);
    }
}
```

### 3.2 Map 阶段

Map 阶段将输入数据分割成键值对，进行初步处理。

```java
// Mapper 示例
public static class MyMapper 
    extends Mapper<Object, Text, Text, IntWritable> {
    
    private final static IntWritable one = new IntWritable(1);
    private Text word = new Text();
    
    public void map(Object key, Text value, Context context) 
            throws IOException, InterruptedException {
        String line = value.toString();
        String[] words = line.split("\\s+");
        
        for (String w : words) {
            word.set(w);
            context.write(word, one);
        }
    }
}
```

### 3.3 Shuffle 阶段

Shuffle 是 MapReduce 的核心过程，负责在 Map 和 Reduce 之间传输数据。

```java
// Shuffle 过程示意
/*
Map 阶段输出
├── Partition 分区
├── Sort 排序
├── Combiner 本地聚合
└── Spill 溢写到磁盘

Reduce 阶段输入
├── Copy 拉取数据
├── Merge 合并数据
└── Sort 排序数据
*/
```

### 3.4 Reduce 阶段

Reduce 阶段对 Map 阶段的输出进行汇总处理。

```java
// Reducer 示例
public static class MyReducer 
    extends Reducer<Text, IntWritable, Text, IntWritable> {
    
    private IntWritable result = new IntWritable();
    
    public void reduce(Text key, Iterable<IntWritable> values, 
                       Context context) throws IOException, InterruptedException {
        int sum = 0;
        for (IntWritable val : values) {
            sum += val.get();
        }
        result.set(sum);
        context.write(key, result);
    }
}
```

### 4.2 Partitioner

Partitioner 决定 Map 输出的键值对发送到哪个 Reducer。

```java
// 自定义 Partitioner
public class MyPartitioner extends Partitioner<Text, IntWritable> {
    
    @Override
    public int getPartition(Text key, IntWritable value, int numPartitions) {
        // 根据键的哈希值分区
        return (key.hashCode() & Integer.MAX_VALUE) % numPartitions;
    }
}

// 使用自定义 Partitioner
job.setPartitionerClass(MyPartitioner.class);
job.setNumReduceTasks(3);
```

### 4.3 Combiner

Combiner 是在 Map 端进行的本地聚合，减少网络传输。

```java
// Combiner 示例（通常与 Reducer 相同）

        result.set(sum);
```

## 6. MapReduce 面试题

### 6.1 基础概念

**Q1: MapReduce 的工作原理是什么？**

MapReduce 将计算分为两个阶段：
- **Map 阶段**：读取输入数据，处理并输出中间键值对
- **Reduce 阶段**：对中间结果进行聚合处理，输出最终结果

**Q2: MapReduce 的 Shuffle 过程是怎样的？**

Shuffle 过程包括：
1. **Map 端**：分区、排序、溢写
2. **Reduce 端**：拉取、合并、排序

### 6.2 技术原理

**Q3: Combiner 的作用是什么？**

Combiner 是 Map 端的本地聚合器，用于：
- 减少 Map 输出到 Reduce 的数据量
- 降低网络传输开销
- 提高作业执行效率

**Q4: 如何设置 Map 和 Reduce 的数量？**

```java
// 设置 Map 任务数量（通过 InputSplit 控制）
FileInputFormat.setMaxInputSplitSize(job, 128 * 1024 * 1024);  // 128MB

// 设置 Reduce 任务数量
job.setNumReduceTasks(10);
```

### 6.3 优化问题

**Q5: 如何优化 MapReduce 作业性能？**

- 合理设置 Map 和 Reduce 任务数量
- 使用 Combiner 减少数据传输
- 启用数据压缩
- 优化数据本地化
- 调整内存配置
- 使用合适的序列化方式

**Q6: MapReduce 中数据倾斜如何处理？**

- 自定义 Partitioner 实现负载均衡
- 对热点 Key 进行特殊处理
- 增加 Reduce 任务数量
- 使用二次排序

## 7. 代码示例

### 7.1 WordCount 完整示例

```java
import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.Path;
import org.apache.hadoop.io.*;
import org.apache.hadoop.mapreduce.*;
import org.apache.hadoop.mapreduce.lib.input.FileInputFormat;
import org.apache.hadoop.mapreduce.lib.output.FileOutputFormat;

public class WordCount {
    
    public static class TokenizerMapper 
        extends Mapper<Object, Text, Text, IntWritable>{
        
        private final static IntWritable one = new IntWritable(1);
        private Text word = new Text();
        
        public void map(Object key, Text value, Context context
                        ) throws IOException, InterruptedException {
            String[] tokens = value.toString().split("\\s+");
            for (String token : tokens) {
                word.set(token);
                context.write(word, one);
            }
        }
    }
    
    public static class IntSumReducer 
        extends Reducer<Text,IntWritable,Text,IntWritable> {
        
        private IntWritable result = new IntWritable();
        
        public void reduce(Text key, Iterable<IntWritable> values, 
                           Context context) throws IOException, InterruptedException {
            int sum = 0;
            for (IntWritable val : values) {
                sum += val.get();
            }
            result.set(sum);
            context.write(key, result);
        }
    }
    
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        Job job = Job.getInstance(conf, "word count");
        
        job.setJarByClass(WordCount.class);
        job.setMapperClass(TokenizerMapper.class);
        job.setCombinerClass(IntSumReducer.class);
        job.setReducerClass(IntSumReducer.class);
        
        job.setOutputKeyClass(Text.class);
        job.setOutputValueClass(IntWritable.class);
        
        FileInputFormat.addInputPath(job, new Path(args[0]));
        FileOutputFormat.setOutputPath(job, new Path(args[1]));
        
        System.exit(job.waitForCompletion(true) ? 0 : 1);
    }
}
```

### 7.2 二次排序示例

```java
import org.apache.hadoop.io.*;

// 自定义键类型，实现二次排序
public class CompositeKey implements WritableComparable<CompositeKey> {
    private Text firstKey;
    private IntWritable secondKey;
    
    public CompositeKey() {
        this.firstKey = new Text();
        this.secondKey = new IntWritable();
    }
    
    public CompositeKey(Text firstKey, IntWritable secondKey) {
        this.firstKey = firstKey;
        this.secondKey = secondKey;
    }
    
    @Override
    public void write(DataOutput out) throws IOException {
        firstKey.write(out);
        secondKey.write(out);
    }
    
    @Override
    public void readFields(DataInput in) throws IOException {
        firstKey.readFields(in);
        secondKey.readFields(in);
    }
    
    @Override
    public int compareTo(CompositeKey other) {
        int cmp = firstKey.compareTo(other.firstKey);
        if (cmp != 0) {
            return cmp;
        }
        return secondKey.compareTo(other.secondKey);
    }
    
    // Getter 方法
    public Text getFirstKey() { return firstKey; }
    public IntWritable getSecondKey() { return secondKey; }
}

// 自定义分区器
public class CompositeKeyPartitioner extends Partitioner<CompositeKey, Text> {
    
    @Override
    public int getPartition(CompositeKey key, Text value, int numPartitions) {
        return (key.getFirstKey().hashCode() & Integer.MAX_VALUE) % numPartitions;
    }
}

// 自定义分组比较器
public class CompositeKeyGroupingComparator extends WritableComparator {
    
    protected CompositeKeyGroupingComparator() {
        super(CompositeKey.class, true);
    }
    
    @Override
    public int compare(WritableComparable a, WritableComparable b) {
        CompositeKey key1 = (CompositeKey) a;
        CompositeKey key2 = (CompositeKey) b;
        return key1.getFirstKey().compareTo(key2.getFirstKey());
    }
}

// 使用示例
public class SecondarySortJob {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        Job job = Job.getInstance(conf, "secondary sort");
        
        job.setJarByClass(SecondarySortJob.class);

        job.setPartitionerClass(CompositeKeyPartitioner.class);
        job.setGroupingComparatorClass(CompositeKeyGroupingComparator.class);
        
        job.setMapOutputKeyClass(CompositeKey.class);
        job.setMapOutputValueClass(Text.class);
        
        job.setOutputKeyClass(Text.class);
        job.setOutputValueClass(Text.class);

    }
}
```

### 7.3 自定义 InputFormat 示例

```java
import org.apache.hadoop.fs.*;
import org.apache.hadoop.io.*;
import org.apache.hadoop.mapreduce.*;
import org.apache.hadoop.mapreduce.lib.input.*;

public class CustomInputFormat extends FileInputFormat<Text, Text> {
    
    @Override
    public RecordReader<Text, Text> createRecordReader(
            InputSplit split, TaskAttemptContext context) {
        return new CustomRecordReader();
    }
    
    @Override
    protected boolean isSplitable(JobContext context, Path filename) {
        return false;  // 不支持文件分割
    }
}

public class CustomRecordReader extends RecordReader<Text, Text> {
    private LineRecordReader lineReader;
    private Text currentKey;
    private Text currentValue;
    
    public CustomRecordReader() {
        lineReader = new LineRecordReader();
    }
    
    @Override
    public void initialize(InputSplit split, TaskAttemptContext context) 
            throws IOException, InterruptedException {
        lineReader.initialize(split, context);
    }
    
    @Override
    public boolean nextKeyValue() throws IOException, InterruptedException {
        if (!lineReader.nextKeyValue()) {
            return false;
        }
        
        Text line = lineReader.getCurrentValue();
        String[] parts = line.toString().split(",");
        
        if (parts.length >= 2) {
            currentKey = new Text(parts[0]);
            currentValue = new Text(parts[1]);
            return true;
        }
        
        return false;
    }
    
    @Override
    public Text getCurrentKey() {
        return currentKey;
    }
    
    @Override
    public Text getCurrentValue() {
        return currentValue;
    }
    
    @Override
    public float getProgress() throws IOException, InterruptedException {
        return lineReader.getProgress();
    }
    
    @Override
    public void close() throws IOException {
        lineReader.close();
    }
}

// 使用自定义 InputFormat
public class CustomInputFormatJob {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        Job job = Job.getInstance(conf, "custom input format");
        
        job.setJarByClass(CustomInputFormatJob.class);
        job.setMapperClass(MyMapper.class);
        job.setReducerClass(MyReducer.class);
        
        job.setInputFormatClass(CustomInputFormat.class);
        
        job.setOutputKeyClass(Text.class);
        job.setOutputValueClass(Text.class);
        
        FileInputFormat.addInputPath(job, new Path(args[0]));
        FileOutputFormat.setOutputPath(job, new Path(args[1]));
        
        System.exit(job.waitForCompletion(true) ? 0 : 1);
    }
}
```

## 8. 总结

MapReduce 作为大数据计算的基础框架，具有以下重要特点：

1. **编程简单**：只需实现 Map 和 Reduce 函数
2. **自动并行**：框架自动处理任务分发和并行执行
3. **容错性强**：任务失败自动重试，保证作业完成
4. **可扩展性好**：支持大规模集群，处理 PB 级数据

虽然 Spark 等新一代计算框架在性能上有所提升，但 MapReduce 仍然是理解分布式计算原理的重要基础。掌握 MapReduce 的工作原理和优化技巧，对于构建高效的大数据处理系统具有重要意义。
