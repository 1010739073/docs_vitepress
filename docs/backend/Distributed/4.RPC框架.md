# RPC 框架

## 一、RPC 基础

### 1.1 什么是 RPC

RPC（Remote Procedure Call，远程过程调用）是一种计算机通信协议，允许程序调用另一个地址空间（通常是另一台机器）的函数或方法，而开发者无需显式编写网络通信代码。

**核心思想**：
- 让远程调用像本地调用一样简单
- 屏蔽网络通信的复杂性
- 隐藏底层的序列化和传输细节

### 1.2 RPC 调用流程

```
客户端                              服务端
  │                                   │
  │  1. 调用本地代理 (User userService) │
  │                                   │
  │  2. 序列化参数                     │
  │  ───────────────────────────────► │  3. 反序列化参数
  │                                   │  4. 调用真实服务
  │  5. 反序列化结果 ◄──────────────── │  5. 序列化结果
  │                                   │
  │  6. 返回结果                       │
  │                                   │
```

**详细步骤**：
1. 客户端调用本地代理
2. 代理将方法名和参数序列化
3. 通过网络传输到服务端
4. 服务端反序列化请求
5. 服务端调用真实方法
6. 服务端将结果序列化
7. 通过网络返回给客户端
8. 客户端反序列化结果

### 1.3 RPC vs HTTP/RESTful

| 特性 | RPC | RESTful |
|------|-----|---------|
| 传输协议 | TCP（自定义）| HTTP/HTTPS |
| 数据格式 | 二进制（高效）| JSON/XML（可读） |
| 性能 | 高 | 较低 |
| 调用方式 | 方法调用 | HTTP 请求（GET/POST） |
| 接口定义 | IDL（Thrift/Protobuf）| OpenAPI/Swagger |
| 跨语言 | 需要代码生成 | 原生支持 |
| 测试 | 需要客户端 | 浏览器可直接测试 |
| 适用场景 | 内部服务调用 | 对外 API |

**选择建议**：
- 内部微服务通信 → RPC（Dubbo、gRPC）
- 对外暴露 API → RESTful
- 移动端/Web 调用 → RESTful

### 2.2 动态代理

**作用**：将远程调用伪装成本地调用

**实现方式**：

#### JDK 动态代理

```java
// RPC 服务接口
public interface UserService {
    User getUserById(Long id);
}

// 动态代理生成
public class RpcProxy implements InvocationHandler {
    private Class<?> interfaceClass;
    private String serverAddress;

    public <T> T create(Class<T> interfaceClass) {
        this.interfaceClass = interfaceClass;
        return (T) Proxy.newProxyInstance(
            interfaceClass.getClassLoader(),
            new Class[]{interfaceClass},
            this
        );
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) {
        // 1. 构建请求
        RpcRequest request = new RpcRequest();
        request.setInterfaceName(method.getDeclaringClass().getName());
        request.setMethodName(method.getName());
        request.setParameterTypes(method.getParameterTypes());
        request.setParameters(args);

        // 2. 发送请求
        RpcResponse response = sendRequest(request);

        // 3. 返回结果
        return response.getResult();
    }
}

// 使用
UserService userService = new RpcProxy().create(UserService.class);
User user = userService.getUserById(1L); // 看起来像本地调用
```

#### CGLIB 动态代理

```java
// CGLIB 通过继承实现代理
public class CglibProxy implements MethodInterceptor {
    @Override
    public Object intercept(Object obj, Method method, Object[] args, MethodProxy proxy) {
        // 类似 JDK 动态代理
        return null;
    }
}

Enhancer enhancer = new Enhancer();
enhancer.setSuperclass(UserService.class);
enhancer.setCallback(new CglibProxy());
UserService userService = (UserService) enhancer.create();
```

**区别**：
- JDK 代理：只能代理接口
- CGLIB 代理：可以代理类（通过继承）

### 2.3 序列化

**作用**：将对象转换为字节流，便于网络传输

**常见序列化方式对比**：

| 序列化方式 | 优点 | 缺点 | 适用场景 |
|-----------|------|------|---------|
| Java 原生 | 简单 | 效率低、占用空间大 | 不推荐 |
| Hessian | 高效、跨语言 | 不支持所有类型 | 内部服务 |
| Protobuf | 极致性能、跨语言 | 需要定义 .proto 文件 | 高性能场景 |
| Kryo | 高性能 | 不支持跨语言 | Java 内部 |
| JSON | 可读性好、跨语言 | 效率低 | 对外 API |
| Avro | 动态模式、支持演进 | 复杂 | 大数据场景 |

**Protobuf 示例**：

```protobuf
// user.proto
syntax = "proto3";

message User {
    int64 id = 1;
    string name = 2;
    int32 age = 3;
}

service UserService {
    rpc getUserById (GetUserRequest) returns (User);
}

message GetUserRequest {
    int64 id = 1;
}
```

```bash
# 生成代码
protoc --java_out=. user.proto
```

### 2.5 负载均衡

**策略**：

| 策略 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| 随机 | 随机选择 | 简单 | 可能不均匀 |
| 轮询 | 依次选择 | 均匀 | 不考虑负载 |
| 加权轮询 | 按权重选择 | 灵活 | 需要配置权重 |
| 最小连接数 | 选择连接数最少的 | 负载均衡 | 需要维护连接数 |
| 一致性哈希 | 根据 key 哈希 | 相同 key 到同一节点 | 节点变化影响大 |

**Dubbo 负载均衡实现**：

```java
// 随机负载均衡
public class RandomLoadBalance extends AbstractLoadBalance {
    @Override
    protected <T> Invoker<T> doSelect(List<Invoker<T>> invokers, URL url, Invocation invocation) {
        int length = invokers.size();
        return invokers.get(ThreadLocalRandom.current().nextInt(length));
    }
}

// 一致性哈希负载均衡
public class ConsistentHashLoadBalance extends AbstractLoadBalance {
    private final ConcurrentMap<String, ConsistentHashSelector<?>> selectors = new ConcurrentHashMap<>();

    @Override
    protected <T> Invoker<T> doSelect(List<Invoker<T>> invokers, URL url, Invocation invocation) {
        String key = invokers.get(0).getUrl().getServiceKey() + "." + invocation.getMethodName();
        ConsistentHashSelector<T> selector = (ConsistentHashSelector<T>) selectors.get(key);

        // 重新构建 selector
        if (selector == null || selector.identityHashCode != invokers.hashCode()) {
            selectors.put(key, new ConsistentHashSelector<>(invokers, invocation.getMethodName()));
            selector = (ConsistentHashSelector<T>) selectors.get(key);
        }

        return selector.select(invocation.getArguments());
    }
}
```

### 2.6 服务容错

**策略**：

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| Failover（失败重试） | 失败后重试其他节点 | 读操作 |
| Failfast（快速失败） | 失败后立即返回错误 | 写操作 |
| Failsafe（失败安全） | 失败后忽略错误 | 日志记录 |
| Failback（失败恢复） | 失败后异步重试 | 非核心业务 |
| Forking（并行调用） | 并行调用多个节点，取最快结果 | 实时性要求高 |
| Broadcast（广播调用） | 调用所有节点 | 缓存刷新 |

**Dubbo 集群容错实现**：

```java
// Failover 集群容错
public class FailoverClusterInvoker<T> extends AbstractClusterInvoker<T> {
    private static final Logger logger = LoggerFactory.getLogger(FailoverClusterInvoker.class);

    public FailoverClusterInvoker(Directory<T> directory) {
        super(directory);
    }

    @Override
    public Result doInvoke(Invocation invocation, List<Invoker<T>> invokers, LoadBalance loadbalance) {
        // 获取重试次数
        int len = getUrl().getMethodParameter(invocation.getMethodName(), Constants.RETRIES_KEY, Constants.DEFAULT_RETRIES) + 1;
        if (len <= 0) {
            len = 1;
        }

        RpcException exception = null;

        // 重试
        for (int i = 0; i < len; i++) {
            // 负载均衡选择
            Invoker<T> invoker = select(loadbalance, invocation, invokers, null);
            try {
                return invoker.invoke(invocation);
            } catch (RpcException e) {
                exception = e;
                logger.warn("Failover invoke...", e);
            }
        }
        throw exception;
    }
}
```

## 三、Dubbo

### 3.1 架构

```
┌─────────────────────────────────────────────────────────┐
│                      容器                               │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐   │
│  │ Service │  │ Protocol│  │ Cluster │  │ Monitor │   │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘   │
└─────────────────────────────────────────────────────────┘
                            │
        ┌───────────────────┼───────────────────┐
        │                   │                   │
┌───────▼────────┐  ┌──────▼───────┐  ┌───────▼────────┐
│   Container    │  │   Registry   │  │   Monitor      │
│ 服务容器       │  │  注册中心     │  │   监控中心     │
└────────────────┘  └──────────────┘  └────────────────┘
```

**核心角色**：
- **Container**：服务容器，负责启动、加载、运行服务
- **Provider**：服务提供者，暴露服务
- **Consumer**：服务消费者，调用远程服务
- **Registry**：注册中心，服务注册与发现
- **Monitor**：监控中心，统计调用次数和调用时间
- **Protocol**：远程调用层，封装 RPC 调用
- **Invocation**：调用过程模型
- **Invoker**：执行者
- **Export/Reference**：服务暴露/引用

### 3.2 服务提供者

```xml
<!-- dubbo-provider.xml -->
<beans xmlns="...">
    <!-- 提供方应用信息 -->
    <dubbo:application name="user-provider" />

    <!-- 注册中心 -->
    <dubbo:registry address="zookeeper://127.0.0.1:2181" />

    <!-- 用 dubbo 协议在 20880 端口暴露服务 -->
    <dubbo:protocol name="dubbo" port="20880" />

    <!-- 声明需要暴露的服务接口 -->
    <dubbo:service interface="com.example.UserService" ref="userService" />

    <!-- 实现类 -->
    <bean id="userService" class="com.example.UserServiceImpl" />
</beans>
```


### 3.5 Dubbo SPI

Dubbo 实现了自己的 SPI（Service Provider Interface）机制，比 Java SPI 更强大。

**Java SPI 缺点**：
- 无法按需加载
- 不支持依赖注入
- 不支持 AOP

**Dubbo SPI 特点**：
- 按需加载
- 支持依赖注入
- 支持 AOP

**示例**：

```java
// 定义接口
@SPI("default")
public interface LoadBalance {
    <T> Invoker<T> select(List<Invoker<T>> invokers, URL url, Invocation invocation);
}

// 实现类
public class RandomLoadBalance implements LoadBalance {
    // ...
}

// 配置文件：META-INF/dubbo/com.example.LoadBalance
default=com.example.RandomLoadBalance
roundrobin=com.example.RoundRobinLoadBalance
leastactive=com.example.LeastActiveLoadBalance

// 使用
LoadBalance loadBalance = ExtensionLoader.getExtensionLoader(LoadBalance.class)
    .getExtension("random");
```

### 3.6 Dubbo 3.0 新特性

- **Triple 协议**：基于 HTTP/2，更好的云原生支持

- **多注册中心**：支持多注册中心

## 四、gRPC

### 4.1 概述

gRPC 是 Google 开源的高性能 RPC 框架，基于 HTTP/2 和 Protobuf。

**特点**：
- 基于 HTTP/2
- 使用 Protobuf 序列化
- 支持多种语言
- 支持流式传输
- 双向流通信

### 4.4 服务定义

```protobuf
// user.proto
syntax = "proto3";

package user;

service UserService {
    // 简单 RPC
    rpc GetUser (GetUserRequest) returns (User);

    // 服务端流式 RPC
    rpc ListUsers (ListUsersRequest) returns (stream User);

    // 客户端流式 RPC
    rpc CreateUser (stream User) returns (CreateUserResponse);

    // 双向流式 RPC
    rpc ChatStream (stream ChatMessage) returns (stream ChatMessage);
}

message User {
    int64 id = 1;
    string name = 2;
    int32 age = 3;
}

message GetUserRequest {
    int64 id = 1;
}

message ListUsersRequest {
    int32 page = 1;
    int32 size = 2;
}

message CreateUserResponse {
    bool success = 1;
    string message = 2;
}

message ChatMessage {
    string content = 1;
    int64 timestamp = 2;
}
```

### 4.2 服务端实现

```java
// 服务端实现
public class UserServiceImpl extends UserServiceGrpc.UserServiceImplBase {

    @Override
    public void getUser(GetUserRequest request,
                        StreamObserver<User> responseObserver) {
        long id = request.getId();
        User user = User.newBuilder()
            .setId(id)
            .setName("张三")
            .setAge(25)
            .build();

        responseObserver.onNext(user);
        responseObserver.onCompleted();
    }

    @Override
    public void listUsers(ListUsersRequest request,
                          StreamObserver<User> responseObserver) {
        // 服务端流式返回
        for (int i = 0; i < 10; i++) {
            User user = User.newBuilder()
                .setId(i)
                .setName("User" + i)
                .setAge(20 + i)
                .build();
            responseObserver.onNext(user);
        }
        responseObserver.onCompleted();
    }

    @Override
    public StreamObserver<User> createUser(StreamObserver<CreateUserResponse> responseObserver) {
        // 客户端流式接收
        return new StreamObserver<User>() {
            int count = 0;

            @Override
            public void onNext(User user) {
                count++;
                // 处理每个用户
                System.out.println("Received: " + user.getName());
            }

            @Override
            public void onError(Throwable t) {
                responseObserver.onError(t);
            }

            @Override
            public void onCompleted() {
                CreateUserResponse response = CreateUserResponse.newBuilder()
                    .setSuccess(true)
                    .setMessage("Created " + count + " users")
                    .build();
                responseObserver.onNext(response);
                responseObserver.onCompleted();
            }
        };
    }

    @Override
    public StreamObserver<ChatMessage> chatStream(StreamObserver<ChatMessage> responseObserver) {
        // 双向流
        return new StreamObserver<ChatMessage>() {
            @Override
            public void onNext(ChatMessage message) {
                // 回显消息
                responseObserver.onNext(message);
            }

            @Override
            public void onError(Throwable t) {
                responseObserver.onError(t);
            }

            @Override
            public void onCompleted() {
                responseObserver.onCompleted();
            }
        };
    }
}

// 启动服务端
public class GrpcServer {
    public static void main(String[] args) throws Exception {
        Server server = ServerBuilder.forPort(9090)
            .addService(new UserServiceImpl())
            .build();

        server.start();
        server.awaitTermination();
    }
}
```

```java
// 服务端拦截器
public class AuthInterceptor implements ServerInterceptor {
    @Override
    public <ReqT, RespT> ServerCall.Listener<ReqT> interceptCall(
        ServerCall<ReqT, RespT> call,
        Metadata headers,
        ServerCallHandler<ReqT, RespT> next) {

        // 鉴权逻辑
        String token = headers.get(Metadata.Key.of("authorization", Metadata.ASCII_STRING_MARSHALLER));
        if (!validateToken(token)) {
            call.close(Status.UNAUTHENTICATED, headers);
            return new ServerCall.Listener<ReqT>() {};
        }

        return next.startCall(call, headers);
    }

    private boolean validateToken(String token) {
        // 验证 token
        return true;
    }
}

// 添加拦截器
Server server = ServerBuilder.forPort(9090)
    .addService(ServerInterceptors.intercept(new UserServiceImpl(), new AuthInterceptor()))
    .build();

// 客户端拦截器
public class TokenInterceptor implements ClientInterceptor {
    @Override
    public <ReqT, RespT> ClientCall<ReqT, RespT> interceptCall(
        MethodDescriptor<ReqT, RespT> method,
        CallOptions callOptions,
        Channel next) {

        return new ForwardingClientCall.SimpleForwardingClientCall<ReqT, RespT>(
            next.newCall(method, callOptions)) {

            @Override
            public void start(Listener<RespT> responseListener, Metadata headers) {
                // 添加 token
                headers.put(Metadata.Key.of("authorization", Metadata.ASCII_STRING_MARSHALLER), "Bearer xxx");
                super.start(responseListener, headers);
            }
        };
    }
}

// 使用拦截器

    .withInterceptors(new TokenInterceptor());
```

## 五、RPC 与 RESTful 混合使用

### 5.1 架构设计

```
                    ┌─────────────┐
                    │   网关       │
                    └──────┬──────┘
                           │
            ┌──────────────┼──────────────┐
            │              │              │
     ┌──────▼──────┐ ┌────▼────┐  ┌──────▼──────┐
     │  RESTful   │ │  RPC    │  │  gRPC      │
     │  （对外）   │ │ （内部） │  │  （内部）   │
     └─────────────┘ └─────────┘  └─────────────┘
```

**原则**：
- 对外 API 使用 RESTful（便于调用、调试）
- 内部服务使用 RPC（高性能）
- 流式处理使用 gRPC

### 5.2 Spring Cloud + Dubbo

```java
// 对外提供 RESTful 接口
@RestController
@RequestMapping("/api")
public class UserController {

    @DubboReference
    private UserService userService;  // 内部使用 Dubbo 调用
```
