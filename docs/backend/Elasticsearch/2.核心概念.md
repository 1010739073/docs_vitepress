# 2. 核心概念

## 基本概念

### 索引 (Index)

索引是 Elasticsearch 中存储数据的基本单位，类似于关系数据库中的数据库。

**特点**：
- 包含多个类型相同的文档
- 拥有独立的配置和映射
- 可以设置副本数和分片数

**索引命名规则**：
- 只能包含小写字母
- 不能以下划线开头
- 不能包含逗号

### 类型 (Type)

> **注意**：Elasticsearch 7.x 及以上版本已移除类型概念

在 6.x 及以下版本中，类型用于区分索引中的不同文档类型。

### 文档 (Document)

文档是 Elasticsearch 中最小的数据单元，类似于关系数据库中的行。

**特点**：
- 使用 JSON 格式存储
- 每个文档有唯一 ID
- 可以包含多个字段

**文档结构**：
```json
{
  "_index": "products",
  "_type": "_doc",
  "_id": "1",
  "_source": {
    "title": "iPhone 15",
    "price": 5999,
    "category": "手机",
    "created_at": "2023-09-22"
  }
}
```

### 字段 (Field)

字段是文档中的属性，类似于关系数据库中的列。

**字段类型**：
| 类型分类 | 具体类型 | 说明 |
|---------|---------|------|
| 字符串 | text, keyword | text用于全文搜索，keyword用于精确匹配 |
| 数值 | long, integer, short, byte | 整数类型 |
| 浮点 | double, float, half_float | 浮点数类型 |
| 日期 | date | 日期时间类型 |
| 布尔 | boolean | 布尔类型 |
| 数组 | array | 数组类型 |
| 对象 | object | 嵌套对象 |

### 分片 (Shard)

分片是索引的水平切分，用于分布式存储和并行搜索。

**分片类型**：
- **主分片 (Primary Shard)**：原始数据的分片
- **副本分片 (Replica Shard)**：主分片的副本

**分片数量配置**：
- `number_of_shards`：主分片数，默认5，创建后不可修改
- `number_of_replicas`：副本数，默认1，可以动态修改

### 节点 (Node)

节点是 Elasticsearch 集群中的单个服务器实例。

**节点类型**：
| 类型 | 说明 |
|------|------|
| Master Node | 管理集群状态、分片分配 |
| Data Node | 存储数据、处理搜索请求 |
| Ingest Node | 数据预处理 |
| Coordinating Node | 协调节点，路由请求 |
| ML Node | 机器学习节点 |

### 集群 (Cluster)

集群是由多个节点组成的分布式系统。

**集群特性**：
- 自动发现节点
- 自动分配分片
- 故障自动转移

## 映射 (Mapping)

映射定义了文档中字段的数据类型和索引方式。

### 映射类型

**动态映射**：
```json
PUT /products
{
  "mappings": {
    "dynamic": "true"  // 自动检测字段类型
  }
}
```

**静态映射**：
```json
PUT /products
{
  "mappings": {
    "properties": {
      "title": { "type": "text" },
      "price": { "type": "float" },
      "category": { "type": "keyword" }
    }
  }
}
```

### 字段属性

| 属性 | 说明 |
|------|------|
| `type` | 字段类型 |
| `index` | 是否创建索引 |
| `analyzer` | 分词器 |
| `search_analyzer` | 搜索时分词器 |
| `fields` | 多字段支持 |
| `copy_to` | 复制到其他字段 |

## 分析器 (Analyzer)

分析器用于将文本转换为索引可搜索的词项。

### 分析器组成

```
┌─────────────────────────────────────────────┐
│              Analyzer                       │
│  ┌─────────────┐  ┌─────────────┐          │
│  │  Character  │→│  Tokenizer  │→│ Token   │
│  │   Filter    │  │             │  │ Filters │
│  └─────────────┘  └─────────────┘  └────────┘
└─────────────────────────────────────────────┘
```

### 内置分析器

| 分析器 | 说明 |
|--------|------|
| `standard` | 标准分析器，适用于大多数语言 |
| `simple` | 简单分词，按非字母字符分割 |
| `whitespace` | 按空白字符分割 |
| `stop` | 去除停用词 |
| `keyword` | 不分词，作为整体 |

### 中文分词器

| 分词器 | 说明 |
|--------|------|
| `ik_max_word` | 细粒度分词 |
| `ik_smart` | 粗粒度分词 |
| `pinyin` | 拼音分词 |

## 查询类型

### 全文查询

| 查询类型 | 说明 |
|----------|------|
| `match` | 全文匹配 |
| `match_phrase` | 短语匹配 |
| `multi_match` | 多字段匹配 |
| `query_string` | 查询字符串 |

### 精确查询

| 查询类型 | 说明 |
|----------|------|
| `term` | 精确匹配 |
| `terms` | 多值精确匹配 |
| `range` | 范围查询 |
| `exists` | 存在性查询 |

### 复合查询

| 查询类型 | 说明 |
|----------|------|
| `bool` | 布尔组合查询 |
| `boosting` | 加权查询 |
| `constant_score` | 恒定分数查询 |

## 聚合 (Aggregation)

聚合用于对数据进行分组和统计分析。

### 聚合类型

| 类型 | 说明 |
|------|------|
| `terms` | 分组统计 |
| `range` | 范围分组 |
| `date_histogram` | 日期直方图 |
| `avg/sum/max/min/count` | 数值聚合 |

### 聚合示例

```json
GET /sales/_search
{
  "aggs": {
    "by_category": {
      "terms": { "field": "category" },
      "aggs": {
        "avg_price": { "avg": { "field": "price" } }
      }
    }
  }
}
```
