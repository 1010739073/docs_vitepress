# Go 语言高频面试题与重要知识点

## Go 语言特点

- **编译型语言**：直接编译为机器码，执行效率高
- **垃圾回收**：自动内存管理，减轻开发者负担
- **并发支持**：内置 goroutine 和 channel，简化并发编程
- **静态类型**：编译时类型检查，减少运行时错误
- **简洁语法**：去除了传统语言的冗余语法元素
- **跨平台**：支持多种操作系统和架构

## 语言基础

### 1. 变量与类型

- **问题**：Go 语言的基本数据类型有哪些？
- **答案**：
  - 布尔型：`bool`
  - 数值型：`int`、`int8`、`int16`、`int32`、`int64`、`uint`、`uint8`、`uint16`、`uint32`、`uint64`、`uintptr`、`float32`、`float64`、`complex64`、`complex128`
  - 字符串：`string`
  - 派生类型：指针、数组、切片、映射、通道、结构体、接口

- **问题**：Go 语言中变量声明的方式有哪些？
- **答案**：
  - `var name type`
  - `var name type = value`
  - `var name = value`
  - `name := value`（短变量声明）

### 2. 控制结构

- **问题**：Go 语言的 if 语句有什么特点？
- **答案**：Go 语言的 if 语句可以在条件判断前执行一个简短的语句，例如：`if err := doSomething(); err != nil { ... }`

- **问题**：Go 语言的 for 循环有几种形式？
- **答案**：
  - 基本 for 循环：`for i := 0; i < n; i++ { ... }`
  - 类似 while 循环：`for condition { ... }`
  - 无限循环：`for { ... }`
  - range 循环：`for index, value := range collection { ... }`

### 3. 函数

- **问题**：Go 语言的函数有什么特点？
- **答案**：
  - 支持多返回值
  - 支持函数作为参数和返回值
  - 支持可变参数
  - 支持匿名函数和闭包
  - 不支持函数重载

- **问题**：什么是 defer 语句？它的执行顺序是怎样的？
- **答案**：defer 语句用于延迟执行函数调用，通常用于资源清理。defer 语句的执行顺序是后进先出（LIFO）。

### 4. 结构体与方法

- **问题**：如何定义结构体和方法？
- **答案**：
  ```go
  type Person struct {
      Name string
      Age  int
  }
  
  func (p *Person) SayHello() {
      fmt.Printf("Hello, my name is %s\n", p.Name)
  }
  ```

- **问题**：值接收者和指针接收者有什么区别？
- **答案**：值接收者会复制结构体，指针接收者会修改原结构体。指针接收者可以避免大结构体的复制开销，也可以修改结构体的状态。

## 并发编程

### 1. Goroutine

- **问题**：什么是 goroutine？如何创建？
- **答案**：goroutine 是 Go 语言的轻量级线程，由 Go 运行时管理。使用 `go` 关键字创建：`go function()`

- **问题**：goroutine 与线程的区别？
- **答案**：
  - goroutine 是用户态线程，线程是内核态线程
  - goroutine 开销小，内存占用少（约 2KB）
  - goroutine 由 Go 运行时调度，线程由操作系统调度
  - goroutine 间切换成本低

### 2. Channel

- **问题**：什么是 channel？如何使用？
- **答案**：channel 是 goroutine 之间的通信机制，用于在 goroutine 之间传递数据。
  ```go
  // 创建 channel
  ch := make(chan int)
  
  // 发送数据
  ch <- value
  
  // 接收数据
  value := <-ch
  ```

- **问题**：channel 的类型有哪些？
- **答案**：
  - 无缓冲 channel：`make(chan T)`
  - 有缓冲 channel：`make(chan T, size)`
  - 单向 channel：`chan<- T`（只发送）、`<-chan T`（只接收）

### 3. 并发控制

- **问题**：如何实现 goroutine 的同步？
- **答案**：
  - 使用 channel 进行通信
  - 使用 sync.WaitGroup
  - 使用 sync.Mutex 和 sync.RWMutex
  - 使用 sync.Once
  - 使用 context 包

- **问题**：什么是死锁？如何避免？
- **答案**：死锁是指两个或多个 goroutine 相互等待对方释放资源的情况。避免死锁的方法：
  - 避免嵌套锁
  - 使用带超时的 channel 操作
  - 使用 context 控制 goroutine 生命周期
  - 按固定顺序获取锁

## 内存管理

### 1. 垃圾回收

- **问题**：Go 语言的垃圾回收机制是怎样的？
- **答案**：Go 使用三色标记-清除算法进行垃圾回收，主要步骤：
  - 标记：从根对象开始，标记所有可达对象
  - 清除：回收未标记的对象
  - 并发执行：垃圾回收与用户代码并发执行，减少停顿时间

- **问题**：如何优化垃圾回收？
- **答案**：
  - 减少内存分配
  - 避免频繁创建大对象
  - 使用对象池复用对象
  - 合理使用 sync.Pool

### 2. 内存逃逸

- **问题**：什么是内存逃逸？
- **答案**：内存逃逸是指变量从栈内存逃逸到堆内存的现象。当变量的生命周期无法在编译时确定时，编译器会将其分配到堆上。

- **问题**：如何检测内存逃逸？
- **答案**：使用 `-gcflags=-m` 编译选项查看逃逸分析结果。

## 大厂高并发面试题与实战场景

### 1. 高并发场景：秒杀系统设计

**问题**：如何使用 Go 语言设计一个高并发的秒杀系统？

**详细答案**：
秒杀系统的核心挑战是：
- 高并发：短时间内大量用户同时请求
- 超卖问题：库存不能为负数
- 性能要求：响应时间要快

**技术方案**：
1. **Redis 预减库存**：使用 Redis 的原子操作预减库存
2. **消息队列异步处理**：将订单请求放入消息队列异步处理
3. **数据库乐观锁**：防止超卖
4. **限流降级**：保护系统不被打垮

**代码示例**：

```go
package main

import (
	"context"
	"fmt"
	"sync"
	"time"
	"github.com/go-redis/redis/v8"
)

type SeckillService struct {
	redisClient *redis.Client
	mu          sync.Mutex
}

func NewSeckillService() *SeckillService {
	return &SeckillService{
		redisClient: redis.NewClient(&redis.Options{
			Addr: "localhost:6379",
		}),
	}
}

func (s *SeckillService) PreReduceStock(ctx context.Context, productId string, userId string) (bool, error) {
	key := fmt.Sprintf("stock:%s", productId)
	
	// 使用 Lua 脚本保证原子性
	luaScript := `
		local stock = redis.call('GET', KEYS[1])
		if not stock then
			return -1
		end
		stock = tonumber(stock)
		if stock <= 0 then
			return 0
		end
		redis.call('DECR', KEYS[1])
		return 1
	`
	
	result, err := s.redisClient.Eval(ctx, luaScript, []string{key}).Int()
	if err != nil {
		return false, err
	}
	
	if result == 1 {
		// 预减库存成功，将订单放入消息队列
		err = s.sendToQueue(ctx, productId, userId)
		return err == nil, err
	}
	
	return false, nil
}

func (s *SeckillService) sendToQueue(ctx context.Context, productId string, userId string) error {
	// 实现消息队列发送逻辑
	fmt.Printf("订单已入队: productId=%s, userId=%s\n", productId, userId)
	return nil
}

// 模拟高并发秒杀
func main() {
	service := NewSeckillService()
	ctx := context.Background()
	
	// 初始化库存
	service.redisClient.Set(ctx, "stock:1001", 100, 0)
	
	var wg sync.WaitGroup
	successCount := 0
	var mu sync.Mutex
	
	// 模拟 1000 个并发用户
	for i := 0; i < 1000; i++ {
		wg.Add(1)
		go func(userId int) {
			defer wg.Done()
			success, err := service.PreReduceStock(ctx, "1001", fmt.Sprintf("user_%d", userId))
			if err == nil && success {
				mu.Lock()
				successCount++
				mu.Unlock()
			}
		}(i)
	}
	
	wg.Wait()
	fmt.Printf("成功抢购人数: %d\n", successCount)
}
```

### 2. 高并发场景：连接池设计

**问题**：如何设计一个高性能的数据库连接池？

**详细答案**：
连接池的核心要点：
- 复用连接，减少创建和销毁的开销
- 控制最大连接数，防止资源耗尽
- 支持连接的健康检查
- 支持连接的自动回收

**代码示例**：

```go
package main

import (
	"database/sql"
	"fmt"
	"sync"
	"time"
	_ "github.com/go-sql-driver/mysql"
)

type ConnectionPool struct {
	pool      chan *sql.DB
	maxConn   int
	minConn   int
	mu        sync.Mutex
	dbConfig  string
}

func NewConnectionPool(dbConfig string, maxConn, minConn int) (*ConnectionPool, error) {
	pool := &ConnectionPool{
		pool:     make(chan *sql.DB, maxConn),
		maxConn:  maxConn,
		minConn:  minConn,
		dbConfig: dbConfig,
	}
	
	// 初始化最小连接数
	for i := 0; i < minConn; i++ {
		db, err := pool.createConnection()
		if err != nil {
			return nil, err
		}
		pool.pool <- db
	}
	
	return pool, nil
}

func (p *ConnectionPool) createConnection() (*sql.DB, error) {
	db, err := sql.Open("mysql", p.dbConfig)
	if err != nil {
		return nil, err
	}
	
	// 设置连接参数
	db.SetMaxOpenConns(1)
	db.SetMaxIdleConns(1)
	db.SetConnMaxLifetime(time.Hour)
	
	// 测试连接
	if err := db.Ping(); err != nil {
		return nil, err
	}
	
	return db, nil
}

func (p *ConnectionPool) Get() (*sql.DB, error) {
	select {
	case db := <-p.pool:
		// 检查连接是否健康
		if err := db.Ping(); err != nil {
			return p.createConnection()
		}
		return db, nil
	default:
		// 池中没有可用连接，创建新连接
		return p.createConnection()
	}
}

func (p *ConnectionPool) Put(db *sql.DB) {
	select {
	case p.pool <- db:
		// 连接放回池中
	default:
		// 池已满，关闭连接
		db.Close()
	}
}

func (p *ConnectionPool) Close() {
	p.mu.Lock()
	defer p.mu.Unlock()
	
	close(p.pool)
	for db := range p.pool {
		db.Close()
	}
}

// 使用示例
func main() {
	pool, err := NewConnectionPool("user:password@tcp(localhost:3306)/database", 100, 10)
	if err != nil {
		panic(err)
	}
	defer pool.Close()
	
	var wg sync.WaitGroup
	
	// 模拟高并发查询
	for i := 0; i < 1000; i++ {
		wg.Add(1)
		go func(id int) {
			defer wg.Done()
			
			db, err := pool.Get()
			if err != nil {
				fmt.Printf("获取连接失败: %v\n", err)
				return
			}
			defer pool.Put(db)
			
			var result string
			err = db.QueryRow("SELECT name FROM users WHERE id = ?", id%100+1).Scan(&result)
			if err != nil {
				fmt.Printf("查询失败: %v\n", err)
				return
			}
			
			fmt.Printf("查询结果: %s\n", result)
		}(i)
	}
	
	wg.Wait()
}
```

### 3. 高并发场景：限流器实现

**问题**：如何实现一个高性能的限流器？

**详细答案**：
限流器的常见算法：
- **令牌桶算法**：以固定速率放入令牌，请求需要获取令牌才能通过
- **漏桶算法**：以固定速率处理请求，多余请求被丢弃
- **固定窗口计数器**：在固定时间窗口内限制请求数量
- **滑动窗口计数器**：更精确的计数器，避免边界问题

**代码示例（令牌桶算法）**：

```go
package main

import (
	"fmt"
	"sync"
	"time"
)

type TokenBucket struct {
	capacity    int64       // 桶容量
	tokens      int64       // 当前令牌数
	rate        int64       // 令牌生成速率（每秒）
	lastTime    time.Time   // 上次添加令牌的时间
	mu          sync.Mutex // 互斥锁
}

func NewTokenBucket(capacity, rate int64) *TokenBucket {
	return &TokenBucket{
		capacity: capacity,
		tokens:   capacity,
		rate:     rate,
		lastTime: time.Now(),
	}
}

func (tb *TokenBucket) Allow() bool {
	tb.mu.Lock()
	defer tb.mu.Unlock()
	
	now := time.Now()
	elapsed := now.Sub(tb.lastTime).Seconds()
	tb.lastTime = now
	
	// 计算应该添加的令牌数
	addTokens := int64(elapsed * float64(tb.rate))
	tb.tokens += addTokens
	
	// 令牌数不能超过容量
	if tb.tokens > tb.capacity {
		tb.tokens = tb.capacity
	}
	
	// 如果有令牌，则允许请求
	if tb.tokens > 0 {
		tb.tokens--
		return true
	}
	
	return false
}

// 滑动窗口限流器
type SlidingWindowLimiter struct {
	window    time.Duration // 时间窗口大小
	maxCount  int64         // 最大请求数
	requests  []time.Time   // 请求时间戳
	mu        sync.Mutex
}

func NewSlidingWindowLimiter(window time.Duration, maxCount int64) *SlidingWindowLimiter {
	return &SlidingWindowLimiter{
		window:   window,
		maxCount: maxCount,
		requests: make([]time.Time, 0),
	}
}

func (swl *SlidingWindowLimiter) Allow() bool {
	swl.mu.Lock()
	defer swl.mu.Unlock()
	
	now := time.Now()
	
	// 移除窗口外的请求
	for len(swl.requests) > 0 && now.Sub(swl.requests[0]) > swl.window {
		swl.requests = swl.requests[1:]
	}
	
	// 检查是否超过最大请求数
	if int64(len(swl.requests)) >= swl.maxCount {
		return false
	}
	
	// 添加当前请求
	swl.requests = append(swl.requests, now)
	return true
}

// 使用示例
func main() {
	// 令牌桶限流器：容量 100，每秒生成 10 个令牌
	tb := NewTokenBucket(100, 10)
	
	// 滑动窗口限流器：1 秒内最多 100 个请求
	swl := NewSlidingWindowLimiter(time.Second, 100)
	
	var wg sync.WaitGroup
	
	// 模拟高并发请求
	for i := 0; i < 200; i++ {
		wg.Add(1)
		go func(id int) {
			defer wg.Done()
			
			// 使用令牌桶限流
			if tb.Allow() {
				fmt.Printf("请求 %d 通过令牌桶限流\n", id)
			} else {
				fmt.Printf("请求 %d 被令牌桶限流\n", id)
			}
			
			// 使用滑动窗口限流
			if swl.Allow() {
				fmt.Printf("请求 %d 通过滑动窗口限流\n", id)
			} else {
				fmt.Printf("请求 %d 被滑动窗口限流\n", id)
			}
		}(i)
	}
	
	wg.Wait()
}
```

### 4. 高并发场景：分布式锁实现

**问题**：如何实现一个分布式锁？

**详细答案**：
分布式锁的核心要求：
- 互斥性：同一时间只有一个客户端能持有锁
- 避免死锁：锁应该有超时时间
- 高可用：锁服务应该高可用
- 可重入性：同一个客户端可以多次获取锁

**代码示例（基于 Redis）**：

```go
package main

import (
	"context"
	"fmt"
	"time"
	"github.com/go-redis/redis/v8"
)

type DistributedLock struct {
	client    *redis.Client
	key       string
	value     string
	ttl       time.Duration
	ctx       context.Context
}

func NewDistributedLock(client *redis.Client, key string, ttl time.Duration) *DistributedLock {
	return &DistributedLock{
		client: client,
		key:    key,
		value:  fmt.Sprintf("%d", time.Now().UnixNano()),
		ttl:    ttl,
		ctx:    context.Background(),
	}
}

func (dl *DistributedLock) Lock() (bool, error) {
	// 使用 SET NX EX 命令实现分布式锁
	result, err := dl.client.SetNX(dl.ctx, dl.key, dl.value, dl.ttl).Result()
	if err != nil {
		return false, err
	}
	
	return result, nil
}

func (dl *DistributedLock) Unlock() error {
	// 使用 Lua 脚本确保只有锁的持有者才能释放锁
	luaScript := `
		if redis.call("GET", KEYS[1]) == ARGV[1] then
			return redis.call("DEL", KEYS[1])
		else
			return 0
		end
	`
	
	_, err := dl.client.Eval(dl.ctx, luaScript, []string{dl.key}, dl.value).Result()
	return err
}

func (dl *DistributedLock) ExtendTTL() error {
	// 延长锁的过期时间
	luaScript := `
		if redis.call("GET", KEYS[1]) == ARGV[1] then
			return redis.call("EXPIRE", KEYS[1], ARGV[2])
		else
			return 0
		end
	`
	
	_, err := dl.client.Eval(dl.ctx, luaScript, []string{dl.key}, dl.value, int(dl.ttl.Seconds())).Result()
	return err
}

// 使用示例
func main() {
	client := redis.NewClient(&redis.Options{
		Addr: "localhost:6379",
	})
	
	lock := NewDistributedLock(client, "my_lock", 10*time.Second)
	
	// 获取锁
	acquired, err := lock.Lock()
	if err != nil {
		panic(err)
	}
	
	if acquired {
		fmt.Println("获取锁成功")
		
		// 执行业务逻辑
		time.Sleep(5 * time.Second)
		
		// 延长锁的过期时间
		err = lock.ExtendTTL()
		if err != nil {
			fmt.Printf("延长锁失败: %v\n", err)
		}
		
		// 释放锁
		err = lock.Unlock()
		if err != nil {
			fmt.Printf("释放锁失败: %v\n", err)
		}
	} else {
		fmt.Println("获取锁失败")
	}
}
```

### 5. 高并发场景：Worker Pool 模式

**问题**：如何实现一个高效的 Worker Pool？

**详细答案**：
Worker Pool 的核心要点：
- 固定数量的 worker goroutine
- 任务队列
- 优雅关闭
- 错误处理

**代码示例**：

```go
package main

import (
	"fmt"
	"sync"
	"time"
)

type Task struct {
	ID   int
	Data interface{}
}

type WorkerPool struct {
	workers    int
	taskQueue  chan Task
	quit       chan struct{}
	wg         sync.WaitGroup
}

func NewWorkerPool(workers int, queueSize int) *WorkerPool {
	return &WorkerPool{
		workers:   workers,
		taskQueue: make(chan Task, queueSize),
		quit:      make(chan struct{}),
	}
}

func (wp *WorkerPool) Start() {
	for i := 0; i < wp.workers; i++ {
		wp.wg.Add(1)
		go wp.worker(i)
	}
}

func (wp *WorkerPool) worker(id int) {
	defer wp.wg.Done()
	
	for {
		select {
		case task := <-wp.taskQueue:
			// 处理任务
			fmt.Printf("Worker %d 处理任务 %d\n", id, task.ID)
			time.Sleep(100 * time.Millisecond) // 模拟任务处理
		case <-wp.quit:
			fmt.Printf("Worker %d 退出\n", id)
			return
		}
	}
}

func (wp *WorkerPool) Submit(task Task) {
	wp.taskQueue <- task
}

func (wp *WorkerPool) Stop() {
	close(wp.quit)
	wp.wg.Wait()
	close(wp.taskQueue)
}

// 使用示例
func main() {
	pool := NewWorkerPool(10, 100)
	pool.Start()
	
	// 提交任务
	for i := 0; i < 100; i++ {
		pool.Submit(Task{
			ID:   i,
			Data: fmt.Sprintf("Task %d", i),
		})
	}
	
	// 等待所有任务完成
	time.Sleep(2 * time.Second)
	
	// 停止 worker pool
	pool.Stop()
}
```

### 6. 高并发场景：缓存击穿、穿透、雪崩解决方案

**问题**：如何解决缓存击穿、穿透、雪崩问题？

**详细答案**：

**缓存击穿**：
- **问题**：热点 key 过期，大量请求同时访问数据库
- **解决方案**：
  - 设置热点 key 永不过期
  - 使用互斥锁，只让一个请求去查询数据库
  - 提前预热热点数据

**缓存穿透**：
- **问题**：查询不存在的数据，请求直接打到数据库
- **解决方案**：
  - 布隆过滤器
  - 缓存空值
  - 请求参数校验

**缓存雪崩**：
- **问题**：大量 key 同时过期，数据库压力骤增
- **解决方案**：
  - 设置不同的过期时间
  - 使用互斥锁
  - 构建高可用的缓存集群

**代码示例**：

```go
package main

import (
	"context"
	"fmt"
	"sync"
	"time"
	"github.com/go-redis/redis/v8"
)

type CacheService struct {
	redisClient *redis.Client
	mu          sync.Map // 用于缓存击穿的互斥锁
}

func NewCacheService() *CacheService {
	return &CacheService{
		redisClient: redis.NewClient(&redis.Options{
			Addr: "localhost:6379",
		}),
	}
}

// 解决缓存击穿：使用互斥锁
func (cs *CacheService) GetWithMutex(ctx context.Context, key string) (string, error) {
	// 先查缓存
	val, err := cs.redisClient.Get(ctx, key).Result()
	if err == redis.Nil {
		// 缓存未命中，使用互斥锁
		mu, _ := cs.mu.LoadOrStore(key, &sync.Mutex{})
		mutex := mu.(*sync.Mutex)
		
		mutex.Lock()
		defer mutex.Unlock()
		
		// 再次检查缓存（双重检查）
		val, err = cs.redisClient.Get(ctx, key).Result()
		if err == redis.Nil {
			// 查询数据库
			val, err = cs.queryFromDB(key)
			if err != nil {
				return "", err
			}
			
			// 设置缓存，设置随机过期时间防止雪崩
			expiration := time.Hour + time.Duration(time.Now().UnixNano()%int64(time.Minute))
			cs.redisClient.Set(ctx, key, val, expiration)
		}
	} else if err != nil {
		return "", err
	}
	
	return val, nil
}

// 解决缓存穿透：缓存空值
func (cs *CacheService) GetWithNullCache(ctx context.Context, key string) (string, error) {
	val, err := cs.redisClient.Get(ctx, key).Result()
	if err == redis.Nil {
		// 查询数据库
		val, err = cs.queryFromDB(key)
		if err != nil {
			// 缓存空值，防止缓存穿透
			cs.redisClient.Set(ctx, key, "NULL", 5*time.Minute)
			return "", err
		}
		
		// 设置缓存
		cs.redisClient.Set(ctx, key, val, time.Hour)
	} else if err != nil {
		return "", err
	} else if val == "NULL" {
		return "", fmt.Errorf("data not found")
	}
	
	return val, nil
}

func (cs *CacheService) queryFromDB(key string) (string, error) {
	// 模拟数据库查询
	time.Sleep(100 * time.Millisecond)
	return fmt.Sprintf("value_for_%s", key), nil
}

// 使用示例
func main() {
	cs := NewCacheService()
	ctx := context.Background()
	
	var wg sync.WaitGroup
	
	// 模拟高并发请求
	for i := 0; i < 100; i++ {
		wg.Add(1)
		go func(id int) {
			defer wg.Done()
			
			val, err := cs.GetWithMutex(ctx, "hot_key")
			if err != nil {
				fmt.Printf("请求 %d 失败: %v\n", id, err)
				return
			}
			
			fmt.Printf("请求 %d 成功: %s\n", id, val)
		}(i)
	}
	
	wg.Wait()
}
```

### 7. 高并发场景：消息队列实现

**问题**：如何实现一个高性能的消息队列？

**详细答案**：
消息队列的核心功能：
- 异步处理
- 削峰填谷
- 解耦系统
- 可靠性保证

**代码示例（基于 Channel）**：

```go
package main

import (
	"context"
	"fmt"
	"sync"
	"time"
)

type Message struct {
	ID   int
	Data interface{}
}

type MessageQueue struct {
	queue      chan Message
	workers    int
	wg         sync.WaitGroup
	ctx        context.Context
	cancel     context.CancelFunc
}

func NewMessageQueue(queueSize, workers int) *MessageQueue {
	ctx, cancel := context.WithCancel(context.Background())
	return &MessageQueue{
		queue:   make(chan Message, queueSize),
		workers: workers,
		ctx:     ctx,
		cancel:  cancel,
	}
}

func (mq *MessageQueue) Start() {
	for i := 0; i < mq.workers; i++ {
		mq.wg.Add(1)
		go mq.worker(i)
	}
}

func (mq *MessageQueue) worker(id int) {
	defer mq.wg.Done()
	
	for {
		select {
		case msg := <-mq.queue:
			// 处理消息
			err := mq.processMessage(msg)
			if err != nil {
				fmt.Printf("Worker %d 处理消息 %d 失败: %v\n", id, msg.ID, err)
			} else {
				fmt.Printf("Worker %d 处理消息 %d 成功\n", id, msg.ID)
			}
		case <-mq.ctx.Done():
			fmt.Printf("Worker %d 退出\n", id)
			return
		}
	}
}

func (mq *MessageQueue) processMessage(msg Message) error {
	// 模拟消息处理
	time.Sleep(50 * time.Millisecond)
	return nil
}

func (mq *MessageQueue) Publish(msg Message) error {
	select {
	case mq.queue <- msg:
		return nil
	default:
		return fmt.Errorf("队列已满")
	}
}

func (mq *MessageQueue) Stop() {
	mq.cancel()
	mq.wg.Wait()
	close(mq.queue)
}

// 使用示例
func main() {
	mq := NewMessageQueue(1000, 10)
	mq.Start()
	
	// 发布消息
	for i := 0; i < 100; i++ {
		msg := Message{
			ID:   i,
			Data: fmt.Sprintf("Message %d", i),
		}
		
		err := mq.Publish(msg)
		if err != nil {
			fmt.Printf("发布消息 %d 失败: %v\n", i, err)
		}
	}
	
	// 等待消息处理完成
	time.Sleep(2 * time.Second)
	
	// 停止消息队列
	mq.Stop()
}
```

### 8. 高并发场景：Rate Limiter 实现

**问题**：如何实现一个基于 Redis 的分布式限流器？

**详细答案**：
分布式限流器的核心要点：
- 原子性操作
- 支持多种限流算法
- 高性能
- 可扩展

**代码示例**：

```go
package main

import (
	"context"
	"fmt"
	"time"
	"github.com/go-redis/redis/v8"
)

type RedisRateLimiter struct {
	client *redis.Client
	ctx    context.Context
}

func NewRedisRateLimiter(client *redis.Client) *RedisRateLimiter {
	return &RedisRateLimiter{
		client: client,
		ctx:    context.Background(),
	}
}

// 固定窗口限流
func (rl *RedisRateLimiter) FixedWindow(key string, limit int64, window time.Duration) (bool, error) {
	luaScript := `
		local current = redis.call("INCR", KEYS[1])
		if current == 1 then
			redis.call("EXPIRE", KEYS[1], ARGV[1])
		end
		if current > tonumber(ARGV[2]) then
			return 0
		else
			return 1
		end
	`
	
	result, err := rl.client.Eval(rl.ctx, luaScript, []string{key}, int(window.Seconds()), limit).Int()
	if err != nil {
		return false, err
	}
	
	return result == 1, nil
}

// 滑动窗口限流
func (rl *RedisRateLimiter) SlidingWindow(key string, limit int64, window time.Duration) (bool, error) {
	now := time.Now().UnixMilli()
	start := now - window.Milliseconds()
	
	luaScript := `
		redis.call("ZREMRANGEBYSCORE", KEYS[1], 0, ARGV[1])
		local count = redis.call("ZCARD", KEYS[1])
		if count < tonumber(ARGV[2]) then
			redis.call("ZADD", KEYS[1], ARGV[3], ARGV[3])
			redis.call("EXPIRE", KEYS[1], ARGV[4])
			return 1
		else
			return 0
		end
	`
	
	result, err := rl.client.Eval(rl.ctx, luaScript, []string{key}, start, limit, now, int(window.Seconds())).Int()
	if err != nil {
		return false, err
	}
	
	return result == 1, nil
}

// 令牌桶限流
func (rl *RedisRateLimiter) TokenBucket(key string, capacity, rate int64) (bool, error) {
	luaScript := `
		local capacity = tonumber(ARGV[1])
		local rate = tonumber(ARGV[2])
		local now = tonumber(ARGV[3])
		
		local info = redis.call("HMGET", KEYS[1], "tokens", "last_time")
		local tokens = tonumber(info[1])
		local last_time = tonumber(info[2])
		
		if tokens == nil then
			tokens = capacity
			last_time = now
		end
		
		local delta = math.max(0, now - last_time)
		local filled_tokens = math.min(capacity, tokens + delta * rate / 1000)
		
		if filled_tokens >= 1 then
			local new_tokens = filled_tokens - 1
			redis.call("HMSET", KEYS[1], "tokens", new_tokens, "last_time", now)
			redis.call("EXPIRE", KEYS[1], 3600)
			return 1
		else
			redis.call("HMSET", KEYS[1], "tokens", filled_tokens, "last_time", now)
			redis.call("EXPIRE", KEYS[1], 3600)
			return 0
		end
	`
	
	result, err := rl.client.Eval(rl.ctx, luaScript, []string{key}, capacity, rate, time.Now().UnixMilli()).Int()
	if err != nil {
		return false, err
	}
	
	return result == 1, nil
}

// 使用示例
func main() {
	client := redis.NewClient(&redis.Options{
		Addr: "localhost:6379",
	})
	
	limiter := NewRedisRateLimiter(client)
	
	// 固定窗口限流：1 分钟内最多 100 个请求
	allowed, err := limiter.FixedWindow("api_limit", 100, time.Minute)
	if err != nil {
		panic(err)
	}
	fmt.Printf("固定窗口限流结果: %v\n", allowed)
	
	// 滑动窗口限流：1 分钟内最多 100 个请求
	allowed, err = limiter.SlidingWindow("api_limit", 100, time.Minute)
	if err != nil {
		panic(err)
	}
	fmt.Printf("滑动窗口限流结果: %v\n", allowed)
	
	// 令牌桶限流：容量 100，每秒生成 10 个令牌
	allowed, err = limiter.TokenBucket("api_limit", 100, 10)
	if err != nil {
		panic(err)
	}
	fmt.Printf("令牌桶限流结果: %v\n", allowed)
}
```
