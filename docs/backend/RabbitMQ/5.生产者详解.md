# RabbitMQ 生产者详解

## 一、消息发送流程

### 1.1 发送流程图

```
Producer
   │
   ├─ 1. 创建消息
   │
   ├─ 2. 设置消息属性
   │
   ├─ 3. 发送到 Exchange
   │
   ├─ 4. Exchange 路由到 Queue
   │
   ├─ 5. Queue 存储消息
   │
   └─ 6. 返回发送结果
```

### 1.2 基本发送

```java
@Autowired
private RabbitTemplate rabbitTemplate;

// 方式一：简单发送
rabbitTemplate.convertAndSend("my.queue", "Hello World");

// 方式二：指定交换机和路由键
rabbitTemplate.convertAndSend("my.exchange", "routing.key", "Hello World");

// 方式三：发送对象
User user = new User("张三", 25);
rabbitTemplate.convertAndSend("user.exchange", "user.created", user);

// 方式四：发送字节数组
byte[] body = "Hello".getBytes(StandardCharsets.UTF_8);
rabbitTemplate.send("", "my.queue", new Message(body));
```

## 三、发送确认（Publisher Confirm）

### 3.1 确认机制

RabbitMQ 提供了两种确认机制：

| 机制 | 说明 | 场景 |
|------|------|------|
| **Confirm** | 消息是否到达 Exchange | 确保消息被接收 |
| **Return** | 消息是否被路由到 Queue | 确保消息被路由 |

### 3.2 Confirm 配置

```yaml
spring:
  rabbitmq:
    publisher-confirm-type: correlated  # 开启 Confirm
    publisher-returns: true             # 开启 Return
    template:
      mandatory: true  # 消息无法路由时返回
```

### 3.3 Confirm 回调

```java
@Configuration
public class RabbitMQConfig {

    @Bean
    public RabbitTemplate rabbitTemplate(ConnectionFactory connectionFactory) {
        RabbitTemplate template = new RabbitTemplate(connectionFactory);

        // Confirm 回调
        template.setConfirmCallback((correlationData, ack, cause) -> {
            if (ack) {
                log.info("消息发送成功, correlationId: {}",
                    correlationData != null ? correlationData.getId() : null);
            } else {
                log.error("消息发送失败, correlationId: {}, 原因: {}",
                    correlationData != null ? correlationData.getId() : null, cause);
                // 处理发送失败，如重试或记录日志
            }
        });

        // Return 回调
        template.setReturnsCallback(returnedMessage -> {
            log.error("消息未被路由, exchange: {}, routingKey: {}, message: {}",
                returnedMessage.getExchange(),
                returnedMessage.getRoutingKey(),
                new String(returnedMessage.getMessage().getBody()));
            // 处理未路由的消息
        });

        return template;
    }
}
```

### 3.4 带关联数据的发送

```java
public void sendMessageWithConfirm(String message) {
    CorrelationData correlationData = new CorrelationData(UUID.randomUUID().toString());
    correlationData.setFuture(new CompletableFuture<>());

    rabbitTemplate.convertAndSend("my.exchange", "routing.key", message, correlationData);

    // 异步获取确认结果
    correlationData.getFuture().thenConfirm(confirm -> {
        if (confirm.isAck()) {
            log.info("消息发送成功");
        } else {
            log.error("消息发送失败: {}", confirm.getReason());
        }
    });
}
```

## 四、事务机制

### 4.1 事务发送

RabbitMQ 支持事务发送，但性能较差，不推荐使用。

```java
@Autowired
private RabbitTemplate rabbitTemplate;

public void sendWithTransaction() {
    // 开启事务
    rabbitTemplate.setChannelTransacted(true);

    try {
        // 发送多条消息
        rabbitTemplate.convertAndSend("my.exchange", "routing.key", "Message 1");
        rabbitTemplate.convertAndSend("my.exchange", "routing.key", "Message 2");
        rabbitTemplate.convertAndSend("my.exchange", "routing.key", "Message 3");

        // 提交事务
        rabbitTemplate.execute(channel -> {
            channel.txCommit();
            return null;
        });
    } catch (Exception e) {
        // 回滚事务
        rabbitTemplate.execute(channel -> {
            channel.txRollback();
            return null;
        });
        throw e;
    }
}
```

### 4.2 Channel 级别事务

```java
try (Connection connection = factory.newConnection();
     Channel channel = connection.createChannel()) {

    // 开启事务
    channel.txSelect();

    try {
        // 发送消息
        channel.basicPublish("my.exchange", "routing.key", null, "Message 1".getBytes());
        channel.basicPublish("my.exchange", "routing.key", null, "Message 2".getBytes());

        // 提交事务
        channel.txCommit();
    } catch (Exception e) {
        // 回滚事务
        channel.txRollback();
        throw e;
    }
}
```

### 4.3 事务 vs Confirm

| 特性 | 事务 | Confirm |
|------|------|---------|
| 性能 | 差（同步阻塞）| 好（异步）|
| 可靠性 | 强 | 强 |
| 使用场景 | 对性能要求不高的场景 | 生产环境推荐 |

**建议**：优先使用 Confirm 机制。

## 五、消息重试

### 5.1 自动重试

```yaml
spring:
  rabbitmq:
    template:
      retry:
        enabled: true
        initial-interval: 1000    # 初始重试间隔
        max-interval: 10000       # 最大重试间隔
        multiplier: 2.0           # 间隔倍数
        max-attempts: 3           # 最大重试次数
```

### 5.2 手动重试

```java
@Component
public class RetryProducer {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    @Value("${rabbitmq.retry.max-attempts:3}")
    private int maxAttempts;

    @Value("${rabbitmq.retry.initial-delay:1000}")
    private long initialDelay;

    public void sendWithRetry(String exchange, String routingKey, Object message) {
        sendWithRetry(exchange, routingKey, message, 0);
    }

    private void sendWithRetry(String exchange, String routingKey,
                               Object message, int attempt) {
        try {
            rabbitTemplate.convertAndSend(exchange, routingKey, message);
            log.info("消息发送成功");
        } catch (Exception e) {
            if (attempt < maxAttempts) {
                long delay = initialDelay * (long) Math.pow(2, attempt);
                log.warn("消息发送失败，{}ms 后重试 (第 {} 次)", delay, attempt + 1);

                try {
                    Thread.sleep(delay);
                } catch (InterruptedException ie) {
                    Thread.currentThread().interrupt();
                }

                sendWithRetry(exchange, routingKey, message, attempt + 1);
            } else {
                log.error("消息发送失败，已达最大重试次数", e);
                // 记录到数据库或发送告警
                saveToDeadLetter(exchange, routingKey, message, e);
            }
        }
    }

    private void saveToDeadLetter(String exchange, String routingKey,
                                  Object message, Exception e) {
        // 保存到死信队列或数据库
    }
}
```

### 5.3 指数退避重试

```java
public class ExponentialBackoffRetry {

    private static final long INITIAL_DELAY = 1000;  // 1秒
    private static final long MAX_DELAY = 60000;     // 60秒
    private static final double MULTIPLIER = 2.0;    // 倍数
    private static final int MAX_ATTEMPTS = 5;       // 最大次数

    public static void retry(Runnable task) {
        retry(task, 0);
    }

    private static void retry(Runnable task, int attempt) {
        try {
            task.run();
        } catch (Exception e) {
            if (attempt < MAX_ATTEMPTS) {
                long delay = Math.min(
                    (long) (INITIAL_DELAY * Math.pow(MULTIPLIER, attempt)),
                    MAX_DELAY
                );
                log.warn("失败，{}ms 后重试 (第 {} 次)", delay, attempt + 1);

                try {
                    Thread.sleep(delay);
                } catch (InterruptedException ie) {
                    Thread.currentThread().interrupt();
                    return;
                }

                retry(task, attempt + 1);
            } else {
                log.error("达到最大重试次数", e);
                throw new RuntimeException("重试失败", e);
            }
        }
    }
}

// 使用
ExponentialBackoffRetry.retry(() -> {
    rabbitTemplate.convertAndSend("my.exchange", "routing.key", message);
});
```

## 七、消息压缩

### 7.1 Gzip 压缩

```java
public class MessageCompressor {

    public static Message compress(Message message) throws IOException {
        byte[] compressed = compress(message.getBody());
        MessageProperties properties = message.getMessageProperties();
        properties.setHeader("compressed", "gzip");
        properties.setContentEncoding("gzip");
        return new Message(compressed, properties);
    }

    public static Message decompress(Message message) throws IOException {
        String compression = (String) message.getMessageProperties()
            .getHeaders().get("compressed");

        if ("gzip".equals(compression)) {
            byte[] decompressed = decompress(message.getBody());
            MessageProperties properties = message.getMessageProperties();
            properties.getHeaders().remove("compressed");
            properties.setContentEncoding(null);
            return new Message(decompressed, properties);
        }

        return message;
    }

    private static byte[] compress(byte[] data) throws IOException {
        ByteArrayOutputStream bos = new ByteArrayOutputStream(data.length);
        try (GZIPOutputStream gzip = new GZIPOutputStream(bos)) {
            gzip.write(data);
        }
        return bos.toByteArray();
    }

    private static byte[] decompress(byte[] data) throws IOException {
        ByteArrayInputStream bis = new ByteArrayInputStream(data);
        ByteArrayOutputStream bos = new ByteArrayOutputStream();
        byte[] buffer = new byte[1024];
        int len;

        try (GZIPInputStream gzip = new GZIPInputStream(bis)) {
            while ((len = gzip.read(buffer)) > 0) {
                bos.write(buffer, 0, len);
            }
        }

        return bos.toByteArray();
    }
}

// 使用
public void sendCompressedMessage(String message) throws IOException {
    Message original = new Message(message.getBytes());
    Message compressed = MessageCompressor.compress(original);
    rabbitTemplate.send("", "my.queue", compressed);
}
```

```java
public void sendBatchWithConfirm(String exchange, String routingKey, List<Object> messages) {
    List<CorrelationData> correlationDataList = new ArrayList<>();

    // 批量发送
    messages.forEach(message -> {
        CorrelationData correlationData = new CorrelationData(UUID.randomUUID().toString());
        correlationDataList.add(correlationData);
        rabbitTemplate.convertAndSend(exchange, routingKey, message, correlationData);
    });

    // 等待所有 Confirm
    CompletableFuture.allOf(correlationDataList.stream()
        .map(CorrelationData::getFuture)
        .toArray(CompletableFuture[]::new)
    ).join();
}
```

## 九、发送监控

### 9.1 发送统计

```java
@Component
public class SendMonitor {

    private final AtomicLong sendCount = new AtomicLong(0);
    private final AtomicLong successCount = new AtomicLong(0);
    private final AtomicLong failureCount = new AtomicLong(0);

    @Autowired
    private RabbitTemplate rabbitTemplate;

    @PostConstruct
    public void init() {
        rabbitTemplate.setConfirmCallback((correlationData, ack, cause) -> {
            long count = sendCount.incrementAndGet();
            if (ack) {
                successCount.incrementAndGet();
            } else {
                failureCount.incrementAndGet();
            }

            if (count % 1000 == 0) {
                log.info("发送统计 - 总数: {}, 成功: {}, 失败: {}, 成功率: {}%",
                    count, successCount.get(), failureCount.get(),
                    successCount.get() * 100.0 / count);
            }
        });
    }

    public Map<String, Object> getStatistics() {
        Map<String, Object> stats = new HashMap<>();
        stats.put("total", sendCount.get());
        stats.put("success", successCount.get());
        stats.put("failure", failureCount.get());
        long total = sendCount.get();
        if (total > 0) {
            stats.put("successRate", successCount.get() * 100.0 / total);
        }
        return stats;
    }
}
```

### 9.2 发送耗时监控

```java
@Component
public class SendLatencyMonitor {

    private final MeterRegistry meterRegistry;

    public SendLatencyMonitor(MeterRegistry meterRegistry) {
        this.meterRegistry = meterRegistry;
    }

    @Around("execution(* org.springframework.amqp.rabbit.core.RabbitTemplate.convertAndSend(..))")
    public Object monitorSendLatency(ProceedingJoinPoint joinPoint) throws Throwable {
        long start = System.currentTimeMillis();
        try {
            Object result = joinPoint.proceed();
            long latency = System.currentTimeMillis() - start;

            // 记录到 Micrometer
            Timer.Sample sample = Timer.start(meterRegistry);
            sample.stop(Timer.builder("rabbitmq.send.latency")
                .tag("exchange", (String) joinPoint.getArgs()[0])
                .tag("routing_key", (String) joinPoint.getArgs()[1])
                .register(meterRegistry));

            return result;
        } catch (Exception e) {
            long latency = System.currentTimeMillis() - start;
            log.error("发送失败, 耗时: {}ms", latency, e);
            throw e;
        }
    }
}
```

## 十、最佳实践

### 10.1 发送建议

| 建议 | 说明 |
|------|------|
| 使用 Confirm | 确保消息可靠发送 |
| 设置消息 ID | 便于追踪和重试 |
| 合理设置 TTL | 避免消息无限堆积 |
| 使用批量发送 | 提高吞吐量 |
| 异步处理 Confirm | 不阻塞发送流程 |
| 监控发送指标 | 及时发现问题 |

### 10.2 代码模板

```java
@Service
@Slf4j
public class ProducerService {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    public void sendMessage(String exchange, String routingKey, Object message) {
        try {
            // 设置消息 ID
            String messageId = UUID.randomUUID().toString();

            // 设置消息属性
            MessageProperties properties = new MessageProperties();
            properties.setMessageId(messageId);
            properties.setTimestamp(new Date());
            properties.setDeliveryMode(MessageDeliveryMode.PERSISTENT);

            // 创建关联数据
            CorrelationData correlationData = new CorrelationData(messageId);

            // 发送消息
            rabbitTemplate.convertAndSend(exchange, routingKey,
                new Message(objectToJson(message).getBytes(), properties),
                correlationData);

            log.info("消息发送成功, messageId: {}, exchange: {}, routingKey: {}",
                messageId, exchange, routingKey);

        } catch (Exception e) {
            log.error("消息发送异常, exchange: {}, routingKey: {}",
                exchange, routingKey, e);
            throw new RuntimeException("消息发送失败", e);
        }
    }

    private String objectToJson(Object obj) {
        // JSON 序列化
        return JSON.toJSONString(obj);
    }
}
```
