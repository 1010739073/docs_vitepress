# RabbitMQ 常规面试题

## 一、基础概念

### Q1: 什么是 RabbitMQ？它有哪些特点？

**答案：**

RabbitMQ 是一个开源的消息代理软件，实现了 AMQP 协议，用于在应用程序之间传递消息。

**主要特点：**

| 特点 | 说明 |
|------|------|
| **可靠性** | 支持消息持久化、消息确认、事务等机制 |
| **灵活性** | 支持多种消息路由模式（Direct、Fanout、Topic、Headers）|
| **高可用性** | 支持集群部署、镜像队列、仲裁队列 |
| **多协议支持** | 支持 AMQP、STOMP、MQTT、HTTP 等 |
| **多语言支持** | 提供 Java、Python、Go、Node.js 等多种语言客户端 |
| **管理界面** | 提供基于 Web 的管理控制台 |
| **插件系统** | 丰富的插件生态（延迟消息、监控、联邦等）|

### Q2: RabbitMQ 的核心组件有哪些？

**答案：**

| 组件 | 说明 |
|------|------|
| **Producer（生产者）** | 创建消息并发布到 Exchange |
| **Exchange（交换机）** | 接收消息并根据路由规则分发到 Queue |
| **Queue（队列）** | 存储消息的容器 |
| **Binding（绑定）** | Exchange 和 Queue 之间的关联关系 |
| **Routing Key（路由键）** | Exchange 路由消息的规则 |
| **Consumer（消费者）** | 从 Queue 获取消息并处理 |
| **Connection（连接）** | 客户端与 RabbitMQ 的 TCP 连接 |
| **Channel（通道）** | Connection 内部的逻辑连接，用于多路复用 |
| **Virtual Host（虚拟主机）** | 逻辑隔离单位 |
| **Broker（代理）** | RabbitMQ 服务器实例 |

### Q3: Exchange 有哪些类型？区别是什么？

**答案：**

| 类型 | 说明 | 路由规则 | 适用场景 |
|------|------|----------|----------|
| **Direct** | 直连交换机 | 完全匹配 Routing Key | 点对点消息路由 |
| **Fanout** | 扇形交换机 | 忽略 Routing Key，广播 | 消息广播 |
| **Topic** | 主题交换机 | 通配符匹配（`*` 和 `#`）| 灵活的路由规则 |
| **Headers** | 头交换机 | 根据消息头属性匹配 | 复杂的路由规则 |

**示例代码：**

```java
// Direct Exchange
@Bean
public DirectExchange directExchange() {
    return new DirectExchange("direct.exchange");
}

// Fanout Exchange
@Bean
public FanoutExchange fanoutExchange() {
    return new FanoutExchange("fanout.exchange");
}

// Topic Exchange
@Bean
public TopicExchange topicExchange() {
    return new TopicExchange("topic.exchange");
}

// Headers Exchange
@Bean
public HeadersExchange headersExchange() {
    return new HeadersExchange("headers.exchange");
}
```

### Q4: 什么是 Virtual Host？为什么需要它？

**答案：**

Virtual Host 是 RabbitMQ 的逻辑隔离单位，类似于数据库中的数据库概念。

**作用：**
- **资源隔离**：不同 vhost 中的 Exchange、Queue 互不干扰
- **权限控制**：可以为不同用户分配不同 vhost 的权限
- **多租户**：一个 RabbitMQ 服务器可以服务多个应用

**示例：**

```bash
# 创建 vhost
rabbitmqctl add_vhost /app1
rabbitmqctl add_vhost /app2

# 授予用户权限
rabbitmqctl set_permissions -p /app1 user1 ".*" ".*" ".*"
rabbitmqctl set_permissions -p /app2 user2 ".*" ".*" ".*"
```

## 二、消息模型

### Q5: RabbitMQ 有哪些消息模型？

**答案：**

| 模型 | 交换机类型 | 说明 | 适用场景 |
|------|-----------|------|----------|
| **简单队列** | Default | 一对一 | 简单任务 |
| **工作队列** | Default | 多消费者竞争消费 | 任务分发 |
| **发布订阅** | Fanout | 一对多广播 | 系统广播 |
| **路由** | Direct | 按路由键分发 | 日志分级 |
| **主题** | Topic | 通配符路由 | 复杂路由 |

### Q6: 如何保证消息不丢失？

**答案：**

需要从三个阶段保证：

**1. 发送阶段**
```yaml
spring:
  rabbitmq:
    publisher-confirm-type: correlated  # 开启 Confirm
    publisher-returns: true             # 开启 Return
    template:
      mandatory: true  # 消息无法路由时返回
```

```java
// Confirm 回调
template.setConfirmCallback((correlationData, ack, cause) -> {
    if (!ack) {
        // 处理发送失败
        retry(correlationData);
    }
});

// Return 回调
template.setReturnsCallback(returnedMessage -> {
    // 处理未路由的消息
    handleUnrouted(returnedMessage);
});
```

**2. 存储阶段**
```java
// 队列持久化
QueueBuilder.durable("my.queue").build();

// 消息持久化
MessageBuilder
    .withBody("Hello".getBytes())
    .setDeliveryMode(MessageDeliveryMode.PERSISTENT)
    .build();

// 交换机持久化
new DirectExchange("my.exchange", true, false);
```

**3. 消费阶段**
```yaml
spring:
  rabbitmq:
    listener:
      simple:
        acknowledge-mode: manual  # 手动 ACK
```

```java
@RabbitListener(queues = "my.queue")
public void receiveMessage(Message message, Channel channel,
                         @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
    try {
        processMessage(message);
        channel.basicAck(deliveryTag, false);
    } catch (Exception e) {
        channel.basicNack(deliveryTag, false, true); // 重新入队
    }
}
```

### Q7: 如何保证消息不重复消费？

**答案：**

**核心思想**：实现幂等性

**方案一：数据库唯一索引**

```java
public void processOrder(Order order) {
    try {
        orderDao.insert(order); // order_id 有唯一索引
    } catch (DuplicateKeyException e) {
        log.warn("重复订单: {}", order.getId());
    }
}
```

**方案二：Redis 缓存**

```java
@RabbitListener(queues = "my.queue")
public void receiveMessage(String message) {
    String messageId = getMessageId(message);

    // 检查是否已处理
    if (redisTemplate.hasKey("processed:" + messageId)) {
        log.info("消息已处理，跳过: {}", messageId);
        return;
    }

    try {
        // 处理消息
        processMessage(message);

        // 标记已处理
        redisTemplate.opsForValue().set("processed:" + messageId, "1", 24, TimeUnit.HOURS);
    } catch (Exception e) {
        log.error("处理失败", e);
        throw e;
    }
}
```

**方案三：状态机**

```java
public enum OrderStatus {
    CREATED, PROCESSING, COMPLETED, CANCELLED
}

public void processOrder(Order order) {
    int updated = orderDao.updateStatus(
        order.getId(),
        OrderStatus.CREATED,
        OrderStatus.PROCESSING
    );

    if (updated == 0) {
        // 状态不是 CREATED，说明已处理
        return;
    }

    try {
        doProcess(order);
        orderDao.updateStatus(order.getId(), OrderStatus.COMPLETED);
    } catch (Exception e) {
        orderDao.updateStatus(order.getId(), OrderStatus.CREATED);
        throw e;
    }
}
```

### Q8: 如何保证消息顺序性？

**答案：**

**方案一：单队列单消费者**

```java
// 所有消息发送到同一个队列
rabbitTemplate.convertAndSend("", "order.queue", message);

// 只有一个消费者
@RabbitListener(queues = "order.queue", concurrency = "1")
public void receiveMessage(String message) {
    processMessage(message);
}
```

**方案二：分区有序**

```java
// 相同 orderId 的消息发送到同一个队列
public void sendOrderMessage(OrderMessage message) {
    int hash = Math.abs(message.getOrderId().hashCode());
    int queueIndex = hash % queueCount;

    String queueName = "order.queue." + queueIndex;
    rabbitTemplate.convertAndSend("", queueName, message);
}

// 每个队列一个消费者
@RabbitListener(queues = "order.queue.0", concurrency = "1")
public void receiveFromQueue0(String message) { ... }

@RabbitListener(queues = "order.queue.1", concurrency = "1")
public void receiveFromQueue1(String message) { ... }
```

## 三、高级特性

### Q9: 什么是死信队列？如何配置？

**答案：**

死信队列用于处理无法正常消费的消息。

**成为死信的情况：**
1. 消息被拒绝（basicNack/basicReject，且 requeue=false）
2. 消息 TTL 到期
3. 队列达到最大长度
4. 队列被删除

**配置死信队列：**

```java
@Configuration
public class DeadLetterConfig {

    // 死信交换机和队列
    @Bean
    public DirectExchange dlxExchange() {
        return new DirectExchange("dlx.exchange");
    }

    @Bean
    public Queue dlxQueue() {
        return QueueBuilder.durable("dlx.queue").build();
    }

    @Bean
    public Binding dlxBinding() {
        return BindingBuilder.bind(dlxQueue())
            .to(dlxExchange())
            .with("dlx.routing.key");
    }

    // 业务队列，绑定死信交换机
    @Bean
    public Queue businessQueue() {
        Map<String, Object> args = new HashMap<>();
        args.put("x-dead-letter-exchange", "dlx.exchange");
        args.put("x-dead-letter-routing-key", "dlx.routing.key");
        args.put("x-message-ttl", 60000); // 消息 TTL

        return QueueBuilder.durable("business.queue")
            .withArguments(args)
            .build();
    }
}
```

### Q10: 如何实现延迟队列？

**答案：**

**方案一：基于死信队列**

```java
@Configuration
public class DelayQueueConfig {

    // 延迟队列（TTL 队列）
    @Bean
    public Queue delayQueue() {
        Map<String, Object> args = new HashMap<>();
        args.put("x-message-ttl", 30000); // 30 秒
        args.put("x-dead-letter-exchange", "dlx.exchange");
        args.put("x-dead-letter-routing-key", "target.key");

        return QueueBuilder.durable("delay.queue")
            .withArguments(args)
            .build();
    }

    // 目标队列
    @Bean
    public Queue targetQueue() {
        return QueueBuilder.durable("target.queue").build();
    }

    @Bean
    public Binding targetBinding() {
        return BindingBuilder.bind(targetQueue())
            .to(dlxExchange())
            .with("target.key");
    }
}
```

**方案二：使用延迟插件**

```bash
# 安装插件
rabbitmq-plugins enable rabbitmq_delayed_message_exchange
```

```java
@Bean
public CustomExchange delayedExchange() {
    Map<String, Object> args = new HashMap<>();
    args.put("x-delayed-type", "direct");

    return new CustomExchange("delayed.exchange", "x-delayed-message", true, false, args);
}

// 发送延迟消息
Message msg = MessageBuilder
    .withBody(JSON.toJSONBytes(message))
    .setHeader("x-delay", 30000) // 延迟 30 秒
    .build();

rabbitTemplate.convertAndSend("delayed.exchange", "delayed.key", msg);
```

### Q11: 什么是镜像队列？如何配置？

**答案：**

镜像队列是 RabbitMQ 的高可用解决方案，队列数据在多个节点上有副本。

**配置镜像策略：**

```bash
# 所有队列都是镜像队列，复制到所有节点
rabbitmqctl set_policy ha-all "^" \
  '{"ha-mode":"all","ha-sync-mode":"automatic"}'

# 只对 ha. 开头的队列做镜像
rabbitmqctl set_policy ha-named "^ha\." \
  '{"ha-mode":"all","ha-sync-mode":"automatic"}'
```

**Java 配置：**

```java
@Bean
public Queue mirroredQueue() {
    return QueueBuilder.durable("ha.my.queue").build();
    // 队列名匹配策略，自动成为镜像队列
}
```

### Q12: 什么是仲裁队列？和镜像队列的区别？

**答案：**

仲裁队列（Quorum Queue）是 RabbitMQ 3.8+ 引入的新队列类型，使用 Raft 共识算法。

**对比：**

| 特性 | 镜像队列 | 仲裁队列 |
|------|----------|----------|
| 一致性 | 最终一致 | 强一致 |
| 故障恢复 | 主从选举 | Raft 选举 |
| 性能 | 较低 | 较高 |
| 消息去重 | 不支持 | 支持 |
| 投递限制 | 不支持 | 支持 |

**创建仲裁队列：**

```java
@Bean
public Queue quorumQueue() {
    return QueueBuilder
        .durable("quorum.queue")
        .quorum()  // 设置为仲裁队列
        .build();
}
```

## 四、性能优化

### Q13: 如何提高 RabbitMQ 性能？

**答案：**

**1. 生产者优化**

```java
// 连接复用
CachingConnectionFactory factory = new CachingConnectionFactory();
factory.setChannelCacheSize(25);

// 批量发送
BatchingRabbitTemplate batchTemplate = new BatchingRabbitTemplate(...);
batchTemplate.setBatchSize(100);

// 异步 Confirm
template.setConfirmCallback(...);
```

**2. 消费者优化**

```yaml
spring:
  rabbitmq:
    listener:
      simple:
        prefetch: 10           # 合理的 prefetch
        concurrency: 10        # 增加消费者数量
        batch: true           # 批量消费
```

**3. 队列优化**

```java
// 使用非持久化消息（允许丢失）
MessageBuilder
    .withBody("Hello".getBytes())
    .setDeliveryMode(MessageDeliveryMode.NON_PERSISTENT)
    .build();

// 使用仲裁队列（性能更好）
QueueBuilder.durable("queue").quorum().build();
```

**4. 系统优化**

```bash
# 调整文件描述符限制
ulimit -n 65536

# 调整内存限制
# rabbitmq.conf
vm_memory_high_watermark.relative = 0.6

# 调整磁盘限制
disk_free_limit.absolute = 2GB
```

### Q14: 什么是 Prefetch？如何设置？

**答案：**

Prefetch 控制消费者未确认的消息数量。

**作用：**
- 公平分发：设置 prefetch=1，避免慢消费者积压消息
- 提高吞吐量：设置较大的 prefetch，减少网络往返

**配置：**

```yaml
spring:
  rabbitmq:
    listener:
      simple:
        prefetch: 10  # 每个消费者最多接收 10 条未确认的消息
```

```java
@Bean
public SimpleRabbitListenerContainerFactory rabbitListenerContainerFactory(
        ConnectionFactory connectionFactory) {
    SimpleRabbitListenerContainerFactory factory =
        new SimpleRabbitListenerContainerFactory();
    factory.setConnectionFactory(connectionFactory);
    factory.setPrefetch(1); // 公平分发
    return factory;
}
```

## 五、运维管理

### Q15: RabbitMQ 的消息如何持久化？

**答案：**

RabbitMQ 提供了两种持久化方式：

**1. 队列持久化**

```java
@Bean
public Queue durableQueue() {
    return QueueBuilder.durable("my.queue").build();
}
```

**2. 消息持久化**

```java
Message message = MessageBuilder
    .withBody("Hello".getBytes())
    .setDeliveryMode(MessageDeliveryMode.PERSISTENT)
    .build();
```

**3. 交换机持久化**

```java
@Bean
public DirectExchange durableExchange() {
    return new DirectExchange("my.exchange", true, false);
}
```

**注意：** 只有队列、交换机、消息都持久化，才能保证 RabbitMQ 重启后消息不丢失。

### Q16: 如何监控 RabbitMQ？

**答案：**

**1. Web 管理界面**

访问 `http://localhost:15672`，查看：
- 概览：连接、通道、队列统计
- 队列：消息数、消费速率
- 连接：连接列表
- 通道：通道列表

**2. 命令行工具**

```bash
# 查看集群状态
rabbitmqctl cluster_status

# 查看队列
rabbitmqctl list_queues

# 查看连接
rabbitmqctl list_connections

# 查看消费者
rabbitmqctl list_consumers
```

**3. Prometheus + Grafana**

```bash
# 启用 Prometheus 插件
rabbitmq-plugins enable rabbitmq_prometheus

# 访问指标
http://localhost:15692/metrics
```

**4. 关键监控指标**

| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| `queue_messages` | 队列消息数 | > 10000 |
| `queue_messages_unacked` | 未确认消息数 | > 1000 |
| `channel_consumers` | 消费者数量 | = 0 |
| `connection_count` | 连接数 | > 1000 |
| `fd_used` | 文件描述符使用数 | > 80% |

### Q17: RabbitMQ 集群有哪些类型？

**答案：**

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| **普通集群** | 数据只在一个节点，其他节点只有元数据 | 测试环境 |
| **镜像集群** | 数据在多个节点有副本 | 生产环境（旧项目）|
| **仲裁队列** | 使用 Raft 协议，强一致性 | 生产环境（新项目）|

**普通集群部署：**

```bash

rabbitmqctl stop_app
rabbitmqctl reset
rabbitmqctl join_cluster rabbit@node1
rabbitmqctl start_app
```

**镜像集群配置：**

```bash

  '{"ha-mode":"all","ha-sync-mode":"automatic"}'
```

**仲裁队列配置：**

@Bean
public Queue quorumQueue() {
