# RabbitMQ 集群架构

## 一、集群基础

### 1.1 为什么需要集群

| 问题 | 单机 | 集群 |
|------|------|------|
| 高可用性 | 单点故障 | 故障自动转移 |
| 可扩展性 | 垂直扩展 | 水平扩展 |
| 吞吐量 | 受限于单机 | 多机分担 |
| 容量 | 受限于单机 | 多机汇总 |

### 1.2 集群类型

#### 普通集群（Classic Cluster）

```
Node 1              Node 2              Node 3
  │                   │                   │
  ├─ Queue A         ├─ Queue A(元数据)  ├─ Queue A(元数据)
  │                   │                   │
  └─ Queue B         └─ Queue B(元数据)  └─ Queue B(元数据)

特点：
- 数据只在一个节点
- 其他节点只有元数据
- 节点故障时，该节点的队列不可用
```

#### 镜像集群（Mirrored Cluster）

```
Node 1              Node 2              Node 3
  │                   │                   │
  ├─ Queue A         ├─ Queue A(镜像)   ├─ Queue A(镜像)
  │                   │                   │
  └─ Queue B         └─ Queue B(镜像)   └─ Queue B(镜像)

特点：
- 数据在多个节点有副本
- 节点故障时，其他节点自动接管
- 写入性能较低（需要同步）
```

#### 仲裁队列集群（Quorum Queue）

```
Node 1              Node 2              Node 3
  │                   │                   │
  └─ Quorum A ───────┴─ Quorum A ───────┴─ Quorum A
      (Leader)           (Follower)          (Follower)

特点：
- 使用 Raft 协议
- 数据一致性更强
- 更好的故障恢复
```

## 二、普通集群部署

### 2.1 节点准备

```bash
# Node 1
hostnamectl set-hostname rabbitmq1

# Node 2
hostnamectl set-hostname rabbitmq2

# Node 3
hostnamectl set-hostname rabbitmq3

# 修改 /etc/hosts
echo "192.168.1.101 rabbitmq1" >> /etc/hosts
echo "192.168.1.102 rabbitmq2" >> /etc/hosts
echo "192.168.1.103 rabbitmq3" >> /etc/hosts
```

### 2.2 同步 Erlang Cookie

```bash
# 在 Node 1 上
cat /var/lib/rabbitmq/.erlang.cookie

# 复制到 Node 2 和 Node 3
# 确保所有节点的 .erlang.cookie 内容一致且权限正确
chmod 400 /var/lib/rabbitmq/.erlang.cookie
chown rabbitmq:rabbitmq /var/lib/rabbitmq/.erlang.cookie
```

### 2.3 启动节点

```bash
# 所有节点启动 RabbitMQ
systemctl start rabbitmq-server
systemctl enable rabbitmq-server
```

### 2.4 组建集群

```bash
# 在 Node 2 上加入 Node 1
rabbitmqctl stop_app
rabbitmqctl reset
rabbitmqctl join_cluster rabbit@rabbitmq1
rabbitmqctl start_app

# 在 Node 3 上加入 Node 1
rabbitmqctl stop_app
rabbitmqctl reset
rabbitmqctl join_cluster rabbit@rabbitmq1
rabbitmqctl start_app
```

### 2.6 节点类型

```bash
# 将节点设置为 RAM 节点（不推荐）
rabbitmqctl stop_app
rabbitmqctl change_cluster_node_type ram
rabbitmqctl start_app

# 将节点设置为磁盘节点（推荐）
rabbitmqctl stop_app
rabbitmqctl change_cluster_node_type disc
rabbitmqctl start_app
```

**建议**：至少保留一个磁盘节点，其他可以是 RAM 节点。

## 三、镜像队列

### 3.1 配置镜像策略

```bash
# 策略：所有队列都是镜像队列，复制到所有节点
rabbitmqctl set_policy ha-all "^" \
  '{"ha-mode":"all","ha-sync-mode":"automatic"}'

# 策略：只对 ha. 开头的队列做镜像
rabbitmqctl set_policy ha-named "^ha\." \
  '{"ha-mode":"all","ha-sync-mode":"automatic"}'

# 策略：指定镜像节点数
rabbitmqctl set_policy ha-two "^" \
  '{"ha-mode":"exactly","ha-count":2,"ha-sync-mode":"automatic"}'

# 策略：指定镜像节点
rabbitmqctl set_policy ha-nodes "^" \
  '{"ha-mode":"nodes","ha-params":["rabbit@node1","rabbit@node2"],"ha-sync-mode":"automatic"}'
```

### 3.2 策略参数说明

| 参数 | 说明 | 可选值 |
|------|------|--------|
| `ha-mode` | 镜像模式 | `all`、`exactly`、`nodes` |
| `ha-sync-mode` | 同步模式 | `manual`、`automatic` |
| `ha-sync-batch-size` | 同步批次大小 | 数字 |
| `ha-count` | 镜像节点数（exactly 模式）| 数字 |
| `ha-params` | 镜像节点列表（nodes 模式）| 节点列表 |

### 3.3 Java 配置镜像队列

```java
@Configuration
public class MirroredQueueConfig {

    @Bean
    public Queue mirroredQueue() {
        return QueueBuilder
            .durable("ha.my.queue")
            .build();
        // 队列名匹配策略，自动成为镜像队列
    }
}
```

### 3.4 故障转移

```
正常状态：
Node 1 (Master) ──→ Node 2 (Slave) ──→ Node 3 (Slave)
        │                 │                  │
    Queue A          Queue A(镜像)      Queue A(镜像)

Node 1 故障：
Node 1 (Down)      Node 2 (Master) ←── Node 3 (Slave)
        ↓                  ↑                  ↑
    不可用         Queue A(接管)       Queue A(镜像)
```

## 四、仲裁队列（Quorum Queue）

### 4.1 创建仲裁队列

```java
@Configuration
public class QuorumQueueConfig {

    @Bean
    public Queue quorumQueue() {
        return QueueBuilder
            .durable("quorum.my.queue")
            .quorum()  // 设置为仲裁队列
            .build();
    }
}
```

### 4.2 仲裁队列参数

```java
@Bean
public Queue quorumQueueWithParams() {
    Map<String, Object> args = new HashMap<>();
    args.put("x-quorum-initial-group-size", 3); // 初始副本数
    args.put("x-delivery-limit", 5);           // 投递限制

    return QueueBuilder
        .durable("quorum.queue")
        .quorum()
        .withArguments(args)
        .build();
}
```

### 4.3 镜像队列 vs 仲裁队列

| 特性 | 镜像队列 | 仲裁队列 |
|------|----------|----------|
| 一致性 | 最终一致 | 强一致 |
| 故障恢复 | 主从选举 | Raft 选举 |
| 性能 | 较低 | 较高 |
| 消息去重 | 不支持 | 支持 |
| 投递限制 | 不支持 | 支持 |
| 适用场景 | 传统场景 | 新项目推荐 |

## 五、联邦（Federation）

### 5.1 什么是联邦

联邦允许不同 RabbitMQ 集群之间共享消息。

```
Cluster A (北京)                    Cluster B (上海)
    │                                  │
    ├── Exchange A                     ├── Exchange B
    │   └── Queue A                   │   └── Queue B
    │                                  │
    └── Federation Link ───────────────┘
             (单向/双向同步)
```

### 5.2 配置联邦

```bash
# 启用联邦插件
rabbitmq-plugins enable rabbitmq_federation
rabbitmq-plugins enable rabbitmq_federation_management

# 添加上游服务器
rabbitmqctl set_parameter federation-upstream my-upstream \
  '{"uri":"amqp://user:pass@remote-host:5672","expires":3600000}'

# 添加上游策略
rabbitmqctl set_policy --apply-to exchanges federate "^fed\." \
  '{"federation-upstream-set":"all"}'
```

### 5.3 使用联邦

```java
// 在下游集群创建队列，名称以 "fed." 开头
@Bean
public Queue federatedQueue() {
    return QueueBuilder
        .durable("fed.my.queue")
        .build();
}

// 上游集群发送消息，下游自动接收
rabbitTemplate.convertAndSend("fed.exchange", "fed.key", message);
```

## 六、Shovel（铲子）

### 6.1 什么是 Shovel

Shovel 用于在不同 RabbitMQ 集群/节点之间复制或迁移消息。

```
Cluster A                          Cluster B
    │                                  │
    ├── Source Queue                   ├── Target Queue
    │                                  │
    └── Shovel ───────────────────────→
         (单向消息复制)
```

### 6.2 配置 Shovel

```bash
# 启用 Shovel 插件
rabbitmq-plugins enable rabbitmq_shovel
rabbitmq-plugins enable rabbitmq_shovel_management

# 配置 Shovel
rabbitmqctl set_parameter shovel my-shovel '{
  "src-uri": "amqp://user:pass@source-host:5672",
  "src-queue": "source.queue",
  "dest-uri": "amqp://user:pass@dest-host:5672",
  "dest-queue": "dest.queue"
}'
```

### 6.3 动态 Shovel

通过管理界面配置动态 Shovel。

## 七、负载均衡

### 7.1 客户端负载均衡

```java
@Configuration
public class RabbitMQConfig {

    @Bean
    public ConnectionFactory connectionFactory() {
        List<Address> addresses = Arrays.asList(
            new Address("rabbitmq1", 5672),
            new Address("rabbitmq2", 5672),
            new Address("rabbitmq3", 5672)
        );

        CachingConnectionFactory factory = new CachingConnectionFactory();
        factory.setAddresses(addresses);
        factory.setUsername("guest");
        factory.setPassword("guest");

        return factory;
    }
}
```

### 7.2 HAProxy 负载均衡

```bash
# /etc/haproxy/haproxy.cfg

frontend rabbitmq_frontend
    bind *:5672
    default_backend rabbitmq_backend

backend rabbitmq_backend
    balance roundrobin
    server rabbitmq1 192.168.1.101:5672 check
    server rabbitmq2 192.168.1.102:5672 check
    server rabbitmq3 192.168.1.103:5672 check
```

### 7.3 Nginx 负载均衡

```nginx
# /etc/nginx/nginx.conf

stream {
    upstream rabbitmq_backend {
        server 192.168.1.101:5672;
        server 192.168.1.102:5672;
        server 192.168.1.103:5672;
    }

    server {
        listen 5672;
        proxy_pass rabbitmq_backend;
    }
}
```

## 九、集群最佳实践

### 9.1 节点规划

| 环境 | 节点数 | 说明 |
|------|--------|------|
| 开发环境 | 1 | 单节点即可 |
| 测试环境 | 2 | 主备模式 |
| 生产环境 | 3+ | 奇数个节点，至少 3 个 |

### 9.2 队列类型选择

```
新项目 → 仲裁队列（推荐）
旧项目 → 镜像队列（可继续使用）
无状态队列 → 普通队列
```

### 9.3 部署建议

1. **节点数量**：奇数个节点，至少 3 个
2. **节点分布**：不同物理机/机架
3. **磁盘节点**：至少保留 1 个
4. **监控**：部署完整的监控告警
5. **备份**：定期备份数据
6. **测试**：定期进行故障演练
