# Linux 大厂面试题与实战命令

## 一、Linux 基础原理

### 1. Linux 系统架构

**问题**：Linux 系统的架构是怎样的？

**详细答案**：
Linux 系统采用分层架构，主要分为：

1. **用户空间**：
   - 应用程序
   - 系统库（glibc）
   - Shell

2. **内核空间**：
   - 系统调用接口
   - 进程管理
   - 内存管理
   - 文件系统
   - 网络协议栈
   - 设备驱动

3. **硬件层**：
   - CPU
   - 内存
   - 磁盘
   - 网络设备

**代码示例**：

```bash
# 查看系统信息
uname -a                    # 查看内核版本
cat /etc/os-release        # 查看操作系统版本
hostnamectl                # 查看系统详细信息

# 查看硬件信息
lscpu                      # 查看 CPU 信息
free -h                    # 查看内存信息
df -h                      # 查看磁盘信息
lsblk                      # 查看块设备信息
lspci                      # 查看 PCI 设备信息
lsusb                      # 查看 USB 设备信息
```

### 2. Linux 文件系统

**问题**：Linux 文件系统的结构是怎样的？

**详细答案**：
Linux 文件系统采用树形结构，主要目录包括：

1. **/**：根目录
2. **/bin**：二进制可执行文件
3. **/sbin**：系统二进制文件
4. **/etc**：配置文件
5. **/home**：用户主目录
6. **/root**：root 用户主目录
7. **/var**：可变数据文件
8. **/tmp**：临时文件
9. **/usr**：用户程序和数据
10. **/opt**：可选软件包
11. **/proc**：虚拟文件系统（进程信息）
12. **/sys**：虚拟文件系统（内核信息）

**代码示例**：

```bash
# 查看文件系统
df -h                      # 查看磁盘使用情况
du -sh /path/to/dir        # 查看目录大小
tree /path/to/dir          # 以树形结构显示目录

# 文件操作
ls -la                     # 列出文件详细信息
cd /path/to/dir            # 切换目录
pwd                        # 显示当前目录
mkdir -p /path/to/dir      # 创建目录（包括父目录）
rm -rf /path/to/dir        # 删除目录及其内容
cp -r /src /dest           # 复制目录
mv /src /dest              # 移动或重命名文件
touch file.txt             # 创建空文件
cat file.txt               # 查看文件内容
less file.txt              # 分页查看文件内容
tail -f file.log           # 实时查看日志文件
```

## 二、Linux 进程管理

### 1. 进程查看与管理

**问题**：如何查看和管理 Linux 进程？

**详细答案**：
进程管理的常用命令：

1. **查看进程**：
   - `ps`：查看当前进程
   - `top`：实时查看进程
   - `htop`：交互式进程查看器
   - `pstree`：以树形结构显示进程

2. **管理进程**：
   - `kill`：终止进程
   - `killall`：按名称终止进程
   - `pkill`：按模式匹配终止进程
   - `nohup`：后台运行进程

**代码示例**：

```bash
# 查看进程
ps aux                     # 查看所有进程
ps -ef                     # 查看所有进程（完整格式）
ps aux | grep nginx        # 查找特定进程
top                        # 实时查看进程
htop                       # 交互式进程查看器
pstree                     # 以树形结构显示进程

# 管理进程
kill 1234                  # 终止 PID 为 1234 的进程
kill -9 1234               # 强制终止进程
killall nginx              # 终止所有 nginx 进程
pkill -f "nginx"           # 终止匹配 nginx 的进程

# 后台运行进程
nohup command &            # 后台运行命令，忽略 SIGHUP 信号
command > /dev/null 2>&1 & # 后台运行命令，丢弃输出

# 查看进程详细信息
cat /proc/1234/status      # 查看 PID 1234 的进程状态
cat /proc/1234/cmdline     # 查看进程命令行
lsof -p 1234               # 查看进程打开的文件
```

### 2. 端口占用查看与处理

**问题**：如何查看端口占用情况并处理？

**详细答案**：
端口占用查看与处理的常用命令：

1. **查看端口占用**：
   - `netstat`：查看网络连接
   - `ss`：查看网络套接字
   - `lsof`：查看打开的文件（包括网络套接字）

2. **处理端口占用**：
   - 找到占用端口的进程
   - 终止进程
   - 修改服务配置使用其他端口

**代码示例**：

```bash
# 查看端口占用（netstat）
netstat -tlnp              # 查看所有监听的 TCP 端口
netstat -ulnp              # 查看所有监听的 UDP 端口
netstat -tlnp | grep :80   # 查看端口 80 的占用情况
netstat -anp | grep ESTABLISHED  # 查看已建立的连接

# 查看端口占用（ss）
ss -tlnp                   # 查看所有监听的 TCP 端口
ss -ulnp                   # 查看所有监听的 UDP 端口
ss -tlnp | grep :3306      # 查看端口 3306 的占用情况
ss -s                      # 查看网络统计信息

# 查看端口占用（lsof）
lsof -i :80                # 查看端口 80 的占用情况
lsof -i :443               # 查看端口 443 的占用情况
lsof -i TCP                # 查看所有 TCP 连接
lsof -i UDP                # 查看所有 UDP 连接

# 查看特定进程的端口占用
lsof -p 1234               # 查看 PID 1234 的进程打开的文件
lsof -c nginx              # 查看 nginx 进程打开的文件

# 查杀占用端口的进程
# 方法 1：找到进程 PID 后 kill
lsof -i :80                # 查看端口 80 的占用情况
kill -9 <PID>              # 终止进程

# 方法 2：使用 fuser
fuser -k 80/tcp            # 终止占用端口 80 的进程

# 方法 3：使用 killall
killall nginx              # 终止所有 nginx 进程

# 查看端口范围
cat /proc/sys/net/ipv4/ip_local_port_range  # 查看本地端口范围

# 修改端口范围（需要 root 权限）
echo "1024 65535" > /proc/sys/net/ipv4/ip_local_port_range

# 查看端口监听状态
ss -tlnp | grep LISTEN     # 查看所有监听状态的端口
netstat -tlnp | grep LISTEN # 查看所有监听状态的端口

# 查看端口连接数
netstat -an | grep :80 | wc -l  # 统计端口 80 的连接数
ss -tn | grep :80 | wc -l      # 统计端口 80 的连接数

# 查看端口连接状态分布
netstat -an | awk '/^tcp/ {print $6}' | sort | uniq -c | sort -rn

# 查看端口流量
iftop                      # 查看网络接口流量
nethogs                    # 查看进程网络流量
```

### 3. 进程资源监控

**问题**：如何监控进程的资源使用情况？

**详细答案**：
进程资源监控的常用命令：

1. **CPU 使用率**：
   - `top`：实时查看 CPU 使用率
   - `ps`：查看进程 CPU 使用率
   - `mpstat`：查看 CPU 统计信息

2. **内存使用率**：
   - `free`：查看内存使用情况
   - `ps`：查看进程内存使用情况
   - `vmstat`：查看虚拟内存统计

3. **磁盘 I/O**：
   - `iostat`：查看磁盘 I/O 统计
   - `iotop`：查看进程磁盘 I/O

**代码示例**：

```bash
# CPU 监控
top                        # 实时查看 CPU 使用率
top -p 1234                # 查看 PID 1234 的 CPU 使用率
ps aux --sort=-%cpu | head -10  # 查看 CPU 使用率最高的 10 个进程
mpstat 1 5                 # 每秒显示 CPU 统计信息，共 5 次

# 内存监控
free -h                    # 查看内存使用情况
free -m                    # 以 MB 为单位查看内存
ps aux --sort=-%mem | head -10  # 查看内存使用率最高的 10 个进程
vmstat 1 5                 # 每秒显示虚拟内存统计，共 5 次

# 磁盘 I/O 监控
iostat -x 1 5              # 每秒显示磁盘 I/O 统计，共 5 次
iotop                      # 查看进程磁盘 I/O
iotop -o                   # 只显示有 I/O 操作的进程

# 综合监控
htop                       # 交互式系统监控
glances                    # 跨平台系统监控工具
dstat                      # 多功能系统监控工具

# 查看进程详细信息
cat /proc/1234/status      # 查看进程状态
cat /proc/1234/stat        # 查看进程统计信息
cat /proc/1234/maps        # 查看进程内存映射

# 查看进程打开的文件
lsof -p 1234               # 查看 PID 1234 打开的文件
lsof -c nginx              # 查看 nginx 进程打开的文件

# 查看进程网络连接
lsof -i -a -p 1234         # 查看 PID 1234 的网络连接
netstat -anp | grep 1234   # 查看 PID 1234 的网络连接
```

## 三、Linux 网络管理

### 1. 网络配置与查看

**问题**：如何配置和查看 Linux 网络？

**详细答案**：
网络配置与查看的常用命令：

1. **查看网络配置**：
   - `ifconfig`：查看网络接口配置
   - `ip`：查看和配置网络接口
   - `nmcli`：NetworkManager 命令行工具

2. **配置网络**：
   - `ifconfig`：配置网络接口
   - `ip`：配置网络接口
   - `nmcli`：配置网络连接

**代码示例**：

```bash
# 查看网络配置
ifconfig                   # 查看所有网络接口
ifconfig eth0              # 查看 eth0 接口配置
ip addr show               # 查看所有网络接口
ip addr show eth0          # 查看 eth0 接口配置
nmcli device show          # 查看所有设备
nmcli connection show      # 查看所有连接

# 配置网络接口
ifconfig eth0 192.168.1.100 netmask 255.255.255.0  # 配置 IP 地址
ip addr add 192.168.1.100/24 dev eth0               # 配置 IP 地址
ip link set eth0 up                                # 启用网络接口
ip link set eth0 down                              # 禁用网络接口

# 查看路由表
route -n                   # 查看路由表
ip route show              # 查看路由表
ip route add default via 192.168.1.1              # 添加默认路由
ip route del default via 192.168.1.1              # 删除默认路由

# 查看网络连接
netstat -an                # 查看所有网络连接
netstat -tn                # 查看 TCP 连接
netstat -un                # 查看 UDP 连接
ss -tlnp                   # 查看监听的 TCP 端口
ss -ulnp                   # 查看监听的 UDP 端口

# 测试网络连通性
ping 8.8.8.8               # 测试网络连通性
ping -c 4 8.8.8.8          # 发送 4 个 ping 包
traceroute 8.8.8.8         # 跟踪路由路径
mtr 8.8.8.8                # 实时跟踪路由路径

# 查看 DNS 配置
cat /etc/resolv.conf       # 查看 DNS 配置
nslookup google.com        # 查询 DNS 记录
dig google.com             # 查询 DNS 记录（详细信息）
host google.com            # 查询 DNS 记录

# 查看网络流量
iftop                      # 查看网络接口流量
nethogs                    # 查看进程网络流量
tcpdump -i eth0            # 抓包分析
```

### 2. 防火墙配置

**问题**：如何配置 Linux 防火墙？

**详细答案**：
防火墙配置的常用命令：

1. **iptables**：
   - 查看 iptables 规则
   - 添加 iptables 规则
   - 删除 iptables 规则

2. **firewalld**：
   - 查看 firewalld 规则
   - 添加 firewalld 规则
   - 删除 firewalld 规则

3. **ufw**：
   - 查看 ufw 规则
   - 添加 ufw 规则
   - 删除 ufw 规则

**代码示例**：

```bash
# iptables 配置
iptables -L -n             # 查看 iptables 规则
iptables -L -n -v          # 查看 iptables 规则（详细信息）
iptables -F                # 清空所有规则
iptables -X                # 删除所有自定义链

# 添加规则
iptables -A INPUT -p tcp --dport 22 -j ACCEPT      # 允许 SSH
iptables -A INPUT -p tcp --dport 80 -j ACCEPT      # 允许 HTTP
iptables -A INPUT -p tcp --dport 443 -j ACCEPT     # 允许 HTTPS
iptables -A INPUT -j DROP                          # 拒绝其他连接

# 删除规则
iptables -D INPUT -p tcp --dport 22 -j ACCEPT      # 删除规则

# 保存规则
iptables-save > /etc/iptables/rules.v4             # 保存规则
iptables-restore < /etc/iptables/rules.v4          # 恢复规则

# firewalld 配置
firewall-cmd --state                                # 查看 firewalld 状态
firewall-cmd --list-all                             # 查看所有规则
firewall-cmd --list-ports                           # 查看开放的端口
firewall-cmd --add-port=80/tcp --permanent         # 开放端口 80
firewall-cmd --remove-port=80/tcp --permanent      # 关闭端口 80
firewall-cmd --reload                               # 重新加载规则

# ufw 配置
ufw status                                          # 查看 ufw 状态
ufw enable                                          # 启用 ufw
ufw disable                                         # 禁用 ufw
ufw allow 22/tcp                                    # 允许 SSH
ufw allow 80/tcp                                    # 允许 HTTP
ufw allow 443/tcp                                   # 允许 HTTPS
ufw deny 3306/tcp                                   # 拒绝 MySQL
ufw delete allow 22/tcp                             # 删除规则
```

### 3. 网络故障排查

**问题**：如何排查 Linux 网络故障？

**详细答案**：
网络故障排查的常用命令：

1. **网络连通性测试**：
   - `ping`：测试网络连通性
   - `traceroute`：跟踪路由路径
   - `mtr`：实时跟踪路由路径

2. **网络抓包分析**：
   - `tcpdump`：抓包分析
   - `wireshark`：图形化抓包工具

3. **网络性能测试**：
   - `iperf`：网络性能测试
   - `netperf`：网络性能测试

**代码示例**：

```bash
# 网络连通性测试
ping 8.8.8.8               # 测试网络连通性
ping -c 4 8.8.8.8          # 发送 4 个 ping 包
ping -i 0.1 8.8.8.8        # 每 0.1 秒发送一个 ping 包
ping -s 1000 8.8.8.8       # 发送 1000 字节的 ping 包

# 跟踪路由路径
traceroute 8.8.8.8         # 跟踪路由路径
traceroute -n 8.8.8.8       # 不解析主机名
mtr 8.8.8.8                # 实时跟踪路由路径
mtr -r -c 10 8.8.8.8       # 报告模式，发送 10 个包

# 网络抓包分析
tcpdump -i eth0            # 抓取 eth0 接口的所有包
tcpdump -i eth0 port 80    # 抓取端口 80 的包
tcpdump -i eth0 host 192.168.1.1  # 抓取特定主机的包
tcpdump -i eth0 -w capture.pcap   # 抓包并保存到文件
tcpdump -r capture.pcap           # 读取抓包文件

# 网络性能测试
iperf -s                   # 启动 iperf 服务器
iperf -c 192.168.1.1       # 连接到 iperf 服务器
iperf -c 192.168.1.1 -t 30 # 测试 30 秒
iperf -c 192.168.1.1 -u    # 使用 UDP 测试

# 查看 DNS 解析
nslookup google.com        # 查询 DNS 记录
dig google.com             # 查询 DNS 记录（详细信息）
dig google.com +trace      # 跟踪 DNS 解析过程

# 查看网络统计
netstat -s                 # 查看网络统计信息
ss -s                      # 查看网络统计信息
cat /proc/net/snmp         # 查看 SNMP 统计信息

# 查看网络接口统计
ifconfig eth0              # 查看网络接口统计
ip -s link show eth0       # 查看网络接口统计
ethtool eth0               # 查看网络接口详细信息
```

## 四、Linux 系统管理

### 1. 用户与权限管理

**问题**：如何管理 Linux 用户和权限？

**详细答案**：
用户与权限管理的常用命令：

1. **用户管理**：
   - `useradd`：添加用户
   - `userdel`：删除用户
   - `usermod`：修改用户
   - `passwd`：修改密码

2. **组管理**：
   - `groupadd`：添加组
   - `groupdel`：删除组
   - `groupmod`：修改组

3. **权限管理**：
   - `chmod`：修改文件权限
   - `chown`：修改文件所有者
   - `chgrp`：修改文件所属组

**代码示例**：

```bash
# 用户管理
useradd username           # 添加用户
useradd -m username        # 添加用户并创建主目录
userdel username           # 删除用户
userdel -r username        # 删除用户及其主目录
usermod -aG group username # 将用户添加到组
passwd username            # 修改用户密码

# 组管理
groupadd groupname         # 添加组
groupdel groupname         # 删除组
groupmod -n newname oldname # 重命名组

# 查看用户信息
id username                # 查看用户 ID 和组信息
groups username            # 查看用户所属的组
cat /etc/passwd            # 查看所有用户
cat /etc/group             # 查看所有组
cat /etc/shadow            # 查看用户密码（需要 root 权限）

# 权限管理
chmod 755 file.txt         # 修改文件权限
chmod u+x file.txt         # 为所有者添加执行权限
chmod g+w file.txt         # 为组添加写权限
chmod o-r file.txt         # 为其他用户移除读权限
chmod -R 755 /path/to/dir  # 递归修改目录权限

chown user:group file.txt  # 修改文件所有者和所属组
chown user file.txt        # 修改文件所有者
chgrp group file.txt        # 修改文件所属组
chown -R user:group /path/to/dir  # 递归修改目录所有者和所属组

# 查看文件权限
ls -la                     # 查看文件详细信息
ls -ld /path/to/dir        # 查看目录权限
stat file.txt              # 查看文件详细统计信息

# sudo 管理
visudo                     # 编辑 sudo 配置文件
sudo -l                    # 查看当前用户的 sudo 权限
sudo -u username command   # 以指定用户身份执行命令
```

### 2. 系统服务管理

**问题**：如何管理 Linux 系统服务？

**详细答案**：
系统服务管理的常用命令：

1. **systemd**：
   - `systemctl`：管理系统服务
   - `journalctl`：查看系统日志

2. **init.d**：
   - `service`：管理系统服务
   - `chkconfig`：管理系统服务

**代码示例**：

```bash
# systemd 服务管理
systemctl status nginx    # 查看 nginx 服务状态
systemctl start nginx     # 启动 nginx 服务
systemctl stop nginx      # 停止 nginx 服务
systemctl restart nginx   # 重启 nginx 服务
systemctl reload nginx    # 重新加载 nginx 配置
systemctl enable nginx    # 设置 nginx 开机自启
systemctl disable nginx   # 取消 nginx 开机自启
systemctl is-enabled nginx # 查看 nginx 是否开机自启

# 查看所有服务
systemctl list-units --type=service  # 查看所有服务
systemctl list-units --type=service --state=running  # 查看运行中的服务
systemctl list-unit-files --type=service  # 查看所有服务文件

# 查看服务日志
journalctl -u nginx       # 查看 nginx 服务日志
journalctl -u nginx -f    # 实时查看 nginx 服务日志
journalctl -u nginx --since today  # 查看今天的 nginx 日志
journalctl -u nginx --since "1 hour ago"  # 查看最近 1 小时的日志

# init.d 服务管理
service nginx status      # 查看 nginx 服务状态
service nginx start       # 启动 nginx 服务
service nginx stop        # 停止 nginx 服务
service nginx restart     # 重启 nginx 服务

# chkconfig 管理
chkconfig --list          # 查看所有服务
chkconfig nginx on        # 设置 nginx 开机自启
chkconfig nginx off       # 取消 nginx 开机自启
chkconfig --level 3 nginx on  # 在运行级别 3 设置 nginx 开机自启

# 查看运行级别
runlevel                  # 查看当前运行级别
systemctl get-default    # 查看默认运行级别
```

### 3. 系统日志管理

**问题**：如何管理 Linux 系统日志？

**详细答案**：
系统日志管理的常用命令：

1. **查看日志**：
   - `journalctl`：查看 systemd 日志
   - `dmesg`：查看内核日志
   - `tail`：查看日志文件

2. **日志轮转**：
   - `logrotate`：日志轮转工具
   - 配置 logrotate

**代码示例**：

```bash
# 查看 systemd 日志
journalctl                 # 查看所有日志
journalctl -n 100          # 查看最近 100 条日志
journalctl -f              # 实时查看日志
journalctl -u nginx        # 查看 nginx 服务日志
journalctl -p err          # 查看错误日志
journalctl --since today   # 查看今天的日志
journalctl --since "1 hour ago"  # 查看最近 1 小时的日志
journalctl --since "2024-01-01" --until "2024-01-02"  # 查看指定时间段的日志

# 查看内核日志
dmesg                      # 查看内核日志
dmesg | tail -n 100        # 查看最近 100 条内核日志
dmesg -T                   # 以人类可读的时间格式显示
dmesg | grep -i error      # 查找错误信息

# 查看应用日志
tail -f /var/log/nginx/access.log   # 实时查看 nginx 访问日志
tail -f /var/log/nginx/error.log    # 实时查看 nginx 错误日志
tail -n 100 /var/log/syslog        # 查看最近 100 条系统日志
grep "error" /var/log/syslog        # 查找错误信息

# 日志轮转配置
cat /etc/logrotate.conf  # 查看 logrotate 配置
cat /etc/logrotate.d/nginx  # 查看 nginx 日志轮转配置

# 手动执行日志轮转
logrotate -f /etc/logrotate.conf  # 强制执行日志轮转
logrotate -d /etc/logrotate.conf  # 调试模式，不实际执行

# 查看日志文件大小
du -sh /var/log/*         # 查看日志目录大小
ls -lh /var/log/          # 查看日志文件大小

# 清理日志
> /var/log/nginx/access.log  # 清空日志文件
rm /var/log/nginx/access.log.1  # 删除旧的日志文件
```

## 五、Linux 性能优化

### 1. 系统性能监控

**问题**：如何监控 Linux 系统性能？

**详细答案**：
系统性能监控的常用命令：

1. **CPU 监控**：
   - `top`：实时查看 CPU 使用率
   - `mpstat`：查看 CPU 统计信息
   - `sar`：系统活动报告

2. **内存监控**：
   - `free`：查看内存使用情况
   - `vmstat`：查看虚拟内存统计
   - `sar`：系统活动报告

3. **磁盘 I/O 监控**：
   - `iostat`：查看磁盘 I/O 统计
   - `iotop`：查看进程磁盘 I/O
   - `sar`：系统活动报告

4. **网络监控**：
   - `iftop`：查看网络接口流量
   - `nethogs`：查看进程网络流量
   - `sar`：系统活动报告

**代码示例**：

```bash
# CPU 监控
top                        # 实时查看 CPU 使用率
top -p 1234                # 查看 PID 1234 的 CPU 使用率
mpstat 1 5                 # 每秒显示 CPU 统计信息，共 5 次
mpstat -P ALL 1 5          # 显示所有 CPU 的统计信息
sar -u 1 5                 # 每秒显示 CPU 使用率，共 5 次

# 内存监控
free -h                    # 查看内存使用情况
free -m                    # 以 MB 为单位查看内存
vmstat 1 5                 # 每秒显示虚拟内存统计，共 5 次
sar -r 1 5                 # 每秒显示内存使用率，共 5 次

# 磁盘 I/O 监控
iostat -x 1 5              # 每秒显示磁盘 I/O 统计，共 5 次
iostat -d 1 5              # 每秒显示磁盘使用率，共 5 次
iotop                      # 查看进程磁盘 I/O
iotop -o                   # 只显示有 I/O 操作的进程
sar -d 1 5                 # 每秒显示磁盘 I/O 统计，共 5 次

# 网络监控
iftop                      # 查看网络接口流量
nethogs                    # 查看进程网络流量
sar -n DEV 1 5             # 每秒显示网络统计，共 5 次
sar -n EDEV 1 5            # 每秒显示网络错误统计，共 5 次

# 综合监控
htop                       # 交互式系统监控
glances                    # 跨平台系统监控工具
dstat                      # 多功能系统监控工具
atop                       # 高级系统监控工具

# 进程监控
ps aux --sort=-%cpu | head -10  # 查看 CPU 使用率最高的 10 个进程
ps aux --sort=-%mem | head -10  # 查看内存使用率最高的 10 个进程
pstree                     # 以树形结构显示进程
lsof -p 1234               # 查看 PID 1234 打开的文件
```

### 2. 系统性能优化

**问题**：如何优化 Linux 系统性能？

**详细答案**：
系统性能优化的方法：

1. **内核参数优化**：
   - 修改 `/etc/sysctl.conf`
   - 使用 `sysctl` 命令

2. **文件系统优化**：
   - 选择合适的文件系统
   - 调整挂载选项

3. **网络参数优化**：
   - 调整 TCP 参数
   - 调整网络缓冲区大小

**代码示例**：

```bash
# 查看内核参数
sysctl -a                  # 查看所有内核参数
sysctl net.ipv4.tcp_max_syn_backlog  # 查看特定内核参数

# 修改内核参数（临时）
sysctl -w net.ipv4.tcp_max_syn_backlog=8192  # 临时修改内核参数

# 修改内核参数（永久）
echo "net.ipv4.tcp_max_syn_backlog=8192" >> /etc/sysctl.conf
sysctl -p                  # 重新加载内核参数配置

# 常用内核参数优化
cat >> /etc/sysctl.conf << EOF
# 网络参数优化
net.ipv4.tcp_max_syn_backlog = 8192
net.core.somaxconn = 65535
net.ipv4.tcp_tw_reuse = 1
net.ipv4.tcp_fin_timeout = 30
net.ipv4.tcp_keepalive_time = 1200
net.ipv4.ip_local_port_range = 1024 65535

# 文件描述符限制
fs.file-max = 65535

# 内存参数优化
vm.swappiness = 10
vm.dirty_ratio = 15
vm.dirty_background_ratio = 5
EOF

sysctl -p                  # 重新加载内核参数配置

# 文件描述符限制
ulimit -n                  # 查看当前用户的文件描述符限制
ulimit -n 65535            # 临时修改文件描述符限制

# 永久修改文件描述符限制
echo "* soft nofile 65535" >> /etc/security/limits.conf
echo "* hard nofile 65535" >> /etc/security/limits.conf

# 磁盘挂载选项优化
# 在 /etc/fstab 中添加 noatime 选项
/dev/sdb1 /data ext4 defaults,noatime 0 0

# 重新挂载文件系统
mount -o remount,noatime /data

# 查看磁盘 I/O 调度器
cat /sys/block/sda/queue/scheduler

# 修改磁盘 I/O 调度器
echo deadline > /sys/block/sda/queue/scheduler

# 查看透明大页
cat /sys/kernel/mm/transparent_hugepage/enabled

# 禁用透明大页
echo never > /sys/kernel/mm/transparent_hugepage/enabled
```
