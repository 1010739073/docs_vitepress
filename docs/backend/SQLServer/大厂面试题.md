# SQL Server 大厂面试题

## 一、基础概念

### 1. 什么是 SQL Server？它有哪些特点？

**答案：**
SQL Server 是 Microsoft 开发的关系型数据库管理系统（RDBMS）。

**主要特点：**
- **高性能**：优化的查询引擎，支持列存储索引
- **高可用**：支持 AlwaysOn、数据库镜像、复制等
- **安全性**：集成 Windows 身份验证，支持行级安全、透明数据加密
- **商业智能**：集成 SSAS、SSIS、SSRS
- **云服务**：Azure SQL Database 提供托管服务

### 2. SQL Server 有哪些版本？如何选择？

**答案：**
| 版本 | 适用场景 | 限制 |
|-----|---------|------|
| Express | 学习、小型应用 | 10GB 数据库限制 |
| Developer | 开发测试 | 功能完整，不可生产 |
| Standard | 中小型企业 | 部分高级功能限制 |
| Enterprise | 大型企业 | 无限制 |
| Azure SQL | 云部署 | 托管服务 |

### 3. SQL Server 的数据类型有哪些？

**答案：**
- **整数**：TINYINT、SMALLINT、INT、BIGINT
- **小数**：DECIMAL、NUMERIC、FLOAT、REAL
- **字符串**：CHAR、VARCHAR、NCHAR、NVARCHAR
- **日期时间**：DATE、TIME、DATETIME、DATETIME2
- **二进制**：BINARY、VARBINARY
- **其他**：BIT、UNIQUEIDENTIFIER、XML、JSON

### 4. CHAR 和 VARCHAR 的区别？NCHAR 和 NVARCHAR 的区别？

**答案：**
| 特性 | CHAR | VARCHAR |
|-----|------|---------|
| 存储 | 定长 | 变长 |
| 空间 | 固定分配 | 实际长度+2字节 |
| 适用 | 固定长度数据 | 变长数据 |

- **NCHAR/NVARCHAR**：Unicode 编码，支持中文，每个字符占 2 字节
- **CHAR/VARCHAR**：非 Unicode，中文可能乱码

### 5. DATETIME 和 DATETIME2 的区别？

**答案：**
| 特性 | DATETIME | DATETIME2 |
|-----|----------|-----------|
| 精度 | 3.33 毫秒 | 100 纳秒 |
| 范围 | 1753-9999 | 0001-9999 |
| 存储 | 8 字节 | 6-8 字节 |
| 推荐 | 旧系统 | 新系统 |

## 二、SQL 查询

### 6. 什么是索引？有哪些类型？

**答案：**
索引是加速数据检索的数据结构。

**类型：**
- **聚集索引**：数据物理排序，每个表只能有一个
- **非聚集索引**：独立于数据存储，可多个
- **唯一索引**：确保值唯一
- **覆盖索引**：包含查询所需所有列
- **复合索引**：多列索引
- **筛选索引**：部分数据索引
- **列存储索引**：适合分析查询
- **全文索引**：文本搜索

### 7. 聚集索引和非聚集索引的区别？

**答案：**
| 特性 | 聚集索引 | 非聚集索引 |
|-----|---------|-----------|
| 数据存储 | 叶子节点是数据 | 叶子节点是行定位符 |
| 数量 | 每个表一个 | 最多 999 个 |
| 大小 | 就是表大小 | 额外存储空间 |
| 查询 | 快 | 可能需要回表 |

### 8. 什么是覆盖索引？

**答案：**
覆盖索引包含查询所需的所有列，查询时无需回表查询数据。

```sql
-- 覆盖索引示例
CREATE INDEX IX_Orders_UserID ON Orders(UserID)
INCLUDE (OrderDate, TotalAmount);

-- 查询时无需回表
SELECT UserID, OrderDate, TotalAmount 
FROM Orders WHERE UserID = 1;
```

### 9. 索引查找和索引扫描的区别？

**答案：**
- **索引查找（Index Seek）**：使用 B-Tree 直接定位，高效
- **索引扫描（Index Scan）**：扫描整个索引，较低效
- **表扫描（Table Scan）**：全表扫描，最低效

### 10. 如何优化慢查询？

**答案：**
1. 添加合适的索引
2. 避免在索引列上使用函数
3. 避免隐式类型转换
4. 使用 EXISTS 代替 IN
5. 避免 SELECT *
6. 使用覆盖索引
7. 更新统计信息
8. 重写查询逻辑

## 三、事务与并发

### 11. 什么是事务？ACID 特性是什么？

**答案：**
事务是一组操作的逻辑单元。

**ACID：**
- **原子性（Atomicity）**：要么全成功，要么全失败
- **一致性（Consistency）**：数据完整性约束不被破坏
- **隔离性（Isolation）**：并发事务相互隔离
- **持久性（Durability）**：提交后永久保存

### 12. SQL Server 有哪些隔离级别？

**答案：**
| 隔离级别 | 脏读 | 不可重复读 | 幻读 |
|---------|------|-----------|------|
| READ UNCOMMITTED | ✓ | ✓ | ✓ |
| READ COMMITTED | ✗ | ✓ | ✓ | (默认)
| REPEATABLE READ | ✗ | ✗ | ✓ |
| SNAPSHOT | ✗ | ✗ | ✗ |
| SERIALIZABLE | ✗ | ✗ | ✗ |

### 13. 什么是脏读、不可重复读、幻读？

**答案：**
- **脏读**：读取到其他事务未提交的数据
- **不可重复读**：同一事务中两次读取同一数据结果不同
- **幻读**：同一事务中两次查询，结果集行数不同

### 14. 什么是 MVCC？

**答案：**
MVCC（多版本并发控制）通过维护数据多个版本实现并发控制。

**SQL Server 实现：**
- 使用快照隔离级别
- 通过 tempdb 存储行版本
- 读操作不阻塞写操作

### 15. 什么是死锁？如何预防？

**答案：**
死锁是两个或多个事务相互等待对方释放资源。

**预防方法：**
1. 按相同顺序访问资源
2. 保持事务简短
3. 使用较低的隔离级别
4. 使用死锁优先级
5. 使用行版本控制

## 四、存储过程与编程

### 16. 存储过程和函数的区别？

**答案：**
| 特性 | 存储过程 | 函数 |
|-----|---------|------|
| 返回值 | 可以没有 | 必须有 |
| 参数 | 输入、输出、输入输出 | 输入 |
| 调用 | EXEC | 在 SQL 中使用 |
| DML 操作 | 可以 | 有限制 |
| 事务 | 可以包含 | 有限制 |

### 17. 什么是触发器？有哪些类型？

**答案：**
触发器是自动执行的存储过程。

**类型：**
- **DML 触发器**：INSERT、UPDATE、DELETE
  - AFTER 触发器
  - INSTEAD OF 触发器
- **DDL 触发器**：CREATE、ALTER、DROP
- **登录触发器**：用户登录时

### 18. 什么是游标？什么时候使用？

**答案：**
游标用于逐行处理结果集。

**使用场景：**
- 需要逐行处理数据
- 复杂业务逻辑

**注意：**
- 尽量避免使用游标
- 使用集合操作代替
- 如果必须使用，使用 FAST_FORWARD

## 五、性能优化

### 19. 如何查看执行计划？

**答案：**
```sql
-- 图形化执行计划（SSMS）
-- 点击"包括实际的执行计划"或按 Ctrl+M

-- 文本执行计划
SET SHOWPLAN_TEXT ON;
SET SHOWPLAN_XML ON;

-- 实际执行计划
SET STATISTICS IO ON;
SET STATISTICS TIME ON;
```

### 20. 什么是参数嗅探？如何解决？

**答案：**
参数嗅探是存储过程首次执行的参数值影响执行计划。

**解决方法：**
1. 使用本地变量
2. 使用 OPTION(RECOMPILE)
3. 使用 OPTIMIZE FOR UNKNOWN
4. 使用提示强制索引

### 21. 什么是统计信息？为什么重要？

**答案：**
统计信息帮助优化器估计行数，选择最优执行计划。

**维护：**
```sql
UPDATE STATISTICS table_name;
UPDATE STATISTICS table_name index_name WITH FULLSCAN;
```

## 六、备份与恢复

### 22. SQL Server 有哪些备份类型？

**答案：**
- **完整备份**：备份整个数据库
- **差异备份**：备份自上次完整备份后的变化
- **事务日志备份**：备份事务日志
- **文件/文件组备份**：备份特定文件
- **仅复制备份**：不影响备份链

### 23. 什么是恢复模式？有哪些？

**答案：**
| 恢复模式 | 日志记录 | 恢复能力 |
|---------|---------|---------|
| 简单 | 最小 | 完整+差异 |
| 完整 | 完整 | 时间点恢复 |
| 大容量日志 | 批量最小 | 有限 |

### 24. 如何恢复数据库到特定时间点？

**答案：**
```sql
-- 1. 恢复完整备份（NORECOVERY）
RESTORE DATABASE MyDB FROM DISK = 'full.bak' WITH NORECOVERY;

-- 2. 恢复差异备份（如果有，NORECOVERY）
RESTORE DATABASE MyDB FROM DISK = 'diff.bak' WITH NORECOVERY;

-- 3. 恢复日志到特定时间点
RESTORE LOG MyDB FROM DISK = 'log.trn' 
WITH STOPAT = '2024-01-15 14:30:00', RECOVERY;
```

## 七、高可用

### 25. SQL Server 有哪些高可用方案？

**答案：**
| 方案 | RTO | RPO | 自动故障转移 |
|-----|-----|-----|------------|
| AlwaysOn AG | 秒级 | 0 | 是 |
| 数据库镜像 | 秒级 | 0 | 是（带见证） |
| 日志传送 | 分钟级 | 分钟级 | 否 |
| 故障转移群集 | 分钟级 | 0 | 是 |
| 复制 | 秒级 | 分钟级 | 否 |

### 26. 什么是 AlwaysOn 可用性组？

**答案：**
AlwaysOn 可用性组是 SQL Server 的高可用和灾难恢复解决方案。

**特点：**
- 支持多个辅助副本
- 可读辅助副本
- 自动故障转移
- 数据库级别故障转移

## 八、高级主题

### 27. 什么是分区表？

**答案：**
分区表将大表分成更小的、可管理的部分。

**优点：**
- 提高查询性能
- 简化维护
- 快速数据加载和删除

### 28. 什么是列存储索引？

**答案：**
列存储索引按列存储数据，适合分析查询。

**优点：**
- 高压缩率
- 查询性能提升 10-100 倍
- 适合数据仓库

### 29. 什么是内存优化表？

**答案：**
内存优化表将整个表存储在内存中。

**优点：**
- 极高的性能
- 无锁并发控制
- 适合高频事务处理

### 30. SQL Server 2019/2022 有哪些新特性？

**答案：**
**SQL Server 2019：**
- 大数据群集
- 智能查询处理
- 加速数据库恢复
- UTF-8 支持

**SQL Server 2022：**
- 智能查询处理增强
- 不可变账本
- JSON 功能增强
- 多写入可用性组

## 九、场景题

### 31. 数据库查询很慢，如何排查？

**答案：**
1. 使用执行计划查看瓶颈
2. 检查是否缺少索引
3. 查看等待统计
4. 检查统计信息是否过期
5. 查看是否有阻塞
6. 检查参数嗅探问题
7. 查看硬件资源（CPU、内存、IO）

### 32. 数据库服务器 CPU 100%，如何处理？

**答案：**
1. 查看当前运行的查询
2. 找出 CPU 消耗高的查询
3. 检查是否有死循环或长时间运行的查询
4. 查看执行计划，优化查询
5. 检查参数嗅探问题
6. 考虑添加索引
7. 必要时终止问题查询

### 33. 如何设计一个高并发系统的数据库架构？

**答案：**
1. 使用读写分离（AlwaysOn AG）
2. 数据库分片
3. 使用缓存（Redis）
4. 优化索引和查询
5. 使用连接池
6. 合理设置隔离级别
7. 使用队列削峰

### 34. 数据库死锁频繁发生，如何解决？

**答案：**
1. 按相同顺序访问资源
2. 保持事务简短
3. 使用较低的隔离级别
4. 使用行版本控制
5. 捕获死锁错误并重试
6. 分析死锁图，找出原因

### 35. 如何进行数据库性能调优？

**答案：**
1. 收集性能基线
2. 识别瓶颈（CPU、内存、IO、锁）
3. 优化查询和索引
4. 调整服务器配置
5. 优化数据库设计
6. 实施分区策略
7. 持续监控和调优
