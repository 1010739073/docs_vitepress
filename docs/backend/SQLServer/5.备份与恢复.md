# SQL Server 备份与恢复

## 1. 备份类型

### 备份类型概述

SQL Server 支持以下备份类型：

| 备份类型 | 描述 | 恢复模式 | 时间 |
|----------|------|----------|------|
| 完整备份 | 备份整个数据库 | 所有模式 | 最长 |
| 差异备份 | 备份自上次完整备份以来的变化 | 完整、大容量日志 | 较短 |
| 事务日志备份 | 备份事务日志 | 完整、大容量日志 | 最短 |
| 文件/文件组备份 | 备份特定文件或文件组 | 所有模式 | 中等 |
| 仅复制备份 | 不影响备份序列 | 所有模式 | 取决于类型 |

## 2. 完整备份

### 执行完整备份

```sql
-- 完整数据库备份
BACKUP DATABASE TestDB
TO DISK = 'C:\Backups\TestDB_Full.bak'
WITH FORMAT,
     MEDIANAME = 'TestDB_Backups',
     NAME = 'Full Backup of TestDB',
     COMPRESSION;  -- 启用压缩

-- 查看备份信息
RESTORE HEADERONLY FROM DISK = 'C:\Backups\TestDB_Full.bak';
```

### 使用 T-SQL 脚本备份

```sql
-- 动态生成备份文件名（带时间戳）
DECLARE @BackupPath NVARCHAR(500) = 'C:\Backups';
DECLARE @DatabaseName NVARCHAR(100) = 'TestDB';
DECLARE @BackupFileName NVARCHAR(500);
DECLARE @BackupDate NVARCHAR(20) = REPLACE(CONVERT(NVARCHAR, GETDATE(), 120), ':', '-');

SET @BackupFileName = @BackupPath + @DatabaseName + '_Full_' + @BackupDate + '.bak';

-- 执行备份
BACKUP DATABASE @DatabaseName
TO DISK = @BackupFileName
WITH FORMAT,
     NAME = @DatabaseName + ' Full Backup',
     COMPRESSION;

PRINT '备份完成: ' + @BackupFileName;
```

## 3. 差异备份

### 执行差异备份

```sql
-- 差异备份（必须在完整备份之后执行）
BACKUP DATABASE TestDB
TO DISK = 'C:\Backups\TestDB_Diff.bak'
WITH DIFFERENTIAL,
     MEDIANAME = 'TestDB_Backups',
     NAME = 'Differential Backup of TestDB',
     COMPRESSION;

-- 查看备份类型
SELECT 
    DatabaseName,
    BackupType,
    BackupStartDate,
    BackupFinishDate,
    BackupSize / 1024 / 1024 AS BackupSizeMB
FROM msdb.dbo.backupset
WHERE DatabaseName = 'TestDB'
ORDER BY BackupFinishDate DESC;
```

## 4. 事务日志备份

### 执行日志备份

```sql
-- 事务日志备份（必须在完整备份之后执行）
BACKUP LOG TestDB
TO DISK = 'C:\Backups\TestDB_Log.bak'
WITH NAME = 'Log Backup of TestDB',
     COMPRESSION;

-- 查看日志备份链
SELECT 
    DatabaseName,
    FirstLSN,
    LastLSN,
    CheckpointLSN,
    BackupStartDate
FROM msdb.dbo.backupset
WHERE DatabaseName = 'TestDB'
  AND Type = 'L'  -- L = Log backup
ORDER BY BackupStartDate DESC;
```

## 5. 恢复模式

### 恢复模式配置

```sql
-- 查看当前恢复模式
SELECT 
    name AS DatabaseName,
    recovery_model_desc AS RecoveryModel
FROM sys.databases
WHERE name = 'TestDB';

-- 设置为完整恢复模式
ALTER DATABASE TestDB SET RECOVERY FULL;

-- 设置为大容量日志恢复模式
ALTER DATABASE TestDB SET RECOVERY BULK_LOGGED;

-- 设置为简单恢复模式
ALTER DATABASE TestDB SET RECOVERY SIMPLE;
```

### 恢复模式对比

| 恢复模式 | 完整备份 | 差异备份 | 日志备份 | 时间点恢复 | 适用场景 |
|----------|----------|----------|----------|------------|----------|
| FULL | 支持 | 支持 | 支持 | 支持 | 生产环境 |
| BULK_LOGGED | 支持 | 支持 | 支持 | 有限 | 批量操作 |
| SIMPLE | 支持 | 支持 | 不支持 | 不支持 | 开发/测试 |

## 6. 恢复数据库

### 完整恢复

```sql
-- 恢复完整备份
RESTORE DATABASE TestDB
FROM DISK = 'C:\Backups\TestDB_Full.bak'
WITH REPLACE,
     STATS = 10;  -- 每 10% 显示进度

-- 恢复到指定位置
RESTORE DATABASE TestDB
FROM DISK = 'C:\Backups\TestDB_Full.bak'
WITH REPLACE,
     MOVE 'TestDB' TO 'D:\Data\TestDB.mdf',
     MOVE 'TestDB_Log' TO 'D:\Log\TestDB_Log.ldf',
     STATS = 10;
```

### 完整 + 差异恢复

```sql
-- 1. 恢复完整备份（使用 NORECOVERY）
RESTORE DATABASE TestDB
FROM DISK = 'C:\Backups\TestDB_Full.bak'
WITH NORECOVERY;

-- 2. 恢复差异备份
RESTORE DATABASE TestDB
FROM DISK = 'C:\Backups\TestDB_Diff.bak'
WITH RECOVERY;
```

### 完整 + 日志恢复（时间点恢复）

```sql
-- 1. 恢复完整备份
RESTORE DATABASE TestDB
FROM DISK = 'C:\Backups\TestDB_Full.bak'
WITH NORECOVERY;

-- 2. 恢复差异备份（如果有）
RESTORE DATABASE TestDB
FROM DISK = 'C:\Backups\TestDB_Diff.bak'
WITH NORECOVERY;

-- 3. 恢复日志备份
RESTORE DATABASE TestDB
FROM DISK = 'C:\Backups\TestDB_Log.bak'
WITH RECOVERY;

-- 时间点恢复
RESTORE DATABASE TestDB
FROM DISK = 'C:\Backups\TestDB_Log.bak'
WITH STOPAT = '2024-01-15 14:30:00',
     RECOVERY;
```

## 7. 自动化备份

### 创建维护计划（T-SQL）

```sql
-- 创建数据库作业进行定期备份
USE msdb;
GO

-- 创建作业
BEGIN TRANSACTION;
DECLARE @ReturnCode INT;
SELECT @ReturnCode = 0;

DECLARE @jobId BINARY(16);
EXEC @ReturnCode = msdb.dbo.sp_add_job 
    @job_name = N'Backup TestDB Database',
    @enabled = 1,
    @job_id = @jobId OUTPUT;

-- 添加作业步骤
EXEC @ReturnCode = msdb.dbo.sp_add_jobstep
    @job_id = @jobId,
    @step_name = N'Full Backup',
    @subsystem = N'TSQL',
    @command = N'BACKUP DATABASE TestDB TO DISK = ''C:\Backups\TestDB_Full.bak'' WITH COMPRESSION',
    @database_name = N'TestDB',
    @on_success_action = 1;

-- 添加作业计划（每天凌晨 2:00 执行）
EXEC @ReturnCode = msdb.dbo.sp_add_jobschedule
    @job_id = @jobId,
    @name = N'Daily Full Backup',
    @freq_type = 4,  -- Daily
    @freq_interval = 1,
    @active_start_time = 20000;  -- 02:00:00

-- 启用作业
EXEC msdb.dbo.sp_add_jobserver @job_id = @jobId;

COMMIT TRANSACTION;
```

### PowerShell 备份脚本

```powershell
# backup-sql.ps1
$serverName = "localhost"
$databaseName = "TestDB"
$backupPath = "C:\Backups"
$backupDate = Get-Date -Format "yyyy-MM-dd-HHmm"
$backupFile = "$backupPath\$databaseName_Full_$backupDate.bak"

# 加载 SMO 程序集
[Reflection.Assembly]::LoadWithPartialName("Microsoft.SqlServer.Smo") | Out-Null

# 创建服务器对象
$server = New-Object Microsoft.SqlServer.Management.Smo.Server($serverName)

# 创建备份对象
$backup = New-Object Microsoft.SqlServer.Management.Smo.Backup
$backup.Action = [Microsoft.SqlServer.Management.Smo.BackupActionType]::Database
$backup.BackupSetDescription = "Full backup of $databaseName"
$backup.BackupSetName = "$databaseName Full Backup"
$backup.Database = $databaseName
$backup.Devices.AddDevice($backupFile, [Microsoft.SqlServer.Management.Smo.DeviceType]::File)
$backup.CompressionOption = [Microsoft.SqlServer.Management.Smo.BackupCompressionOptions]::On

# 执行备份
Write-Host "Starting backup..."
$backup.SqlBackup($server)
Write-Host "Backup completed: $backupFile"
```

## 8. 备份验证

### 验证备份完整性

```sql
-- 验证备份文件
RESTORE VERIFYONLY FROM DISK = 'C:\Backups\TestDB_Full.bak';

-- 查看备份详细信息
RESTORE HEADERONLY FROM DISK = 'C:\Backups\TestDB_Full.bak';

RESTORE FILELISTONLY FROM DISK = 'C:\Backups\TestDB_Full.bak';

-- 使用 DBCC 检查数据库
DBCC CHECKDB(TestDB) WITH NO_INFOMSGS;
```

### 监控备份状态

```sql
-- 查看正在进行的备份
SELECT 
    session_id,
    command,
    percent_complete,
    estimated_completion_time,
    DATEADD(ms, estimated_completion_time, GETDATE()) AS estimated_completion
FROM sys.dm_exec_requests
WHERE command LIKE 'BACKUP%';
```

## 10. 备份策略

### 推荐备份策略

```sql
-- 生产环境备份策略
-- 1. 每周日凌晨执行完整备份
-- 2. 每天凌晨执行差异备份
-- 3. 每小时执行日志备份

-- 创建完整备份作业（每周日）
BACKUP DATABASE TestDB
TO DISK = 'C:\Backups\TestDB_Full_Sunday.bak'
WITH COMPRESSION,
     STATS = 10;

-- 创建差异备份作业（每天）
BACKUP DATABASE TestDB
TO DISK = 'C:\Backups\TestDB_Diff_Daily.bak'
WITH DIFFERENTIAL,
     COMPRESSION;

-- 创建日志备份作业（每小时）
BACKUP LOG TestDB
TO DISK = 'C:\Backups\TestDB_Log_Hourly.bak'
WITH COMPRESSION;
```

### 备份保留策略

```sql
-- 清理旧备份文件
DECLARE @BackupPath NVARCHAR(500) = 'C:\Backups';
DECLARE @RetentionDays INT = 30;

DECLARE @SQL NVARCHAR(MAX);
SET @SQL = N'
    EXEC xp_cmdshell ''DEL "' + @BackupPath + 'TestDB_Full_' + 
    REPLACE(CONVERT(NVARCHAR, DATEADD(DAY, -@RetentionDays, GETDATE()), 120), ':', '-') + 
    '.bak"'';
';

EXEC sp_executesql @SQL, N'@RetentionDays INT', @RetentionDays = @RetentionDays;
```

## 11. 灾难恢复

### 系统数据库备份

```sql
-- 备份 master 数据库
BACKUP DATABASE master
TO DISK = 'C:\Backups\master.bak'
WITH COMPRESSION;

-- 备份 msdb 数据库
BACKUP DATABASE msdb
TO DISK = 'C:\Backups\msdb.bak'
WITH COMPRESSION;

-- 备份 model 数据库
BACKUP DATABASE model
TO DISK = 'C:\Backups\model.bak'
WITH COMPRESSION;
```

### 灾难恢复步骤

```sql
-- 1. 重建 master 数据库（如果需要）
-- 2. 启动 SQL Server 单用户模式
-- 3. 恢复 master 数据库
RESTORE DATABASE master
FROM DISK = 'C:\Backups\master.bak'
WITH REPLACE;

-- 4. 恢复 msdb 数据库
RESTORE DATABASE msdb
FROM DISK = 'C:\Backups\msdb.bak'
WITH REPLACE;

-- 5. 恢复 model 数据库
RESTORE DATABASE model
FROM DISK = 'C:\Backups\model.bak'
WITH REPLACE;

-- 6. 恢复用户数据库
RESTORE DATABASE TestDB
FROM DISK = 'C:\Backups\TestDB_Full.bak'
WITH REPLACE;
```

## 12. 最佳实践

1. **备份策略**：
   - 定期执行完整备份（如每周）
   - 频繁执行差异备份（如每天）
   - 高频执行日志备份（如每小时）
   - 保留备份文件足够长的时间

2. **备份位置**：
   - 备份到本地磁盘
   - 同时复制到远程存储
   - 考虑使用云存储

3. **验证备份**：
   - 定期验证备份文件的完整性
   - 测试恢复流程
   - 监控备份作业执行情况

4. **安全性**：
   - 加密备份文件
   - 限制备份文件访问权限
   - 安全存储备份密码

5. **监控告警**：
   - 设置备份失败告警
   - 监控备份文件大小
   - 监控备份完成时间
