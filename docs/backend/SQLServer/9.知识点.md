# SQL Server 知识点总结

## 一、基础概念

### 1.1 数据类型速查

| 类型 | 说明 | 范围/大小 |
|-----|------|----------|
| **BIT** | 布尔值 | 0, 1, NULL |
| **TINYINT** | 小整数 | 0~255，1字节 |
| **SMALLINT** | 短整数 | -32768~32767，2字节 |
| **INT** | 整数 | -21亿~21亿，4字节 |
| **BIGINT** | 长整数 | -9京~9京，8字节 |
| **DECIMAL(p,s)** | 精确小数 | p:精度，s:小数位 |
| **FLOAT** | 浮点数 | 近似值，8字节 |
| **DATE** | 日期 | 0001-01-01~9999-12-31 |
| **DATETIME** | 日期时间 | 1753-01-01~9999-12-31 |
| **DATETIME2** | 高精度日期时间 | 0001-01-01~9999-12-31，100纳秒精度 |

## 二、SQL 语句速查

### 2.1 DDL（数据定义语言）

```sql
-- 数据库操作
CREATE DATABASE MyDB;
ALTER DATABASE MyDB SET RECOVERY FULL;
DROP DATABASE MyDB;

-- 表操作
CREATE TABLE Users (
    UserID INT IDENTITY(1,1) PRIMARY KEY,
    UserName NVARCHAR(50) NOT NULL,
    Email NVARCHAR(100) UNIQUE,
    Age INT CHECK (Age >= 0),
    CreatedDate DATETIME2 DEFAULT GETDATE(),
    IsActive BIT DEFAULT 1
);

ALTER TABLE Users ADD Phone NVARCHAR(20);
ALTER TABLE Users ALTER COLUMN UserName NVARCHAR(100);
ALTER TABLE Users DROP COLUMN Phone;

DROP TABLE Users;

-- 索引操作
CREATE INDEX IX_Users_Name ON Users(UserName);
CREATE UNIQUE INDEX IX_Users_Email ON Users(Email);
CREATE CLUSTERED INDEX IX_Orders_Date ON Orders(OrderDate);

ALTER INDEX IX_Users_Name ON Users REBUILD;
DROP INDEX IX_Users_Name ON Users;

-- 视图操作
CREATE VIEW vw_ActiveUsers AS
SELECT * FROM Users WHERE IsActive = 1;

ALTER VIEW vw_ActiveUsers AS
SELECT UserID, UserName, Email FROM Users WHERE IsActive = 1;

DROP VIEW vw_ActiveUsers;
```

### 2.2 DML（数据操作语言）

```sql
-- 插入
INSERT INTO Users (UserName, Email) VALUES ('张三', 'zs@example.com');
INSERT INTO Users VALUES ('李四', 'ls@example.com', 25, GETDATE(), 1);
INSERT INTO Users SELECT Name, Email FROM TempUsers;

-- 更新
UPDATE Users SET Age = 26 WHERE UserID = 1;
UPDATE U SET U.Age = U.Age + 1 FROM Users U JOIN Orders O ON U.UserID = O.UserID;

-- 删除
DELETE FROM Users WHERE UserID = 1;
TRUNCATE TABLE Users;  -- 快速清空，不可回滚

-- 合并（UPSERT）
MERGE Users AS target
USING (SELECT 1 AS ID, '张三' AS Name) AS source
ON target.UserID = source.ID
WHEN MATCHED THEN UPDATE SET UserName = source.Name
WHEN NOT MATCHED THEN INSERT (UserName) VALUES (source.Name);
```

### 2.3 DQL（数据查询语言）

```sql
-- 基础查询
SELECT * FROM Users;
SELECT DISTINCT Status FROM Users;
SELECT TOP 10 * FROM Users;
SELECT TOP 10 PERCENT * FROM Users;

-- 条件查询
SELECT * FROM Users WHERE Age > 18;
SELECT * FROM Users WHERE Age BETWEEN 18 AND 60;
SELECT * FROM Users WHERE UserName LIKE '%张%';
SELECT * FROM Users WHERE UserName IN ('张三', '李四');

-- 排序
SELECT * FROM Users ORDER BY Age DESC;
SELECT * FROM Users ORDER BY Age OFFSET 10 ROWS FETCH NEXT 10 ROWS ONLY;

-- 分组
SELECT Status, COUNT(*) AS Count FROM Users GROUP BY Status;
SELECT Status, COUNT(*) AS Count FROM Users GROUP BY Status HAVING COUNT(*) > 5;

-- 连接
SELECT * FROM Users U INNER JOIN Orders O ON U.UserID = O.UserID;
SELECT * FROM Users U LEFT JOIN Orders O ON U.UserID = O.UserID;
SELECT * FROM Users U RIGHT JOIN Orders O ON U.UserID = O.UserID;
SELECT * FROM Users U FULL OUTER JOIN Orders O ON U.UserID = O.UserID;
SELECT * FROM Users U CROSS JOIN Departments D;

-- 子查询
SELECT * FROM Users WHERE UserID IN (SELECT UserID FROM Orders);
SELECT * FROM Users U WHERE EXISTS (SELECT 1 FROM Orders O WHERE O.UserID = U.UserID);
SELECT * FROM (SELECT UserID, UserName FROM Users) AS T;
```

### 2.4 TCL（事务控制语言）

```sql
-- 事务控制
BEGIN TRANSACTION;  -- 或 BEGIN TRAN
COMMIT TRANSACTION; -- 或 COMMIT
ROLLBACK TRANSACTION; -- 或 ROLLBACK
SAVE TRANSACTION SavePoint1;

-- 事务属性
SET XACT_ABORT ON;  -- 错误时自动回滚

```

## 三、索引与优化

### 3.1 索引类型

| 索引类型 | 特点 | 适用场景 |
|---------|------|---------|
| **聚集索引** | 数据物理排序，唯一 | 主键、范围查询 |
| **非聚集索引** | 独立于数据 | 频繁查询列 |
| **唯一索引** | 确保值唯一 | 唯一约束 |
| **覆盖索引** | 包含所有查询列 | 避免回表 |
| **复合索引** | 多列索引 | 多条件查询 |
| **筛选索引** | 部分数据索引 | 热点数据 |
| **列存储索引** | 列式存储 | 分析查询 |

### 3.2 查询优化原则

1. **避免在索引列上使用函数**
   ```sql
   -- 不好
   WHERE YEAR(CreatedDate) = 2023
   -- 好
   WHERE CreatedDate >= '2023-01-01' AND CreatedDate < '2024-01-01'
   ```

2. **避免隐式转换**
   ```sql
   -- 不好（如果列是NVARCHAR）
   WHERE UserName = '张三'
   -- 好
   WHERE UserName = N'张三'
   ```

3. **使用 EXISTS 代替 IN**
   ```sql
   -- 不好
   WHERE UserID IN (SELECT UserID FROM Orders)
   -- 好
   WHERE EXISTS (SELECT 1 FROM Orders O WHERE O.UserID = Users.UserID)
   ```

4. **避免 SELECT ***
   ```sql
   -- 不好
   SELECT * FROM Users
   -- 好
   SELECT UserID, UserName FROM Users
   ```

## 四、事务与并发

### 4.1 隔离级别

| 隔离级别 | 脏读 | 不可重复读 | 幻读 | 并发性 |
|---------|------|-----------|------|-------|
| READ UNCOMMITTED | ✓ | ✓ | ✓ | 最高 |
| READ COMMITTED | ✗ | ✓ | ✓ | 高 |
| REPEATABLE READ | ✗ | ✗ | ✓ | 中 |
| SNAPSHOT | ✗ | ✗ | ✗ | 中 |
| SERIALIZABLE | ✗ | ✗ | ✗ | 最低 |

### 4.2 锁类型

| 锁类型 | 缩写 | 说明 |
|-------|------|------|
| 共享锁 | S | 读取时获取 |
| 排他锁 | X | 修改时获取 |
| 更新锁 | U | 更新前获取 |
| 意向锁 | I | 表示下级资源加锁 |
| 架构锁 | Sch | DDL操作时使用 |

## 五、备份与恢复

### 5.1 备份类型

```sql
-- 完整备份
BACKUP DATABASE MyDB TO DISK = 'C:\Backups\MyDB_Full.bak';

-- 差异备份
BACKUP DATABASE MyDB TO DISK = 'C:\Backups\MyDB_Diff.bak' WITH DIFFERENTIAL;

-- 日志备份
BACKUP LOG MyDB TO DISK = 'C:\Backups\MyDB_Log.trn';

-- 文件备份
BACKUP DATABASE MyDB FILEGROUP = 'PRIMARY' TO DISK = 'C:\Backups\MyDB_FG.bak';
```

### 5.2 恢复模式

| 模式 | 日志记录 | 恢复能力 | 适用场景 |
|-----|---------|---------|---------|
| SIMPLE | 最小 | 完整+差异 | 开发测试 |
| FULL | 完整 | 时间点恢复 | 生产环境 |
| BULK_LOGGED | 批量操作最小 | 有限 | 大批量导入 |

## 六、存储过程与函数

### 6.1 存储过程

```sql
-- 创建存储过程
CREATE PROCEDURE sp_GetUserByID
    @UserID INT,
    @UserName NVARCHAR(50) OUTPUT
AS
BEGIN
    SET NOCOUNT ON;
    SELECT @UserName = UserName FROM Users WHERE UserID = @UserID;
END;

-- 执行
DECLARE @Name NVARCHAR(50);
EXEC sp_GetUserByID 1, @Name OUTPUT;
```

### 6.2 函数类型

| 类型 | 返回值 | 特点 |
|-----|-------|------|
| 标量函数 | 单个值 | 可在SELECT中使用 |
| 内联表值函数 | 表 | 性能好，可优化 |
| 多语句表值函数 | 表 | 支持复杂逻辑 |
