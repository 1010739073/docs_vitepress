# Nginx 大厂面试题与高并发场景

## 一、Nginx 基础原理

### 1. Nginx 架构

**问题**：Nginx 的架构是怎样的？

**详细答案**：
Nginx 采用事件驱动、异步非阻塞的架构，主要特点：

1. **Master-Worker 模式**：
   - Master 进程：管理 Worker 进程
   - Worker 进程：处理客户端请求
   - 多进程模型，避免线程安全问题

2. **事件驱动模型**：
   - 基于 epoll（Linux）或 kqueue（BSD）
   - 单个 Worker 可以处理数万个并发连接
   - 非阻塞 I/O，提高并发性能

3. **模块化设计**：
   - 核心模块：事件处理、配置解析
   - HTTP 模块：处理 HTTP 请求
   - Upstream 模块：反向代理
   - 其他模块：负载均衡、缓存、限流等

**配置示例**：

```nginx
# nginx.conf
user nginx;
worker_processes auto;
worker_rlimit_nofile 65535;

events {
    worker_connections 10240;
    use epoll;
    multi_accept on;
}

http {
    include mime.types;
    default_type application/octet-stream;
    
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    
    # Gzip 压缩
    gzip on;
    gzip_min_length 1k;
    gzip_types text/plain text/css application/json application/javascript;
    
    # 虚拟主机
    server {
        listen 80;
        server_name example.com;
        
        location / {
            root /var/www/html;
            index index.html index.htm;
        }
    }
}
```

### 2. Nginx 工作原理

**问题**：Nginx 是如何处理请求的？

**详细答案**：
Nginx 处理请求的流程：

1. **请求接收**：
   - 监听端口，接收客户端连接
   - 将连接分配给 Worker 进程

2. **请求处理**：
   - 解析 HTTP 请求
   - 匹配 location 规则
   - 执行相应的处理逻辑

3. **响应返回**：
   - 生成 HTTP 响应
   - 发送给客户端
   - 关闭或保持连接

**代码示例**：

```nginx
# 请求处理流程示例
http {
    # 全局配置
    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';
    
    access_log /var/log/nginx/access.log main;
    error_log /var/log/nginx/error.log warn;
    
    # 上游服务器
    upstream backend {
        server 192.168.1.10:8080 weight=3;
        server 192.168.1.11:8080;
        server 192.168.1.12:8080 backup;
        keepalive 32;
    }
    
    server {
        listen 80;
        server_name example.com;
        
        # 重定向到 HTTPS
        return 301 https://$server_name$request_uri;
    }
    
    server {
        listen 443 ssl http2;
        server_name example.com;
        
        ssl_certificate /etc/nginx/ssl/cert.pem;
        ssl_certificate_key /etc/nginx/ssl/key.pem;
        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_ciphers HIGH:!aNULL:!MD5;
        
        # 静态文件
        location /static/ {
            alias /var/www/static/;
            expires 30d;
            add_header Cache-Control "public, immutable";
        }
        
        # 动态请求
        location /api/ {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            
            # 连接优化
            proxy_http_version 1.1;
            proxy_set_header Connection "";
            proxy_connect_timeout 60s;
            proxy_send_timeout 60s;
            proxy_read_timeout 60s;
        }
        
        # 健康检查
        location /health {
            access_log off;
            return 200 "healthy\n";
            add_header Content-Type text/plain;
        }
    }
}
```

## 二、Nginx 高并发场景

### 1. 高并发负载均衡

**问题**：如何使用 Nginx 实现高并发负载均衡？

**详细答案**：
Nginx 负载均衡的策略：

1. **轮询（Round Robin）**：
   - 默认策略
   - 按顺序分配请求

2. **最少连接（Least Connections）**：
   - 分配给连接数最少的服务器
   - 适用于长连接场景

3. **IP Hash**：
   - 根据客户端 IP 分配
   - 保证同一客户端访问同一服务器

4. **加权轮询（Weighted Round Robin）**：
   - 根据权重分配请求
   - 适用于服务器性能不均的场景

**配置示例**：

```nginx
# 轮询负载均衡
upstream backend_round_robin {
    server 192.168.1.10:8080;
    server 192.168.1.11:8080;
    server 192.168.1.12:8080;
}

# 最少连接负载均衡
upstream backend_least_conn {
    least_conn;
    server 192.168.1.10:8080;
    server 192.168.1.11:8080;
    server 192.168.1.12:8080;
}

# IP Hash 负载均衡
upstream backend_ip_hash {
    ip_hash;
    server 192.168.1.10:8080;
    server 192.168.1.11:8080;
    server 192.168.1.12:8080;
}

# 加权轮询负载均衡
upstream backend_weighted {
    server 192.168.1.10:8080 weight=3;
    server 192.168.1.11:8080 weight=2;
    server 192.168.1.12:8080 weight=1;
}

# 完整配置示例
http {
    # 负载均衡配置
    upstream backend {
        least_conn;
        server 192.168.1.10:8080 weight=3 max_fails=3 fail_timeout=30s;
        server 192.168.1.11:8080 weight=2 max_fails=3 fail_timeout=30s;
        server 192.168.1.12:8080 weight=1 max_fails=3 fail_timeout=30s backup;
        keepalive 32;
        keepalive_timeout 60s;
        keepalive_requests 100;
    }
    
    server {
        listen 80;
        server_name example.com;
        
        location / {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            
            # 连接优化
            proxy_http_version 1.1;
            proxy_set_header Connection "";
            proxy_connect_timeout 60s;
            proxy_send_timeout 60s;
            proxy_read_timeout 60s;
            
            # 缓冲优化
            proxy_buffering on;
            proxy_buffer_size 4k;
            proxy_buffers 8 4k;
            proxy_busy_buffers_size 8k;
        }
    }
}
```

### 2. 高并发缓存优化

**问题**：如何使用 Nginx 缓存提高高并发性能？

**详细答案**：
Nginx 缓存优化的方法：

1. **静态文件缓存**：
   - 使用 expires 指令
   - 设置 Cache-Control 头

2. **代理缓存**：
   - 使用 proxy_cache
   - 缓存上游服务器的响应

3. **FastCGI 缓存**：
   - 使用 fastcgi_cache
   - 缓存 FastCGI 响应

4. **Open File Cache**：
   - 缓存文件描述符
   - 减少文件打开开销

**配置示例**：

```nginx
# 静态文件缓存
http {
    # 静态文件缓存
    server {
        listen 80;
        server_name example.com;
        
        # 静态文件缓存
        location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|woff|woff2|ttf|eot)$ {
            expires 30d;
            add_header Cache-Control "public, immutable";
            access_log off;
        }
        
        # HTML 文件缓存
        location ~* \.html$ {
            expires 1h;
            add_header Cache-Control "public";
        }
    }
}

# 代理缓存
http {
    # 定义缓存路径和参数
    proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=1g inactive=60m use_temp_path=off;
    
    # 定义缓存键
    proxy_cache_key "$scheme$request_method$host$request_uri";
    
    server {
        listen 80;
        server_name example.com;
        
        # 启用代理缓存
        location / {
            proxy_cache my_cache;
            proxy_cache_valid 200 302 10m;
            proxy_cache_valid 404 1m;
            proxy_cache_use_stale error timeout updating http_500 http_502 http_503 http_504;
            proxy_cache_background_update on;
            proxy_cache_lock on;
            
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        }
        
        # 清除缓存
        location ~ /purge(/.*) {
            allow 127.0.0.1;
            deny all;
            proxy_cache_purge my_cache "$scheme$request_method$host$1";
        }
    }
}

# FastCGI 缓存
http {
    # 定义 FastCGI 缓存路径和参数
    fastcgi_cache_path /var/cache/nginx/fastcgi levels=1:2 keys_zone=fastcgi_cache:10m max_size=1g inactive=60m;
    
    # 定义缓存键
    fastcgi_cache_key "$scheme$request_method$host$request_uri";
    
    server {
        listen 80;
        server_name example.com;
        
        # 启用 FastCGI 缓存
        location ~ \.php$ {
            fastcgi_cache fastcgi_cache;
            fastcgi_cache_valid 200 60m;
            fastcgi_cache_bypass $skip_cache;
            fastcgi_no_cache $skip_cache;
            
            fastcgi_pass unix:/var/run/php-fpm.sock;
            fastcgi_index index.php;
            fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
            include fastcgi_params;
        }
    }
}

# Open File Cache
http {
    # 定义 open_file_cache
    open_file_cache max=10000 inactive=30s;
    open_file_cache_valid 60s;
    open_file_cache_min_uses 2;
    open_file_cache_errors on;
    
    server {
        listen 80;
        server_name example.com;
        
        location / {
            root /var/www/html;
            index index.html;
        }
    }
}
```

### 3. 高并发限流

**问题**：如何使用 Nginx 实现高并发限流？

**详细答案**：
Nginx 限流的方法：

1. **连接限流**：
   - limit_conn_zone
   - limit_conn

2. **请求限流**：
   - limit_req_zone
   - limit_req

3. **带宽限流**：
   - limit_rate
   - limit_rate_after

**配置示例**：

```nginx
# 连接限流
http {
    # 定义连接限流区域
    limit_conn_zone $binary_remote_addr zone=conn_limit:10m;
    
    server {
        listen 80;
        server_name example.com;
        
        # 限制每个 IP 的并发连接数
        limit_conn conn_limit 10;
        
        location / {
            root /var/www/html;
            index index.html;
        }
    }
}

# 请求限流
http {
    # 定义请求限流区域
    limit_req_zone $binary_remote_addr zone=req_limit:10m rate=10r/s;
    
    server {
        listen 80;
        server_name example.com;
        
        # 限制每个 IP 的请求速率
        limit_req zone=req_limit burst=20 nodelay;
        
        location / {
            root /var/www/html;
            index index.html;
        }
        
        # 对 API 接口应用更严格的限流
        location /api/ {
            limit_req zone=req_limit burst=5 nodelay;
            proxy_pass http://backend;
        }
    }
}

# 带宽限流
http {
    server {
        listen 80;
        server_name example.com;
        
        # 限制下载速度
        location /download/ {
            limit_rate_after 10m;
            limit_rate 1m;
            
            root /var/www/download;
        }
        
        # 对视频文件应用带宽限制
        location ~* \.(mp4|avi|mkv)$ {
            limit_rate_after 5m;
            limit_rate 500k;
            
            root /var/www/videos;
        }
    }
}

# 综合限流配置
http {
    # 连接限流
    limit_conn_zone $binary_remote_addr zone=conn_limit:10m;
    
    # 请求限流
    limit_req_zone $binary_remote_addr zone=req_limit:10m rate=10r/s;
    
    # 白名单
    geo $limit {
        default 1;
        192.168.1.0/24 0;
        10.0.0.0/8 0;
    }
    
    server {
        listen 80;
        server_name example.com;
        
        # 应用限流
        limit_conn conn_limit 10;
        limit_req zone=req_limit burst=20 nodelay;
        
        location / {
            if ($limit = 0) {
                set $limit_key "";
            }
            
            root /var/www/html;
            index index.html;
        }
    }
}
```

### 4. 高并发 SSL 优化

**问题**：如何优化 Nginx 的 SSL 性能？

**详细答案**：
SSL 优化的方法：

1. **SSL 会话缓存**：
   - ssl_session_cache
   - ssl_session_timeout

2. **SSL 会话票据**：
   - ssl_session_tickets

3. **SSL 协议和加密套件**：
   - 使用 TLS 1.2 和 TLS 1.3
   - 选择高效的加密套件

4. **OCSP Stapling**：
   - ssl_stapling
   - ssl_stapling_verify

**配置示例**：

```nginx
# SSL 优化配置
http {
    # SSL 会话缓存
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 1d;
    ssl_session_tickets off;
    
    # SSL 协议和加密套件
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384';
    ssl_prefer_server_ciphers off;
    
    # OCSP Stapling
    ssl_stapling on;
    ssl_stapling_verify on;
    ssl_trusted_certificate /etc/nginx/ssl/chain.pem;
    resolver 8.8.8.8 8.8.4.4 valid=300s;
    resolver_timeout 5s;
    
    server {
        listen 443 ssl http2;
        server_name example.com;
        
        # SSL 证书
        ssl_certificate /etc/nginx/ssl/cert.pem;
        ssl_certificate_key /etc/nginx/ssl/key.pem;
        
        # SSL 优化
        ssl_buffer_size 4k;
        ssl_ecdh_curve secp384r1;
        
        location / {
            root /var/www/html;
            index index.html;
        }
    }
}
```

## 四、Nginx 安全配置

### 1. 安全头配置

**问题**：如何配置 Nginx 的安全头？

**详细答案**：
安全头配置的方法：

1. **常用安全头**：
   - X-Frame-Options
   - X-Content-Type-Options
   - X-XSS-Protection
   - Strict-Transport-Security
   - Content-Security-Policy

2. **其他安全措施**：
   - 隐藏版本号
   - 限制请求方法
   - 防止目录遍历

**配置示例**：

```nginx
# 安全头配置
http {
    # 隐藏版本号
    server_tokens off;
    
    server {
        listen 80;
        server_name example.com;
        
        # 安全头
        add_header X-Frame-Options "SAMEORIGIN" always;
        add_header X-Content-Type-Options "nosniff" always;
        add_header X-XSS-Protection "1; mode=block" always;
        add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
        add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline';" always;
        add_header Referrer-Policy "strict-origin-when-cross-origin" always;
        
        # 限制请求方法
        if ($request_method !~ ^(GET|HEAD|POST)$ ) {
            return 405;
        }
        
        # 防止目录遍历
        location ~* \.(htaccess|htpasswd|ini|log|sh|sql|bak|conf)$ {
            deny all;
        }
        
        location / {
            root /var/www/html;
            index index.html;
        }
    }
}
```

### 2. 访问控制

**问题**：如何配置 Nginx 的访问控制？

**详细答案**：
访问控制的方法：

1. **IP 访问控制**：
   - allow/deny 指令
   - geo 模块

2. **基本认证**：
   - auth_basic
   - auth_basic_user_file

3. **IP 白名单/黑名单**：
   - 使用 include 包含配置文件

**配置示例**：

```nginx
# IP 访问控制
server {
    listen 80;
    server_name example.com;
    
    # 只允许特定 IP 访问
    allow 192.168.1.0/24;
    allow 10.0.0.0/8;
    deny all;
    
    location / {
        root /var/www/html;
        index index.html;
    }
}

# 基本认证
server {
    listen 80;
    server_name example.com;
    
    # 启用基本认证
    auth_basic "Restricted Area";
    auth_basic_user_file /etc/nginx/.htpasswd;
    
    location / {
        root /var/www/html;
        index index.html;
    }
}

# IP 白名单/黑名单
http {
    # 定义 IP 白名单
    geo $whitelist {
        default 0;
        192.168.1.0/24 1;
        10.0.0.0/8 1;
    }
    
    server {
        listen 80;
        server_name example.com;
        
        # 只允许白名单 IP 访问
        if ($whitelist = 0) {
            return 403;
        }
        
        location / {
            root /var/www/html;
            index index.html;
        }
    }
}
```

## 五、Nginx 监控与调优

### 1. 性能监控

**问题**：如何监控 Nginx 的性能？

**详细答案**：
Nginx 性能监控的方法：

1. **内置监控**：
   - stub_status 模块
   - access_log 和 error_log

2. **第三方工具**：
   - Prometheus + Grafana
   - ELK Stack
   - Datadog

**配置示例**：

```nginx
# 内置监控
```

### 2. 性能调优

**问题**：如何调优 Nginx 的性能？

**详细答案**：
Nginx 性能调优的方法：

1. **系统级优化**：
   - 调整文件描述符限制
   - 调整内核参数

2. **Nginx 级优化**：
   - 优化 Worker 进程
   - 优化网络参数
   - 优化缓存配置

3. **应用级优化**：
   - 优化后端服务器
   - 使用 CDN
   - 启用压缩

**配置示例**：

```nginx
# 系统级优化（需要在系统层面配置）
# /etc/sysctl.conf
net.core.somaxconn = 65535
net.ipv4.tcp_max_syn_backlog = 8192
net.ipv4.tcp_tw_reuse = 1
net.ipv4.ip_local_port_range = 1024 65535

# Nginx 级优化
user nginx;
worker_processes auto;
worker_priority -5;
worker_rlimit_nofile 65535;

events {
    worker_connections 10240;
    use epoll;
    multi_accept on;
    accept_mutex off;
}

http {
    # 网络优化
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    
    # 缓冲优化
    client_body_buffer_size 128k;
    client_header_buffer_size 1k;
    large_client_header_buffers 4 4k;
    client_max_body_size 10m;
    
    # 超时优化
    client_body_timeout 12;
    client_header_timeout 12;
    send_timeout 10;
    
    # Keep-Alive 优化
    keepalive_timeout 65;
    keepalive_requests 100;
    
    # Gzip 压缩
    gzip on;
    gzip_min_length 1k;
    gzip_comp_level 6;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    
    # 隐藏版本号
    server_tokens off;
    
    server {
        listen 80;
        server_name example.com;
        
        location / {
            root /var/www/html;
            index index.html;
        }
    }
}
```
