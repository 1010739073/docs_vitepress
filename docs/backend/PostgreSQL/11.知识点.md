## 二、SQL 语句速查

### 2.1 DDL（数据定义语言）

```sql
-- 数据库操作
CREATE DATABASE mydb;
DROP DATABASE mydb;
\c mydb  -- 切换数据库

-- 表操作
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT NOW()
);

ALTER TABLE users ADD COLUMN age INTEGER;
ALTER TABLE users DROP COLUMN age;
ALTER TABLE users RENAME TO customers;

DROP TABLE users;
TRUNCATE TABLE users RESTART IDENTITY;

-- 索引操作
CREATE INDEX idx_name ON users(name);
CREATE UNIQUE INDEX idx_email ON users(email);
DROP INDEX idx_name;

-- 视图操作
CREATE VIEW active_users AS SELECT * FROM users WHERE is_active = TRUE;
DROP VIEW active_users;
```

### 2.2 DML（数据操作语言）

```sql
-- 插入
INSERT INTO users (name, email) VALUES ('张三', 'zs@example.com');
INSERT INTO users VALUES ('李四', 'ls@example.com');
INSERT INTO users SELECT * FROM temp_users;

-- 更新
UPDATE users SET age = 25 WHERE id = 1;
UPDATE users SET age = age + 1 WHERE is_active = TRUE;

-- 删除
DELETE FROM users WHERE id = 1;
TRUNCATE TABLE users;

-- UPSERT (INSERT ON CONFLICT)
INSERT INTO users (id, name) VALUES (1, '张三')
ON CONFLICT (id) DO UPDATE SET name = EXCLUDED.name;

INSERT INTO users (id, name) VALUES (1, '张三')
ON CONFLICT (id) DO NOTHING;
```

### 2.3 DQL（数据查询语言）

```sql
-- 基础查询
SELECT * FROM users;
SELECT DISTINCT status FROM users;
SELECT * FROM users LIMIT 10 OFFSET 0;

-- 条件查询
SELECT * FROM users WHERE age > 18;
SELECT * FROM users WHERE name LIKE '%张%';
SELECT * FROM users WHERE name ILIKE '%zhang%';  -- 不区分大小写
SELECT * FROM users WHERE age BETWEEN 18 AND 60;
SELECT * FROM users WHERE id IN (1, 2, 3);

-- 排序和分页
SELECT * FROM users ORDER BY age DESC;
SELECT * FROM users ORDER BY age LIMIT 10 OFFSET 20;

-- 分组
SELECT status, COUNT(*) FROM users GROUP BY status;
SELECT status, COUNT(*) FROM users GROUP BY status HAVING COUNT(*) > 5;

-- 连接
SELECT * FROM users u JOIN orders o ON u.id = o.user_id;
SELECT * FROM users u LEFT JOIN orders o ON u.id = o.user_id;
SELECT * FROM users u FULL JOIN orders o ON u.id = o.user_id;
SELECT * FROM users u CROSS JOIN roles r;

-- 子查询
SELECT * FROM users WHERE id IN (SELECT user_id FROM orders);
SELECT * FROM users u WHERE EXISTS (SELECT 1 FROM orders o WHERE o.user_id = u.id);
```

## 三、索引与优化

### 3.1 索引类型

| 索引类型 | 适用场景 | 示例 |
|---------|---------|------|
| **B-tree** | 等值、范围查询 | 默认索引 |
| **Hash** | 等值查询 | 很少使用 |
| **GiST** | 几何、范围类型 | 地理数据 |
| **GIN** | 数组、JSONB、全文搜索 | jsonb_ops |
| **BRIN** | 大表顺序数据 | 时序数据 |

### 3.2 查询优化原则

1. **避免在索引列上使用函数**
   ```sql
   -- 不好
   WHERE EXTRACT(YEAR FROM created_at) = 2023
   -- 好
   WHERE created_at >= '2023-01-01' AND created_at < '2024-01-01'
   ```

2. **使用 EXISTS 代替 IN**
   ```sql
   -- 不好
   WHERE id IN (SELECT user_id FROM orders)
   -- 好
   WHERE EXISTS (SELECT 1 FROM orders o WHERE o.user_id = users.id)
   ```

3. **避免 SELECT ***
   ```sql
   -- 不好
   SELECT * FROM users
   -- 好
   SELECT id, name FROM users
   ```

## 四、事务与并发

### 4.1 隔离级别

| 隔离级别 | 脏读 | 不可重复读 | 幻读 |
|---------|------|-----------|------|
| READ UNCOMMITTED | ✗ | ✓ | ✓ |
| READ COMMITTED | ✗ | ✓ | ✓ | (默认)
| REPEATABLE READ | ✗ | ✗ | ✗ | (PG中)
| SERIALIZABLE | ✗ | ✗ | ✗ |

### 4.2 事务控制

```sql
BEGIN;  -- 开始事务
COMMIT;  -- 提交事务
ROLLBACK;  -- 回滚事务
SAVEPOINT sp1;  -- 保存点
ROLLBACK TO SAVEPOINT sp1;  -- 回滚到保存点

-- 设置隔离级别
BEGIN ISOLATION LEVEL SERIALIZABLE;
SET TRANSACTION ISOLATION LEVEL REPEATABLE READ;
```

## 七、高可用

### 7.1 流复制

```bash
# 主服务器配置
wal_level = replica
max_wal_senders = 10
wal_keep_size = 1GB

# 备服务器设置
pg_basebackup -h primary -U replicator -D /data -Fp -Xs -P -R
```

### 7.2 逻辑复制

```sql
-- 发布
CREATE PUBLICATION my_pub FOR TABLE users, orders;

-- 订阅
CREATE SUBSCRIPTION my_sub 
CONNECTION 'host=primary dbname=mydb user=replicator' 
PUBLICATION my_pub;
```
