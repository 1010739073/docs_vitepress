# SQL Server 索引与性能

## 1. 索引基础

### 什么是索引

索引是数据库表中一列或多列的值进行排序的一种结构，类似于书籍的目录。合理使用索引可以大大提高查询性能。

### 索引类型

| 类型 | 描述 | 适用场景 |
|------|------|----------|
| 聚集索引 | 数据按索引顺序存储 | 主键、经常查询的列 |
| 非聚集索引 | 独立于数据的索引结构 | 外键、查询条件 |
| 唯一索引 | 索引列值唯一 | 邮箱、用户名 |
| 全文索引 | 文本搜索 | 大文本内容 |
| 列存储索引 | 数据分析 | 报表查询 |

## 2. 创建和管理索引

### 创建聚集索引

```sql
-- 创建表时指定主键（自动创建聚集索引）
CREATE TABLE Users (
    UserID INT PRIMARY KEY IDENTITY(1,1),
    UserName NVARCHAR(50) NOT NULL,
    Email NVARCHAR(100)
);

-- 在已有表上创建聚集索引
CREATE CLUSTERED INDEX IX_Users_UserID ON Users(UserID);
```

### 创建非聚集索引

```sql
-- 创建单列索引
CREATE NONCLUSTERED INDEX IX_Users_Email ON Users(Email);

-- 创建多列索引
CREATE NONCLUSTERED INDEX IX_Users_NameAge ON Users(UserName, Age);

-- 创建唯一索引
CREATE UNIQUE NONCLUSTERED INDEX IX_Users_Email ON Users(Email);

-- 创建过滤索引
CREATE NONCLUSTERED INDEX IX_Users_ActiveUsers 
ON Users(UserID) 
WHERE IsActive = 1;
```

### 创建包含索引

```sql
-- 包含索引可以提高查询性能，避免回表操作
CREATE NONCLUSTERED INDEX IX_Orders_UserDate 
ON Orders(UserID, OrderDate)
INCLUDE (Amount, Status);
```

### 查看索引

```sql
-- 查看表的索引
SELECT 
    i.name AS IndexName,
    i.type_desc AS IndexType,
    i.is_unique AS IsUnique,
    i.is_primary_key AS IsPrimaryKey
FROM sys.indexes i
WHERE i.object_id = OBJECT_ID('Users');

-- 查看索引统计信息
SELECT 
    OBJECT_NAME(i.object_id) AS TableName,
    i.name AS IndexName,
    s.user_seeks,
    s.user_scans,
    s.user_lookups,
    s.user_updates
FROM sys.indexes i
LEFT JOIN sys.dm_db_index_usage_stats s 
    ON i.object_id = s.object_id 
    AND i.index_id = s.index_id
WHERE OBJECT_NAME(i.object_id) = 'Users';
```

### 删除索引

```sql
-- 删除索引
DROP INDEX IX_Users_Email ON Users;

-- 删除表时自动删除所有索引
DROP TABLE Users;
```

## 3. 索引设计原则

### 何时创建索引

1. **高选择性列**：列的唯一值比例高（如邮箱、用户名）
2. **频繁查询的列**：经常出现在 WHERE、JOIN、ORDER BY 子句中
3. **外键列**：关联查询的性能关键
4. **覆盖查询列**：将查询列包含在索引中

### 何时不创建索引

1. **小表**：数据量少，全表扫描更快
2. **频繁更新的列**：索引会降低写操作性能
3. **低选择性列**：如性别、状态等
4. **大文本列**：索引会占用大量空间

### 索引优化建议

```sql
-- 创建索引前检查列的选择性
SELECT 
    COUNT(DISTINCT Email) AS UniqueEmails,
    COUNT(*) AS TotalRows,
    CAST(COUNT(DISTINCT Email) AS FLOAT) / COUNT(*) AS Selectivity
FROM Users;

-- 选择性大于 0.1（10%）适合创建索引
```

## 4. 查询性能优化

### 查询计划分析

```sql
-- 查看执行计划
SET SHOWPLAN_TEXT ON;
SELECT * FROM Users WHERE Email = 'test@example.com';
SET SHOWPLAN_TEXT OFF;

-- 使用 SET STATISTICS 查看查询统计
SET STATISTICS IO ON;
SET STATISTICS TIME ON;
SELECT * FROM Users WHERE Age > 25;
SET STATISTICS IO OFF;
SET STATISTICS TIME OFF;
```

### 查询优化技巧

```sql
-- 使用索引列在查询前面
-- 好
SELECT * FROM Users WHERE Email = 'test@example.com' AND UserName = 'Test';

-- 不好
SELECT * FROM Users WHERE UserName = 'Test' AND Email = 'test@example.com';

-- 避免 SELECT *
-- 好
SELECT UserID, UserName FROM Users;

-- 不好
SELECT * FROM Users;

-- 使用参数化查询
DECLARE @Email NVARCHAR(100) = 'test@example.com';
SELECT * FROM Users WHERE Email = @Email;

-- 避免在列上使用函数
-- 好
SELECT * FROM Users WHERE CreatedDate >= '2024-01-01';

-- 不好
SELECT * FROM Users WHERE YEAR(CreatedDate) = 2024;
```

### 使用临时表和表变量

```sql
-- 临时表
CREATE TABLE #TempUsers (
    UserID INT,
    UserName NVARCHAR(50)
);

INSERT INTO #TempUsers
SELECT UserID, UserName FROM Users WHERE Age > 25;

-- 使用临时表
SELECT * FROM #TempUsers;

-- 删除临时表
DROP TABLE #TempUsers;

-- 表变量
DECLARE @UserTable TABLE (
    UserID INT,
    UserName NVARCHAR(50)
);

INSERT INTO @UserTable
SELECT UserID, UserName FROM Users WHERE Age > 25;
```

## 5. 分区表

### 创建分区表

```sql
-- 创建分区函数
CREATE PARTITION FUNCTION PF_OrderDate(DATE)
AS RANGE RIGHT FOR VALUES ('2024-01-01', '2025-01-01');

-- 创建分区方案
CREATE PARTITION SCHEME PS_OrderDate
AS PARTITION PF_OrderDate
ALL TO ([PRIMARY]);

-- 创建分区表
CREATE TABLE Orders (
    OrderID INT IDENTITY(1,1),
    UserID INT,
    OrderDate DATE NOT NULL,
    Amount DECIMAL(10,2),
    PRIMARY KEY (OrderID, OrderDate)
) ON PS_OrderDate(OrderDate);
```

### 管理分区

```sql
-- 查看分区信息
SELECT 
    p.partition_number,
    p.rows,
    f.value AS BoundaryValue
FROM sys.partitions p
INNER JOIN sys.partition_functions f 
    ON 1=1
WHERE p.object_id = OBJECT_ID('Orders');

-- 合并分区
ALTER PARTITION FUNCTION PF_OrderDate()
MERGE RANGE ('2024-01-01');

-- 分割分区
ALTER PARTITION FUNCTION PF_OrderDate()
SPLIT RANGE ('2026-01-01');
```

## 6. 统计信息

### 创建统计信息

```sql
-- 创建统计信息
CREATE STATISTICS ST_Users_Age ON Users(Age);

-- 更新统计信息
UPDATE STATISTICS Users;

-- 更新特定统计信息
UPDATE STATISTICS Users ST_Users_Age;

-- 设置自动更新统计信息
ALTER DATABASE YourDB SET AUTO_UPDATE_STATISTICS ON;
```

### 查看统计信息

```sql
-- 查看统计信息
SELECT 
    s.name AS StatName,
    s.stats_date AS LastUpdated,
    st.name AS TableName,
    c.name AS ColumnName
FROM sys.stats s
INNER JOIN sys.tables st ON s.object_id = st.object_id
INNER JOIN sys.stats_columns sc ON s.stats_id = sc.stats_id AND s.object_id = sc.object_id
INNER JOIN sys.columns c ON sc.column_id = c.column_id AND sc.object_id = c.object_id
WHERE st.name = 'Users';
```

## 7. 性能监控

### 性能计数器

```sql
-- 查看缓存命中率
SELECT 
    CASE 
        WHEN cntr_type = 537003264 THEN 'Buffer Cache Hit Ratio'
        WHEN cntr_type = 272696576 THEN 'Page Lookups'
        WHEN cntr_type = 272696576 THEN 'Page Reads'
    END AS CounterName,
    cntr_value
FROM sys.dm_os_performance_counters
WHERE counter_name IN (
    'Buffer cache hit ratio',
    'Page lookups/sec',
    'Page reads/sec'
);
```

### 查询性能统计

```sql
-- 查看最耗时的查询
SELECT TOP 10
    qs.total_elapsed_time / qs.execution_count AS avg_elapsed_time,
    qs.execution_count,
    qs.total_elapsed_time,
    SUBSTRING(st.text, (qs.statement_start_offset/2) + 1,
        ((CASE qs.statement_end_offset
            WHEN -1 THEN DATALENGTH(st.text)
            ELSE qs.statement_end_offset
        END - qs.statement_start_offset)/2) + 1) AS query_text
FROM sys.dm_exec_query_stats qs
CROSS APPLY sys.dm_exec_sql_text(qs.sql_handle) st
ORDER BY avg_elapsed_time DESC;
```

## 8. Go 性能优化示例

```go
package main

import (
    "database/sql"
    "fmt"
    "log"
    "time"
    
    _ "github.com/microsoft/go-mssqldb"
)

func main() {
    connString := "server=localhost;user id=sa;password=YourPassword;database=TestDB"
    db, err := sql.Open("sqlserver", connString)
    if err != nil {
        log.Fatal(err)
    }
    defer db.Close()
    
    // 使用预编译语句
    query := `SELECT UserID, UserName, Email FROM Users WHERE Age > @Age`
    stmt, err := db.Prepare(query)
    if err != nil {
        log.Fatal(err)
    }
    defer stmt.Close()
    
    start := time.Now()
    rows, err := stmt.Query(sql.Named("Age", 25))
    if err != nil {
        log.Fatal(err)
    }
    defer rows.Close()
    
    for rows.Next() {
        var userID int
        var userName, email string
        err := rows.Scan(&userID, &userName, &email)
        if err != nil {
            log.Fatal(err)
        }
    }
    
    elapsed := time.Since(start)
    fmt.Printf("查询耗时: %v
", elapsed)
}
```

## 10. 常见性能问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 查询慢 | 缺少索引 | 创建合适的索引 |
| 缓存命中率低 | 内存不足 | 增加内存或优化查询 |
| 死锁 | 事务过长 | 优化事务，减少持有时间 |
| 索引碎片 | 频繁更新 | 重建或重组索引 |
| 统计信息过时 | 未更新 | 定期更新统计信息 |
