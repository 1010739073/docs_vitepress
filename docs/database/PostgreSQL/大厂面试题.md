# PostgreSQL 大厂面试题

## 一、基础概念

### 1. 什么是 PostgreSQL？它有哪些特点？

**答案：**
PostgreSQL 是一个功能强大的开源对象关系型数据库系统。

**主要特点：**
- **开源免费**：BSD 许可证
- **标准兼容**：高度兼容 SQL 标准
- **扩展性强**：支持自定义数据类型、函数、操作符
- **并发控制**：原生 MVCC，读写不阻塞
- **数据完整性**：支持外键、约束、触发器
- **全文搜索**：内置全文搜索功能
- **地理信息**：PostGIS 扩展
- **JSON 支持**：原生 JSON/JSONB

### 2. PostgreSQL 和 MySQL 的区别？

**答案：**
| 特性 | PostgreSQL | MySQL |
|-----|-----------|-------|
| 许可证 | PostgreSQL | GPL |
| SQL 标准 | 更兼容 | 部分兼容 |
| 扩展性 | 极强 | 中等 |
| JSON 支持 | JSONB 二进制 | JSON 文本 |
| 并发控制 | MVCC | 基于锁 |
| 存储过程 | PL/pgSQL | 多种语言 |
| 地理信息 | PostGIS | 有限支持 |

### 3. PostgreSQL 的数据类型有哪些？

**答案：**
- **整数**：SMALLINT、INTEGER、BIGINT、SERIAL
- **小数**：DECIMAL、NUMERIC、REAL、DOUBLE
- **字符串**：CHAR、VARCHAR、TEXT
- **日期时间**：DATE、TIME、TIMESTAMP、TIMESTAMPTZ、INTERVAL
- **布尔**：BOOLEAN
- **二进制**：BYTEA
- **JSON**：JSON、JSONB
- **数组**：ARRAY
- **UUID**：UUID
- **范围**：INT4RANGE、TSRANGE、DATERANGE

### 4. VARCHAR 和 TEXT 的区别？

**答案：**
- **VARCHAR(n)**：变长字符串，最大 n 字符，有长度限制
- **TEXT**：无长度限制的变长字符串

**性能：** 两者在 PostgreSQL 中性能相同，TEXT 更灵活。

### 5. JSON 和 JSONB 的区别？

**答案：**
| 特性 | JSON | JSONB |
|-----|------|-------|
| 存储 | 文本 | 二进制 |
| 解析 | 使用时解析 | 存储时解析 |
| 性能 | 较慢 | 较快 |
| 索引 | 不支持 | 支持 GIN |
| 空格 | 保留 | 去除 |
| 键顺序 | 保留 | 不保留 |
| 重复键 | 保留 | 去重 |

**推荐**：查询频繁使用 JSONB，只需存储使用 JSON。

## 二、SQL 查询

### 6. 什么是索引？PostgreSQL 有哪些索引类型？

**答案：**
索引是加速数据检索的数据结构。

**类型：**
- **B-tree**：默认，适合等值和范围查询
- **Hash**：等值查询（很少使用）
- **GiST**：几何、范围类型
- **SP-GiST**：高维数据
- **GIN**：数组、JSONB、全文搜索
- **BRIN**：大表顺序数据

### 7. B-tree 索引的原理？

**答案：**
B-tree（平衡树）索引：
- 所有叶子节点在同一层
- 支持等值查询、范围查询、排序
- 自动维护平衡
- 时间复杂度 O(log n)

### 8. 什么时候使用 GIN 索引？

**答案：**
GIN（通用倒排索引）适合：
- 数组包含查询
- JSONB 查询
- 全文搜索
- 范围类型查询

```sql
-- 数组查询
CREATE INDEX idx_tags ON articles USING GIN(tags);
SELECT * FROM articles WHERE tags @> ARRAY['sql'];

-- JSONB 查询
CREATE INDEX idx_data ON table_name USING GIN(data);
SELECT * FROM table_name WHERE data @> '{"key": "value"}';
```

### 9. 什么是覆盖索引？

**答案：**
覆盖索引包含查询所需的所有列，查询时无需回表。

```sql
-- PostgreSQL 不支持 INCLUDE 语法，但可以通过复合索引实现
CREATE INDEX idx_cover ON users(username, email, age);

-- 查询时无需回表
SELECT username, email, age FROM users WHERE username = '张三';
```

### 10. 如何查看和分析执行计划？

**答案：**
```sql
-- 基础 EXPLAIN
EXPLAIN SELECT * FROM users WHERE age > 25;

-- 实际执行（包含时间和缓冲区统计）
EXPLAIN (ANALYZE, BUFFERS, FORMAT TEXT)
SELECT * FROM users WHERE age > 25;

-- JSON 格式
EXPLAIN (ANALYZE, BUFFERS, FORMAT JSON)
SELECT * FROM users WHERE age > 25;
```

## 三、事务与并发

### 11. 什么是 MVCC？PostgreSQL 如何实现？

**答案：**
MVCC（多版本并发控制）通过维护数据多个版本实现并发控制。

**PostgreSQL 实现：**
- 每行数据有 `xmin`（插入事务ID）和 `xmax`（删除事务ID）
- 查询时根据事务ID判断可见性
- 无需读锁，读写不阻塞
- 需要定期 VACUUM 清理死元组

### 12. PostgreSQL 的隔离级别有哪些？

**答案：**
| 隔离级别 | 脏读 | 不可重复读 | 幻读 |
|---------|------|-----------|------|
| READ UNCOMMITTED | ✗ | ✓ | ✓ | (同 READ COMMITTED) |
| READ COMMITTED | ✗ | ✓ | ✓ | (默认) |
| REPEATABLE READ | ✗ | ✗ | ✗ | (PG中防止幻读) |
| SERIALIZABLE | ✗ | ✗ | ✗ | |

### 13. 什么是 WAL（预写日志）？

**答案：**
WAL（Write-Ahead Logging）是先写日志后写数据的机制。

**作用：**
- 保证数据持久性
- 支持时间点恢复
- 支持流复制
- 提高写入性能

### 14. 什么是死锁？如何预防？

**答案：**
死锁是两个事务相互等待对方释放资源。

**PostgreSQL 处理：**
- 自动检测死锁
- 终止其中一个事务

**预防方法：**
1. 按相同顺序访问资源
2. 保持事务简短
3. 使用 NOWAIT
4. 使用咨询锁

### 15. 什么是表膨胀？如何解决？

**答案：**
表膨胀是由于 MVCC 机制导致的死元组积累。

**解决：**
```sql
-- 普通清理
VACUUM users;

-- 完全清理（会锁表）
VACUUM FULL users;

-- 使用 pg_repack（不锁表）
pg_repack -d mydb -t users
```

## 四、存储过程与函数

### 16. PostgreSQL 函数和存储过程的区别？

**答案：**
| 特性 | 函数 | 存储过程 |
|-----|------|---------|
| 返回值 | 必须有 | 可以没有 |
| 事务控制 | 有限制 | 完整控制 |
| 调用方式 | SELECT/表达式 | CALL |
| 版本 | 一直支持 | PG 11+ |

### 17. 什么是 PL/pgSQL？

**答案：**
PL/pgSQL 是 PostgreSQL 的过程语言。

**特点：**
- 类似 Oracle PL/SQL
- 支持变量、控制结构、异常处理
- 可以创建函数、存储过程、触发器

### 18. 什么是触发器？有哪些类型？

**答案：**
触发器是自动执行的函数。

**类型：**
- **BEFORE/AFTER**：在操作前/后触发
- **INSTEAD OF**：替代操作（用于视图）
- **行级触发器**：每行触发（FOR EACH ROW）
- **语句级触发器**：每条语句触发（FOR EACH STATEMENT）

## 五、性能优化

### 19. 如何优化慢查询？

**答案：**
1. 使用 EXPLAIN ANALYZE 分析执行计划
2. 添加合适的索引
3. 避免在索引列上使用函数
4. 使用 EXISTS 代替 IN
5. 更新统计信息（ANALYZE）
6. 重写查询逻辑
7. 使用分区表
8. 调整配置参数

### 20. 什么是统计信息？为什么重要？

**答案：**
统计信息帮助优化器估计行数，选择最优执行计划。

**维护：**
```sql
-- 更新统计信息
ANALYZE users;
ANALYZE users(username, email);

-- 自动收集
-- autovacuum 会自动运行 ANALYZE
```

### 21. 什么是连接池？为什么需要？

**答案：**
连接池是管理数据库连接的缓存。

**作用：**
- 减少连接开销
- 限制连接数
- 提高性能

**常用工具：**
- PgBouncer
- HikariCP（Java）
- SQLAlchemy Pool（Python）

## 六、备份与恢复

### 22. PostgreSQL 有哪些备份方式？

**答案：**
- **逻辑备份**：pg_dump、pg_dumpall
- **物理备份**：pg_basebackup
- **增量备份**：WAL 归档、PG 17+ 增量备份

### 23. pg_dump 和 pg_basebackup 的区别？

**答案：**
| 特性 | pg_dump | pg_basebackup |
|-----|---------|--------------|
| 类型 | 逻辑备份 | 物理备份 |
| 内容 | SQL 语句 | 二进制文件 |
| 恢复 | 灵活（表级） | 完整恢复 |
| 大小 | 较小 | 较大 |
| 速度 | 较慢 | 较快 |
| 使用场景 | 迁移、部分恢复 | 灾难恢复 |

### 24. 什么是 PITR（时间点恢复）？

**答案：**
PITR（Point-In-Time Recovery）允许将数据库恢复到特定时间点。

**步骤：**
1. 基础备份（pg_basebackup）
2. WAL 归档配置
3. 恢复时指定时间点

### 25. 什么是流复制？

**答案：**
流复制是主备实时同步的机制。

**配置：**
```bash
# 主服务器
wal_level = replica
max_wal_senders = 10

# 备服务器
pg_basebackup -h primary -U replicator -D /data -Fp -Xs -P -R
```

## 七、高可用

### 26. PostgreSQL 有哪些高可用方案？

**答案：**
| 方案 | 特点 |
|-----|------|
| 流复制 | 实时同步，读写分离 |
| 逻辑复制 | 表级复制，跨版本 |
| Patroni | 自动故障转移 |
| repmgr | 复制管理工具 |
| Pgpool-II | 连接池 + 负载均衡 |

### 27. 什么是逻辑复制？

**答案：**
逻辑复制是基于行的复制机制。

**特点：**
- 表级复制
- 跨版本复制
- 支持多主复制
- 可以选择性复制

```sql
-- 发布
CREATE PUBLICATION my_pub FOR TABLE users, orders;

-- 订阅
CREATE SUBSCRIPTION my_sub 
CONNECTION 'host=primary dbname=mydb' 
PUBLICATION my_pub;
```

### 28. 什么是复制槽？

**答案：**
复制槽确保主服务器保留备服务器需要的 WAL 文件。

**作用：**
- 防止 WAL 被过早删除
- 支持断点续传

```sql
-- 创建复制槽
SELECT * FROM pg_create_physical_replication_slot('slot_name');

-- 查看复制槽
SELECT * FROM pg_replication_slots;
```

## 八、高级主题

### 29. 什么是分区表？

**答案：**
分区表将大表分成更小的、可管理的部分。

**类型：**
- 范围分区（RANGE）
- 列表分区（LIST）
- 哈希分区（HASH）

```sql
-- 创建分区表
CREATE TABLE measurements (
    id SERIAL,
    logdate DATE NOT NULL,
    value NUMERIC
) PARTITION BY RANGE (logdate);

-- 创建分区
CREATE TABLE measurements_y2024m01 PARTITION OF measurements
    FOR VALUES FROM ('2024-01-01') TO ('2024-02-01');
```

### 30. 什么是并行查询？

**答案：**
并行查询使用多个 CPU 核心同时执行查询。

**配置：**
```sql
max_parallel_workers_per_gather = 4
max_parallel_workers = 8
```

**使用：**
```sql
EXPLAIN (ANALYZE)
SELECT COUNT(*) FROM large_table;
-- Gather
--    Workers Planned: 2
```

### 31. PostgreSQL 有哪些扩展？

**答案：**
- **PostGIS**：地理信息
- **pg_stat_statements**：SQL 统计
- **pg_trgm**：模糊搜索
- **uuid-ossp**：UUID 生成
- **hstore**：键值存储
- **citext**：不区分大小写文本

```sql
-- 安装扩展
CREATE EXTENSION postgis;
CREATE EXTENSION pg_stat_statements;
```

## 九、场景题

### 32. 数据库查询很慢，如何排查？

**答案：**
1. 使用 EXPLAIN ANALYZE 查看执行计划
2. 检查是否缺少索引
3. 查看 pg_stat_statements 慢查询
4. 检查统计信息是否过期
5. 查看是否有锁等待
6. 检查表膨胀情况
7. 查看硬件资源

### 33. 数据库连接数过多，如何处理？

**答案：**
1. 查看当前连接（pg_stat_activity）
2. 终止空闲连接
3. 增加 max_connections
4. 使用连接池（PgBouncer）
5. 优化应用程序连接使用

### 34. 如何设计一个支持千万级数据量的表？

**答案：**
1. 使用分区表
2. 设计合理的索引
3. 使用 BRIN 索引（时序数据）
4. 定期归档历史数据
5. 使用表空间分布 IO
6. 考虑读写分离

### 35. 数据库死锁频繁发生，如何解决？

**答案：**
1. 按相同顺序访问资源
2. 保持事务简短
3. 使用 NOWAIT 或 SKIP LOCKED
4. 使用较低的隔离级别
5. 捕获死锁错误并重试
6. 分析死锁日志

### 36. 如何进行数据库性能调优？

**答案：**
1. 收集性能基线
2. 识别瓶颈（CPU、内存、IO、锁）
3. 优化查询和索引
4. 调整配置参数（shared_buffers、work_mem 等）
5. 优化数据库设计
6. 使用分区策略
7. 持续监控和调优

### 37. 什么是 VACUUM？为什么重要？

**答案：**
VACUUM 清理死元组，回收空间。

**类型：**
- **VACUUM**：标记死元组可重用
- **VACUUM FULL**：压缩表，回收空间（锁表）
- **ANALYZE**：更新统计信息
- **VACUUM ANALYZE**：同时进行

**重要性：**
- 防止表膨胀
- 更新统计信息
- 防止事务ID回卷

### 38. 如何监控 PostgreSQL？

**答案：**
**系统视图：**
- pg_stat_activity
- pg_stat_database
- pg_stat_user_tables
- pg_stat_user_indexes
- pg_stat_replication

**工具：**
- pgAdmin
- Grafana + Prometheus
- pgwatch2
- pganalyze

### 39. 什么是事务ID回卷？如何预防？

**答案：**
事务ID 是 32 位整数，会用完并回卷。

**预防：**
- 定期运行 VACUUM
- 监控 age(datfrozenxid)
- 设置 autovacuum 参数

```sql
-- 检查事务ID 年龄
SELECT datname, age(datfrozenxid) FROM pg_database;
```
