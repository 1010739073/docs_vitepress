# PostgreSQL 知识点总结

## 二、SQL 语句速查

### 2.1 DDL（数据定义语言）

```sql
-- 数据库操作
CREATE DATABASE mydb;
DROP DATABASE mydb;
\c mydb  -- 切换数据库

-- 表操作
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT NOW()
);

ALTER TABLE users ADD COLUMN age INTEGER;
ALTER TABLE users DROP COLUMN age;
ALTER TABLE users RENAME TO customers;

DROP TABLE users;
TRUNCATE TABLE users RESTART IDENTITY;

-- 索引操作
CREATE INDEX idx_name ON users(name);
CREATE UNIQUE INDEX idx_email ON users(email);
DROP INDEX idx_name;

-- 视图操作
CREATE VIEW active_users AS SELECT * FROM users WHERE is_active = TRUE;
DROP VIEW active_users;
```

### 2.2 DML（数据操作语言）

```sql
-- 插入
INSERT INTO users (name, email) VALUES ('张三', 'zs@example.com');
INSERT INTO users VALUES ('李四', 'ls@example.com');
INSERT INTO users SELECT * FROM temp_users;

-- 更新
UPDATE users SET age = 25 WHERE id = 1;
UPDATE users SET age = age + 1 WHERE is_active = TRUE;

-- 删除
DELETE FROM users WHERE id = 1;
TRUNCATE TABLE users;

-- UPSERT (INSERT ON CONFLICT)
INSERT INTO users (id, name) VALUES (1, '张三')
ON CONFLICT (id) DO UPDATE SET name = EXCLUDED.name;

INSERT INTO users (id, name) VALUES (1, '张三')
ON CONFLICT (id) DO NOTHING;
```

### 2.3 DQL（数据查询语言）

```sql
-- 基础查询
SELECT * FROM users;
SELECT DISTINCT status FROM users;
SELECT * FROM users LIMIT 10 OFFSET 0;

-- 条件查询
SELECT * FROM users WHERE age > 18;
SELECT * FROM users WHERE name LIKE '%张%';
SELECT * FROM users WHERE name ILIKE '%zhang%';  -- 不区分大小写
SELECT * FROM users WHERE age BETWEEN 18 AND 60;
SELECT * FROM users WHERE id IN (1, 2, 3);

-- 排序和分页
SELECT * FROM users ORDER BY age DESC;
SELECT * FROM users ORDER BY age LIMIT 10 OFFSET 20;

-- 分组
SELECT status, COUNT(*) FROM users GROUP BY status;
SELECT status, COUNT(*) FROM users GROUP BY status HAVING COUNT(*) > 5;

-- 连接
SELECT * FROM users u JOIN orders o ON u.id = o.user_id;
SELECT * FROM users u LEFT JOIN orders o ON u.id = o.user_id;
SELECT * FROM users u FULL JOIN orders o ON u.id = o.user_id;
SELECT * FROM users u CROSS JOIN roles r;

-- 子查询
SELECT * FROM users WHERE id IN (SELECT user_id FROM orders);
SELECT * FROM users u WHERE EXISTS (SELECT 1 FROM orders o WHERE o.user_id = u.id);
```

## 三、索引与优化

### 3.1 索引类型

| 索引类型 | 适用场景 | 示例 |
|---------|---------|------|
| **B-tree** | 等值、范围查询 | 默认索引 |
| **Hash** | 等值查询 | 很少使用 |
| **GiST** | 几何、范围类型 | 地理数据 |
| **GIN** | 数组、JSONB、全文搜索 | jsonb_ops |
| **BRIN** | 大表顺序数据 | 时序数据 |

### 3.2 查询优化原则

1. **避免在索引列上使用函数**
   ```sql
   -- 不好
   WHERE EXTRACT(YEAR FROM created_at) = 2023
   -- 好
   WHERE created_at >= '2023-01-01' AND created_at < '2024-01-01'
   ```

2. **使用 EXISTS 代替 IN**
   ```sql
   -- 不好
   WHERE id IN (SELECT user_id FROM orders)
   -- 好
   WHERE EXISTS (SELECT 1 FROM orders o WHERE o.user_id = users.id)
   ```

3. **避免 SELECT ***
   ```sql
   -- 不好
   SELECT * FROM users
   -- 好
   SELECT id, name FROM users
   ```

## 四、事务与并发

### 4.1 隔离级别

| 隔离级别 | 脏读 | 不可重复读 | 幻读 |
|---------|------|-----------|------|
| READ UNCOMMITTED | ✗ | ✓ | ✓ |
| READ COMMITTED | ✗ | ✓ | ✓ | (默认)
| REPEATABLE READ | ✗ | ✗ | ✗ | (PG中)
| SERIALIZABLE | ✗ | ✗ | ✗ |

### 4.2 事务控制

```sql
BEGIN;  -- 开始事务
COMMIT;  -- 提交事务
ROLLBACK;  -- 回滚事务
SAVEPOINT sp1;  -- 保存点
ROLLBACK TO SAVEPOINT sp1;  -- 回滚到保存点

-- 设置隔离级别
BEGIN ISOLATION LEVEL SERIALIZABLE;
SET TRANSACTION ISOLATION LEVEL REPEATABLE READ;
```

## 六、系统视图

### 6.1 系统目录和信息视图

PostgreSQL 的系统元数据主要放在每个数据库内部的系统 schema 中。日常排查可以查询它们，但不要直接 `UPDATE`、`DELETE` 或手动修改 `pg_catalog` 中的系统表，否则可能导致数据库元数据异常。

| schema | 说明 |
|-------|------|
| `pg_catalog` | PostgreSQL 自带系统 schema，保存系统表、系统视图、内置函数、数据类型等元数据 |
| `information_schema` | SQL 标准定义的信息 schema，用标准方式展示表、字段等元数据，兼容性更好 |

常见系统目录表：

| 系统表 | 存储内容 |
|-------|----------|
| `pg_database` | 当前实例中的数据库信息，例如数据库名、编码、属主、连接权限 |
| `pg_class` | 表、索引、序列、视图、物化视图等关系对象信息 |
| `pg_namespace` | schema 信息，例如 `public`、`pg_catalog`、`information_schema` |
| `pg_attribute` | 表字段信息，例如字段名、字段类型、字段序号、是否删除 |
| `pg_type` | 数据类型信息，包括内置类型、自定义类型、枚举类型等 |
| `pg_roles` | 角色和用户信息，常用于查看角色名、权限属性 |
| `pg_index` | 索引和表字段之间的关联信息 |
| `pg_constraint` | 主键、外键、唯一约束、检查约束等约束信息 |
| `pg_proc` | 函数和存储过程信息 |

常见统计视图和信息视图：

| 视图 | 存储或展示内容 |
|-----|----------------|
| `pg_stat_activity` | 当前连接、会话状态、正在执行的 SQL、等待事件等信息 |
| `pg_stat_database` | 数据库级统计信息，例如事务提交、回滚、读写块、连接数 |
| `pg_stat_user_tables` | 用户表访问统计，例如扫描次数、插入行数、更新行数、删除行数 |
| `pg_stat_user_indexes` | 用户索引访问统计，例如索引扫描次数和读取行数 |
| `pg_indexes` | 索引名称、所属表、schema 和索引定义 |
| `information_schema.tables` | 标准 SQL 视图，用于查看表和视图信息 |
| `information_schema.columns` | 标准 SQL 视图，用于查看字段名、数据类型、是否允许为空等信息 |

### 6.2 常用查询模板

```sql
-- 查看实例中有哪些数据库
SELECT datname, datdba, encoding
FROM pg_database;

-- 查看当前数据库有哪些 schema
SELECT nspname
FROM pg_namespace;

-- 查看 public schema 下有哪些表
SELECT tablename
FROM pg_tables
WHERE schemaname = 'public';

-- 查看 public schema 下的字段信息
SELECT table_name, column_name, data_type, is_nullable
FROM information_schema.columns
WHERE table_schema = 'public'
ORDER BY table_name, ordinal_position;

-- 查看当前连接和正在执行的 SQL
SELECT pid, usename, datname, state, query
FROM pg_stat_activity;

-- 查看用户表统计信息
SELECT schemaname, relname, seq_scan, idx_scan, n_live_tup, n_dead_tup
FROM pg_stat_user_tables
ORDER BY n_dead_tup DESC;

-- 查看索引定义
SELECT schemaname, tablename, indexname, indexdef
FROM pg_indexes
WHERE schemaname = 'public';
```

### 6.3 常用系统视图

```sql
-- 数据库信息
SELECT * FROM pg_database;
SELECT * FROM pg_tables;
SELECT * FROM pg_indexes;

-- 性能监控
SELECT * FROM pg_stat_activity;
SELECT * FROM pg_stat_database;
SELECT * FROM pg_stat_user_tables;
SELECT * FROM pg_stat_user_indexes;
SELECT * FROM pg_stat_statements;

-- 锁和复制信息
SELECT * FROM pg_locks;
SELECT * FROM pg_stat_replication;
```

## 七、高可用

### 7.1 流复制

```bash
# 主服务器配置
wal_level = replica
max_wal_senders = 10
wal_keep_size = 1GB

# 备服务器设置
pg_basebackup -h primary -U replicator -D /data -Fp -Xs -P -R
```

### 7.2 逻辑复制

```sql
-- 发布
CREATE PUBLICATION my_pub FOR TABLE users, orders;

-- 订阅
CREATE SUBSCRIPTION my_sub 
CONNECTION 'host=primary dbname=mydb user=replicator' 
PUBLICATION my_pub;
```
